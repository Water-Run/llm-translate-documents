1. 概述

校验和 VFS 扩展是一个VFS shim，它向 SQLite 数据库中每一页的末尾添加一个 8 字节的校验和。 在每页写入时添加校验和，在每页读取时验证校验和。校验和的目的是帮助检测由 大容量存储设备中的随机位翻转引起的数据库损坏。

校验和 VFS 扩展需要 SQLite 版本 3.32.0（2020-05-22）或更高版本。它无法 与早期版本的 SQLite 一起使用。

2. 编译

校验和 VFS 模块是一个可加载扩展。它没有包含在合并源中。 必须在编译时或运行时将其添加到 SQLite。校验和 VFS 模块的源代码位于 ext/misc/cksumvfs.c源文件中， 位于SQLite 源树中。

要将校验和 VFS 模块构建为可在运行时加载的扩展，请使用类似以下的命令：

  • (linux) → gcc -fPIC -shared cksumvfs.c -o cksumvfs.so
  • (mac) → clang -fPIC -dynamiclib cksumvfs.c -o cksumvfs.dylib
  • (windows) → cl cksumvfs.c -link -dll -out:cksumvfs.dll

当然，您可能需要根据项目需要添加其他编译器选项。

要将此扩展静态链接到您的产品中， 像编译其他 C 语言模块一样编译，但添加 "-DSQLITE_CKSUMVFS_STATIC" 选项，使该模块知道 它是静态链接而非动态链接。

3. 加载

要将此扩展作为共享库加载，首先必须 打开一个用于传递给sqlite3_load_extension() API 调用的 虚拟 SQLite 数据库连接。然后调用 sqlite3_load_extension() API 并关闭该虚拟数据库连接。 之后打开的所有数据库连接都将包含此扩展。例如： 

sqlite3 *db;
sqlite3_open(":memory:", &db);
sqlite3_load_extension(db, "./cksumvfs");
sqlite3_close(db);

如果此扩展是使用 -DSQLITE_CKSUMVFS_STATIC 编译并 与应用程序静态链接的，请使用以下单个 API 调用进行初始化： 

sqlite3_cksumvfs_init();

Cksumvfs 是一个VFS shim。加载后，“cksmvfs” 成为新的 默认 VFS，并将先前的默认 VFS 作为堆栈中的下一个 VFS。 这通常是您所需要的。然而，在 多个 VFS shim 同时加载的复杂情况下， 确保 cksumvfs 以正确的顺序加载从而在默认 VFS Shim 堆栈中排在正确位置可能非常重要。

4. 使用

使用sqlite3_open()或 sqlite3_open_v2()接口按通常方式打开数据库连接。 普通数据库文件（没有校验和）将照常工作。包含 校验和的数据库在遇到包含无效校验和的页面时将返回 SQLITE_IOERR_DATA 错误。

只有当数据库的保留字节值精确为 8 时， 校验和才有效。保留字节的默认值为 0。 因此，新创建的数据库文件默认不会包含校验和。要创建包含校验和的数据库，请通过运行类似如下的代码将 保留字节值更改为 8： 

int n = 8;
sqlite3_file_control(db, 0, SQLITE_FCNTL_RESERVE_BYTES, &n);

如果您在创建新数据库文件后立即执行此操作， 在写入其他内容之前，这可能就是您需要做的全部。 否则，上述 API 调用之后应执行： 

sqlite3_exec(db, "VACUUM", 0, 0, 0);

即使您不需要，也运行 VACUUM 也无妨。 如果数据库处于 WAL 模式，应关闭并重新打开所有数据库连接，然后再继续。

在命令行中，使用“.filectrl reserve_bytes 8”命令， 然后执行“VACUUM;”。

请注意，SQLite 允许增加保留字节的数量，但不允许减少。 因此，如果数据库文件已有大于 8 的保留字节值，则无法 在该数据库上激活校验和，除非导出并恢复数据库文件。 还应注意，其他扩展可能也会使用保留字节。 校验和将与这些其他扩展不兼容。

5. 校验和验证

如果任何校验和不正确，“PRAGMA quick_check”命令 将发现它。要验证校验和确实已启用并正在运行，请使用如下 SQL： 

SELECT count(*), verify_checksum(data)
  FROM sqlite_dbpage
 GROUP BY 2;

verify_checksum() 函数有三种可能的输出：1、0 和 NULL。 如果校验和正确，返回 1。如果校验和不正确，返回 0。 如果页面无法读取，返回 NULL。如果启用了校验和， 当校验和错误时读取将失败，因此 verify_checksum() 在出现错误校验和时的通常结果是 NULL。

如果一切正常，上述查询应该只返回一行，其第二列为 1。 任何其他结果都表明或是存在校验和错误，或是校验和验证已禁用。

6. 控制校验和验证

cksumvfs 扩展实现了一个新的 PRAGMA 语句，可用于禁用、重新启用或查询校验和验证的状态： 

PRAGMA checksum_verification;          -- 查询状态
PRAGMA checksum_verification=OFF;      -- 禁用验证
PRAGMA checksum_verification=ON;       -- 重新启用验证

"checksum_verification" pragma 如果启用校验和验证则返回 "1"（true），如果禁用则返回 "0"（false）。 在此上下文中，“验证”指的是在读取时检测到校验和不匹配时 引发 SQLITE_IOERR_DATA 错误的功能。只要数据库的保留字节值为 8， 校验和始终会保持最新状态，无论此 pragma 的设置如何。 可以禁用校验和验证（例如）以便对先前报告校验和错误的数据库执行取证分析。

如果数据库文件的保留字节值不是 8， 则 "checksum_verification" pragma 始终返回 "0"。 如果未加载 cksumvfs 扩展，则该 pragma 将不返回任何行。
