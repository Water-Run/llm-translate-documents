SQLITE_MEMSTAT 虚拟表

1. 概览

SQLITE_MEMSTAT 扩展实现了一个同名虚拟表，提供对sqlite3_status64()和 sqlite3_db_status()接口的 SQL 访问。

SQLITE_STMT 扩展也可以在运行时加载，只需使用位于https://sqlite.org/src/file/ext/misc/memstat.c的源代码， 并按照如何编译可加载扩展中的说明将扩展编译为共享库或 DLL。

2. 用法

SQLITE_MEMSTAT 虚拟表是一个只读表，可用于查询当前 SQLite 实例的性能特征（主要是正在使用的内存量）。 SQLITE_MEMSTATE 表本质上是围绕 C 语言接口sqlite3_status64()和sqlite3_db_status()的封装。 如果将memstat.c源文件与 -DSQLITE_ENABLE_ZIPVFS 选项一起编译， 那么 SQLITE_MEMSTAT 还会执行一些文件控制调用，以提取 ZIPVFS 子系统的内存使用信息，只要该子系统已获授权、安装并在使用中。

SQLITE_MEMSTAT 表的模式如下： 

CREATE TABLE sqlite_memstat(
  name TEXT,
  schema TEXT,
  value INT,
  hiwtr INT
);

SQLITE_MEMSTAT 表的每一行都对应对sqlite3_status64()或sqlite3_db_status()接口的单次调用。 该行的 NAME 列标识传递给这些接口的“操作”。例如，如果以 SQLITE_STATUS_MEMORY_USED调用sqlite3_status64()， 则 NAME 列为 'MEMORY_USED'。或者如果以SQLITE_DBSTATUS_CACHE_USED调用 sqlite3_db_status()，则 NAME 列为 "DB_CACHE_USED"。

除非使用sqlite3_file_control()接口查询 ZIPVFS 后端，否则 SCHEMA 列为 NULL。 由于只有在 memstat.c 模块使用 -DSQLITE_ENABLE_ZIPVFS 编译且ZIPVFS 正在使用的情况下才会发生这种情况， SCHEMA 通常为 NULL。

VALUE 和 HIWTR 列报告该度量的当前值及其“高水位线”。高水位线是该度量自上次重置以来观察到的最高值。SQLITE_MEMSTAT 虚拟表不提供重置高水位线的机制。

根据所查询的参数不同，VALUE 或 HIWTR 值可能未定义。例如，SQLITE_STATUS_MALLOC_SIZE只对高水位线有意义， 而SQLITE_DBSTATUS_CACHE_USED只对当前值有意义。 对于 VALUE 或 HIWTR 中某一项无意义的行，该值返回 NULL。
