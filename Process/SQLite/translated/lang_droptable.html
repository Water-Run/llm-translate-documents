DROP TABLE

drop-table-stmt：

DROP TABLE 语句删除通过 CREATE TABLE 语句添加的表。所指定的名称即为表名。被删除的表会从数据库模式和磁盘文件中完全移除，且无法恢复。与该表关联的所有索引和触发器也会被删除。

可选的 IF EXISTS 子句可抑制在表不存在时通常会产生的错误。

如果启用了 外键约束，则 DROP TABLE 命令在从数据库模式中删除表之前，会隐式执行一次 DELETE FROM 命令。附加到该表上的任何触发器会在隐式 DELETE FROM 执行之前从数据库模式中删除，因此这不会导致任何触发器被触发。相比之下，隐式 DELETE FROM 会触发任何配置的 外键操作。如果隐式 DELETE FROM 在 DROP TABLE 命令中执行时违反了任何立即外键约束，系统会返回错误并且不会删除表。如果隐式 DELETE FROM 导致任何延迟外键约束被违反，而且在事务提交时这些违规仍然存在，则在提交时会返回错误。
