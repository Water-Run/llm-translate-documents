如何将 MinGW 构建为交叉编译器

MinGW，也称为 Minimalist GNU For Windows， 是一种流行的 GCC 编译器的版本，用于构建 Win95/Win98/WinNT 可执行文件。有关详情请参阅官网。

本页介绍如何在 Linux 下从源代码构建 MinGW 作为交叉编译器。这样一来，您就可以在熟悉的 Unix 桌面环境中轻松创建 WinNT 二进制文件。

以下是步骤：

  1. 获取源代码副本。您需要 binutils、编译器和 MinGW 运行时。它们可分别下载。 截至本文撰写时，Mumit Khan 已在一个 FTP 站点为您收集了所需的一切： ftp://ftp.nanotech.wisc.edu/pub/khan/gnu-win32/mingw32/snapshots/gcc-2.95.2-1 您需要的三个文件是：

    • binutils-19990818-1-src.tar.gz
    • gcc-2.95.2-1-src.tar.gz
    • mingw-20000203.zip

    将所有下载文件放在一个不会频繁使用的目录中。后续步骤 将假设所有下载都位于名为 ~/mingw/download 的目录中。

  2. 创建一个用于安装新的编译器套件的目录，并使该目录可写。 根据选择的目录，可能需要切换到 root。下面的示例 shell 命令 将假设安装目录是 /opt/mingw 且您的用户 ID 是 drh。

    su
mkdir /opt/mingw
chown drh /opt/mingw
exit

  3. 将源码 tarball 解压到一个单独的目录。

    mkdir ~/mingw/src
cd ~/mingw/src
tar xzf ../download/binutils-*.tar.gz
tar xzf ../download/gcc-*.tar.gz
unzip ../download/mingw-*.zip

  4. 创建一个用于放置所有构建产物的目录。

    mkdir ~/mingw/bld

  5. 配置并构建 binutils，并将结果添加到您的 PATH。

    mkdir ~/mingw/bld/binutils
cd ~/mingw/bld/binutils
../../src/binutils/configure --prefix=/opt/mingw --target=i386-mingw32 -v
make 2>&1 | tee make.out
make install 2>&1 | tee make-install.out
export PATH=$PATH:/opt/mingw/bin

  6. 在尝试构建编译器之前，手动将运行时头文件复制到安装目录中。

    mkdir /opt/mingw/i386-mingw32/include
cd ~/mingw/src/mingw-runtime*/mingw/include
cp -r * /opt/mingw/i386-mingw32/include

  7. 配置并构建编译器

    mkdir ~/mingw/bld/gcc
cd ~/mingw/bld/gcc
../../src/gcc-*/configure --prefix=/opt/mingw --target=i386-mingw32 -v
cd gcc
make installdirs
cd ..
make 2>&1 | tee make.out
make install

  8. 配置并构建 MinGW 运行时

    mkdir ~/mingw/bld/runtime
cd ~/mingw/bld/runtime
../../src/mingw-runtime*/configure --prefix=/opt/mingw --target=i386-mingw32 -v
make install-target-w32api
make install

至此完成...
