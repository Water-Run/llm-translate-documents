1. 介绍

生成列（有时也称为“计算列”）是表中的列，其值是同一行中其他列的函数。 生成列可以读取，但不能直接写入。更改生成列值的唯一方法是修改用于 计算该生成列的其他列的值。

2. 语法

从语法上讲，生成列通过“GENERATED ALWAYS”列约束 指定。例如：

CREATE TABLE t1(
   a INTEGER PRIMARY KEY,
   b INT,
   c TEXT,
   d INT GENERATED ALWAYS AS (a*abs(b)) VIRTUAL,
   e TEXT GENERATED ALWAYS AS (substr(c,b,b+1)) STORED
);

上面的语句有三个普通列：“a”（主键）、“b”和“c”，以及两个生成列“d”和“e”。

约束开头的“GENERATED ALWAYS”关键字以及末尾的“VIRTUAL”或“STORED” 关键字都是可选的。只有“AS”关键字和括号中的表达式是必需的。如果省略末尾的 “VIRTUAL”或“STORED”关键字，则默认是 VIRTUAL。因此，上面的示例语句 可以简化为：

CREATE TABLE t1(
   a INTEGER PRIMARY KEY,
   b INT,
   c TEXT,
   d INT AS (a*abs(b)),
   e TEXT AS (substr(c,b,b+1)) STORED
);

2.1. VIRTUAL 与 STORED 列

生成列可以是 VIRTUAL 或 STORED。VIRTUAL 列的值在读取时计算，而 STORED 列的值在写入行时计算。STORED 列占用数据库文件中的空间，而 VIRTUAL 列在读取时 则需要更多的 CPU 周期。

从 SQL 的角度来看，STORED 和 VIRTUAL 列几乎完全相同。针对任一类生成列的查询 都会产生相同的结果。唯一的功能差异是无法通过 ALTER TABLE ADD COLUMN 命令添加新的 STORED 列。只能使用 ALTER TABLE 添加 VIRTUAL 列。

2.2. 功能

  1. 生成列可以具有数据类型。SQLite 会尝试使用与普通列相同的 亲和力 规则将生成表达式的结果转换为该数据类型。

  2. 生成列可以像普通列一样具有 NOT NULL、CHECK 和 UNIQUE 约束，以及外键约束。

  3. 生成列可以像普通列一样参与索引。

  4. 生成列的表达式可以引用表中声明的任何其他列，包括其他生成列， 只要该表达式不直接或间接引用回自身。

  5. 生成列可以出现在表定义的任意位置。生成列可以与普通列混合出现。不必像上面 示例所示那样将生成列放在表定义列列表的末尾。

2.3. 限制

  1. 生成列不能有默认值（不能使用“DEFAULT”子句）。生成列的值始终由紧随“AS”关键字之后的表达式指定。

  2. 生成列不能作为主键的一部分。 （未来版本的 SQLite 可能会对 STORED 列放宽此限制。）

  3. 生成列的表达式只能引用常量字面值和同一行内的列，并且只能使用标量 确定性函数。表达式不得使用子查询、聚合函数、窗口函数或 表值函数。

  4. 生成列的表达式可以引用同一行中的其他生成列，但任何生成列都不能直接或间接地依赖自身。

  5. 生成列的表达式不得直接引用ROWID，但可以引用 INTEGER PRIMARY KEY 列，该列通常与 ROWID 相同。

  6. 每个表必须至少有一个非生成列。

  7. 无法通过ALTER TABLE ADD COLUMN 添加 STORED 列。 不过可以添加 VIRTUAL 列。

  8. 生成列的数据类型和排序序列仅由列定义中的数据类型和 COLLATE 子句决定。GENERATED ALWAYS AS 表达式的数据类型 和排序序列不会影响列本身的数据类型和排序序列。

  9. 生成列不包含在 PRAGMA table_info 语句提供的列列表中。但它们包含在更新的 PRAGMA table_xinfo 语句的输出中。

3. 兼容性

生成列支持始于 SQLite 3.31.0 版本（2020-01-22）。如果早期版本的 SQLite 尝试读取 包含生成列模式的数据库文件，那么早期版本会将生成列语法识别为错误，并报告数据库模式损坏。

说明一下：SQLite 3.31.0 版本可以读取和写入由任何早期版本（追溯至 SQLite 3.0.0（2004-06-18））创建的任何数据库。而早期版本的 SQLite（在 3.31.0 之前） 也可以读取和写入由 3.31.0 及以后版本创建的数据库，只要该数据库模式不包含 早期版本无法理解的特性，例如生成列。只有当你使用 3.31.0 或更高版本创建一个包含 生成列的新数据库，然后尝试使用不理解生成列的早期版本的 SQLite 读取或写入该数据库文件时，才会出现问题。
