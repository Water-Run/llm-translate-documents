创建虚拟表

create-virtual-table-stmt:

虚拟表是指向外部存储或计算引擎的接口，它看起来像一个表，但实际上并不在数据库文件中存储信息。

通常，你可以对虚拟表执行和普通表相同的任何操作，只是不能在虚拟表上创建索引或触发器。某些虚拟表的实现可能会施加额外限制，例如许多虚拟表是只读的。

module-name 是实现虚拟表的对象的名称。在发出 CREATE VIRTUAL TABLE 语句之前，必须使用 sqlite3_create_module() 或 sqlite3_create_module_v2() 将 module-name 注册到 SQLite 数据库连接中。该模块接受零个或多个用逗号分隔的参数。这些参数几乎可以是任何文本，只要括号配对即可。参数语法足够通用，可以让这些参数在传统的 CREATE TABLE 语句中看起来像 列定义。SQLite 会将模块参数直接传递给模块实现的 xCreate 和 xConnect 方法，而不会进行任何解释。解析和解释参数是模块实现的责任。

虚拟表通过普通的 DROP TABLE 语句销毁。不存在 DROP VIRTUAL TABLE 语句。
