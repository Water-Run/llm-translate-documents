Base85() SQL 函数

1. 概述

base85() 函数是作为 SQLite 的可加载扩展实现的 SQL 函数。该函数将二进制 BLOB 转换为等效的 base-85 文本编码，或将该 base-85 文本编码转换回 BLOB。

base85() 函数不是 SQLite 的标准组成部分。它必须作为单独的扩展加载。base85() 的源代码位于 SQLite 源树的base85.c 源文件中，即ext/misc/ 文件夹。

base85() 函数未包含在 SQLite 库的标准构建中，但在CLI中默认加载。这是CLI的典型行为，它加载的扩展超出了标准 SQLite 库所提供的内容。

与base64() 表示类似，base85() 函数可用于通过任何合理的 ASCII 通道格式化二进制内容而不受干扰。它在 CSV 中表现良好，也可以写成 TCL 花括号括起来的字面量或 SQL 字符串字面量。不适合直接用于类似 XML 的文档。

2. 编码

所使用的编码类似 Ascii85，但由作者（Larry Brasfield）在 1984 年左右在 VAX 主机上开发，早于 Mozilla、Adobe、ZMODEM 或其他 Ascii85 变体来源。此外，这还是一个独立实现的 base85 系统。因此，该作者已将其合理地置于公共领域。

Base85 数字从 7 位 USASCII 码集中选取，按编码顺序排除控制字符和空格 ! " ' ( ) { | } ~ Del，表示数字值 0 到 84（十进制）。

4 字节一组，按大端 32 位值解释，表示为 5 位 base85 数字（最高位到最低位顺序）。1-3 字节时表示为 2-4 位数字，仍为大端但表示 8-24 位值。（使用大端可最方便地过渡到小于 4 字节的字节组。该字节组也可视为 base-256 数字。）0 字节组用 0 位表示，反之亦然。不使用填充字符；编码的 base85 数字序列（即“组”）长度与解码后的二进制长度一一对应。

任何不在 base85 数字集内的字符都用于分隔组。当 base85 被流式传输或存储在不定大小容器中时，使用换行符将其拆分为不超过 80 位的子序列，以便可以使用 fgets() 读取。

3. 特性

  1. base85() 函数始终接受单个参数，必须是 TEXT（用于 base85 到二进制转换）或 BLOB（用于二进制到 base85 转换）。

  2. 如果 base85() 的参数是 BLOB，则返回值为上述编码所描述的 TEXT。

  3. 如果 base85() 的参数是上述所述的 TEXT，则返回值为与该 base85 文本对应的二进制 BLOB。

  4. 如果 base85() 的参数不是 TEXT 或 BLOB，则会引发错误。
