CSV 虚拟表

1. 概述

CSV 虚拟表读取 RFC 4180 格式的逗号分隔值，并将其内容当作 SQL 表的行和列返回。

CSV 虚拟表对需要批量加载大量逗号分隔值内容的应用程序非常有用。 CSV 虚拟表也可作为实现其他虚拟表的模板源文件。

CSV 虚拟表未内置于 SQLite 混合体中。 它可通过 单独的源文件 提供，可编译为可加载扩展。 从命令行 shell 使用 CSV 虚拟表的典型示例可能如下所示： 

.load ./csv
CREATE VIRTUAL TABLE temp.t1 USING csv(filename='thefile.csv');
SELECT * FROM t1;

上面脚本的第一行让命令行 shell读取并激活用于 CSV 的运行时可加载扩展。对于应用程序，等效的 C 语言 API 为 sqlite3_load_extension()。 请注意扩展文件名中省略了文件扩展名（例如 ".dll"、".so" 或 ".dylib"）。 省略文件扩展名并非必须，但有助于使脚本跨平台。SQLite 会自动追加适当的扩展名。

上面的第二行创建了名为 "t1" 的虚拟表，用于读取参数中指定的文件内容。列的数量和名称通过读取内容的第一行自动确定。 CSV 虚拟表的其他选项允许从字符串而非单独文件获取 CSV 内容，并让程序员更好地控制列的数量和名称。 这些选项在下文中详细说明。CSV 虚拟表通常创建为 TEMP 表，使其仅在当前数据库连接中存在，而不会成为数据库模式的永久部分。 请注意 SQLite 中不存在 "CREATE TEMP VIRTUAL TABLE" 命令。 相反，应在虚拟表名称前加上 "temp." 模式前缀。

示例的第三行展示了如何使用虚拟表来读取 CSV 文件的所有内容。 这可能是虚拟表最简单的用法。CSV 虚拟表可以像普通虚拟表一样在任何地方使用。 可以在子查询中使用 CSV 虚拟表，或使用公用表表达式，也可以根据需要添加 WHERE、GROUP BY、HAVING、ORDER BY 和 LIMIT 子句。

2. 参数

上面的示例展示了 CSV 虚拟表的单个 filename='thefile.csv' 参数。但也可以使用其他参数。

  • filename=FILENAME

    filename= 参数指定了 CSV 内容所在的外部文件。每个 CSV 虚拟表必须同时具有 filename= 参数或 data= 参数中的一个，且不能两者兼有。

  • data=TEXT

    data= 参数指定 TEXT 是 CSV 文件的字面内容。

  • schema=SCHEMA

    schema= 参数指定一个CREATE TABLE 语句，CSV 虚拟表将该语句传递给 sqlite3_declare_vtab() 接口，以定义虚拟表中各列的名称。

  • columns=N

    columns=N 参数指定 CSV 文件中的列数。 如果输入数据包含多于此数量的列，则多余列将被忽略。若输入数据包含列数少于指定值，则会用 NULL 填充额外的列。 如果省略 columns=N 参数，则会读取 CSV 文件的第一行，以确定列数。

  • header=BOOLEAN
    或仅
    header

    如果 header 参数为 true，则 CSV 文件的第一行将被视为表头而非数据。CSV 文件的第二行将成为第一行数据。 如果省略 schema= 选项，那么 CSV 文件的第一行会决定列名。

3. 列名

虚拟表的列名主要由 schema= 参数决定。 如果省略了 schema= 参数，但 header 为 true，则 CSV 文件第一行中的值将成为列名。 如果省略了 schema= 参数且 header 为 false，则列名依次为 “c0”、“c1”、“c2”，依此类推。
