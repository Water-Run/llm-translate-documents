sqldiff.exe：数据库差异实用程序

1. 使用方法

sqldiff.exe 二进制文件是一个命令行实用程序，用于显示 SQLite 数据库之间的内容差异。示例用法： 

sqldiff [options] database1.sqlite database2.sqlite

常见的输出是一个将 database1.sqlite（“源”数据库）转换为 database2.sqlite（“目标”数据库）所需的 SQL 脚本。此行为可以通过命令行开关进行调整：

--changeset FILE

不要将更改写入标准输出。相反，将一个（二进制）更改集文件写入 FILE。可以使用 SQLite 的 会话扩展 来解释该更改集。

--lib LIBRARY
-L LIBRARY

在计算差异之前，将共享库或 DLL 文件 LIBRARY 加载到 SQLite 中。这可以用于添加架构所需的应用程序定义的 排序序列。

--primarykey

使用架构定义的 PRIMARY KEY 而不是 rowid 来配对源和目标数据库中的行。（下文会有更多说明。）

--schema

只显示架构中的列名和表差异，而不显示表内容。

--summary

显示每个表更改了多少行，但不显示实际更改。

--table TABLE

仅显示 TABLE 的内容差异，而不是整个数据库的差异。

--transaction

将 SQL 输出包装在一个大事务中。

--vtab

添加对处理 FTS3、FTS5 和 rtree 虚拟表的支持。详见下文。

2. 工作原理

sqldiff.exe 实用程序通过在源和目标中查找逻辑“配对”行来工作。默认行为是如果两个行所属的表同名，且它们具有相同的 rowid，或者在 WITHOUT ROWID 表的情况下具有相同的 PRIMARY KEY，则将它们视为配对。配对行内容的任何差异会输出为 UPDATE。无法配对的源数据库中的行会输出为 DELETE。无法配对的目标数据库中的行会输出为 INSERT。

--primarykey 标志会稍微修改配对算法，使得架构声明的 PRIMARY KEY 始终用于配对，即使在具有 rowid 的表上也是如此。通常这是查找差异的更好选择，但在某些 PRIMARY KEY 列设置为 NULL 的行的情况下可能会遗漏差异。

3. 限制

  1. sqldiff.exe 实用程序不会为以下任一情况计算更改集：rowid 无法访问的 rowid 表；或者没有显式主键的表。启用 --changeset 选项时，sqldiff 会将这些表从比较中忽略。此类表的示例有：

    CREATE TABLE NilChangeset (
   -- inaccessible rowid due to hiding its aliases
   "rowid" TEXT,
   "oid" TEXT,
   "_rowid_" TEXT
);
    和 
    CREATE TABLE NilChangeset (
   -- no explicit primary key
   "authorId" TEXT,
   "bookId" TEXT
);

    当 sqldiff 专门对这些表进行比较时，不会发生错误。但结果可能出乎意料。例如，以下调用：

    sqldiff --changeset CHANGESET_OUT --table NilChangeset db1.sdb db2.sdb

    将产生一个名为“CHANGESET_OUT”的空文件。有关详细信息，请参见 会话限制。

  2. sqldiff.exe 实用程序（当前）不会显示 TRIGGER 或 VIEW 的差异。

  3. sqldiff 实用程序并未设计用于支持模式迁移，并且在列定义不同的情况下表现出容忍。在比较内容之前，通常只比较同名表的列名及其顺序。

    不过，可以使用名称为“sqlite_schema”的单表比较选项来显示或检测两个数据库之间的详细架构差异。在这样做时，输出不应直接用于修改数据库。

  4. 默认情况下，不会报告虚拟表的架构或内容差异。

    然而，如果某个 虚拟表 实现通过在数据库内创建真实表（有时称为“影子”表）来存储数据，那么 sqldiff.exe 确实会计算这些表之间的差异。如果随后在与源数据库不 完全 相同的数据库上运行生成的 SQL 脚本，可能会出现意想不到的影响。对于 SQLite 附带的多个虚拟表（FTS3、FTS5、rtree 等），这些意外影响可能包括虚拟表内容的损坏。

    如果向 sqldiff.exe 传递 --vtab 选项，则会忽略属于 FTS3、FTS5 或 rtree 虚拟表的所有底层影子表，而直接包含虚拟表之间的差异。
