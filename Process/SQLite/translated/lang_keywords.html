SQLite Keywords

SQL 标准指定了大量关键字，这些关键字不能用作表、索引、列、数据库、用户定义函数、排序规则、虚拟表模块或任何其他具名对象的名称。关键字列表非常长，很少有人能记住全部。对于大多数 SQL 代码，最安全的做法是永远不要使用任何英语单词作为用户定义对象的名称。

如果你想把关键字用作名称，就必须给它加上引号。SQLite 中有四种引用关键字的方式：

'keyword' 带有单引号的关键字是字符串字面量。
"keyword" 带有双引号的关键字是标识符。
[keyword] 用方括号括起来的关键字是标识符。这不是标准 SQL。这个引用机制用于 MS Access 和 SQL Server，并被包含在 SQLite 中以确保兼容性。
`keyword` 用反引号（ASCII 码 96）括起来的关键字是标识符。这不是标准 SQL。这个引用机制用于 MySQL，并被包含在 SQLite 中以确保兼容性。

为了在处理历史 SQL 语句时更具弹性，SQLite 有时会突破上述引用规则：

  • 如果在允许使用标识符但不允许使用字符串字面量的上下文中使用带单引号的关键字（例如：'key' 或 'glob'），那么该标记会被理解为标识符，而非字符串字面量。

  • 如果在无法解析为标识符但允许字符串字面量的上下文中使用带双引号的关键字（例如："key" 或 "glob"），那么该标记会被理解为字符串字面量，而非标识符。

我们提醒程序员不要使用前面两条例外中描述的情况。我们强调它们仅存在于让旧的、不规范的 SQL 语句能够正确运行。未来版本的 SQLite 可能会对上述被例外接受的不规范语句抛出错误。

SQLite 会在引入新特性时不时添加新关键字。因此，为了防止未来增强破坏你的代码，即使没有必要，也应该通常给任何英语单词的标识符加上引号。
