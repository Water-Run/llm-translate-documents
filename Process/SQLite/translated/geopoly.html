1. 概述

Geopoly 模块是 R-Tree 扩展 的替代接口，使用 GeoJSON 表示法 （RFC-7946）来描述二维 多边形。Geopoly 包含用于检测一个多边形是否被包含在另一个多边形内或与之重叠、 计算多边形所包围的面积、对多边形执行线性变换、 将多边形渲染为 SVG，以及其他 类似操作的函数。

Geopoly 的源代码包含在 合并文件 中。然而， 根据配置选项和你使用的具体 SQLite 版本，Geopoly 扩展 可能默认启用，也可能未启用。为了确保在你的构建中启用 Geopoly， 请添加 -DSQLITE_ENABLE_GEOPOLY=1 编译时选项。

Geopoly 作用于“简单”多边形——即边界不会自相交的多边形。Geopoly 因此 扩展了 R-Tree 扩展 的功能，后者只能处理矩形区域。 另一方面，R-Tree 扩展 可以处理 1 到 5 维坐标，而 Geopoly 则仅限于二维形状。

Geopoly 模块中的每个多边形都可以关联任意数量的辅助数据字段。

1.1. GeoJSON

GeoJSON 标准是一套使用 JSON 交换 地理空间信息的语法。GeoJSON 是一个丰富的标准， 几乎可以描述任何类型的地理空间内容。

Geopoly 模块只理解 GeoJSON 的一个小子集，但却是关键的子集。 特别是，GeoJSON 理解描述简单多边形的顶点 JSON 数组。

多边形由其顶点定义。 每个顶点是一个包含两个数值的 JSON 数组，分别代表该顶点的 X 和 Y 坐标。 一个多边形是至少四个这样顶点的 JSON 数组， 因此是数组中的数组。 数组中的第一个和最后一个顶点必须相同。 多边形遵循右手法则：当从一个顶点追踪到下一个顶点时，线右侧的区域属于外部， 而线左侧的区域属于多边形内部。 换句话说，顶点的净旋转方向是逆时针。

例如，下列 JSON 描述了一个等腰三角形，其位于 X 轴上，面积为 0.5： 

[[0,0],[1,0],[0.5,1],[0,0]]

三角形有三个顶点，但该三角形的 GeoJSON 描述包含四个顶点，因为第一和最后一个顶点是重复的。

1.2. 二进制存储格式

在内部，Geopoly 将多边形以二进制格式（SQL BLOB）存储。下文提供了二进制格式的细节。 所有 Geopoly 接口都可以接受 GeoJSON 格式或二进制格式的多边形。

2. 使用 Geopoly 扩展

Geopoly 表按如下方式创建： 

CREATE VIRTUAL TABLE newtab USING geopoly(a,b,c);

上述语句创建一个名为 “newtab” 的 newtab geopoly 表。 每个 geopoly 表都包含一个内建的整数 “rowid” 列 以及一个 “_shape” 列，该列存储与该行对应的多边形。 上述示例还定义了三个辅助数据列 “a”、“b” 和 “c”，它们可以存储应用需要与每个多边形关联的额外 信息。如果无需存储辅助信息，则可以省略辅助列列表。

使用普通的 INSERT 语句将新多边形存入表： 

INSERT INTO newtab(_shape) VALUES('[[0,0],[1,0],[0.5,1],[0,0]]');

UPDATE 和 DELETE 语句的工作方式类似。

2.1. 查询

要使用索引的地理空间搜索查询 geopoly 表， 在 WHERE 子句中将 geopoly_overlap() 或 geopoly_within() 作为布尔函数使用， 并将 “_shape” 列作为函数的第一个参数。 例如： 

SELECT * FROM newtab WHERE geopoly_overlap(_shape, $query_polygon);

上述示例会返回所有与 $query_polygon 参数中的多边形重叠的 _shape 行。geopoly_within() 函数的工作方式类似， 但仅返回 _shape 完全包含在 $query_polygon 内的行。

在 WHERE 子句中包含裸露的 geopoly_overlap() 或 geopoly_within() 函数的查询（以及 DELETE 和 UPDATE 语句） 会利用底层的 R*Tree 数据结构进行快速查找，只需检查表中的部分行。 查找的行数当然取决于 $query_polygon 的大小。 较大的 $query_polygon 通常需要检查比小的多更多的行。

针对 geopoly 表的 rowid 的查询也非常快速，即便表有大量行。 然而，辅助数据列都不是索引，因此针对辅助数据列的查询将涉及全表扫描。

3. 特殊函数

Geopoly 模块定义了一些新 SQL 函数，可用于处理多边形。所有传递给这些函数的多边形参数都可以是 GeoJSON 格式或内部二进制格式。

3.1. geopoly_overlap(P1,P2) 函数

如果 P1 和 P2 都是多边形，那么 geopoly_overlap(P1,P2) 函数在 P1 与 P2 有任意重叠时返回非零整数， 如果 P1 与 P2 完全不相交则返回零。如果 P1 或 P2 不是多边形，该例程返回 NULL。

geopoly_overlap(P1,P2) 函数的特殊之处在于 geopoly 虚拟表知道如何使用 R*Tree 索引来优化 WHERE 子句中 将 geopoly_overlap() 作为布尔函数的查询。只有 geopoly_overlap(P1,P2) 和 geopoly_within(P1,P2) 函数具有这一能力。

3.2. geopoly_within(P1,P2) 函数

如果 P1 和 P2 都是多边形，那么 geopoly_within(P1,P2) 函数在 P1 完全包含在 P2 内时返回非零整数， 如果 P1 的任意部分位于 P2 之外则返回零。如果 P1 和 P2 是同一个多边形，该例程返回非零。 如果 P1 或 P2 不是多边形，该例程返回 NULL。

geopoly_within(P1,P2) 函数的特殊之处在于 geopoly 虚拟表知道如何使用 R*Tree 索引来优化 WHERE 子句中 将 geopoly_within() 作为布尔函数的查询。只有 geopoly_within(P1,P2) 和 geopoly_overlap(P1,P2) 函数具有这一能力。

3.3. geopoly_area(P) 函数

如果 P 是多边形，那么 geopoly_area(P) 返回该多边形所包围的面积。如果 P 不是多边形，geopoly_area(P) 返回 NULL。

3.4. geopoly_blob(P) 函数

如果 P 是多边形，那么 geopoly_blob(P) 返回该多边形的二进制编码（BLOB）。 如果 P 不是多边形，geopoly_blob(P) 返回 NULL。

3.5. geopoly_json(P) 函数

如果 P 是多边形，那么 geopoly_json(P) 返回该多边形的 GeoJSON 表示（TEXT 字符串）。 如果 P 不是多边形，geopoly_json(P) 返回 NULL。

3.6. geopoly_svg(P,...) 函数

如果 P 是多边形，那么 geopoly_svg(P,...) 返回一个文本字符串，该字符串是该多边形的 可缩放矢量图形（SVG） 表示。如果存在多个参数，则第二个及之后的参数会作为属性添加到每个 SVG 图元上。例如： 

SELECT geopoly_svg($polygon,'class="poly"','style="fill:blue;"');

如果 P 不是多边形，geopoly_svg(P,...) 返回 NULL。

请注意，geopoly 使用传统的右手笛卡尔坐标系，原点在左下角，而 SVG 使用左手坐标系，原点在左上角。 geopoly_svg() 例程不会尝试转换坐标系，因此显示的图像会镜像并旋转。如果这不可接受， 可以在将多边形传递给 geopoly_svg() 之前使用 geopoly_xform() 例程将输出从笛卡尔坐标转换为 SVG 坐标。

3.7. geopoly_bbox(P) 与 geopoly_group_bbox(P) 函数

如果 P 是多边形，那么 geopoly_bbox(P) 返回一个新的多边形，该多边形是完全包围 P 的最小（轴对齐）矩形。 如果 P 不是多边形，geopoly_bbox(P) 返回 NULL。

geopoly_group_bbox(P) 函数是 geopoly_bbox(P) 的聚合版本。 geopoly_group_bbox(P) 函数返回聚合期间出现的所有 P 值所能包围的最小矩形。

3.8. geopoly_contains_point(P,X,Y) 函数

如果 P 是多边形，那么 geopoly_contains_point(P,X,Y) 当且仅当坐标 X,Y 在多边形 P 内部或边界上时返回非零整数。 如果 P 不是多边形，geopoly_contains_point(P,X,Y) 返回 NULL。

3.9. geopoly_xform(P,A,B,C,D,E,F) 函数

geopoly_xform(P,A,B,C,D,E,F) 函数返回一个新的多边形，该多边形是 P 的仿射变换， 变换由 A,B,C,D,E,F 定义。如果 P 不是有效多边形，该例程返回 NULL。

变换根据以下公式转换多边形的每个顶点： 

x1 = A*x0 + B*y0 + E
y1 = C*x0 + D*y0 + F

因此，例如，如果想在不改变形状的情况下平移多边形 DX、DY，可使用： 

geopoly_xform($polygon, 1, 0, 0, 1, $DX, $DY)

要围绕点 0,0 旋转多边形 R 弧度： 

geopoly_xform($polygon, cos($R), sin($R), -sin($R), cos($R), 0, 0)

请注意，翻转多边形的变换可能会导致顶点顺序反转。 换句话说，该变换可能使顶点顺序由逆时针变为顺时针。 这可以在变换后将结果传入 geopoly_ccw() 函数来修正。

3.10. geopoly_regular(X,Y,R,N) 函数

geopoly_regular(X,Y,R,N) 函数返回一个凸的、简单的、正则的、 等边等角的 N 边形，该多边形以 X,Y 为中心，外接圆半径为 R。 如果 R 为负或 N 小于 3，则函数返回 NULL。N 的值被限制为 1000， 因此即使 N 值超过 1000，该例程也永远不会绘制超过 1000 条边的多边形。

例如，下面的图形：

由下列脚本生成： 

SELECT '<svg width="600" height="300">';
WITH t1(x,y,n,color) AS (VALUES
   (100,100,3,'red'),
   (200,100,4,'orange'),
   (300,100,5,'green'),
   (400,100,6,'blue'),
   (500,100,7,'purple'),
   (100,200,8,'red'),
   (200,200,10,'orange'),
   (300,200,12,'green'),
   (400,200,16,'blue'),
   (500,200,20,'purple')
)
SELECT
   geopoly_svg(geopoly_regular(x,y,40,n),
        printf('style="fill:none;stroke:%s;stroke-width:2"',color))
   || printf(' <text x="%d" y="%d" alignment-baseline="central" text-anchor="middle">%d</text>',x,y+6,n)
  FROM t1;
SELECT '</svg>';

3.11. geopoly_ccw(J) 函数

geopoly_ccw(J) 函数返回具有逆时针（CCW）旋转的多边形 J。

RFC-7946 要求多边形使用 CCW 旋转。 但规范也指出许多旧版 GeoJSON 文件没有遵循规范， 并包含顺时针（CW）旋转的多边形。geopoly_ccw() 函数对正在读取旧版 GeoJSON 脚本的应用非常有用。 如果传入 geopoly_ccw() 的输入是格式正确的多边形，函数不会做任何更改。 然而，如果输入多边形的循环顺序相反，则 geopoly_ccw() 会反转循环顺序，使其符合规范， 并且可以与 Geopoly 模块一起正常工作。

4. 实现细节

Geopoly 模块是对 R-Tree 扩展 的扩展。Geopoly 使用与 R-Tree 扩展 相同的底层逻辑和影子表。 Geopoly 仅仅呈现了一个不同的接口，并提供了一些额外的逻辑 用于计算多边形的解码、重叠和包含关系。

4.1. 多边形的二进制编码

Geopoly 在内部使用二进制格式存储所有多边形。一个二进制多边形由 4 字节的头部 后跟一组坐标对组成，每个坐标的每个维度都是 32 位浮点数。

头部的第一个字节是标志字节。标志字节的最低有效位决定头部后面 的坐标对是以大端还是小端存储。最低有效位为 0 表示大端，为 1 表示小端。 头部第一个字节的其他位保留供将来扩展使用。

头部的接下来的三个字节记录多边形中顶点的数量，作为一个大端整数。 因此，每个多边形的顶点数量上限约为 1600 万。

头部之后是坐标对数组。每个坐标都是 32 位浮点数。 坐标使用 32 位浮点值意味着地表上的任何点都可以用约 2.5 米的分辨率进行映射。 如果地图限于单个大陆或国家，当然可以实现更高分辨率。 请注意，geopoly 模块中坐标的分辨率与地球表面由于潮汐力引起的日常移动量 在数量级上相似。

二进制格式中的坐标列表不包含重复项。 最后一个坐标不像 GeoJSON 那样重复第一个坐标。 因此，二进制表示中的坐标对数量总是比 GeoJSON 表示少一项。

4.2. 影子表

Geopoly 模块基于 R-Tree 扩展 构建，并使用 相同的底层影子表和算法。对于索引目的，每个多边形在影子表中表示为一个矩形边界框。 底层的 R-Tree 实现使用边界框来限制搜索空间。然后 geoploy_overlap() 和/或 geopoly_within() 例程进一步 将搜索缩小到确切答案。
