1. 概述

SQLite 的 SQL 语言解析器是使用一个名为“Lemon”的代码生成程序生成的。Lemon 程序读取输入语言的语法，并输出实现该语言解析器的 C 代码。

1.1. Lemon 源文件及文档

Lemon 没有自己的源代码仓库。Lemon 只是 SQLite 源树中的几个文件：

  • lemon.html → Lemon 原始的详细使用说明文档和程序员参考。

  • lemon.c → 阅读语法文件并生成相应解析器 C 代码的实用程序的源代码。

  • lempar.c → 生成的解析器 C 代码的模板。Lemon 实用程序读取该模板并插入额外代码以生成解析器。

2. Lemon 优势

Lemon 生成 LALR(1) 解析器。它的运行方式类似于更熟悉的工具 Yacc 和 Bison，但 Lemon 添加了重要改进，包括：

  • Lemon 的语法比 Yacc 的更不容易出错——使用语义值的符号名称，而不是 Yacc 那样的“$1”式位置表示法。

  • 在 Lemon 中，分词器调用解析器。Yacc 则是相反，由解析器调用分词器。Lemon 的方式是可重入且线程安全的，而 Yacc 使用全局变量，因此两者都不是。由于某些 SQL 语句会递归调用解析器（例如在解析 CREATE TABLE 语句时，SQLite 会递归调用解析器以生成 INSERT 语句来在 sqlite_schema 表中插入新条目），可重入性对 SQLite 尤其重要。

  • Lemon 有“非终结符析构器”的概念，可在语法错误或其他中止解析之后用来回收内存或其他资源。

2.1. SQLite 中的 Lemon 用法

SQLite 中在两个地方使用 Lemon。

Lemon 的主要用途是创建 SQL 语言解析器。一个语法文件 (parse.y) 被 Lemon 编译成 parse.c 和 parse.h。parse.c 文件在不做进一步修改的情况下并入 合并包 中。

Lemon 还用于为 FTS5 扩展中的查询模式表达式生成解析器。在这种情况下，输入语法文件是 fts5parse.y。

2.2. Lemon 为 SQLite 的定制

将代码生成工具包含在项目中带来的一个优势是，这些工具可以针对整个项目的特定需求进行优化。Lemon 就享受到了这种效果。多年来，Lemon 解析器生成器不断扩展和增强，为 SQLite 提供了新功能与性能提升。为 SQLite 专门设计的 Lemon 特定增强包括：

  • Lemon 具有“回退”令牌的概念。SQL 语言包含大量关键字，这些关键字可能与标识符名称发生冲突。Lemon 能够指定一些关键字可以“回退”为标识符。如果关键字在输入令牌流中的上下文本会导致语法错误，令牌就会在引发语法错误之前自动转换为其回退形式。此功能允许解析器对作为标识符使用的保留字表现出极大的容忍度，而这在 SQL 语言中是一个经常出现的问题。

  • 为了配合 SQLite 的 100% MC/DC 测试 目标，由 Lemon 生成的解析器代码没有不可达的分支，并包含额外的（编译时选择的）可用于测量测试覆盖率的插桩。

  • Lemon 支持语法文件规则的条件编译，从而根据编译时选项生成不同的解析器。

  • 作为一项性能优化，Lemon 输入语法中的归约动作允许包含形式为 “/*A-overwrites-Z*/” 的注释，以指示规则右侧的语义值 “A” 可以直接覆盖左侧的语义值 “Z”。这项简单的优化减少了用于解析输入语法的下推自动机中的栈操作次数，从而提升了解析器的性能，也使生成的代码略微变小。

SQL 语句的解析是任何 SQL 数据库引擎中 CPU 周期的主要消耗来源。不断优化 SQLite 的努力促使开发者大量时间用于微调 Lemon 以生成更快的解析器。这些努力不仅惠及 SQLite，也对所有 Lemon 解析器生成器用户有利。但如果 Lemon 是一个单独维护的工具，就更难协调地同时修改 SQLite 和 Lemon，因而实现的优化也会更少。因此，将解析器生成工具包含在 SQLite 的源树中，对于该工具本身和 SQLite 都被证明是一个净收益。

3. Lemon 历史

Lemon 最初由 D. Richard Hipp（同时也是 SQLite 的创建者）在 1987 年到 1992 年间于杜克大学攻读研究生期间编写。Lemon 的最初创建日期已无从考证，但很可能是在 1990 年前后。Lemon 生成 LALR(1) 解析器。曾有一个名为 “Lime” 的配套 LL(1) 解析器生成工具，但 Lime 的源代码已经丢失。

Lemon 的源代码最初写成多个独立的源文件，后来才合并成一个名为 “lemon.c” 的源文件。

Lemon 和 SQLite 的作者 Hipp 报告称，他在学习 John Ousterhout 的 Tcl 原始源代码后 C 编程技能得到了极大提升。Hipp 在 1993 年发现并研究 Tcl。Lemon 编写于此之前，SQLite 编写于此之后。这两个产品在编码风格上存在明显差异，SQLite 看起来更加清晰、易读且更易维护。
