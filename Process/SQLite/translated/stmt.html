SQLITE_STMT 虚拟表

1. 概述

SQLITE_STMT 扩展实现了一个仅同名虚拟表，用于提供与数据库连接关联的所有已预编译语句的信息。

SQLITE_STMT 扩展包含在合并源中，但默认情况被禁用。使用SQLITE_ENABLE_STMTVTAB编译时选项即可启用 SQLITE_STMT 扩展。 SQLITE_STMT 扩展也可以在运行时加载，方法是使用位于 https://sqlite.org/src/file/ext/misc/stmt.c 的源代码将扩展编译为共享库或 DLL，并按照可加载扩展的编译说明进行操作。

在命令行 shell的默认构建中，SQLITE_STMT 扩展是启用的。

2. 用法

SQLITE_STMT 虚拟表是一个只读表，可直接查询以获取当前数据库连接上所有已预编译语句的信息。例如： 

SELECT * FROM sqlite_stmt;

如上所示的语句可以在调用sqlite3_close()之前立即运行，以确认所有已预编译语句都已被终结，并帮助识别和追踪那些“泄漏”而未被终结的语句。

SQLITE_STMT 虚拟表还可用于访问有关已预编译语句的性能信息，以便优化应用程序。 例如，要确定从未运行过的已预编译语句正在使用多少内存，可以运行： 

SELECT sum(mem) FROM sqlite_stmt WHERE run=0;

2.1. 列

SQLITE_STMT 虚拟表提供的列可由下面展示的假想 CREATE TABLE 语句概括： 

CREATE TABLE sqlite_stmt(
  sql    TEXT,    -- 原始 SQL 文本
  ncol   INT,     -- 输出列数
  ro     BOOLEAN, -- 对于“只读”语句为真
  busy   BOOLEAN, -- 语句当前是否正在运行
  nscan  INT,     -- 全表扫描步骤次数
  nsort  INT,     -- 排序操作次数
  naidx  INT,     -- 自动索引插入次数
  nstep  INT,     -- 字节码引擎步骤次数
  reprep INT,     -- 重新准备操作次数
  run    INT,     -- 语句运行次数
  mem    INT      -- 该语句使用的堆内存
);

未来版本可能会添加新的输出列，也可能更改旧列的顺序。 下面提供了 SQLITE_STMT 虚拟表中每一列含义的详细说明：

  • sql： 预编译语句的原始 SQL 文本。如果该语句是通过sqlite3_prepare() 接口编译的，则 SQL 文本可能未被保存，此时该列为 NULL。

  • ncol： 查询结果集中的列数。对于 DML 语句，该列值为 0。

  • ro： “只读”列。如果 SQL 语句是查询，则该列为真（非零）；如果是 DML 语句，则为假（零）。

  • busy： 如果预编译语句当前正在运行，则本字段为真。换句话说，如果已对该预编译语句调用过至少一次sqlite3_step()，但尚未调用sqlite3_reset()对其进行重置，则本字段为真。

  • nscan： 该字段表示字节码引擎作为全表扫描过程访问表的次数。该字段值较大可能表示可以通过添加索引来提升性能。该字段等价于SQLITE_STMTSTATUS_FULLSCAN_STEP。

  • nsort： 该字段表示字节码引擎进行排序操作的次数。该字段值为正可能表示可以通过增加索引使查询结果自然按期望顺序出现，从而提升性能。 该字段等价于SQLITE_STMTSTATUS_SORT。

  • naidx： 该字段表示已插入自动索引的行数。该字段值为正可能表示可以创建一个命名索引来替代自动索引，以改善性能。 该字段等价于SQLITE_STMTSTATUS_AUTOINDEX。

  • nstep： 该字段表示为该预编译语句执行的字节码引擎操作次数。该字段可用作语句所用 CPU 时间的近似指标。 该字段等价于SQLITE_STMTSTATUS_VM_STEP。

  • reprep： 该字段表示由于模式更改或参数绑定更改而需要重新准备语句的次数。 该字段等价于SQLITE_STMTSTATUS_REPREPARE。

  • run： 该字段表示语句的运行次数。 该字段等价于SQLITE_STMTSTATUS_RUN。

  • mem： 该字段表示预编译语句所占用的堆内存字节数。 该字段等价于SQLITE_STMTSTATUS_MEMUSED。
