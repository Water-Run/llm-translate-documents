关于 SQLite

SQLite 是一个进程内库，实现了一个 自包含、 无服务器、 零配置、 事务型 SQL 数据库引擎。 SQLite 的代码属于 公有领域，因此可免费用于 任何用途，无论是商业还是私人。 SQLite 是世界上部署最广泛的 数据库，应用数量多到难以计数，包括多个 高知名度项目。

SQLite 是嵌入式 SQL 数据库引擎。 与大多数其他 SQL 数据库不同，SQLite 没有单独的 服务器进程。SQLite 直接读写普通磁盘 文件。一个完整的 SQL 数据库包含多个表、索引、 触发器和视图，全部存储在一个磁盘文件中。 数据库的 文件格式 跨平台——你可以在 32 位与 64 位系统之间自由复制数据库，或在 大端 与 小端 架构之间自由复制。这些特性使得 SQLite 成为 应用文件格式 的热门选择。 SQLite 数据库文件被美国国会图书馆列为 推荐的存储格式。 与其把 SQLite 看作是 Oracle 的替代品， 不如把它当作 fopen() 的替代。

SQLite 是一个紧凑的库。 启用所有特性时，库大小 可以小于 900KiB， 具体取决于目标平台和编译器优化设置。 （64 位代码更大。而且有些编译器优化， 例如激进的函数内联与循环展开，会导致 目标代码变得更大。） 内存使用与速度之间存在权衡。 SQLite 通常在你给予更多内存时运行更快。 尽管如此，即使在内存有限的环境中， 性能通常也相当出色。根据使用方式，SQLite 可以 比直接文件系统 I/O 更快。

SQLite 在每次发布前都 经过极其严格的测试，声誉非常可靠。 大部分 SQLite 源代码专注于测试与验证。 一个自动化测试套件运行数百万个测试用例， 涵盖数亿条 SQL 语句，并达到 100% 分支覆盖。 SQLite 能优雅地应对内存分配失败与磁盘 I/O 错误。 即使在系统崩溃或断电后，事务也保持 ACID 属性。 所有这些都经过自动化测试通过模拟 系统故障的特殊测试工具验证。 当然，即便经过如此多的测试，仍然有漏洞。 但与一些类似项目（尤其是商业竞争对手）不同， SQLite 对所有漏洞保持开放与诚实，并提供 漏洞列表 与 每分钟更新的 代码变更时间线。

SQLite 的代码库由一支 国际团队 的开发者全职维护。 开发者不断扩展 SQLite 的能力， 提升其可靠性与性能，同时保持 与已发布接口规范、 SQL 语法 以及数据库 文件格式 的向后兼容性。 源代码对任何人完全免费开放， 但也提供专业支持。

SQLite 项目于 2000-05-09 启动。 未来难以预测，但开发者的目标是 支持 SQLite 至 2050 年。设计决策皆以此为参考。
