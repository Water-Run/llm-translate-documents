单文件跨平台数据库

SQLite 中的数据库是一个单独的磁盘文件¹。 Furthermore，该文件格式是跨平台的。 一个在某台机器上创建的数据库可以复制并在架构不同的另一台机器上使用。 SQLite 数据库在 32 位和 64 位机器之间移动，并且在 大端 和 小端 架构之间都是可移植的。

SQLite 数据库文件格式也很稳定。 所有 SQLite 3 的版本都可以读取和写入由最早的 SQLite 3 发布版本（版本 3.0.0） 创建的数据库文件，这可以追溯到 2004-06-18。这就是“向后兼容”。 开发者承诺对未来所有 SQLite 3 的发布保持数据库文件格式的向后兼容性。 “向前兼容”意味着旧版本的 SQLite 也可以读取和写入由更新版本创建的数据库。 SQLite 通常是向前兼容的，但并非完全。

SQLite 数据库文件格式的稳定性以及该文件格式的跨平台特性相结合，使得 SQLite 数据库文件成为一个出色的应用文件格式。 美国国会图书馆通过将 SQLite 列为推荐存储格式来认可这一点， 用于数字内容的长期保存。

注释：

  1. 临时日志文件是事务控制的一部分，但这些额外的文件并不是稳定状态下数据库的一部分。
