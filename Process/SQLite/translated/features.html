Features Of SQLite

  • 事务 是原子性的、一致的、隔离的并且持久的（ACID）， 即使在系统崩溃或断电之后也能保持。
  • 零配置 —— 不需要设置或管理。
  • 功能齐全的 SQL 实现， 拥有诸如 部分索引、 表达式索引、JSON、 公用表表达式 和 窗口函数 等高级功能。 （省略的功能）
  • 完整的数据库存储在 单个跨平台磁盘文件 中。 非常适合作为 应用程序文件格式。
  • 支持 TB 级数据库以及 GB 级字符串和二进制数据。（见 limits.html。）
  • 小巧的代码 体积： 完全配置下小于 900KiB，省略可选功能后更小。
  • 简单易用的 API。
  • 快速：在某些情况下，SQLite 比 直接文件系统 I/O 更快
  • 使用 ANSI-C 编写。包含 TCL 绑定。 各种其他语言的绑定可单独获得。
  • 源代码注释充分， 拥有 100% 分支测试覆盖率。
  • 提供 单个 ANSI-C 源代码文件， 易于编译，因此也容易 集成到更大的项目中。
  • 自包含： 无外部依赖。
  • 跨平台：Android、*BSD、iOS、Linux、Mac、Solaris、VxWorks 和 Windows（Win32、WinCE、WinRT）开箱即用并得到支持。 易于移植到其他系统。
  • 源码位于 公共领域。 可用于任何目的。
  • 包含独立的 命令行界面 （CLI）客户端，可用于管理 SQLite 数据库。

Suggested Uses For SQLite:

  • 物联网数据库。 SQLite 是手机、PDA、MP3 播放器、机顶盒和其他电子设备中常见的数据库引擎选择。 SQLite 具有小巧的代码体积，充分利用内存、磁盘空间和磁盘带宽，可靠性高， 并且无需数据库管理员进行维护。

  • 应用程序文件格式。 与其使用 fopen() 写入 XML、JSON、CSV 或某种专有格式到 应用使用的磁盘文件，不如使用 SQLite 数据库。 这样无需编写和调试解析器，数据更易访问、跨平台，而且更新具有事务性。 （更多...）

  • 网站数据库。 由于无需配置并将信息存储在普通磁盘文件中，SQLite 是支持中小型网站的常见数据库选择。

  • 企业级关系型数据库的替代品。 SQLite 常被用作企业级关系型数据库的替代，用于演示或测试。 SQLite 快速且无需设置，极大减少了测试的麻烦，使演示灵活且易于启动。
