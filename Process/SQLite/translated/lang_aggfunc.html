下面列出的聚合函数默认可用。还有两个聚合函数归类在JSON SQL 函数之中。应用程序可以通过sqlite3_create_function()接口定义自定义聚合函数。

在任何只接受单个参数的聚合函数中，该参数前可以加关键字 DISTINCT。在这种情况下，会在将值传入聚合函数前先过滤掉重复的元素。例如函数 count(distinct X) 会返回列 X 的不同值数量，而不是列 X 中非 NULL 值的总数。

如果提供了 FILTER 子句，则只有满足 expr 条件为真的行才会被包括在聚合结果中。

如果提供 ORDER BY 子句，该子句就决定聚合输入的处理顺序。对于像 max() 和 count() 这样的聚合函数，输入顺序无关紧要。但对于 string_agg() 和 json_group_object() 之类的函数，ORDER BY 会影响输出结果。如果没有指定 ORDER BY，聚合的输入将以任意顺序出现，并且不同调用之间可能会变化。

另请参阅：标量函数 和 窗口函数。

3. 内置聚合函数说明

avg(X)

avg() 函数返回组内所有非 NULL X 的平均值。不能解析为数字的字符串或 BLOB 会被视为 0。只要存在至少一个非 NULL 输入，avg() 的结果就始终为浮点值，即便所有输入都是整数；如果没有非 NULL 输入，则返回 NULL。avg() 的结果按照 total()/count() 计算，因此所有适用于 total() 的约束也同样适用于 avg()。

count(X)
count(*)

count(X) 函数返回组内 X 不为 NULL 的次数。count(*)（无参数时）返回组中行的总数。

group_concat(X)
group_concat(X,Y)
string_agg(X,Y)

group_concat() 函数返回一个字符串，它是所有非 NULL X 值的连接结果。如果提供参数 Y，则该参数作为 X 各实例之间的分隔符；省略 Y 时默认使用逗号（“,”）。

string_agg(X,Y) 是 group_concat(X,Y) 的别名。string_agg() 与 PostgreSQL 和 SQL Server 兼容，而 group_concat() 与 MySQL 兼容。

除非在最后一个参数之后紧跟 ORDER BY，否则拼接元素的顺序是任意的。

max(X)

max() 聚合函数返回组中所有值的最大值。最大值等同于同一列 ORDER BY 时最后返回的值。当且仅当组中没有非 NULL 值时，max() 才返回 NULL。

median(X)

median() 函数返回组内所有非 NULL X 的中位数。median(X) 等价于 percentile_cont(X,0.5)。有关更多信息，请参阅percentile 扩展文档。

只要大合集使用-DSQLITE_ENABLE_PERCENTILE编译，或通过可加载扩展加载，median() 函数就自 SQLite 3.51.0（2025-11-04）起可用。

min(X)

min() 聚合函数返回组中所有非 NULL 值的最小值。最小值等同于在该列上执行 ORDER BY 时排在最前面的非 NULL 值。只有组中没有非 NULL 值时，min() 才返回 NULL。

percentile(Y,P)

percentile(Y,P) 聚合函数计算一个值 X，使其大于等于 P 百分比的非 NULL 输入且小于等于剩余的值。参数 P 必须在 0.0 到 100.0 之间，且对于所有聚合项必须相同且不可为 NULL。Y 输入必须是 NULL 或数字；NULL 会被忽略，任何非 NULL 但非数字的 Y 会导致错误。

percentile(Y,P) 等价于 percentile_cont(Y,P/100.0)。详细内容请参阅percentile 扩展。

只要大合集使用-DSQLITE_ENABLE_PERCENTILE编译，或通过可加载扩展提供，percentile() 函数自 SQLite 3.51.0（2025-11-04）起可用。

percentile_cont(Y,P)

percentile_cont(Y,P) 聚合函数计算一个值 X，使其大于等于 P 分数的非 NULL 输入且小于等于 1-P 分数的输入。参数 P 必须在 0.0 到 1.0 之间，且所有参与项的 P 值相同且不可为 NULL。Y 输入必须是 NULL 或数字；NULL 会被忽略，任何非 NULL 且非数字的 Y 会引发错误。percentile_cont(Y,P) 与percentile(Y,P) 类似，只是 P 的取值范围是 0.0 到 1.0，因此 percentile_cont(Y,P) 的结果等同于 percentile(Y,P*100)。

只要大合集使用-DSQLITE_ENABLE_PERCENTILE编译，或通过可加载扩展提供，percentile_cont() 函数自 SQLite 3.51.0（2025-11-04）起可用。

percentile_disc(Y,P)

percentile_disc(Y,P) 与 percentile_cont(Y,P) 类似，但其结果总是某个输入值，而且总是选取两个候选值中较小者，而不是对最接近值做加权平均。详细内容请参阅percentile 扩展。

只要大合集使用-DSQLITE_ENABLE_PERCENTILE编译，或通过可加载扩展提供，percentile_disc() 函数自 SQLite 3.51.0（2025-11-04）起可用。

sum(X)
total(X)

sum() 和 total() 聚合函数返回组中所有非 NULL 值之和。如果没有非 NULL 输入行，sum() 返回 NULL，但 total() 返回 0.0。虽然 NULL 在没有行时的求和结果通常无用，但 SQL 标准如此规定，大多数其他 SQL 数据库也这样实现，因此 SQLite 也以相同方式提供 sum() 以保持兼容性。total() 是非标准的函数，可作为解决该设计问题的便捷方式。

total() 的结果始终是浮点数。如果所有非 NULL 输入都为整数，sum() 结果为整数；只要有任意一个输入既不是整数也不是 NULL，sum() 就返回近似的浮点值。

当所有输入为整数或 NULL 且计算过程中发生整数溢出时，sum() 会抛出“integer overflow”异常。如果此前已有浮点数输入，则不会触发溢出。total() 则永远不会抛出整数溢出。

在对浮点数求和时，若数值量级差距很大，结果可能会不精确，因为IEEE 754 浮点数是近似值。可以使用 decimal 扩展中的 decimal_sum(X) 聚合来得到准确求和。请参考示例： 

CREATE TABLE t1(x REAL);
INSERT INTO t1 VALUES(1.55e+308),(1.23),(3.2e-16),(-1.23),(-1.55e308);
SELECT sum(x), decimal_sum(x) FROM t1;

在这个例子中，巨大数值 ±1.55e+308 相互抵消，不过这种抵消要等到求和结束时才发生，在此之前，+1.55e+308 会淹没 3.2e-16，因此 sum() 的结果并不精确。decimal_sum() 聚合会提供精确答案，但需要更多的 CPU 和内存。请注意 decimal_sum() 不是 SQLite 核心内置函数，而是可加载扩展。

如果求和结果超过 IEEE 754 浮点数可表示的范围，可能会返回 +Infinity 或 -Infinity。如果使用了具有不同符号但数值极大的输入，使得 SUM() 或 TOTAL() 无法判断应该返回 +Infinity、-Infinity 还是其他中间值，则结果为 NULL。例如下面的查询就返回 NULL： 

WITH t1(x) AS (VALUES(1.0),(-9e+999),(2.0),(+9e+999),(3.0))
 SELECT sum(x) FROM t1;
