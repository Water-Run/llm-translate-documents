SQLite 3版本的C/C++接口

1.0 概述

SQLite 3.0 是基于 SQLite 2.8.13 代码库的全新版本，但它采用了不兼容的文件格式和 API。SQLite 3.0 的推出是为了满足以下功能需求：

  • 支持 UTF-16。
  • 支持用户自定义文本排序序列。
  • 支持在索引列中存储 BLOB。

必须迁移到 3.0 版本才能实现这些功能，因为每一项都需要对数据库文件格式进行不兼容的更改。其他不兼容的更改，比如对 API 的整理，也是同时引入的，其理念是“一次性完成所有不兼容的改动”。

3.0 版的 API 与 2.X 版类似，但也有一些重要变化。最明显的是所有 API 函数和数据结构前缀由 "sqlite_" 变为 "sqlite3_"。这样可以避免两套 API 之间的混淆，并允许同时链接 SQLite 2.X 与 SQLite 3.0。

尚未统一 UTF-16 字符串在 C 中应使用的数据类型。因此，SQLite 使用通用类型 void* 来表示 UTF-16 字符串。客户端软件可以将 void* 强制转换为其系统所需的任何数据类型。

2.0 C/C++接口

SQLite 3.0 的 API 包含 83 个独立函数，以及若干数据结构和 #define。（完整的API 参考作为独立文档提供。）幸运的是，接口远不像看起来那么复杂。简单程序仍然仅需三个函数：sqlite3_open()、sqlite3_exec() 和 sqlite3_close()。如需对数据库引擎的执行进行更多控制，可以使用 sqlite3_prepare_v2() 将 SQLite 语句编译为字节码，并使用 sqlite3_step() 来执行该字节码。以 sqlite3_column_ 为前缀的一系列例程用于提取查询结果集的信息。许多接口函数成对出现，分别提供 UTF-8 与 UTF-16 版本。此外，还有一些例程用于实现用户自定义的 SQL 函数与文本排序序列。

2.1 打开与关闭数据库

   typedef struct sqlite3 sqlite3;
   int sqlite3_open(const char*, sqlite3**);
   int sqlite3_open16(const void*, sqlite3**);
   int sqlite3_close(sqlite3*);
   const char *sqlite3_errmsg(sqlite3*);
   const void *sqlite3_errmsg16(sqlite3*);
   int sqlite3_errcode(sqlite3*);

sqlite3_open() 返回的是整数错误码，而不是像 2.x 接口那样直接返回 sqlite3 结构的指针。sqlite3_open() 与 sqlite3_open16() 的区别在于 sqlite3_open16() 接受 UTF-16（宿主本机字节序）的数据库文件名。如果需要创建新数据库文件，sqlite3_open16() 会将内部文本表示设为 UTF-16，而 sqlite3_open() 则设置为 UTF-8。

数据库文件的打开或创建被推迟至实际需要该文件时才进行。这样就可以使用 PRAGMA 语句设置诸如本机文本表示与默认页大小之类的选项与参数。

sqlite3_errcode() 返回最近一次主要 API 调用的结果码。sqlite3_errmsg() 返回最近一次错误的英文错误消息。错误消息以 UTF-8 表示，且是临时的——在下一次调用任何 SQLite API 函数之后可能就不存在了。sqlite3_errmsg16() 的作用与 sqlite3_errmsg() 相同，只不过它返回的是以宿主本机字节序表示的 UTF-16 错误消息。

SQLite 3 版本的错误码与 2 版本保持不变，具体如下： 

#define SQLITE_OK           0   /* 成功结果 */
#define SQLITE_ERROR        1   /* SQL 错误或数据库缺失 */
#define SQLITE_INTERNAL     2   /* SQLite 中的内部逻辑错误 */
#define SQLITE_PERM         3   /* 访问权限被拒 */
#define SQLITE_ABORT        4   /* 回调例程请求中止 */
#define SQLITE_BUSY         5   /* 数据库文件被锁定 */
#define SQLITE_LOCKED       6   /* 数据库中的某个表被锁定 */
#define SQLITE_NOMEM        7   /* malloc() 分配失败 */
#define SQLITE_READONLY     8   /* 尝试写入只读数据库 */
#define SQLITE_INTERRUPT    9   /* sqlite_interrupt() 中断了操作 */
#define SQLITE_IOERR       10   /* 发生某种磁盘 I/O 错误 */
#define SQLITE_CORRUPT     11   /* 数据库磁盘映像损坏 */
#define SQLITE_NOTFOUND    12   /* （仅供内部使用）表或记录未找到 */
#define SQLITE_FULL        13   /* 插入失败，因为数据库已满 */
#define SQLITE_CANTOPEN    14   /* 无法打开数据库文件 */
#define SQLITE_PROTOCOL    15   /* 数据库锁协议错误 */
#define SQLITE_EMPTY       16   /* （仅供内部使用）数据库表为空 */
#define SQLITE_SCHEMA      17   /* 数据库模式发生变化 */
#define SQLITE_TOOBIG      18   /* 单条记录的数据过多 */
#define SQLITE_CONSTRAINT  19   /* 由于违反约束而中止 */
#define SQLITE_MISMATCH    20   /* 数据类型不匹配 */
#define SQLITE_MISUSE      21   /* 库使用不当 */
#define SQLITE_NOLFS       22   /* 使用了宿主系统不支持的操作系统特性 */
#define SQLITE_AUTH        23   /* 授权被拒 */
#define SQLITE_ROW         100  /* sqlite_step() 有另一行可用 */
#define SQLITE_DONE        101  /* sqlite_step() 执行完毕 */

2.2 执行 SQL 语句

   typedef int (*sqlite_callback)(void*,int,char**, char**);
   int sqlite3_exec(sqlite3*, const char *sql, sqlite_callback, void*, char**);

sqlite3_exec() 的工作方式与 SQLite 2 中大致相同。传入的第二个参数中可以包含零条或多条 SQL 语句，这些语句会被编译并执行。查询结果会通过回调例程返回。

在 SQLite 3 中，sqlite3_exec 仅是对预处理语句接口的一个包装。 

   typedef struct sqlite3_stmt sqlite3_stmt;
   int sqlite3_prepare(sqlite3*, const char*, int, sqlite3_stmt**, const char**);
   int sqlite3_prepare16(sqlite3*, const void*, int, sqlite3_stmt**, const void**);
   int sqlite3_finalize(sqlite3_stmt*);
   int sqlite3_reset(sqlite3_stmt*);

sqlite3_prepare 接口将单条 SQL 语句编译成字节码以供后续执行。现在该接口是访问数据库的首选方式。

对于 sqlite3_prepare()，SQL 语句应以 UTF-8 字符串传入。sqlite3_prepare16() 的工作方式相同，只不过它期望 SQL 输入是 UTF-16 字符串。输入字符串中仅编译第一条 SQL 语句。若还存在未编译的 SQL 语句，第五个参数会被填充为指向下一条语句的指针。sqlite3_finalize() 用于释放已经准备好的 SQL 语句，所有准备好的语句在关闭数据库前都必须被 finalize。sqlite3_reset() 会将预处理语句复位，使其可以再次执行。

SQL 语句中可以包含形如 "?"、"?nnn" 或 ":aaa" 的标记，其中 "nnn" 是整数，而 "aaa" 是标识符。这些标记表示待填充的字面值（或“通配符”），可通过 sqlite3_bind 接口在之后赋值。每个通配符都有一个编号，编号就是它在语句中的顺序，或在 "?nnn" 形式中的 "nnn"。同一个语句中可以多次出现相同的通配符，这时所有实例都会被赋予相同值。未绑定的通配符默认值为 NULL。 

   int sqlite3_bind_blob(sqlite3_stmt*, int, const void*, int n, void(*)(void*));
   int sqlite3_bind_double(sqlite3_stmt*, int, double);
   int sqlite3_bind_int(sqlite3_stmt*, int, int);
   int sqlite3_bind_int64(sqlite3_stmt*, int, long long int);
   int sqlite3_bind_null(sqlite3_stmt*, int);
   int sqlite3_bind_text(sqlite3_stmt*, int, const char*, int n, void(*)(void*));
   int sqlite3_bind_text16(sqlite3_stmt*, int, const void*, int n, void(*)(void*));
   int sqlite3_bind_value(sqlite3_stmt*, int, const sqlite3_value*);

有若干 sqlite3_bind 系列函数用于给预处理 SQL 语句中的通配符赋值。未绑定的通配符会被当作 NULL。sqlite3_reset() 不会重置绑定值，但在 sqlite3_reset() 之后可以再次绑定新的值。

在 SQL 语句准备完成（并可选地绑定参数）后，使用以下函数执行： 

   int sqlite3_step(sqlite3_stmt*);

sqlite3_step() 在返回结果集中的单行数据时会返回 SQLITE_ROW，在执行完成（无论成功还是出错）后返回 SQLITE_DONE。若无法打开数据库文件，可能会返回 SQLITE_BUSY。如果返回值是 SQLITE_ROW，下面的例程可用来提取该结果集行的信息： 

   const void *sqlite3_column_blob(sqlite3_stmt*, int iCol);
   int sqlite3_column_bytes(sqlite3_stmt*, int iCol);
   int sqlite3_column_bytes16(sqlite3_stmt*, int iCol);
   int sqlite3_column_count(sqlite3_stmt*);
   const char *sqlite3_column_decltype(sqlite3_stmt *, int iCol);
   const void *sqlite3_column_decltype16(sqlite3_stmt *, int iCol);
   double sqlite3_column_double(sqlite3_stmt*, int iCol);
   int sqlite3_column_int(sqlite3_stmt*, int iCol);
   long long int sqlite3_column_int64(sqlite3_stmt*, int iCol);
   const char *sqlite3_column_name(sqlite3_stmt*, int iCol);
   const void *sqlite3_column_name16(sqlite3_stmt*, int iCol);
   const unsigned char *sqlite3_column_text(sqlite3_stmt*, int iCol);
   const void *sqlite3_column_text16(sqlite3_stmt*, int iCol);
   int sqlite3_column_type(sqlite3_stmt*, int iCol);

sqlite3_column_count() 返回结果集中的列数。该函数可在 sqlite3_prepare_v2() 之后任何时间调用。sqlite3_data_count() 的行为类似于 sqlite3_column_count()，但它仅在 sqlite3_step() 之后有效。如果前一次的 sqlite3_step() 返回 SQLITE_DONE 或错误码，那么 sqlite3_data_count() 会返回 0，而 sqlite3_column_count() 仍会继续返回结果集的列数。

返回的数据可以通过其他 sqlite3_column_***() 函数进行检查，它们都以列号为第二个参数。列号从左至右以 0 为起始。注意，这与参数不同，参数的索引从 1 开始。

sqlite3_column_type() 返回第 N 列值所使用的数据类型。返回值为下列之一： 

   #define SQLITE_INTEGER  1
   #define SQLITE_FLOAT    2
   #define SQLITE_TEXT     3
   #define SQLITE_BLOB     4
   #define SQLITE_NULL     5

sqlite3_column_decltype() 返回在 CREATE TABLE 语句中声明的列类型。对于表达式，返回空字符串。sqlite3_column_name() 返回第 N 列的名称。sqlite3_column_bytes() 返回 BLOB 类型列所占字节数，或以 UTF-8 编码的 TEXT 字符串所占字节数。sqlite3_column_bytes16() 对 BLOB 返回相同字节数，但对于 TEXT 字符串返回 UTF-16 编码所占的字节数。

  • sqlite3_column_blob() 返回 BLOB 数据。
  • sqlite3_column_text() 以 UTF-8 返回 TEXT 数据。
  • sqlite3_column_text16() 以 UTF-16 返回 TEXT 数据。
  • sqlite3_column_int() 以宿主机本机整数格式返回 INTEGER 数据。
  • sqlite3_column_int64() 返回 64 位 INTEGER 数据。
  • 最后，sqlite3_column_double() 返回浮点数数据。

无需非得以 sqlite3_column_type() 指定的格式获取数据。如果请求了其他格式，数据会自动转换。

数据格式转换可能导致先前调用 sqlite3_column_blob()、sqlite3_column_text() 和/或 sqlite3_column_text16() 返回的指针失效。指针可能在以下情况中失效：

  • 原始内容为 BLOB，随后调用 sqlite3_column_text() 或 sqlite3_column_text16()。可能需要在字符串末尾添加终止符。

  • 原始内容为 UTF-8 文本，随后调用 sqlite3_column_bytes16() 或 sqlite3_column_text16()。必须将内容转换为 UTF-16。

  • 原始内容为 UTF-16 文本，随后调用 sqlite3_column_bytes() 或 sqlite3_column_text()。必须将内容转换为 UTF-8。

注意：UTF-16BE 与 UTF-16LE 之间的转换总是在原地进行，因此不会使先前的指针失效，尽管指针指向的缓冲区内容会因此被修改。其他类型的转换在可能的情况下也会尝试原地进行，但有时无法做到，在这种情况下先前的指针会失效。

最安全、最容易记住的策略是：假设任何

  • sqlite3_column_blob() 的返回值，
  • sqlite3_column_text() 的返回值，或
  • sqlite3_column_text16() 的返回值，
都可能在随后的
  • sqlite3_column_bytes()
  • sqlite3_column_bytes16()
  • sqlite3_column_text()
  • sqlite3_column_text16()
调用中被使得失效。也就是说，你应当总是在调用 sqlite3_column_blob()、sqlite3_column_text() 或 sqlite3_column_text16() 之前先调用 sqlite3_column_bytes() 或 sqlite3_column_bytes16()。

2.3 用户自定义函数

可以使用如下例程创建用户自定义函数： 

   typedef struct sqlite3_value sqlite3_value;
   int sqlite3_create_function(
     sqlite3 *,
     const char *zFunctionName,
     int nArg,
     int eTextRep,
     void*,
     void (*xFunc)(sqlite3_context*,int,sqlite3_value**),
     void (*xStep)(sqlite3_context*,int,sqlite3_value**),
     void (*xFinal)(sqlite3_context*)
   );
   int sqlite3_create_function16(
     sqlite3*,
     const void *zFunctionName,
     int nArg,
     int eTextRep,
     void*,
     void (*xFunc)(sqlite3_context*,int,sqlite3_value**),
     void (*xStep)(sqlite3_context*,int,sqlite3_value**),
     void (*xFinal)(sqlite3_context*)
   );
   #define SQLITE_UTF8     1
   #define SQLITE_UTF16    2
   #define SQLITE_UTF16BE  3
   #define SQLITE_UTF16LE  4
   #define SQLITE_ANY      5

nArg 参数指定函数的参数个数。值为 0 表示允许任意数量的参数。eTextRep 参数指定该函数期望文本参数使用的表示形式，该参数的值应该是上面定义的几种之一。SQLite 3 允许针对同一函数提供多个实现，分别使用不同的文本表示形式。数据库引擎会选择需要最少文本转换次数的实现。

普通函数只指定 xFunc，并将 xStep 和 xFinal 设为 NULL。聚合函数指定 xStep 与 xFinal，并将 xFunc 设为 NULL。没有单独的 sqlite3_create_aggregate() API。

函数名以 UTF-8 指定。另一个 sqlite3_create_function16() API 与 sqlite3_create_function() 完全相同，只是函数名采用 UTF-16 宿主字节序表示。

注意：函数的参数现在是指向 sqlite3_value 结构的指针，而不是 SQLite 2.X 中的字符串指针。下面的例程用于从这些“值”中提取有用的信息： 

   const void *sqlite3_value_blob(sqlite3_value*);
   int sqlite3_value_bytes(sqlite3_value*);
   int sqlite3_value_bytes16(sqlite3_value*);
   double sqlite3_value_double(sqlite3_value*);
   int sqlite3_value_int(sqlite3_value*);
   long long int sqlite3_value_int64(sqlite3_value*);
   const unsigned char *sqlite3_value_text(sqlite3_value*);
   const void *sqlite3_value_text16(sqlite3_value*);
   int sqlite3_value_type(sqlite3_value*);

函数实现使用如下 API 获取上下文并返回结果： 

   void *sqlite3_aggregate_context(sqlite3_context*, int nbyte);
   void *sqlite3_user_data(sqlite3_context*);
   void sqlite3_result_blob(sqlite3_context*, const void*, int n, void(*)(void*));
   void sqlite3_result_double(sqlite3_context*, double);
   void sqlite3_result_error(sqlite3_context*, const char*, int);
   void sqlite3_result_error16(sqlite3_context*, const void*, int);
   void sqlite3_result_int(sqlite3_context*, int);
   void sqlite3_result_int64(sqlite3_context*, long long int);
   void sqlite3_result_null(sqlite3_context*);
   void sqlite3_result_text(sqlite3_context*, const char*, int n, void(*)(void*));
   void sqlite3_result_text16(sqlite3_context*, const void*, int n, void(*)(void*));
   void sqlite3_result_value(sqlite3_context*, sqlite3_value*);
   void *sqlite3_get_auxdata(sqlite3_context*, int);
   void sqlite3_set_auxdata(sqlite3_context*, int, void*, void (*)(void*));

2.4 用户自定义排序序列

以下例程用于实现用户自定义的排序序列： 

   sqlite3_create_collation(sqlite3*, const char *zName, int eTextRep, void*,
      int(*xCompare)(void*,int,const void*,int,const void*));
   sqlite3_create_collation16(sqlite3*, const void *zName, int eTextRep, void*,
      int(*xCompare)(void*,int,const void*,int,const void*));
   sqlite3_collation_needed(sqlite3*, void*, 
      void(*)(void*,sqlite3*,int eTextRep,const char*));
   sqlite3_collation_needed16(sqlite3*, void*,
      void(*)(void*,sqlite3*,int eTextRep,const void*));

sqlite3_create_collation() 指定排序序列名称以及实现该排序序列的比较函数。比较函数仅用于比较文本值。eTextRep 参数必须是 SQLITE_UTF8、SQLITE_UTF16LE、SQLITE_UTF16BE 或 SQLITE_ANY 中的一种，用来说明比较函数所适用的文本表示形式。对于同一排序序列，可以分别为 UTF-8、UTF-16LE 和 UTF-16BE 提供不同的比较函数。sqlite3_create_collation16() 与 sqlite3_create_collation() 的作用相同，只不过排序名称以 UTF-16 宿主字节序指定，而非 UTF-8。

sqlite3_collation_needed() 注册一个回调，当数据库引擎遇到未知的排序序列时会调用该回调。回调可以查找合适的比较函数并在需要时调用 sqlite_3_create_collation()。回调的第四个参数是排序序列的 UTF-8 名称。对于 sqlite3_collation_need16()，回调会传入 UTF-16 宿主字节序的排序名称。
