UNION 虚拟表

  1. UNION 虚拟表（以下简称“union-vtab”）是一个 虚拟表，它可让多个独立的 rowid 表 看起来像一个大型表。

  2. 参与 union-vtab 的表可以在同一数据库文件中，或在通过 ATTACH 附加到同一数据库连接的若干数据库文件中。

  3. union-vtab 并非内置于 SQLite。它是一个 可加载扩展。union-vtab 的源代码位于 SQLite 源代码树中的单个文件 ext/misc/unionvtab.c。

  4. 创建一个新的 union-vtab 实例如下：

    CREATE VIRTUAL TABLE temp.tabname USING unionvtab(query);

  5. 每个 union-vtab 都必须位于 TEMP 命名空间。因此在 tabname 前的 "temp." 是必须的。只有 union-vtab 本身必须在 TEMP 命名空间中，参与 union 的各个表可以是任何 ATTACH 的数据库。

  6. CREATE VIRTUAL TABLE 语句中的 query 必须是格式正确的 SQL 查询，返回四列且任意数量的行。query 结果中的每一行代表一个将参与 union 的表。

    1. 第一列是包含这些表的数据库的模式名。例如：“main”、“zone512”。
    2. 第二列是表名。
    3. 第三列是该表中任意 rowid 的最小值。
    4. 第四列是该表中任意 rowid 的最大值。

  7. CREATE VIRTUAL TABLE 语句中的 query 可以是 SELECT 语句或 VALUES 子句。

  8. 当首次遇到 CREATE VIRTUAL TABLE 语句时，只执行一次 query，并将该次执行的结果用于之后对 union-vtab 的所有访问。如果 query 的结果发生变化，应该先 DROP 再重新创建 union-vtab，以便重新执行 query。

  9. 参与 union-vtab 的各个表的 rowid 区间不得重叠。

  10. 参与 union-vtab 的所有表必须具有相同的 CREATE TABLE 定义，唯一区别可以是表名不同。

  11. 参与 union-vtab 的所有表必须是 rowid 表。

  12. tabname 的列名和定义与基础表相同。应用程序可以像访问真实基础表一样访问 tabname。

  13. union-vtab 中的任何表都不能包含超出 CREATE VIRTUAL TABLE 语句中 query 所规定的 rowid 范围的条目。

  14. 当查询约束满足下列形式时，union-vtab 会优化对底层真实表的访问。其他类型的约束可能在未来受到优化，但在初始实现中仅优化这些形式。

    • rowid=$id
    • rowid IN query-or-list
    • rowid BETWEEN $lwr AND $upr

    其他类型的约束也可以使用并且可工作，但它们将逐行单独检查，并且（至少在初始阶段）不会被优化。所有约束检查都是完全自动的，无论是否进行了优化。此处提到的优化只是一个性能考虑。无论是否优化，最终结果都是相同的。

  15. union-vtab 是只读的。写入支持可能在以后添加，但初始实现不包括写入功能。

  16. 注意： sqlite3_blob_open() 接口 不 适用于 union-vtab。必须使用普通 SQL 语句从 union-vtab 读取 BLOB 内容。
