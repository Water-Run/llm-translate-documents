SQLite 与其他数据库引擎的 NULL 处理

目标是让 SQLite 以符合标准的方式处理 NULL。 但 SQL 标准中关于如何处理 NULL 的描述似乎模棱两可。 从标准文档中不清楚究竟在所有情况下应该如何处理 NULL。

因此，与其依据标准文档，不如对各种流行的 SQL 引擎进行测试，看看它们如何处理 NULL。 目标是让 SQLite 的行为与其他引擎一致。 志愿者在各种 SQL RDBMS 上开发并运行了一个 SQL 测试脚本，并使用这些测试结果推断出每个引擎如何处理 NULL 值。 最初的测试于 2002 年 5 月运行。 本文件末尾附有测试脚本的副本。

SQLite 最初的编码方式使下面图表中所有问题的答案都为“是”。 但在其他 SQL 引擎上的实验表明，它们都不是这样工作的。 因此 SQLite 被修改为与 Oracle、PostgreSQL 和 DB2 的行为一致； 这意味着在 SELECT DISTINCT 声明和 SELECT 中的 UNION 操作中，NULL 被视为相同。 在 UNIQUE 列中 NULL 仍然被视为不同。 这看似有些武断，但为了与其他引擎兼容的愿望压倒了这一反对意见。

可以让 SQLite 在 SELECT DISTINCT 和 UNION 的上下文中将 NULL 视为不同。 为此，应修改 sqliteInt.h 源文件中 NULL_ALWAYS_DISTINCT 的 #define 值并重新编译。

更新 2003-07-13： 由于本文档最初撰写以来，一些被测试的数据库引擎已经更新，且用户也好心地发送了对下表的修正。 最初的数据展示了各种行为，但随着时间推移，行为范围已趋向 PostgreSQL/Oracle 模型。 唯一显著的差异是 Informix 和 MS-SQL 在 UNIQUE 列中都将 NULL 视为相同。

NULL 在 UNIQUE 列中是不同的，而在 SELECT DISTINCT 和 UNION 中却是相同的，这仍然令人不解。 似乎 NULL 应该要么在所有地方都不同，要么在所有地方都相同。 SQL 标准文档暗示 NULL 在各处都应是不同的。 然而截至本文撰写时，尚无任何测试过的 SQL 引擎在 SELECT DISTINCT 或 UNION 中将 NULL 视为不同。

以下脚本用于收集上述表格的信息。 

-- 我差不多已经决定 SQL 对 NULL 的处理是反复无常的，无法通过逻辑推导得出。
-- 它必须通过实验来发现。为此，我准备了下面的脚本，测试各种 SQL 数据库如何处理 NULL。
-- 我的目标是利用这个脚本收集的信息，使 SQLite 尽可能地像其他数据库。
--
-- 如果你能在自己的数据库引擎上运行这个脚本，并将结果发送到 drh@hwaci.com，那将大有帮助。
-- 请务必说明你用于测试的数据库引擎。谢谢。
--
-- 如果你必须修改任何内容才能让这个脚本在你的数据库引擎上运行，请把修改后的脚本和结果一起寄给我。
--

-- 创建一个带数据的测试表
create table t1(a int, b int, c int);
insert into t1 values(1,0,0);
insert into t1 values(2,0,1);
insert into t1 values(3,1,0);
insert into t1 values(4,1,1);
insert into t1 values(5,null,0);
insert into t1 values(6,null,1);
insert into t1 values(7,null,null);

-- 检查 CASE 在其测试表达式中对 NULL 的处理
select a, case when b<>0 then 1 else 0 end from t1;
select a+10, case when not b<>0 then 1 else 0 end from t1;
select a+20, case when b<>0 and c<>0 then 1 else 0 end from t1;
select a+30, case when not (b<>0 and c<>0) then 1 else 0 end from t1;
select a+40, case when b<>0 or c<>0 then 1 else 0 end from t1;
select a+50, case when not (b<>0 or c<>0) then 1 else 0 end from t1;
select a+60, case b when c then 1 else 0 end from t1;
select a+70, case c when b then 1 else 0 end from t1;

-- 当你把 NULL 乘以零时会发生什么？
select a+80, b*0 from t1;
select a+90, b*c from t1;

-- 其他运算符对 NULL 会产生什么结果？
select a+100, b+c from t1;

-- 测试聚合运算符对 NULL 的处理
select count(*), count(b), sum(b), avg(b), min(b), max(b) from t1;

-- 检查 WHERE 子句中 NULL 的行为
select a+110 from t1 where b<10;
select a+120 from t1 where not b>10;
select a+130 from t1 where b<10 OR c=1;
select a+140 from t1 where b<10 AND c=1;
select a+150 from t1 where not (b<10 AND c=1);
select a+160 from t1 where not (c=1 AND b<10);

-- 检查 DISTINCT 查询中 NULL 的表现
select distinct b from t1;

-- 检查 UNION 查询中 NULL 的表现
select b from t1 union select b from t1;

-- 创建一个带有 UNIQUE 列的新表，检查 NULL 是否被认为是不同的。
create table t2(a int, b int unique);
insert into t2 values(1,1);
insert into t2 values(2,null);
insert into t2 values(3,null);
select * from t2;

drop table t1;
drop table t2;
