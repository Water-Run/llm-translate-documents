1. 概述

zipfile 模块提供对简单 ZIP 存档 的读/写访问。 当前的实现具有以下限制：

  • 不支持加密。
  • 不支持跨多个文件的 ZIP 存档。
  • 不支持 zip64 扩展。
  • 唯一支持的压缩算法是 “deflate”。

这些限制中的部分或全部将来可能被移除。

2. 获取和编译 Zipfile

zipfile 模块的代码位于 ext/misc/zipfile.c 文件中，属于 SQLite 主源代码树。 可以使用如下命令将其编译为 SQLite 可加载扩展： 

gcc -g -fPIC -shared zipfile.c -o zipfile.so

或者，也可以将 zipfile.c 文件编译到应用程序中。 在这种情况下，应在每个新的数据库连接上调用以下函数来注册扩展： 

int sqlite3_zipfile_init(sqlite3 *db, void*, void*);

第一个参数应传入用于注册扩展的数据库句柄。第二和第三个参数都应传入 0。

zipfile 已包含在大多数 命令行 shell 的构建中。

3. 使用 Zipfile

zipfile 模块提供三种类似的接口来访问、更新 和创建 zip 文件存档：

  1. 一个表值函数，可对现有存档提供只读访问，无论是来自文件系统还是内存。
  2. 一个虚拟表，可对存储在文件系统中的存档提供读写访问。
  3. 一个 SQL 聚合函数，可用于在内存中创建新存档。

zipfile 模块提供两种类似的接口用于访问 zip 存档。一个表值函数，可对现有存档提供只读访问；另一个虚拟表接口，可提供读写访问。

3.1. 表值函数（只读访问）

为读取现有的 zip 存档，Zipfile 模块提供了一个 表值函数，它接受一个参数。如果参数 是文本值，则表示要从文件系统读取的 zip 存档路径。 如果参数是 SQL Blob，则表示 zip 存档数据本身。

例如，要检查当前目录下名为“test.zip”的存档内容： 

SELECT * FROM zipfile('test.zip');

或者，在 SQLite shell 工具中（readfile() 函数会从文件系统读取文件内容并作为 Blob 返回）： 

SELECT * FROM zipfile( readfile('test.zip') );

表值函数为 zip 存档中的每条记录（文件、 目录或符号链接）返回一行。每行包含以下列：

列名内容
name zip 文件记录的文件名/路径。
mode UNIX 模式，即 stat(2) 返回的 zip 文件记录（一个整数）。 这标识了记录的类型（文件、目录或符号链接）以及相关的用户/组/全体权限。
mtime UTC 时间戳，自 UNIX 纪元以来的秒数（一个整数）。
sz 解压后的关联数据字节数（一个整数）。
rawdata 与 zip 文件条目关联的原始（可能已压缩）数据（一个 Blob）。
data 如果记录的压缩方法为 0 或 8（见下文），则为关联的未压缩数据。 如果压缩方法不是 0 或 8，则此列为 NULL。
method 用于压缩数据的压缩方法（一个整数）。值 0 表示数据在 zip 存档中未压缩。8 表示原始 deflate 算法。

3.2. 虚拟表接口（读/写访问）

为了创建或修改现有的 zip 文件，必须在数据库模式中创建一个“zipfile”虚拟表。CREATE VIRTUAL TABLE 语句只接受一个参数，即 zip 文件的路径。例如，要向当前目录下的“test.zip”写入，可以使用以下语句创建 zipfile 表： 

CREATE VIRTUAL TABLE temp.zip USING zipfile('test.zip');

这样的虚拟表具有与上一节所述表值函数相同的列。可以像表值函数一样通过 SELECT 语句读取它。

使用虚拟表接口，可以通过向虚拟表插入新行向 zip 存档添加新条目。可以通过删除行删除条目，或通过更新行来修改条目。

3.2.1. 向 Zip 存档添加条目

可以通过插入新行向 zip 存档添加条目。最简单的方式是仅为“name”和“data”列指定值，让 zipfile 为其他字段填充合理的默认值。要将目录插入存档，请将“data”列设置为 NULL。 例如，要将目录“dir1”和包含文本“abcdefghi”的文件“m.txt”添加到“test.zip”存档： 

INSERT INTO temp.zip(name, data) VALUES('dir1', NULL);           -- 添加目录 
INSERT INTO temp.zip(name, data) VALUES('m.txt', 'abcdefghi');   -- 添加常规文件

插入目录时，如果“name”值不以“/”字符结尾，zipfile 模块会自动追加一个“/”。这是为了与其他程序（尤其是“info-zip”）处理 zip 存档时保持兼容。

要插入符号链接，用户还必须提供“mode”值。例如，要添加从“link.txt”指向“m.txt”的符号链接： 

INSERT INTO temp.zip(name, mode, data) VALUES('link.txt', 'lrwxrw-rw-', 'm.txt');

以下规则和注意事项适用于每个 INSERT 语句中指定的值：

列说明
name name 列必须指定一个非 NULL 的文本值。 如果指定的名称已存在于存档中，则会报错。
mode 如果向 mode 列插入 NULL，则根据“sz”、“data”和“rawdata”列的值自动将新条目的模式设置为 33188（-rw-r--r--）或 16877（drwxr-xr-x），具体取决于条目是文件还是目录。

如果指定的值是整数（或看起来像整数的文本），则原样插入。如果该值不是有效的 UNIX 模式，则从存档中提取文件时某些程序可能会出现意外行为。

最后，如果为该列指定的值既不是整数也不是 NULL，则假定它是类似“ls -l”命令输出的 UNIX 权限字符串（例如“-rw-r--r--”、“drwxr-xr-x”等）。在这种情况下，如果无法解析该字符串，则会报错。
mtime 如果向 mtime 列插入 NULL，则新条目的时间戳设置为当前时间。否则，将指定的值解释为整数并按原样使用。
sz 该列必须设置为 NULL。如果向该列插入非 NULL 值，或者使用 UPDATE 语句提供新的非 NULL 值，则会报错。
rawdata 该列必须设置为 NULL。如果向该列插入非 NULL 值，或者使用 UPDATE 语句提供新的非 NULL 值，则会报错。
data 要将目录插入存档，则此字段必须设置为 NULL。在这种情况下，如果“mode”列显式指定了值，则该值必须与目录一致（即应满足 (mode & 0040000)=0040000）。

否则，插入到该字段的值即为常规文件的内容或符号链接的目标路径。
method 此字段必须设置为整数 0 或 8，也可以设置为 NULL。

对于目录条目，该字段的任何值都会被忽略。 否则，如果设置为 0，则文件数据或符号链接目标会原样存储，并将压缩方法设置为 0。 如果设置为 8，则在存储之前会使用 deflate 压缩文件数据或链接目标，并将压缩方法设置为 8。 最后，如果向该字段写入 NULL，zipfile 模块会自动决定在存储前是否压缩数据。

INSERT 语句中显式为 rowid 字段指定值是不支持的。提供的任何值都会被忽略。

3.2.2. 删除 Zip 存档条目

可以通过删除相应的行来从现有 zip 存档中移除记录。例如，要使用上述创建的虚拟表从“test.zip”存档中删除文件“m.txt”： 

DELETE FROM temp.zip WHERE name = 'm.txt';

请注意，从 zip 存档中删除记录并不会回收存档内部使用的空间——它仅仅从存档的“中央目录结构”中移除了一条记录，使得该记录无法访问。 解决该低效的一种方法是基于编辑后的存档内容创建一个新的 zip 存档。例如，在编辑了通过虚拟表 temp.zzz 访问的存档之后： 

-- 创建一个全新的空存档： 
CREATE VIRTUAL TABLE temp.newzip USING zipfile('new.zip');

-- 将现有存档的内容复制到新存档
INSERT INTO temp.newzip(name, mode, mtime, data, method)
    SELECT name, mode, mtime, data, method FROM temp.zzz;

3.2.3. 更新现有 Zip 存档条目

可以使用 UPDATE 语句修改现有的 zip 存档条目。

zipfile 虚拟表的最左侧三列“name”、“mode” 和“mtime”都可以设置为可以插入到该列的任意值（详见上文）。如果“mode”或“mtime”设置为 NULL，则最终值按插入 NULL 时的规则确定——“mtime”取当前时间，“mode”取 33188 或 16877，具体取决于 zipfile 表后面四列所述的值是否表明该条目是目录还是文件。

尝试将 sz 或 rawdata 字段设置为非 NULL 值会报错。

data 和 method 列也可以按上述 INSERT 的描述进行设置。

3.3. zipfile() 聚合函数

可以使用 zipfile() 聚合函数在内存中完全构建新的 zip 存档。聚合函数访问的每一行都会向存档添加一个条目。返回值是包含整个存档镜像的 Blob。

zipfile() 聚合函数可以使用 2、4 或 5 个参数。如果使用 5 个参数调用，那么添加到存档的条目等价于将相同的值插入 zipfile 虚拟表的“name”、“mode”、“mtime”、“data”以及“method”列。

如果使用 2 个参数调用 zipfile()，则添加的条目等价于将相同的两个值插入 zipfile 虚拟表的“name”和“data”列，其他所有值均设为 NULL。如果使用 4 个参数调用，则等价于将这四个值插入“name”、“mode”、“mtime”以及“data”列。换言之，以下查询对等： 

SELECT zipfile(name, data) ...
SELECT zipfile(name, NULL, NULL, data, NULL) ...

SELECT zipfile(name, mode, mtime, data) ...
SELECT zipfile(name, mode, mtime, data, NULL) ...

例如，要创建一个包含两个文本文件“a.txt”和“b.txt”的存档，内容分别为“abc”和“123”： 

WITH contents(name, data) AS (
  VALUES('a.txt', 'abc') UNION ALL
  VALUES('b.txt', '123')
)
SELECT zipfile(name, data) FROM contents;
