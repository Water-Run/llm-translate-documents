会话模块 C 接口

从更改集迭代器获取冲突行的值

int sqlite3changeset_conflict(
  sqlite3_changeset_iter *pIter,  /* Changeset iterator */
  int iVal,                       /* Column number */
  sqlite3_value **ppValue         /* OUT: Value from conflicting row */
);

该函数只能与通过 sqlite3changeset_apply() 并使用 SQLITE_CHANGESET_DATA 或 SQLITE_CHANGESET_CONFLICT 将迭代器对象传递到冲突处理器回调时一起使用。如果对任何其他迭代器调用此函数，将返回 SQLITE_MISUSE，并将 *ppValue 置为 NULL。

参数 iVal 必须大于等于 0，并小于当前变更影响的表中的列数。否则，将返回 SQLITE_RANGE，并将 *ppValue 置为 NULL。

如果成功，此函数会将 *ppValue 设置为指向受保护的 sqlite3_value 对象，该对象包含与当前冲突处理器回调关联的“冲突行”中的第 iVal 个值，并返回 SQLITE_OK。

如果发生其他错误（例如内存不足），将返回 SQLite 错误代码，并将 *ppValue 置为 NULL。

另请参阅 对象、 常量 和 函数 的列表。
