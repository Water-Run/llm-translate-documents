接受 changeset（或 patchset）作为输入的非流式函数要求将整个 changeset 存储在内存中的单个缓冲区中。类似地，那些返回 changeset 或 patchset 的函数也是通过返回一个使用 sqlite3_malloc() 分配的大缓冲区的指针来实现。通常这很方便。但如果一个应用运行在低内存环境中并需要处理非常大的 changeset，则所需的大型连续内存分配会变得繁重。

为了避免这个问题，流式 API 函数不是使用一个大的缓冲区，而是通过回调函数传入输入数据，sessions 模块在需要数据时会逐步调用该回调函数。在所有情况下，都用例如下的一对函数参数替代：

       int (*xInput)(void *pIn, void *pData, int *pnData),
       void *pIn,

每次 sessions 模块调用 xInput 回调时，传递的第一个参数都是提供的 pIn 上下文指针的副本。第二个参数 pData 指向大小为 (*pnData) 字节的缓冲区。假设未发生错误，xInput 方法应将最多 (*pnData) 字节的数据复制到缓冲区，并在返回 SQLITE_OK 之前将 (*pnData) 设置为实际复制的字节数。如果输入完全耗尽，则应将 (*pnData) 设置为零以示意。或者如果发生错误，则应返回 SQLite 错误码。在所有情况下，如果 xInput 回调返回错误，则所有处理会被中止，流式 API 函数将错误码的副本返回给调用方。

在 sqlite3changeset_start_strm() 的情况下，sessions 模块可以在迭代器的生命周期内的任何时点调用 xInput 回调。如果该 xInput 回调返回错误，则迭代器进入错误状态，此后所有对迭代器函数的调用都会立即以 xInput 返回的相同错误码失败。

同样，那些返回 changeset（或 patchset）的流式 API 函数通过回调函数分块返回数据，而不是通过指向单个大缓冲区的指针。在这种情况下，用如下一对参数替代：

       int (*xOutput)(void *pOut, const void *pData, int nData),
       void *pOut

xOutput 回调会被调用零次或多次以向应用返回数据。每次调用传入的第一个参数都是由应用提供的 pOut 指针的副本。第二个参数 pData 指向大小为 nData 字节的缓冲区，其中包含要返回的输出数据块。如果 xOutput 回调成功处理了提供的数据，则应返回 SQLITE_OK 表示成功。否则，它应返回其他 SQLite 错误码。在这种情况下处理会立即中止，流式 API 函数会将 xOutput 错误码的副本返回给应用。

sessions 模块从不在第三个参数小于或等于零的情况下调用 xOutput 回调。除此之外，对于返回数据块的大小不做任何保证。
