会话模块 C 接口

创建遍历变更集的迭代器 

int sqlite3changeset_start(
  sqlite3_changeset_iter **pp,    /* OUT: New changeset iterator handle */
  int nChangeset,                 /* Size of changeset blob in bytes */
  void *pChangeset                /* Pointer to blob containing changeset */
);
int sqlite3changeset_start_v2(
  sqlite3_changeset_iter **pp,    /* OUT: New changeset iterator handle */
  int nChangeset,                 /* Size of changeset blob in bytes */
  void *pChangeset,               /* Pointer to blob containing changeset */
  int flags                       /* SESSION_CHANGESETSTART_* flags */
);

创建用于遍历变更集内容的迭代器。如果成功，*pp 将指向迭代器句柄并返回 SQLITE_OK。否则，如果发生错误，*pp 将被置为零并返回 SQLite 错误代码。

以下函数可用于推进并查询由此函数创建的变更集迭代器：

  • sqlite3changeset_next()
  • sqlite3changeset_op()
  • sqlite3changeset_new()
  • sqlite3changeset_old()

调用者有责任最终通过传递给 sqlite3changeset_finalize() 来销毁迭代器。包含变更集的缓冲区（pChangeset）必须在迭代器销毁之后仍然有效。

假设变更集 blob 是由 sqlite3session_changeset()、sqlite3changeset_concat() 或 sqlite3changeset_invert() 函数创建的，那么所有应用于单个表的变更都会被归为一组。这意味着当应用程序使用此函数创建的迭代器遍历变更集时，与单个表相关的所有变更都会被依次访问。不可能先访问应用于表 X 的更改，再访问表 Y 的更改，然后再访问表 X 的更改。

可以通过将一组 支持的标志 作为第 4 个参数传递来修改 sqlite3changeset_start_v2() 及其流式等价函数的行为。

请注意 sqlite3changeset_start_v2() API 仍然是 实验性 的，因此可能会发生变化。
