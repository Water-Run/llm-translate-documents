会话模块 C 接口

从变更集迭代器获取当前操作

int sqlite3changeset_op(
  sqlite3_changeset_iter *pIter,  /* Iterator object */
  const char **pzTab,             /* OUT: Pointer to table name */
  int *pnCol,                     /* OUT: Number of columns in table */
  int *pOp,                       /* OUT: SQLITE_INSERT, DELETE or UPDATE */
  int *pbIndirect                 /* OUT: True for an 'indirect' change */
);

传递给此函数的 pIter 参数可以是通过 sqlite3changeset_apply() 传递给冲突处理程序的迭代器，也可以是由 sqlite3changeset_start() 创建的迭代器。在后一种情况下，最近一次调用 sqlite3changeset_next() 必须返回 SQLITE_ROW。如果不是这样，该函数将返回 SQLITE_MISUSE。

参数 pOp、pnCol 和 pzTab 不能是 NULL。返回时，这三个输出将通过这些指针设置：

*pOp 将根据迭代器当前指向的更改类型设置为 SQLITE_INSERT、SQLITE_DELETE 或 SQLITE_UPDATE；

*pnCol 将设置为受该更改影响的表的列数；

*pzTab 将设置为指向一个以空字符终止的 UTF-8 编码字符串，该字符串包含当前更改所影响的表名。该缓冲区在调用 sqlite3changeset_next() 或冲突处理程序函数返回之前一直有效。

如果 pbIndirect 非 NULL，则 *pbIndirect 在更改为间接更改时设置为 true（1），否则设置为 false（0）。有关直接更改和间接更改的说明，请参阅 sqlite3session_indirect() 的文档。

如果没有发生错误，将返回 SQLITE_OK。如果发生错误，将返回 SQLite 错误代码。在这种情况下，输出变量的值可能不可靠。
