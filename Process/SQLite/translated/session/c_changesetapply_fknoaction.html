会话模块 C 接口

sqlite3changeset_apply_v2 的标志

#define SQLITE_CHANGESETAPPLY_NOSAVEPOINT   0x0001
#define SQLITE_CHANGESETAPPLY_INVERT        0x0002
#define SQLITE_CHANGESETAPPLY_IGNORENOOP    0x0004
#define SQLITE_CHANGESETAPPLY_FKNOACTION    0x0008

以下标志可以通过第九个参数传递给 sqlite3changeset_apply_v2 和 sqlite3changeset_apply_v2_strm：

SQLITE_CHANGESETAPPLY_NOSAVEPOINT
通常，会话模块会将单次对apply_v2()或apply_v2_strm()的 调用执行的所有操作封装在一个SAVEPOINT中。 如果变更集或补丁集成功应用则提交该SAVEPOINT；若发生错误则回滚。 指定此标志会让会话模块省略此SAVEPOINT。在这种情况下，如果 调用者在调用apply_v2()时已打开事务或SAVEPOINT，则可以通过回滚 部分应用的变更集来恢复。

SQLITE_CHANGESETAPPLY_INVERT
在应用变更集之前先对其进行反转。这等价于在应用之前调用 sqlite3changeset_invert()对变更集进行反转。若与补丁集一起 指定此标志则会引发错误。

SQLITE_CHANGESETAPPLY_IGNORENOOP
对于那些即使应用也不会实际修改数据库的变更，不会调用冲突处理器 回调。具体而言，以下情况不会触发冲突处理器：
  • 删除变更但找不到被删除的行，
  • 更新变更但冲突行中的字段已经与其新值一致，或者
  • 插入变更但冲突行的所有字段都与要插入的行一致。

SQLITE_CHANGESETAPPLY_FKNOACTION
如果设置了此标志，则目标数据库中的所有外键约束都像声明为 “ON UPDATE NO ACTION ON DELETE NO ACTION”那样行为，即便它们 实际上是CASCADE、RESTRICT、SET NULL或SET DEFAULT。
