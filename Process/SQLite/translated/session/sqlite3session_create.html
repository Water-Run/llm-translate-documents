会话模块 C 接口

创建一个新的会话对象

int sqlite3session_create(
  sqlite3 *db,                    /* 数据库句柄 */
  const char *zDb,                /* 数据库名称（例如 “main”） */
  sqlite3_session **ppSession     /* 输出：新的会话对象 */
);

创建一个附加到数据库句柄 db 的新会话对象。 如果成功，则会将新对象的指针写入 *ppSession 并返回 SQLITE_OK。 如果发生错误，则 *ppSession 将设置为 NULL 并返回一个 SQLite 错误代码（例如 SQLITE_NOMEM）。

可以创建多个附加到同一个数据库句柄的会话对象。

使用此函数创建的会话对象应在其附加的数据库句柄自身关闭之前 通过 sqlite3session_delete() 函数将其删除。 如果在删除会话对象之前关闭数据库句柄，则对该会话对象调用任何会话模块函数的结果（包括 sqlite3session_delete()）都是未定义的。

由于会话模块使用 sqlite3_preupdate_hook() API，因此 应用程序无法在附加了一个或多个会话对象的数据库句柄上注册预更新钩子。 也无法创建附加到已经为其定义了预更新钩子的数据库句柄的会话对象。 尝试执行上述任一操作的结果都是未定义的。

会话对象将用于为数据库 zDb 中的表创建更改集，其中 zDb 可为 “main”、“temp” 或附加数据库的名称。 在创建会话对象时，如果数据库 zDb 未附加到数据库，也不会被视为错误。
