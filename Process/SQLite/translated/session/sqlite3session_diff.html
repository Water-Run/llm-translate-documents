会话模块 C 接口

将表之间的差异加载到会话中

int sqlite3session_diff(
  sqlite3_session *pSession,
  const char *zFromDb,
  const char *zTbl,
  char **pzErrMsg
);

如果该表尚未附加到作为第一个参数传入的会话对象，本函数会以与 sqlite3session_attach() 函数相同的方式附加 zTbl 表。如果 zTbl 不存在，或它没有主键，本函数将不执行任何操作（但不会返回错误）。

参数 zFromDb 必须是附加到与包含与本函数附加到会话的表兼容的表的会话对象使用的同一数据库句柄的数据库的名称（例如 “main”、“temp” 等）。当满足下列条件时，一个表被视为兼容：

  • 具有相同的名称，
  • 具有相同的列集合并以相同顺序声明，且
  • 具有相同的 PRIMARY KEY 定义。

如果这些表不兼容，将返回 SQLITE_SCHEMA。如果这些表兼容但没有任何 PRIMARY KEY 列，这并非错误，但不会向会话对象添加任何更改。与其他会话 API 相同，无主键的表会被忽略。

该函数向会话对象添加一组更改，可用于使数据库 zFrom 中的表（称为“源表”）与附加到会话对象的表（称为“目标表”）的内容一致。具体而言：

  • 对于存在于目标表但不存在于源表的每一行（主键），向会话对象添加一条 INSERT 记录。
  • 对于存在于目标表但不存在于源表的每一行（主键），向会话对象添加一条 DELETE 记录。
  • 对于在两个表中都存在但非主键值不同的每一行，向会话对象添加一条 UPDATE 记录。

为更清楚地说明，如果调用本函数后使用 sqlite3session_changeset() 构造更改集， 那么在将该更改集应用于数据库 zFrom 后，这两个兼容表的内容将完全相同。

除非如上所述该函数调用是无操作，否则如果数据库 zFrom 不存在或不包含所需兼容表，将会报错。

如果操作成功，将返回 SQLITE_OK。否则将返回一个 SQLite 错误码。在这种情况下，如果参数 pzErrMsg 不为 NULL，则 *pzErrMsg 可能会被设置为指向一个包含英文错误信息的缓冲区。调用者有责任使用 sqlite3_free() 释放此缓冲区。
