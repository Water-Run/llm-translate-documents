会话模块 C 接口

传递给冲突处理程序的常量

#define SQLITE_CHANGESET_DATA        1
#define SQLITE_CHANGESET_NOTFOUND    2
#define SQLITE_CHANGESET_CONFLICT    3
#define SQLITE_CHANGESET_CONSTRAINT  4
#define SQLITE_CHANGESET_FOREIGN_KEY 5

可能作为冲突处理程序第二个参数传递的值。

SQLITE_CHANGESET_DATA
当处理 DELETE 或 UPDATE 更改时，如果具有所需主键字段的行存在于数据库中，但更新所修改的一个或多个其他(非主键)字段不包含预期的“之前”值，则冲突处理程序会以 CHANGESET_DATA 作为第二个参数被调用。

在这种情况下，冲突的行是具有匹配主键的数据库行。

SQLITE_CHANGESET_NOTFOUND
当处理 DELETE 或 UPDATE 更改时，如果数据库中不存在具有所需主键字段的行，则冲突处理程序会以 CHANGESET_NOTFOUND 作为第二个参数被调用。

在这种情况下不存在冲突行。调用 sqlite3changeset_conflict() API 的结果未定义。

SQLITE_CHANGESET_CONFLICT
在处理 INSERT 更改时，如果该操作会导致主键值重复，则会以 CHANGESET_CONFLICT 作为第二个参数传递给冲突处理程序。

在这种情况下，冲突的行是具有匹配主键的数据库行。

SQLITE_CHANGESET_FOREIGN_KEY
如果启用了外键处理，并且应用更改集后数据库处于包含外键违规的状态，则在更改集提交之前，冲突处理程序会仅被调用一次，并将 CHANGESET_FOREIGN_KEY 作为第二个参数。如果冲突处理程序返回 CHANGESET_OMIT，则包括导致外键约束违规的更改在内的所有更改都会被提交。或者，如果返回 CHANGESET_ABORT，则更改集会被回滚。

不会提供任何当前行或冲突行的信息。唯一可以在提供的 sqlite3_changeset_iter 句柄上调用的函数是 sqlite3changeset_fk_conflicts()。

SQLITE_CHANGESET_CONSTRAINT
如果在应用更改时发生任何其他约束违规（例如 UNIQUE、CHECK 或 NOT NULL 约束），则冲突处理程序会以 CHANGESET_CONSTRAINT 作为第二个参数被调用。

在这种情况下不存在冲突行。调用 sqlite3changeset_conflict() API 的结果未定义。
