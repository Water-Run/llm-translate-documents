会话模块 C 接口

向更改组添加架构

int sqlite3changegroup_schema(sqlite3_changegroup*, sqlite3*, const char *zDb);

此方法可用于可选地强制要求添加到更改组句柄的更改集必须与数据库 zDb（“main”、“temp” 或附加数据库的名称）的架构匹配。如果调用 sqlite3changegroup_add() 添加的更改集与配置的架构不兼容，则返回 SQLITE_SCHEMA，并且更改组对象将处于未定义状态。

更改集架构被视为与数据库架构兼容，与 sqlite3changeset_apply() 的情况相同。具体而言，对于更改集中的每个表，数据库中存在一个具有以下特征的表：

  • 更改集所标识的名称，且
  • 拥有至少与更改集中记录的列数相同的列数，且
  • 主键列与更改集中记录的位置相同。

更改组对象的输出始终具有与使用此函数指定的数据库相同的架构。在向 sqlite3changegroup_add() 传递的更改集中列数少于数据库架构中相应表的情况下，这些列会使用数据库架构中的默认列值进行填充。这样就可以在更改组中组合对于单个表具有不同列数但在其他方面兼容的更改集。
