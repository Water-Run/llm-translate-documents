会话模块 C 接口

向变更组添加更改集

int sqlite3changegroup_add(sqlite3_changegroup*, int nData, void *pData);

将缓冲区 pData（大小为 nData 字节）中的整个更改集（或补丁集）中的所有更改添加到变更组中。

如果缓冲区包含补丁集，则对同一变更组对象的早期调用也必须指定补丁集。或者，如果缓冲区包含更改集，则这些早期调用也必须包含更改集。否则，将返回 SQLITE_ERROR，并且不会向变更组添加任何更改。

更改集和变更组中的行通过其主键列的值进行标识。如果更改集中的一条更改与变更组中已有更改具有相同的主键，则该更改被视为作用于相同的行。

对于尚未出现在变更组中的行，其更改会直接复制进去。或者，如果新的更改集和变更组都包含适用于同一行的更改，则变更组的最终内容取决于各自更改的类型，具体如下：

已有更改 新更改 输出更改
INSERT INSERT 忽略新更改。如果新更改集紧接着已经添加到变更组的更改集被记录，则不会发生这种情况。
INSERT UPDATE INSERT 更改保留在变更组中。INSERT 更改中的值会被修改，仿佛先由现有更改插入该行，然后再根据新更改进行更新。
INSERT DELETE 现有的 INSERT 会从变更组中移除。DELETE 不会被添加。
UPDATE INSERT 忽略新更改。如果新更改集紧接着已经添加到变更组的更改集被记录，则不会发生这种情况。
UPDATE UPDATE 现有的 UPDATE 保留在变更组中。它会被修改，使其随附的值等同于先由现有更改更新该行，然后再由新更改更新一次。
UPDATE DELETE 现有的 UPDATE 会被新的 DELETE 替换到变更组中。
DELETE INSERT 如果新更改插入的行中的一个或多个列值与现有更改删除的行不同，则现有的 DELETE 会被变更组中的 UPDATE 替换。否则，如果插入的行与删除的行完全相同，则现有的 DELETE 会被直接丢弃。
DELETE UPDATE 忽略新更改。如果新更改集紧接着已经添加到变更组的更改集被记录，则不会发生这种情况。
DELETE DELETE 忽略新更改。如果新更改集紧接着已经添加到变更组的更改集被记录，则不会发生这种情况。

如果新的更改集包含对变更组中已存在表的更改，则该表的列数和主键列的位置必须保持一致。如果不一致，则该函数会返回 SQLITE_SCHEMA。除非已经使用 sqlite3changegroup_schema() API 为变更组对象配置了数据库模式，在这种情况下，只要它们在其他方面兼容，就可以将具有不同列数的更改集合并到同一表。

如果输入的更改集看起来被损坏并且检测到了损坏，则返回 SQLITE_CORRUPT。或者，在处理过程中发生内存不足，则该函数返回 SQLITE_NOMEM。

在所有情况下，如果发生错误，则变更组最终内容的状态未定义。如果没有错误，则返回 SQLITE_OK。
