会话模块 C 接口

变基更改集

int sqlite3rebaser_rebase(
  sqlite3_rebaser*,
  int nIn, const void *pIn, 
  int *pnOut, void **ppOut 
);

重要：此接口为 实验性，可能会在未通知的情况下发生更改。

参数 pIn 必须指向一个包含大小为 nIn 字节更改集的缓冲区。此函数会根据作为第一个参数传入的 rebaser 对象的配置，分配并填充一个缓冲区，将更改集重新变基并复制进去。如果成功，(*ppOut) 将指向包含重变基更改集的新缓冲区，(*pnOut) 被设置为其字节大小，并返回 SQLITE_OK。调用者有责任最终使用 sqlite3_free() 释放该新缓冲区。否则，如果发生错误，(*ppOut) 和 (*pnOut) 会被置为零，并返回 SQLite 错误码。

另见 对象、 常量 以及 函数 列表。
