Carray() 表值函数

1. 概览

Carray() 是一个 表值函数，具有单列（命名为“value”）和零行或多行。Carray() 中每一行的“value”来自应用通过 参数绑定 提供的 C 语言数组。通过这种方式，carray() 函数为将 C 语言数组绑定到 SQL 查询提供了便捷机制。

2. 可用性

自 SQLite 3.51.0 版（2025-11-04）起，carray() 扩展已被集成到 合并源码 中，尽管默认是禁用的，只有在用 -DSQLITE_ENABLE_CARRAY 编译 SQLite 时才会起作用。 在 3.51.0 版本之前，carray() 位于一个需要单独编译并通过 可加载扩展 添加到 SQLite 的独立源文件中。

carray() 函数最初在 SQLite 3.14 版（2016-08-08）中引入。sqlite3_carray_bind() 接口和 carray() 的单参数变体在 SQLite 3.34.0 版（2020-12-01）中加入。支持将被视为 BLOB 的 struct iovec 对象数组绑定的能力在 SQLite 3.41.0 版（2023-02-21）中加入。

3. 详细信息

carray() 函数可接受一、二或三参数。推荐使用单参数变体。

3.1. 单参数 CARRAY

单参数形式的 carray() 需要一个名为 “sqlite3_carray_bind()” 的特殊 C 语言接口来附加值： 

  int sqlite3_carray_bind(
    sqlite3_stmt *pStmt,         /* 包含 CARRAY 的语句 */
    int idx,                     /* CARRAY 参数的参数号 */
    void *aData,                 /* 数据数组 */
    int nData,                   /* 数组中的条目数 */
    int mFlags,                  /* 数据类型标志 */
    void (*xDestroy)(void*)      /* aData 的析构器 */
  );

传递给 sqlite3_carray_bind() 的 mFlags 参数必须是以下之一： 

  #define SQLITE_CARRAY_INT32   0
  #define SQLITE_CARRAY_INT64   1
  #define SQLITE_CARRAY_DOUBLE  2
  #define SQLITE_CARRAY_TEXT    3
  #define SQLITE_CARRAY_BLOB    4

SQLITE_CARRAY_INT32 类型表示该数组由 “int” 构成。SQLITE_CARRAY_INT64 表示数组由 “sqlite3_int64” 构成。SQLITE_CARRAY_DOUBLE 表示数组由 “double” 构成。SQLITE_CARRAY_TEXT 表示数组由 “char*” 构成，其中数组的每个元素要么是 NULL 指针，要么是指向以零结尾字符串的指针。

SQLITE_CARRAY_BLOB 参数表示该数组由 “struct iovec” 对象构成。“struct iovec” 类型是标准的 Posix 数据结构，通常通过 “#include <sys/uio.h>” 声明。其格式为： 

struct iovec {
  void  *iov_base; /* 起始地址 */
  size_t iov_len;  /* 要传输的字节数 */
};

目前 mFlags 参数的高位必须都为零，尽管未来的扩展可能会使用这些位。

传递给 sqlite3_carray_bind() 的 xDestroy 参数是一个函数指针，用于释放输入数组。SQLite 在使用完数据后会调用该函数。xDestroy 参数也可以是 sqlite3.h 中定义的以下常量之一：

  • SQLITE_STATIC → 这意味着调用 sqlite3_carray_bind() 的应用保留该数据数组的所有权，并承诺在预处理语句完成之前不会修改或释放这些数据。

  • SQLITE_TRANSIENT → 该特殊值指示 SQLite 在 sqlite3_carray_bind() 接口返回之前会自行制作一份私有副本。

3.2. 多参数 CARRAY

最初的 carray() 设计使用两个或三个参数。该方式仍然为了向后兼容而被支持，不过我们鼓励新的应用使用上述的单参数 carray() 设计。

对于 carray() 的二参数和三参数版本，第一个参数是指向数组的指针。由于 SQL 无法直接指定指针值，因此第一个参数必须是一个通过 参数 来绑定的指针值，并使用 sqlite3_bind_pointer() 接口绑定，指针类型为 “carray”。第二个参数是数组的元素个数。可选的第三个参数是一个字符串，用于确定 C 语言数组元素的数据类型。第三个参数允许的值为：

  1. 'int32'
  2. 'int64'
  3. 'double'
  4. 'char*'
  5. 'struct iovec'

默认数据类型为 'int32'。当 carray() 函数仅有两个参数时，假定数据类型为 'int32'。

4. 用法

carray() 函数可以在查询的 FROM 子句中使用。例如，使用位于地址 \(PTR\) 的 C 语言数组中的 rowid 查询 OBJ 表中的两条记录。 

SELECT obj.* FROM obj, carray($PTR) AS x
 WHERE obj.rowid=x.value;

该查询等价于： 

SELECT * FROM obj WHERE rowid IN carray($PTR);
