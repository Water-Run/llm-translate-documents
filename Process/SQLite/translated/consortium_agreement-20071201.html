SQLite 联盟协议

本《SQLite 联盟协议》（“协议”）由_____年____月_____日（“生效日期”）由___________________（“公司”）与 Hipp, Wyrick & Company, Inc.（一家总部位于夏洛特、地址为 6200 Maple Cove Lane, Charlotte, NC 的乔治亚州公司，“Hwaci”）共同缔结。

鉴于 Hwaci 是一个无服务器、可嵌入、公共领域的 SQL 数据库引擎的开发者，该引擎可在 http://www.sqlite.org/ （“SQLite”）获得；以及

鉴于公司希望支持 SQLite 数据库引擎的独立开发，以确保 SQLite 持续可靠、充满活力且独立，并因此希望提供资金用于 SQLite 代码库的持续研究、开发与维护、SQLite 网站的运营、SQLite 商标的保护与维护，以及在会议和贸易展览上推广 SQLite；以及

鉴于公司希望以 Hwaci 的服务形式获得其对 SQLite 的支持回报；以及

鉴于公司和 Hwaci 均希望 SQLite 能作为一个独立、社区驱动的项目得到支持与维护，并希望确保 SQLite 开发团队保持独立。

因此，鉴于各方在本协议中所作的相互承诺，双方达成如下协议：

1.0 定义

1.1 SQLite 开发者

“SQLite 开发者”指 Hwaci 中参与 SQLite 开发的员工或承包商，包括但不限于 SQLite 架构师。

1.2 SQLite 架构师

“SQLite 架构师”是对 SQLite 代码的更改和维护负有最终责任和权限的 SQLite 开发者。目前的 SQLite 架构师是 SQLite 的原始设计者与开发者 D. Richard Hipp。Hwaci 可在公司批准的情况下不时更换 SQLite 架构师。

1.3 联盟成员

“联盟成员”是指与 Hwaci 签署了与本协议实质上相同协议的公司或组织（包括但不限于公司）。

2.0 协议

2.1 SQLite 开发者的职责与义务

2.1.1 一般支持

SQLite 开发者应无时间限制地就公司通过电子邮件或电话随时提出的有关 SQLite 的问题和意见提供及时、准确的答复、建议和指导。

2.1.2 调试支持

SQLite 开发者应在公司请求时提供直接协助，调试建立于或使用 SQLite 的应用程序。针对公司的调试支持与定制开发的总时长每年限定为 330 小时。

2.1.3 定制开发

SQLite 开发者应在公司要求时按照公司的规范为 SQLite 编写新的扩展或增强功能。这些扩展和增强可以为公司的专有，也可以是开源的，或者在公司与 SQLite 架构师都同意的情况下合并到公共领域的 SQLite 源代码树中。为公司进行定制开发的时间每年限定为 220 小时。

2.1.4 定制构建

SQLite 开发者应在公司请求时根据公司的规范为其提供 SQLite 的特殊构建版本。

2.1.5 旧版支持

SQLite 开发者应在公司请求时为任何历史版本的 SQLite 提供支持和补丁修复。

2.1.6 定制测试

SQLite 开发者应在公司请求时根据公司的规范设置并运行验收测试，这些测试专门用于检验公司所配置与使用的 SQLite。必须在任何 SQLite 发布之前运行并通过这些验收测试。

2.1.7 优先级缺陷修复

SQLite 开发者应优先修复公司报告或公司明确表达关注的 SQLite 缺陷。

2.1.8 代码质量

SQLite 开发者应始终保持 SQLite 源代码有良好的注释和文档，并保持清晰的结构和组织，使经验丰富且称职的程序员在不到几个月的学习后即可理解并维护。

2.1.9 向后兼容性

SQLite 开发者认识到公司在关键任务应用中使用 SQLite，因此应努力确保持续无缺陷、正确运行，并与先前版本保持向后兼容性。

2.1.10 测试覆盖率

SQLite 开发者应维护 SQLite 测试套件，确保提供不少于 95% 的源码覆盖率。

2.1.11 网站

SQLite 开发者应维护 http://www.sqlite.org/ 网站的正常运作，并使所有 SQLite 源代码在该网站上免费提供。

2.2 SQLite 架构师的职责与义务

2.2.1 私下简报

SQLite 架构师应在公司请求时向公司提供任何可能影响公司产品的新缺陷或新功能的私下通报与简报。

2.2.2 指导

SQLite 架构师应协调 SQLite 开发者的活动，并应不时发布关于拟议中的未来变更和增强功能的路线图。

2.2.3 监督

SQLite 架构师应亲自审查所有对 SQLite 源代码的更改，并对这些更改的质量与正确性负责。

2.2.4 连续性

初任 SQLite 架构师应为 SQLite 的原始开发者 D. Richard Hipp。除非因 Hipp 博士的残疾并经公司批准，否则在本协议期限内不得将 SQLite 架构师的角色转交他人。

2.3 Hwaci 的职责与义务

2.3.1 雇主

Hwaci 应作为 SQLite 开发者和 SQLite 架构师的雇主，履行所有雇主所需的税务和法律申报及税款扣缴。Hwaci 应在公司请求时提供 W-9 信息。

2.3.2 责任方

本协议中规定的所有 SQLite 开发者义务均视为 Hwaci 的义务。Hwaci 应以促进本协议中所规定的 SQLite 开发者职责与义务得以履行为目标招聘、雇用并监督 SQLite 开发者。

2.3.3 多个开发者

Hwaci 应在合理范围内用来自联盟成员的资金招聘并雇用尽可能多小时数的 SQLite 开发者。为了避免任何单一 SQLite 开发者的残疾或退出中断对 SQLite 的支持，Hwaci 将保持至少两名知识渊博且称职的 SQLite 开发者在职。

2.3.4 指导

Hwaci 应努力鼓励来自世界各地的独立程序员熟悉 SQLite 的内部结构，以便将来拥有能够支持 SQLite 的人才储备。

2.3.5 审计

Hwaci 应保存并维护公司提供的开发资金使用记录的完整与准确，并应允许公司或其代表——一名经 Hwaci 与公司共同认可的注册会计师——在办公时间内、以合理间隔（每 12 个月最多一次）检查并提取或复制此类记录，仅用于核实 Hwaci 是否遵守本协议的目标和要求。

2.3.6 灾备计划

Hwaci 应在至少两个彼此分隔、且与主要在线 SQLite 仓库之间距离至少 400 公里的独立地点维护所有 SQLite 源文件和文档（当前与历史版本）的备份副本。

2.3.7 商标

Hwaci 应保持 SQLite 商标与 sqlite.org 域名的所有权，并应购买 http://www.sqlite.org/ 网站所需的带宽与服务器空间。

2.3.8 不进行接管或收购

在本协议有效期内，除非得到公司的同意，否则 Hwaci 与 SQLite 架构师不得通过收购、合并或任何其他方式放弃对 SQLite 的开发控制。

2.3.9 新联盟成员

可不时在与本协议相同条款、或在现有联盟成员批准的实质性相似条款下接纳新的联盟成员加入联盟。

2.3.10 足够的人员

Hwaci 应招聘并雇用足够数量且资质良好的 SQLite 开发者，以便轻松覆盖所有联盟成员的定制开发、调试和一般支持服务义务，同时仍为 SQLite 开发者提供充足时间从事 SQLite 的一般维护与扩展。

2.3.11 资金使用

Hwaci 应将本协议项下提供的资金仅用于本协议中规定的 SQLite 开发与维护。在提前收到并以信托方式持有的资金上产生的利息应再投资并用于与本金相同的目的。

2.4 公司的职责与义务

2.4.1 资金

公司应根据第 3.0 节“费用”提供 SQLite 的持续支持与维护资金。

2.4.2 版权免责声明

公司承认在 SQLite 网站 发布的 SQLite 源代码与文档属于公共领域，本协议中的任何内容均不会改变这一事实。

3.0 费用

为了 Hwaci、SQLite 开发者与 SQLite 架构师在本协议中所述义务的履行，公司应每年至少预付 75,000 美元，可按公司自行决定的年度、季度或月度方式支付。

4.0 保密性

4.1 保密信息的定义

“保密信息”是指公司披露的任何专有信息、技术数据、商业秘密或专有技术，包括但不限于研究、产品计划、产品、服务、客户、客户名单、市场、软件、开发、发明、流程、公式、技术、设计、图纸、工程、硬件配置信息、市场、财务或其他商业信息，不论是书面、口头、通过图纸还是通过零部件或设备的检查，直接或间接披露。

4.2 不使用与不披露。

在本协议期间或之后，Hwaci 不得将公司的保密信息用于除履行服务之外的任何目的，也不得向任何第三方披露公司的保密信息。双方确认保密信息将始终为公司的独有财产。Hwaci 应采取一切合理预防措施以防止未经授权的披露保密信息，包括但不限于让每位有权接触任何保密信息的 Hwaci 员工或顾问（如有）签署包含与本协议在公司方面实质相似条款的保密协议。保密信息不包括以下信息：(i)公司披露给 Hwaci 时 Hwaci 已知的，且 Hwaci 有书面记录作为证明；(ii)通过 Hwaci 的非不当行为已成为公众所知且普遍可得；或 (iii) Hwaci 从有权披露该信息的第三方处获得的信息。

4.2.1 法律要求的披露

如根据有管辖权法院颁布的有效传票或命令，或行政或其他政府机构的要求或信息请求，需要 Hwaci 披露任何保密信息，Hwaci 应在未被传票、命令或要求禁止的情况下，迅速通知公司该要求存在及其条款和背景，且应与公司商议是否采取法律上可用的步骤以抗辩或缩小该要求的范围；如必须披露该保密信息，Hwaci 应尽合理最大努力缩小披露范围并取得命令或其他可靠保证，使该保密信息得到保密处理。在传票、命令或要求的条款禁止 Hwaci 通知公司时，Hwaci 应尽合理努力缩小披露范围。

4.3 材料归还。

在本协议终止后或公司提前要求时，Hwaci 应向公司交付所有由 Hwaci 持有或控制的公司财产或保密信息。

5.0 知识产权

5.1 不可转让

公司确认与 SQLite 相关，由 Hwaci 制作、生成、构思或付诸实践的所有可版权化材料、笔记、记录、图纸、设计、发明、改进、开发、发现与商业秘密均归 Hwaci 所有。本协议中任何内容均不得被解释为向公司转让 Hwaci 的任何知识产权。

5.2 可用性与公共领域奉献

SQLite 开发者与 Hwaci 应在其自行判断 SQLite 源代码与文档的质量与适用于公开发布的程度的基础上，使 SQLite 源代码与文档以可下载文件形式公开，并公开声明将 SQLite 源代码与文档中所有知识产权，包括但不限于版权与专利权，归入公共领域。在 SQLite 开发者与 Hwaci 决定不公开发布 SQLite 源代码与文档的情况下，他们应向公司提供其副本，并在本协议项下以其所拥有的全部权利，包括但不限于版权与专利权，授予公司对 SQLite 源代码与文档及其派生作品的永久、不可撤销、全球性、非独占、可再许可的使用、复制、准备派生作品、公开表演与展示的许可。

5.3 商标

Hwaci 应仅将“SQLite”名称用于生效日期所示公共可用项目。Hwaci 可自行决定提交其认为适当的商标申请或注册以保护或记录其权利，并可制定其认为适当的政策许可商标使用。

6.0 陈述与保证

6.1 知识产权清理。

Hwaci 特此声明并保证，Hwaci 将与 SQLite 开发者签署足以使 Hwaci 承担第 5 节义务的协议。

6.2 免责声明。

Hwaci 与公司的工作成果及所有材料按“现状”提供。双方对于本协议标的事项，不就任何口头或书面、明示、隐含或因法规、惯例、交易习惯或商业习惯产生的陈述或保证作出任何保证。各方明确否认任何和所有关于所有权、适销性、特定用途适用性与不侵权的默示保证或条件。

7.0 期限与终止

7.1 期限。

本协议自生效日期起开始，并在生效日期后继续有效 12 个月。其后，双方可在就公司为额外期限赞助支付的费用达成一致的前提下共同同意续签本协议。

7.2 终止。

若任一方重大违约其任何重要义务，并在书面通知后 30 日内未予纠正，则非违约方可选择在采取其他救济措施的同时，向违约方发出书面终止通知，自行终止本协议。

7.3 存续。

在此类终止后，双方彼此间的所有权利与义务均告终止，但以下各节仍然有效：第 4 节（保密）、第 5 节（知识产权）及第 8 节（杂项）。

8. 杂项

8.1 不可转让/具有约束力的协议。

双方确认 Hwaci 服务的独特性质构成双方签订本协议的实质对价。未经公司事先书面同意——该同意不得被无理拒绝——Hwaci 不得全部或部分自愿或依法转让或以其他方式转移本协议或本协议项下的任何权利。除非前述情形，本协议对双方及其各自的继承人和受让人具有约束力且对其有利。任何违反前述规定的转让均属无效。

8.2 通知。

本协议项下或依法要求的任何通知必须为书面，并且必须通过以下方式之一送达： (a) 亲自递送；(b) 通过适当的一级挂号邮件或航空邮件发送；或 (c) 通过隔夜航空快递发送，在每种情况下均须正确贴邮并全额预付至本协议序言中所列的地址。任一方可通过依据本节规定向另一方发送通知变更其通知地址。通知应在亲自实际交付时、在按上述方式寄出三个工作日后或在交付至隔夜航空快递服务后一日视为已送达。

8.3 弃权。

本协议或一方在本协议项下的权利或救济的任何弃权必须采用书面形式方为有效。一方在任何时间未能、忽视或延迟执行本协议条款或其权利或救济，不得被解释为放弃该方在本协议项下的权利，也不得以任何方式影响本协议全部或任何部分的效力或损害该方采取后续行动的权利。任一方对本协议任何权利或救济的行使或执行，不应排除该方对本协议任何其他权利或救济或其依法有权执行的任何其他救济的行使或执行。

8.4 可分割性。

若本协议中的任何条款、条件或规定被认定为无效、非法或不可执行，则各方应本着诚意努力达成可保存本协议意图的修正。如双方未能就该修正达成一致，则该无效、非法或不可执行的条款、条件或规定应从其余条款、条件和规定中剥离，其余部分将在法律允许的最大范围内继续有效并可执行。

8.5 整合。

本协议包含双方就本协议标的事项达成的全部协议，并取代双方在该事项上此前的所有交流、陈述、谅解和协议（无论口头或书面）。本协议不得修改，除非双方均书面签署。

8.6 副本。

本协议可分多份签署，每一份均被视为原件，并且这些副本共同构成同一份协议。

8.7 适用法律。

本协议应依照北卡罗来纳州与美利坚合众国法律解释与构造，不考虑冲突法原则。因本协议引起的所有争议应提交北卡罗来纳州内联邦与州法院的专属管辖，各方在此同意接受该等法院的个人管辖。

8.8 独立承包商。

各方意图为 Hwaci 为独立承包商。本协议的任何内容均不得被解释为将 Hwaci 或其任何员工构成公司的代理人、员工或代表。

9.0 签字

各方已在下方签署本协议，以表明其接受本协议条款。
