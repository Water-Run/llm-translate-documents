SQLite 是一个自包含系统

SQLite 在某种意义上是“独立”或“自包含”的，它几乎不依赖任何外部组件。 它可以在任何操作系统上运行，甚至是被大幅精简的嵌入式操作系统。SQLite 不使用外部库或接口（除了下面描述的少数标准 C 库调用）。 整个 SQLite 库封装在一个单一源代码文件中，构建时不需要特殊设施或工具。

SQLite 的最小构建仅需标准 C 库中的这些例程：

  • memcmp()
  • memcpy()
  • memmove()
  • memset()
  • strcmp()
  • strlen()
  • strncmp()

大多数构建还会使用系统内存分配例程：

  • malloc()
  • realloc()
  • free()

但这些例程是可选的，可以通过编译时选项省略。

SQLite 的默认构建包含用于与底层操作系统通信的适当VFS 对象，这些 VFS 对象会封装操作系统调用，例如 open()、read()、write()、fsync() 等。这些接口在大多数平台上随处可用，且可以设计自定义 VFS 以便在极端简陋的嵌入式设备上运行 SQLite。
