发布历史

本页面提供 SQLite 变更的简要概览。 更多细节请参阅 Fossil 提交日志： https://sqlite.org/src/timeline 和 https://sqlite.org/src/timeline?t=release。 详见编年表，内含发布记录摘要。

2026-01-09 (3.51.2)

  1. 修复前述第17项中新引入的罕见死锁，涉及破损 POSIX 锁检测逻辑。
  2. 修复第6b项优化中新增的 EXISTS-to-JOIN 优化所带来的多个问题。
  3. 其他少量缺陷修复。

2025-11-28 (3.51.1)

  1. 修复 3.51.0 中第6b项优化导致的嵌套 EXISTS 查询结果不正确的问题。
  2. 修复 fts5vocab 虚拟表在 3.51.0 新优化下暴露的潜在缺陷。

2025-11-04 (3.51.0)

  1. sqlite3.h 中新增宏：
    1. SQLITE_SCM_BRANCH → 表示源代码所属分支名称。
    2. SQLITE_SCM_TAGS → 以空格分隔的提交标签列表。
    3. SQLITE_SCM_DATETIME → 提交的 ISO-8601 时间戳。
  2. 新增两个 JSON 函数 jsonb_each() 与 jsonb_tree()， 用法与 json_each()/json_tree() 相同，惟“type”为 'array'/'object' 时“value”列返回 JSONB。
  3. carray 与 percentile 扩展现已内嵌于单体源中， 默认停用，需在编译时加上 -DSQLITE_ENABLE_CARRAY 与/或 -DSQLITE_ENABLE_PERCENTILE。
  4. TCL 接口增强：
    1. 为 eval 命令新增 -asdict 标志，以字典而非数组返回行数据。
    2. 用户自定义函数可通过 break 返回 SQL NULL。
  5. CLI 改进：
    1. ".timer" 精度提升至微秒。
    2. “box”与“column”格式化模式增强，可处理双倍宽字符。
    3. ".imposter" 命令提供可与 VACUUM 协作且无需 --unsafe-testing 的只读 冒充表。
    4. 为 CLI 选项与 .open 命令新增 --ifexists。
    5. ".width" 命令设置列宽最长 30,000，避免滥用。
  6. 性能优化：
    1. 提交读事务消耗更少 CPU 周期。
    2. 提前识别因某些表无行而导致不返回行的 JOIN。
    3. 若标量子查询结果对整体表达式无影响，则避免求值。
    4. 使用具有大 :y 的 "BETWEEN :x FOLLOWING AND :y FOLLOWING" 时， 窗口函数查询更快。
  7. 新增 PRAGMA wal_checkpoint=NOOP; 及 SQLITE_CHECKPOINT_NOOP 参数供 sqlite3_wal_checkpoint_v2() 使用。
  8. 新增sqlite3_set_errmsg() API，供扩展调用。
  9. 新增sqlite3_db_status64()，与现有sqlite3_db_status() 相同，但返回 64 位值。
  10. 在sqlite3_db_status()与sqlite3_db_status64() 中新增SQLITE_DBSTATUS_TEMPBUF_SPILL。
  11. session 扩展新增sqlite3changeset_apply_v3()。
  12. 对内建 printf()与format()： 若省略 '+' 且加上 '#'，且所有显示数字均为 '0'，则负浮点不再前置 '-'，推荐使用 '%#f' 避免 "-0.00"。
  13. 改善 FTS5 生成的错误信息。
  14. 对计算列采用 STRICT 类型。
  15. 加强对 VxWorks 的支持。
  16. JavaScript/WASM 现支持 64 位 WASM。尽管官方构建仍为 32 位，但自建 64 位只需 “make”。
  17. 增强对因应用通过 close() 破坏 Posix 建议锁而导致数据库损坏的防护。
