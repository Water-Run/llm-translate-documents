以下是让您开始尝试 SQLite 的步骤，无需大量繁琐的阅读和配置：

下载代码

使用命令行接口创建新数据库

命令行接口或“CLI”是一个简单的命令行程序，它接受 SQL 输入文本并将其传递给 SQLite 数据库引擎核心进行执行。CLI 程序名为“sqlite3”（在 Windows 上为 “sqlite3.exe”）。请使用 CLI 进行数据库的手动交互。

  • 在 shell 或 DOS 提示符下输入：“sqlite3 test.db”。这将创建一个名为“test.db”的新数据库。（如有需要可使用其他名称。）

  • 在提示符下输入 SQL 命令以创建并填充新数据库。

  • 详见 CLI 详细文档

可在 https://sqlite.org/fiddle 获取在浏览器中运行的 CLI 的 WASM 构建版本。

编写使用 SQLite 的程序

  • 下面是一个简单的 TCL 程序，演示如何使用 SQLite 的 TCL 接口。该程序将第一参数指定的数据库与第二参数提供的 SQL 语句一起执行。需要关注的命令是在第 7 行打开 SQLite 数据库并创建一个名为“db”的新对象的 sqlite3 命令，第 8 行在 db 对象上调用的 eval 方法 以对数据库运行 SQL 命令，以及脚本最后一行关闭数据库连接。

    01  #!/usr/bin/tclsh
02  if {$argc!=2} {
03    puts stderr "Usage: %s DATABASE SQL-STATEMENT"
04    exit 1
05  }
06  package require sqlite3
07  sqlite3 db [lindex $argv 0]
08  db eval [lindex $argv 1] x {
09    foreach v $x(*) {
10      puts "$v = $x($v)"
11    }
12    puts ""
13  }
14  db close

  • 下面是一个简单的 C 程序，演示如何使用 SQLite 的 C/C++ 接口。第一个参数给出数据库名称，第二个参数是一条或多条要对数据库执行的 SQL 语句。需要留意的函数调用包括在第 22 行打开数据库的 sqlite3_open()，第 28 行用于执行 SQL 命令的 sqlite3_exec()，以及第 33 行关闭数据库连接的 sqlite3_close()。

    另请参阅 SQLite C/C++ 接口介绍，了解 SQLite 接口函数的入门概览和路线图。

    01  #include <stdio.h>
02  #include <sqlite3.h>
03  
04  static int callback(void *NotUsed, int argc, char **argv, char **azColName){
05    int i;
06    for(i=0; i<argc; i++){
07      printf("%s = %s\n", azColName[i], argv[i] ? argv[i] : "NULL");
08    }
09    printf("\n");
10    return 0;
11  }
12  
13  int main(int argc, char **argv){
14    sqlite3 *db;
15    char *zErrMsg = 0;
16    int rc;
17  
18    if( argc!=3 ){
19      fprintf(stderr, "Usage: %s DATABASE SQL-STATEMENT\n", argv[0]);
20      return(1);
21    }
22    rc = sqlite3_open(argv[1], &db);
23    if( rc ){
24      fprintf(stderr, "Can't open database: %s\n", sqlite3_errmsg(db));
25      sqlite3_close(db);
26      return(1);
27    }
28    rc = sqlite3_exec(db, argv[2], callback, 0, &zErrMsg);
29    if( rc!=SQLITE_OK ){
30      fprintf(stderr, "SQL error: %s\n", zErrMsg);
31      sqlite3_free(zErrMsg);
32    }
33    sqlite3_close(db);
34    return 0;
35  }
