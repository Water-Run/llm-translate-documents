Indexes On Expressions

通常，SQL 索引引用表的列。但索引也可以建立在包含表列的表达式上。

例如，考虑下列表记录各种“账户”的美元金额变动： 

CREATE TABLE account_change(
  chng_id INTEGER PRIMARY KEY,
  acct_no INTEGER REFERENCES account,
  location INTEGER REFERENCES locations,
  amt INTEGER,  -- in cents
  authority TEXT,
  comment TEXT
);
CREATE INDEX acctchng_magnitude ON account_change(acct_no, abs(amt));

account_change 表中的每条记录都记录账户的存款或取款。存款的 amt 为正，取款的 amt 为负。

acctchng_magnitude 索引覆盖账户编号（acct_no）和金额的绝对值。该索引可以针对变动幅度进行高效查询。例如，若要列出账户编号为 $xyz 的所有超过 $100.00 的变动，可以写： 

SELECT * FROM account_change WHERE acct_no=$xyz AND abs(amt)>=10000;

或者，要按变动幅度递减顺序列出某个特定账户（$xyz）的所有变动，可以这样写： 

SELECT * FROM account_change WHERE acct_no=$xyz
 ORDER BY abs(amt) DESC;

上述示例查询即使没有 acctchng_magnitude 索引也能正常工作。acctchng_magnitude 索引只不过是让查询运行得更快，特别是在每个账户在表中有很多记录的数据库上。

1. 如何使用表达式索引

使用 CREATE INDEX 语句像创建列索引一样在一个或多个表达式上创建新索引。唯一的区别是要索引的元素是表达式而不是列名。

当索引的表达式在查询的 WHERE 子句或 ORDER BY 子句中以 CREATE INDEX 语句中书写的 完全相同 方式出现时，SQLite 查询优化器才会考虑使用表达式索引。查询优化器不会做代数运算。为了将 WHERE 约束和 ORDER BY 项匹配到索引，SQLite 要求表达式相同，除了诸如空白字符差异等细微语法差异。所以如果你有： 

CREATE TABLE t2(x,y,z);
CREATE INDEX t2xy ON t2(x+y);

然后你运行查询： 

SELECT * FROM t2 WHERE y+x=22;

那么该索引将 不会 被使用，因为 CREATE INDEX 语句中的表达式 (x+y) 与查询中出现的表达式 (y+x) 不相同。两个表达式在数学上可能等价，但 SQLite 查询优化器坚持它们必须完全相同，而不能仅仅等价。考虑将查询重写如下： 

SELECT * FROM t2 WHERE x+y=22;

这个第二个查询可能会使用索引，因为现在 WHERE 子句中的表达式 (x+y) 与索引中的表达式完全匹配。

2. 限制条件

CREATE INDEX 语句中的表达式存在一些合理的限制：

  1. CREATE INDEX 语句中的表达式只能引用正在索引的表的列，而不能引用其他表的列。

  2. CREATE INDEX 语句中的表达式可以包含函数调用，但只能调用其输出始终完全由输入参数决定的函数（即确定性函数）。显然，像 random() 这样的函数在索引中效果不好。另外，像 sqlite_version() 这样的函数虽然在任何一个数据库连接中是常量，但在底层数据库文件的整个生命周期中并非常量，因此不能在 CREATE INDEX 语句中使用。

    注意，应用程序定义的 SQL 函数默认被视为非确定性的，除非在注册函数时使用了 SQLITE_DETERMINISTIC 标志，否则不能在 CREATE INDEX 语句中使用。

  3. CREATE INDEX 语句中的表达式不得使用子查询。

  4. 表达式只能在 CREATE INDEX 语句中使用，而不能在 UNIQUE 或 PRIMARY KEY 约束中（这些约束位于 CREATE TABLE 语句中）。

3. 兼容性

表达式索引的能力在 SQLite 的 3.9.0 版本（2015-10-14）中被加入。使用表达式索引的数据库无法由更早版本的 SQLite 使用。
