SQLite 3.37.0 于 2021-11-27 发布

  1. STRICT 表 提供一种有规范的数据类型管理方式，适合偏好这类机制的开发者。
  2. 在新增包含 CHECK 约束 或 生成列 中含有 NOT NULL 约束 的列时，ALTER TABLE ADD COLUMN 现在会将新的约束同数据库中已有的行进行校验，仅在不违反约束的情况下继续执行。
  3. 添加了 PRAGMA table_list 语句。
  4. CLI 改进：
    1. 添加了 .connection 命令，允许 CLI 同时保持多个数据库连接打开。
    2. 添加了 --safe 命令行选项，禁用可能引发影响超出命令行中指定的单一数据库文件之外的副作用的 点命令 和 SQL 语句。
    3. 读取跨多行的 SQL 语句时性能得到提升。
  5. 添加了 sqlite3_autovacuum_pages() 接口。
  6. sqlite3_deserialize() 不支持且从未支持 TEMP 数据库。这一限制现已在文档中说明。
  7. 查询规划器现在会在移除子查询和视图中的 ORDER BY 子句不会改变查询语义时省略这些子句。
  8. generate_series 表值函数扩展已修改为第一个参数（"START"）现在为必需。此举旨在演示如何编写具有必选参数的表值函数。通过 -DZERO_ARGUMENT_GENERATE_SERIES 编译选项可恢复旧行为。
  9. 添加了新的 sqlite3_changes64() 和 sqlite3_total_changes64() 接口。
  10. 为 sqlite3_open_v2() 添加了 SQLITE_OPEN_EXRESCODE 标志选项。
  11. 减少用于保存数据库模式的内存。
