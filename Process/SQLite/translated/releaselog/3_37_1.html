SQLite 版本 3.37.1 发布于 2021-12-30

从版本 3.37.0（2021-11-27）继承的更改：

  1. STRICT 表 提供了一种明确的数据类型管理方式， 适用于偏好这种方式的开发者。
  2. 当添加包含 CHECK 约束 或包含 NOT NULL 约束 的 生成列 的列时， ALTER TABLE ADD COLUMN 会对现有数据库行中的新约束进行检查， 仅在没有违反约束的情况下才继续。
  3. 添加了 PRAGMA table_list 语句。
  4. CLI 改进：
    1. 添加了 .connection 命令，允许 CLI 同时保持多个数据库连接打开。
    2. 添加了 --safe 命令行选项， 禁用可能导致超出命令行指定的单个数据库文件范围以外的副作用的 点命令 和 SQL 语句。
    3. 读取跨多行的 SQL 语句时的性能改进。
  5. 添加了 sqlite3_autovacuum_pages() 接口。
  6. sqlite3_deserialize() 没有（也从未）支持 TEMP 数据库。 这一限制现在在文档中注明。
  7. 查询优化器现在会省略子查询和视图上的 ORDER BY 子句， 只要删除这些子句不会改变查询语义。
  8. 对 generate_series 表值函数扩展进行了调整， 使得第一个参数（"START"）变为必需。 这样做是为了演示如何编写具有必需参数的表值函数。 通过 -DZERO_ARGUMENT_GENERATE_SERIES 编译时选项可恢复旧行为。
  9. 添加了新的 sqlite3_changes64() 和 sqlite3_total_changes64() 接口。
  10. 为 sqlite3_open_v2() 添加了 SQLITE_OPEN_EXRESCODE 标志选项。
  11. 使用更少内存来保存数据库架构。

此特定补丁版本 3.37.1（2021-12-30）中的更改：

  1. 修复了由 UPSERT 在 版本 3.35.0 中的增强引入的一个错误， 该错误可能导致某些罕见但有效的 SQL 生成不正确的字节码， 进而可能导致空指针解引用。
  2. 修复了在读取损坏数据库文件时 FTS5 中可能发生的越界读取。
  3. 提升了 CLI 中 --safe 选项的鲁棒性。
  4. 对 assert() 语句和测试用例进行了其他小修复。
