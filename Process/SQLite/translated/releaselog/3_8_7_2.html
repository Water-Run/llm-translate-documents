SQLite 3.8.7.2 发布于 2014-11-18

  1. 增强 ROLLBACK 命令，使只要模式保持不变，待处理的查询就可以继续。此前，ROLLBACK 会导致所有待处理查询因 SQLITE_ABORT 或 SQLITE_ABORT_ROLLBACK 错误而失败。如果 ROLLBACK 修改了模式，则仍会返回该错误。
  2. 错误修复：确保来自 OP_Column 的 NULL 结果完全为 NULL，并且未设置 MEM_Ephem 位。工单 094d39a4c95ee4。
  3. 错误修复：sqlite3_mprintf() 中的 %c 格式现在可以处理超过 70 的精度。
  4. 错误修复：不要自动从作为 IN 运算符右侧的 SELECT 中移除 DISTINCT 关键字，因为当 SELECT 中还包含 LIMIT 时需要保留它。工单 db87229497。
  5. SQLITE_SOURCE_ID: "2014-11-18 20:57:56 2ab564bf9655b7c7b97ab85cafc8a48329b27f93"
  6. sqlite3.c 的 SHA1: b2a68d5783f48dba6a8cb50d8bf69b238c5ec53a

一个完整的 SQLite 发布列表与年表也都可用。每次提交的详细历史可在SQLite 版本控制站点查阅。
