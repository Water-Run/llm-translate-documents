SQLite 3.25.3 发布于 2018-11-05

来自版本 3.25.0（2018-09-15）的先前更改：

  1. 新增对 窗口函数 的支持
  2. 增强了 ALTER TABLE 命令：
    1. 新增通过 ALTER TABLE table RENAME COLUMN oldname TO newname 在表中重命名列的支持。
    2. 修复了表重命名功能，使其也会更新在 触发器 和 视图 中引用被重命名表的地方。
  3. 查询优化器改进：
    1. 避免在聚合查询中不在聚合函数内且不属于 GROUP BY 子句的列进行不必要的加载。
    2. IN 提前终止优化：当在多列索引上执行查找并且在左边以外的列上使用 IN 运算符时，如果第一个 IN 值没有匹配行，那么在继续下一个 IN 值之前，先检查右侧列是否存在匹配行。
    3. 使用传递性尝试在 WHERE 子句中传播常量值。例如，将 “a=99 AND b=a” 转换为 “a=99 AND b=99”。
  4. 在 Unix 的 VFS 中对每个 inode 使用独立的互斥锁，而不是共享一个互斥锁，以在多线程环境中获得略微更好的并发性。
  5. 增强 PRAGMA integrity_check 命令，以便更好地检测页面空闲列表中的问题。
  6. 在 命令行 shell 的 “.dump” 命令中将无穷大输出为 1e999。
  7. 新增 SQLITE_FCNTL_DATA_VERSION 文件控制。
  8. 新增 Geopoly 模块

    Bug 修复：

  9. 在极为罕见的情况下，ORDER BY LIMIT 优化可能在预处理语句的字节码中导致无限循环，这由查询优化器中的一系列小缺陷共同引起。 修复了票据 9936b2fa443fec03ff25
  10. 在 UPSERT 中重新排列约束检查顺序时，确保在任何约束检查之前对插入内容执行亲和性转换。修复了票据 79cad5e4b2e219dd197242e9e。
  11. 避免在 CLI 的 “.stats on” 命令关闭后的 “.eqp full” 逻辑中使用已关闭的预处理语句。修复了票据 7be932dfa60a8a6b3b26bcf76。
  12. LIKE 优化在左操作数具有数值亲和性且右侧模式为 '/%' 或以 ESCAPE 字符开头时会生成错误字节码，从而得到错误结果。修复了票据 c94369cae9b561b1f996d0054b

来自版本 3.25.1（2018-09-18）的先前更改：

  1. 在 3.25.0 版本中为 ALTER TABLE 添加的额外健全性检查在某些情况下会产生误报， 当正在修改的表包含一个更新虚表的触发器时，误报会导致 ALTER TABLE 回滚，从而保持架构不变。 票据 b41031ea2b537237。
  2. 3.25.0 版本中修复的与 ORDER BY LIMIT 优化相关的无限循环在某些涉及窗口函数的查询中仍然无效。需要进一步修正。票据 510cde277783b5fb

来自版本 3.25.2（2018-09-25）的先前更改：

  1. 添加 PRAGMA legacy_alter_table=ON 命令，使 “ALTER TABLE RENAME” 命令行为与 SQLite 3.24.0 及更早版本相同：触发器和视图主体中对被重命名表的引用不会更新。该 PRAGMA 为期望旧有奇怪行为的旧程序提供兼容性解决方案。
  2. 修复了新 窗口函数 实现中的一个问题，当视图内部使用包含窗口函数的复杂表达式时会导致失效。
  3. 修复了与罕见配置相关的其他各种编译器警告和轻微问题。

此补丁发布版本 3.25.3（2018-11-05）的更改：

  1. 禁止在 CTE 的递归部分使用 窗口函数。票据 e8275b415a2f03bee
  2. 修复 typeof() 和 length() 在虚表上的行为。票据 69d642332d25aa3b7315a6d385
  3. 加强对故意损坏的数据库文件的防御措施。
  4. 修复查询规划器在使用具有冗余列的主键的行值表达式时出现的问题。票据 1a84668dcfdebaf12415d
  5. 修复查询规划器，使其在启用了 SQLITE_ENABLE_STAT4 编译选项的 LEFT JOIN 的 ON 子句中处理 IS NOT NULL 运算符时能正确工作。 65eb38f6e46de8c75e188a17ec
