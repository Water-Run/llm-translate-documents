SQLite 3.9.0 版本发布于 2015-10-14

    策略变更：

  1. SQLite 的版本编号约定已修订为采用新兴的语义化版本管理标准。

    新功能与改进：

  2. 在源代码树和集成包中添加了json1 扩展模块。通过SQLITE_ENABLE_JSON1编译时选项启用支持。
  3. 在集成包中添加了全文搜索版本 5 (FTS5)，并通过SQLITE_ENABLE_FTS5启用。FTS5 至少在再一个发布周期内都将视为“实验性”（可能存在不兼容更改）。
  4. CREATE VIEW 语句现在允许在视图名称后添加可选的列名列表。
  5. 新增了对表达式索引的支持。
  6. 在SELECT 语句的 FROM 子句中新增了对表值函数的支持。
  7. 新增了对同名虚拟表的支持。
  8. VIEW 现在可以在创建时引用未定义的表和函数，当在查询中使用该 VIEW 时会报告缺失的表和函数。
  9. 新增了sqlite3_value_subtype()和sqlite3_result_subtype()接口（由json1 扩展使用）。
  10. 查询优化器现在能够使用 WHERE 子句中包含 AND 连接项的部分索引。
  11. sqlite3_analyzer.exe 工具已更新，可报告每个 B 树的深度，并显示索引和 WITHOUT ROWID 表的平均分支因子。
  12. 增强了dbstat 虚拟表，以便在参数为要分析的模式时可以作为表值函数使用。

    其他更改：

  13. sqlite3_memory_alarm() 接口在 8 年来一直被弃用且未记录，现已变为无操作。

    重要修复：

  14. 修复了 SQLite 加密扩展中的一个严重错误：当VACUUM 命令改变加密随机数的大小时，可能导致数据库变得不可读且无法恢复。
  15. 在sqlite3_initialize() 的实现中添加了内存屏障，以帮助确保线程安全。
  16. 修复了OR 优化，使其始终忽略未使用索引的子计划。
  17. 不要对来自 LEFT JOIN 的 ON 或 USING 子句的项应用 WHERE 子句下推优化。此为票据c2a19d81652f40568c的修复。
  18. SQLITE_SOURCE_ID: "2015-10-14 12:29:53 a721fc0d89495518fe5612e2e3bbc60befd2e90d"
  19. sqlite3.c 的 SHA1: c03e47e152ddb9c342b84ffb39448bf4a2bd4288
