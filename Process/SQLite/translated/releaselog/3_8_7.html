SQLite 3.8.7 版本发布于 2014-10-17

    性能增强：

  1. 多项微优化使得相对于上一个版本，在相同的 CPU 周期内完成的工作量提升了 20.3%。自版本 3.8.0 起的累计性能提升为 61%。（在 Ubuntu 13.10 x64 上使用 gcc 4.8.1 和 -Os，并借助 cachegrind 对 speedtest1.c 工作负载进行测量。实际性能可能有所不同。）
  2. 排序器可以使用辅助线程来提升实时响应能力。此功能默认关闭，可通过 PRAGMA threads 命令或 SQLITE_DEFAULT_WORKER_THREADS 编译时选项启用。
  3. 增强了跳过扫描优化，使其能够跳过位于索引中间的条目，而不仅仅是索引的左侧。
  4. 改进了CAST 运算符的优化。
  5. 查询优化器在使用sqlite_stat4 信息评估计划成本方面进行多项改进。

    新功能：

  6. 新增带 64 位长度参数的接口：sqlite3_malloc64()、sqlite3_realloc64()、sqlite3_bind_blob64()、sqlite3_result_blob64()、sqlite3_bind_text64() 和 sqlite3_result_text64()。
  7. 新增返回从 sqlite3_malloc64() 及其变体分配的内存大小的接口 sqlite3_msize()。
  8. 在 sqlite3_limit() 及 PRAGMA threads 命令中新增 SQLITE_LIMIT_WORKER_THREADS 选项，以配置可用的工作线程数量。
  9. spellfix1 扩展允许应用程序可选地为每个 INSERT 指定 rowid。
  10. 新增 用户认证 扩展。

    错误修复：

  11. 修复了 部分索引 实现中的一个 bug：当在子查询或 视图 中使用部分索引时，可能导致错误结果。问题编号 98d973b8f5。
  12. 修复了查询优化器的一个 bug：当使用 DESC 索引来实现具有相同 GROUP BY 子句的 ORDER BY 时，可能导致表以错误的方向扫描（从而颠倒输出顺序）。问题编号 ba7cbfaedc7e6。
  13. 修复了 sqlite3_trace() 中的一个 bug：当需要重新准备 SQL 语句时，有时会无法打印该语句。问题编号 11d5aa455e0d98f3c1e6a08
  14. 修复了一个错误的 assert() 语句。问题编号 369d57fb8e5ccdff06f1

    测试、调试与分析的变化：

  15. 在启用了 SQLITE_DEBUG、SQLITE_ENABLE_SELECTTRACE 和 SQLITE_ENABLE_WHERETRACE 的情况下，命令行 shell 支持使用 ".selecttrace" 和 ".wheretrace" 命令显示 ASCII 艺术树形语法图。还提供了 sqlite3TreeViewExpr() 和 sqlite3TreeViewSelect() 接口，可在调试器中断点停住后调用以查看解析树。
  16. 不再支持 SQLITE_ENABLE_TREE_EXPLAIN。SELECTTRACE 机制能提供更实用的诊断信息。
  17. 为 命令行 shell 新增配置辅助内存使用的选项：--pagecache、--lookaside 和 --scratch。

  19. sqlite3.c 的 SHA1：56dcf5e931a9e1fa12fc2d600cd91d3bf9b639cd
