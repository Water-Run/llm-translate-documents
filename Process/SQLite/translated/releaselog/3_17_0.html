SQLite 3.17.0 发行于 2017-02-13

  1. 来自 R-Tree 扩展 的性能约提升 25%。
    1. 在可用时使用编译器内置函数（如 __builtin_bswap32() 或 _byteswap_ulong()）进行字节交换。
    2. 使用 sqlite3_blob 键/值访问对象而非 SQL 来获取 R-Tree 节点中的内容。
    3. 其他杂项改进，例如循环展开。
  2. 添加 SQLITE_DEFAULT_LOOKASIDE 编译时选项。
  3. 将默认的 lookaside 大小从 512,125 增加到 1200,100， 这样可以获得更好的性能，同时每个连接只增加 56KB 的额外内存。 对内存敏感的应用程序可以在编译、启动或运行时恢复旧的默认值。
  4. 在可用时使用编译器内置函数 __builtin_sub_overflow()、__builtin_add_overflow() 和 __builtin_mul_overflow()。 （所有编译器内置函数都可以通过 SQLITE_DISABLE_INTRINSIC 编译时选项禁用。）
  5. 添加 SQLITE_ENABLE_NULL_TRIM 编译时选项， 在某些应用中可以显著减小数据库文件，但可能与旧版本 SQLite 不兼容。
  6. 将 SQLITE_DEFAULT_PCACHE_INITSZ 从 100 改为 20，以提升性能。
  7. 添加 SQLITE_UINT64_TYPE 编译时选项，作为 SQLITE_INT64_TYPE 的对应项。
  8. 部分 UPDATE 操作改为单遍处理，而不是两遍。
  9. 增强 session 扩展，以支持 WITHOUT ROWID 表。
  10. 修复在使用数十万行的多行 VALUES 子句创建 视图 时的性能问题和潜在栈溢出。
  11. 添加了 sha1.c 扩展。
  12. 在 命令行界面 中增强 “.mode” 命令，使其在 “line”、 “list”、 “column” 和 “tcl” 模式下恢复默认的列和行分隔符。
  13. 增强 SQLITE_DIRECT_OVERFLOW_READ 选项，使其在 WAL 模式 下工作，只要被读取的页面不在 WAL 文件中即可。
  14. 增强 Lemon 语法分析器生成器，使其可以将解析器对象存储为栈变量，而不是从堆上分配空间，并在 amalgamation 中利用这一改进。
  15. 其他性能提升。约节省 6.5% 的 CPU 周期。

    错误修复：

  16. 如果 LEFT JOIN 的 ON 子句引用了 ON 子句右侧的表，则抛出错误。这与 PostgreSQL 的行为一致。此前 SQLite 会悄悄地将 LEFT JOIN 转换为 INNER JOIN。修复来自工单 25e335f802dd。
  17. 为自动索引的列使用正确的亲和性。工单 7ffd1ca1d2ad4ec。
  18. 确保 sqlite3_blob_reopen() 接口能正确处理短行。修复来自工单 e6e962d6b0f06f46e。
