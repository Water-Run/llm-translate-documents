SQLite 发布 3.50.4（2025-07-30）

版本 3.50.0（2025-05-29）之前的更改：

  1. 添加 sqlite3_setlk_timeout() 接口，为支持阻塞锁的构建设置一个与 sqlite3_busy_timeout() 不同的单独超时。
  2. 对 SQLITE_DBCONFIG_ENABLE_COMMENTS 限制（在 前一个版本 中添加）略作放宽，从现有 sqlite_schema 表中读取架构时始终允许注释。新的 SQL 中才会禁止注释。
  3. 新增 SQL 函数：
    1. unistr()
    2. unistr_quote()
  4. 对于内置 printf() 中的 %Q 和 %q 转换（涵盖 sqlite3_mprintf() API、format() SQL 函数 等），alternate-form-1 标志 (“#”) 使 控制字符 转换为适合 unistr() 的反斜杠转义序列。
  5. CLI 增强：
    1. 避免直接输出大多数 控制字符。
    2. 除非 --escape 模式关闭，否则 .dump 命令 的输出使用新的 unistr() SQL 函数对特殊字符进行编码。
    3. 改进 ".schema --indent" 命令 输出中复杂部分索引的格式。
  6. 对 sqlite3_rsync 的增强：
    1. 取消了数据库必须处于 WAL 模式的要求。
    2. 同步协议得到加强，当双方初始非常相似时减少网络带宽使用。
    3. 只要在以下目录之一安装 sqlite3_rsync 可执行文件： $HOME/bin:/usr/local/bin:/opt/homebrew/bin，即可在 Mac 上使用 sqlite3_rsync 程序，而无需在远程端指定 sqlite3_rsync 可执行文件的完整路径。
  7. JSON 函数的更改：
    1. 错误修复：强制执行 JSON5 的限制，即 "\0" 转义后不得紧跟数字。
    2. 错误修复：当 json_group_object(LABEL,VALUE) 的 LABEL 参数为 NULL 时，结果对象中省略该元素。
    3. 优化：如果 jsonb_set() 或 jsonb_replace() 函数在大型 JSONB 对象内部进行了更改，它们尽力保持 JSONB 大小不变，并尽量少修改内部字节，从而只需写入包含更改字节的页面，而不是所有周围页面，从而减少 I/O。
  8. 改进了在 Cygwin、MinGW 等以及 Termux 上的构建支持。
  9. 修复文档与源代码注释中的错别字。
  10. 其他性能改进。
  11. JavaScript/WASM：
    1. 修复了 OPFS SAHPool VFS 中长期存在的文件名摘要计算错误。3.50.0+ 在该 VFS 中创建的数据库无法被旧版本 VFS 读取，但 3.50.0 可以向后兼容地与旧版本创建的现有数据库一起使用。

版本 3.50.1（2025-06-06）之前的更改：

  1. 修复 jsonb_set() 及类似函数中的长期存在的 bug，该 bug 是由 3.50.0 中新增优化暴露的。
  2. 修复在使用 -DSQLITE_DEFAULT_MEMSTATUS=0 构建时可能出现的看似无害的 ASAN 警告。
  3. 修复 sqlite3_rsync 中的 off-by-one 错误，该错误可能导致复制数据库的最后一页未被传输。
  4. 查询规划器优化：允许 LEFT JOIN 的右侧即使是虚拟表也可以被展开。
  5. 修复 sqlite3_setlk_timeout()，在打开快照事务以及被运行恢复的其他进程阻塞时使用阻塞锁。
  6. 其他在 3.50.0 发布后报告的小修复。

版本 3.50.2（2025-06-28）之前的更改：

  1. 修复 concat_ws() SQL 函数，使其在串联时包含空字符串。 论坛帖子 52503ac21d。
  2. 修复 CLI 使用的文件 I/O 扩展，使其可使用 MinGW 编译链构建。
  3. 如果在脏页已经溢出到 wal 文件之后回滚 savepoint，则避免将没有校验和的帧写入 wal 文件。 论坛帖子 b490f726db。
  4. 修复 Bitvec 对象，以避免在数据库接近最大大小 60 页以内时发生栈溢出。
  5. 修复包含 BLOB 值的 fts5 表上的 UPDATE 问题。
  6. 修复 RIGHT JOIN 上传递 IS 约束的问题。
  7. 如果查询中的聚合项数量超过最大列数，则提前抛出错误，以避免后续的断言故障。
  8. 确保 sqlite3_setlk_timeout() 持有数据库互斥锁。
  9. 修复 API 文档中的错别字。

版本 3.50.3（2025-07-17）之前的更改：

  1. 修复一个可能的内存错误，该错误会在针对以特定方式故意损坏的 FTS5 索引执行查询时发生。
  2. 修复解析器，使其在 CREATE TRIGGER 语句的所有位置都忽略 SQL 注释。该问题由 3.49.0 中引入的 SQLITE_DBCONFIG_ENABLE_COMMENTS 功能引起。
  3. 修复由于 AND 运算符过度优化导致的错误结果。 论坛帖子 f4878de3e。
  4. 修复小的 makefile 问题和文档错别字。
