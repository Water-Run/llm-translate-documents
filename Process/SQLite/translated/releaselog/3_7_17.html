SQLite 3.7.17 版本于 2013-05-20 发布

  1. 增加对 内存映射 I/O 的支持。
  2. 新增 sqlite3_strglob() 便捷接口。
  3. 将 数据库头 中偏移 68 处的整数指定为当 SQLite 被用作 应用程序文件格式 时的 应用程序 ID。新增 PRAGMA application_id 命令以查询和设置该应用程序 ID。
  4. 在 错误日志 中，将回滚恢复报告为 SQLITE_NOTICE_RECOVER_ROLLBACK。将 WAL 恢复的错误日志代码从 SQLITE_OK 改为 SQLITE_NOTICE_RECOVER_WAL。
  5. 在 错误日志 中，将对 未链接数据库文件 和 数据库文件名别名 的危险用法报告为 SQLITE_WARNING 消息。
  6. 新增 SQLITE_TRACE_SIZE_LIMIT 编译时选项。
  7. 将 SQLITE_MAX_SCHEMA_RETRY 的默认值增加到 50，并确保在所有可能迫使语句重试的模式更改处都尊重该值。
  8. 新增名为 “mptester” 的测试框架，用于验证多个进程同时使用同一数据库文件时的正确运行。
  9. 增强 扩展加载 机制，使之在保持向下兼容的同时更加灵活，方式如下：
    1. 如果可加载扩展中没有默认入口点 “sqlite3_extension_init”，则尝试基于共享库文件名的入口点 “sqlite3_X_init”。这样每个扩展可以有不同的入口点，从而无需修改代码即可静态链接。
    2. 传递给 sqlite3_load_extension() 的共享库文件名可以省略后缀，系统会自动添加适当的与架构相关的后缀（“.so”、“ .dylib” 或 “.dll”）。
  10. 向源树中新增多个可加载扩展，包括 amatch、closure、fuzzer、ieee754、nextchar、regexp、spellfix 和 wholenumber。有关每个扩展的功能，请参阅其源文件头部注释。
  11. 增强 FTS3，以避免在 MATCH 运算符右侧存在大量术语时使用过多栈空间。此更改的副作用是 MATCH 运算符一次最多只能容纳 12 个 NEAR 运算符。
  12. 增强 fts4aux 虚拟表，使其可以成为 TEMP 表。
  13. 将 fts3tokenize 虚拟表 添加到 全文搜索 逻辑中。
  14. 查询规划器增强：尽可能利用约束的传递性，将约束移动到连接的外层循环中，从而减少内层循环需要处理的工作量。
  15. 在 Unix 上停止使用 posix_fallocate()，因为它并不适用于所有文件系统。
  16. 改进 Windows VFS 的跟踪和调试功能。
  17. 漏洞修复：在一个 数据库连接 关闭而另一个正在进行写事务时的 共享缓存模式 中，修复一个潜在的 数据库损坏漏洞。票据 e636a050b7
  18. 漏洞修复：仅在没有其他匹配项时，才将结果集中的 AS 名称视为解析 WHERE 子句标识符的候选。在 ORDER BY 子句中，AS 名称优先于任何列名。票据 2500cdb9be05
  19. 漏洞修复：除非所有外层循环都保证只返回不超过一行结果，否则不要允许虚拟表取消 ORDER BY 子句。票据 ba82a4a41eac1。
  20. 漏洞修复：在虚拟表查询中使用 IN 约束时，不要压制 ORDER BY 子句。票据 f69b96e3076e。
  21. 漏洞修复：命令行 Shell 通过 “.quit” 命令终止时会返回 0 退出码的情况。
  22. 漏洞修复：确保 PRAGMA 语句出现在 sqlite3_trace() 输出中。
  23. 漏洞修复：当复合查询使用带有 COLLATE 运算符 的 ORDER BY 子句时，确保排序按照指定的排序规则进行，并且与复合查询关联的比较使用本地排序规则。票据 6709574d2a8d8。
  24. 漏洞修复：在执行更改 rowid 的 UPDATE 时，确保 授权器 回调为列名参数提供指向字符串 “ROWID” 的有效指针。票据 0eb70d77cb05bb2272
  25. 漏洞修复：不要将 WHERE 子句项移入包含在 LEFT JOIN 的 ON 子句内的 OR 表达式中。票据 f2369304e4
  26. 漏洞修复：当尝试执行需要但缺少 排序序列 的操作时，确保始终报告错误。票据 0fc59f908b
