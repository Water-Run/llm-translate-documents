SQLite 3.27.1 发行于 2019-02-08

来自 3.27.0 版本（2019-02-07）的前置更改：

  1. 添加了 VACUUM INTO 命令
  2. 如果使用了 双引号字符串字面量，则在 错误日志 上发出 SQLITE_WARNING 消息。
  3. sqlite3_normalized_sql() 接口可以用于任何通过 sqlite3_prepare_v2() 或 sqlite3_prepare_v3() 创建的预处理语句。现在无需为使用 sqlite3_normalized_sql() 而在 sqlite3_prepare_v3() 中使用 SQLITE_PREPARE_NORMALIZE。
  4. 为 FTS3 与 FTS5 添加了 remove_diacritics=2 选项。
  5. 为 sqlite3_prepare_v3() 添加了 SQLITE_PREPARE_NO_VTAB 选项。使用该选项可防止对 影子表 的循环引用导致资源泄漏。
  6. 对 sqlite3_deserialize() 接口的增强：
    1. 添加了 SQLITE_FCNTL_SIZE_LIMIT 文件控制 接口，用于设定 sqlite3_deserialize 创建的内存数据库大小的上限。默认上限为 1GiB，或由 sqlite3_config(SQLITE_CONFIG_MEMDB_MAXSIZE) 和/或 SQLITE_MEMDB_DEFAULT_MAXSIZE 指定的其他值。
    2. 尊重文档中已描述但之前无效的 SQLITE_DESERIALIZE_READONLY 标志。
    3. 增强 TCL 接口 的 “deserialize” 命令，新增 "--maxsize N" 和 "--readonly BOOLEAN" 选项。
  7. 对 CLI 的增强，主要用于支持 SQLite 库自身的测试与调试：
    1. 为 ".open --hexdb" 添加支持。用于生成 “hexdb” 文本的 "dbtotxt" 工具已加入源代码树。
    2. 为 ".open --deserialize" 添加 "--maxsize N" 选项。
    3. 添加 "--memtrace" 命令行选项，用于显示所有内存分配与释放。
    4. 在启用 SQLITE_DEBUG 的构建中添加 ".eqp trace" 选项，以便一步完成字节码程序列表（含缩进）和 PRAGMA vdbe_trace。
    5. 添加 ".progress" 命令以访问 sqlite3_progress_handler() 接口。
    6. 为 ".backup" 命令添加 "--async" 选项。
    7. 为 ".trace" 命令添加 "--expanded"、"--normalized"、"--plain"、"--profile"、"--row"、"--stmt" 和 "--close" 等选项。
  8. 提高了对恶意 SQL 与恶意破坏数据库交互的鲁棒性。

    错误修复：

  9. 在 IN 运算符上进行表扫描时不再使用部分索引。工单 1d958d90596593a774。
  10. 修复了 查询扁平化器，使其能处理包含使用 窗口函数 的子查询的语句。工单 709fcd17810f65f717
  11. 确保 ALTER TABLE 能修改视图与触发器中 WITH 子句嵌入的表名与列名。
  12. 修复了防止使用括号包裹表值函数的解析器错误。
  13. 修复了 表达式索引 上的 OR 优化 的问题。工单 d96eba87698a428c1d。
  14. 修复了 LEFT JOIN 强度缩减优化 中由于 IS NOT NULL 运算符导致的不当应用问题。工单 5948e09b8c415bc45d。
  15. 修复了 REPLACE 命令使其不再在 NOT NULL 列（即便默认值为 NULL）中悄悄插入 NULL 值。工单 e6f1f2e34dceeb1ed6
  16. 修复了在 相关子查询 中使用 窗口函数 的问题。工单 d0866b26f83e9c55e3
  17. 修复了 ALTER TABLE RENAME COLUMN 命令，使其适用于具有冗余 UNIQUE 约束的表。工单 bc8d94f0fbd633fd9a
  18. 修复了一个在插入使用 表达式索引 的表时导致 zeroblob 值被截断的错误。工单 bb4bdb9f7f654b0bb9

在该补丁版本 3.27.1（2019-02-08）中的更改：

  1. 修复了查询优化器的一个错误：OR 优化 与试图直接使用 表达式索引 中读取的值而不重新计算表达式的优化之间存在不良交互。工单 4e8e4857d32d401f
