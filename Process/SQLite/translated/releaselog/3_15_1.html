SQLite 3.15.1 版本于 2016-11-04 发布

此前版本 3.15.0（2016-10-14）的更改：

  1. 添加对 行值 的支持。
  2. 允许在 部分索引 的 WHERE 子句中使用 确定性 SQL 函数。
  3. 在 unix VFS 上添加了 "modeof=filename" URI 参数
  4. 添加对 SQLITE_DBCONFIG_MAINDBNAME 的支持。
  5. 添加了对 附加 数据库执行 VACUUM 的能力。
  6. 命令行 shell 的增强：
    1. 添加“.testcase”和“.check”点命令。
    2. 在 ".open" 点命令中添加 --new 选项，使得在打开之前会清除数据库中的任何先前内容。
  7. 增强 fts5vocab 虚拟表以高效处理 "ORDER BY term"。
  8. 各种微小优化使得常见负载下的 CPU 使用降低超过 7%。本版本的大部分优化集中在前端的 sqlite3_prepare_v2()。

    错误修复：

  9. 乘法运算符现在能在所有边界情况下正确检测到 64 位整数溢出，并提升为浮点数。修复了票据 1ec41379c9c1e400。
  10. 在使用具有冗余唯一索引的列作为 IN 操作符 左侧时，进行了正确处理。修复了票据 0eab1ac759。
  11. 在 表达式索引 的范围查询中跳过 NULL 条目。修复了票据 4baa46491212947。
  12. 确保 sqlite_sequence 表中的 AUTOINCREMENT 计数器在对 "INSERT ... SELECT" 语句执行“Xfer 优化”时得到初始化。修复了票据 7b3328086a5c116c。
  13. 确保来自检入 559733b09e 的 ORDER BY LIMIT 优化在 INTEGER PRIMARY KEY 上的 IN 操作符下也能工作。修复了票据 96c1454c

本补丁版本 3.15.1（2016-11-04）的更改：

  1. 添加 SQLITE_FCNTL_WIN32_GET_HANDLE 文件控制操作码。
  2. 修复 VACUUM 命令，使其将多余内容溢写到磁盘，而不是全部保存在内存中，从而避免在较大数据库文件上可能出现的内存不足错误。此项修复了解 3.15.0 版本引入的问题。
  3. 修复了一个自 3.8.0（2013-08-26）以来就存在的问题，即在 LEFT JOIN 的 ON 子句中 OR 连接的条件可能导致错误结果。票据 34a579141b2c5ac。
  4. 修复了一个在 LEFT JOIN 的 ON 子句中使用 行值 可能造成错误结果的情况。票据 fef4bb4bd9185ec8f。
