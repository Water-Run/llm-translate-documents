SQLite Release 3.2.3 On 2005-08-21

  1. 新增对 CAST 运算符的支持
  2. Tcl 接口允许将 BLOB 值传递给用户定义的函数
  3. 为 Tcl 接口新增了 “transaction” 方法
  4. 允许列的 DEFAULT 值调用具有常量操作数的函数
  5. 新增 ANALYZE 命令，用于收集索引统计信息并在优化器选择索引时使用这些统计数据
  6. 取消 WHERE 子句中项数的限制（原为 100）
  7. IN 操作符的右侧现在可以是表达式列表，而不仅仅是常量列表
  8. 重新设计优化器，使其能够更好地利用索引
  9. 连接中的表顺序会自动调整，以更好地利用索引
  10. 即使 IN 操作符的左侧不是索引的最左项，它也可以成为优化候选。多个 IN 操作符可以共享同一索引。
  11. 使用 BETWEEN 和 OR 的 WHERE 子句表达式现在也是优化候选
  12. 新增 case_sensitive_like pragma 及 SQLITE_CASE_SENSITIVE_LIKE 编译期选项，用于将其默认值设为 “on”
  13. 在启用 case_sensitive_like pragma 时，也使用索引来辅助 GLOB 和 LIKE 表达式
  14. 新增对反引号引用的支持，以兼容 MySQL
  15. 提升测试覆盖率
  16. 修复了数十个小错误

SQLite 发布的完整列表 （单页）和一个年表也都可用。 每次提交的详细历史可在 SQLite 版本控制站点查看。
