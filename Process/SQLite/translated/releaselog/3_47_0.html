SQLite 发布 3.47.0 于 2024-10-21

  1. 允许在 RAISE 函数 的第二个参数中使用任意表达式。
  2. 如果 ->> 运算符 的右侧值为负数，则从右侧开始计数访问数组元素。
  3. 修复在鲜为人知的 unix-dotfile VFS 中回滚热日志文件时的问题。
  4. 即使 FTS5 表使用未注册的非标准分词器，也可以删除该表。
  5. 修复 group_concat() 聚合函数，使其在仅接收到一个空字符串输入时返回空字符串，而不是 NULL。
  6. 增强 generate_series() 表值函数，使其能够识别并利用对输出值的约束。
  7. 预更新钩子现在能识别通过 ALTER TABLE ADD COLUMN 添加的列是否具有非空默认值。
  8. 性能优化：
    1. 改进与 IN 运算符 关联的子查询的重用，尤其是在由于 谓词下推 导致 IN 运算符被复制的情况下。
    2. 在右侧子查询上使用 Bloom 过滤器（IN 运算符），在有助于提升性能的场景中启用。
    3. 确保像 “SELECT func(a) FROM tab GROUP BY 1” 这样的查询每行只调用一次 func() 函数。
    4. 不会尝试在因为在其他已分析的索引中使用而已知无选择性的列上创建 自动索引。
    5. 调整查询规划器，以便为具有大量维度表的星型查询生成更优计划。
    6. 添加 “order-by-subquery” 优化，尝试在子查询中的 ORDER BY 子句自然得到所需顺序时禁用外层查询的排序操作。
    7. “indexed-subtype-expr” 优化努力使用作为索引一部分的表达式，而不是基于表值重新计算该表达式，只要查询规划器能够证明表达式的 子类型 永不会被使用。
    8. 其他编码细节调整，以获得更快的运行时间。
  9. 与 SQLite 相关的命令行程序增强：
    1. 添加实验性的 sqlite3_rsync 程序。
    2. 向 CLI 添加扩展函数 median()、percentile()、percentile_cont() 和 percentile_disc()。
    3. 向 CLI 添加 .www 点命令。
    4. sqlite3_analyzer 工具现在提供 WITHOUT ROWID 表的统计细分。
    5. sqldiff 工具在其第二个参数不存在时避免创建空数据库。
  10. 增强 sqlite_dbpage 表值函数，使得可以使用 INSERT 增加或减少数据库文件大小。
  11. SQLite 不再使用 “long double” 数据类型，因为硬件对 long double 的支持越来越少，而且 long double 会给某些编译工具链带来挑战。需要扩展精度时，SQLite 改为使用 Dekker 算法。
  12. SQLite 的 TCL 接口 支持 TCL9。虽然不保证但很可能仍然支持 TCL 8.5 及更高版本。建议用户升级到 TCL9。
  13. JavaScript/WASM：
    1. 修复 JavaScript “opfs” VFS 中导致损坏的错误。
    2. 修正 “opfs” VFS 对 “mode=ro” 的处理。
    3. 解决几个浏览器特有的 OPFS 奇异问题。
  14. FTS5 更改：
    1. 添加 fts5_tokenizer_v2 API 和 locale=1 选项，用于创建自定义的本地化分词器以及可以利用它们的 FTS5 表。
    2. 添加 contentless_unindexed=1 选项，用于创建将任何 UNINDEXED 列的值持久存储在数据库中的无内容 FTS5 表。
    3. 即使使用自定义分词器但其实现不可用，也允许删除 FTS5 表。
