SQLite 3.32.2 发布于 2020-06-04

来自 3.32.0 版本（2020-05-22）的先前更改：

  1. 添加了使用 近似 ANALYZE 的支持，可通过 PRAGMA analysis_limit 命令控制。
  2. 添加了 字节码虚拟表。
  3. 将 checksum VFS shim 添加到源代码树中包含的可运行时加载扩展集中。
  4. 添加了 iif() SQL 函数。
  5. INSERT 和 UPDATE 语句现在始终在计算 CHECK 约束 之前应用 列亲和性。 此次缺陷修复理论上可能会给具有非典型 CHECK 约束的旧数据库带来问题，这些约束要求 INSERT 的输入类型与声明的列类型不同。 请参见票据 86ba67afafded936 了解更多信息。
  6. 添加了 sqlite3_create_filename()、sqlite3_free_filename() 和 sqlite3_database_file_object() 接口， 以更好地支持 VFS shim 实现。
  7. 将参数个数的 默认上限 从 999 提高到 32766。
  8. 为 UINT 排序序列 添加了代码，作为可选的 可加载扩展。
  9. CLI 的增强：
    1. 为 .import 命令添加了选项：--csv、--ascii、--skip
    2. .dump 命令现在接受多个 LIKE 模式参数，并输出所有匹配表的并集。
    3. 在调试构建中添加 .oom 命令
    4. 为 .excel、.output 和 .once 命令添加 --bom 选项。
    5. 增强 .filectrl 命令以支持 --schema 选项。
    6. UINT 排序序列 扩展会自动加载
  10. LIKE 操作符的 ESCAPE 子句现在会覆盖通配符字符，行为与 PostgreSQL 保持一致。

来自 3.32.1 版本（2020-05-25）的先前更改：

  1. 修复了两个长期存在的漏洞，攻击者可以利用恶意 SQL 语句让运行 SQLite 的进程崩溃。 这些漏洞在 3.32.0 发布约 24 小时后由第三方发布，但并非该版本特有。
  2. 其它一些次要的编译器警告修复等。
