SQLite 3.41.1 发布于 2023-03-10

来自 3.41.0 版本（2023-02-21）之前的更改：

  1. 查询规划器改进：
    1. 在包含 GROUP BY 子句的聚合查询中利用索引表达式。
    2. 查询规划器在判断索引是否为覆盖索引时更加敏感，并相应调整预测运行时间。
    3. 查询规划器更积极使用协程，而不是展开子查询和视图。
    4. 针对内置表值函数 json_tree() 与 json_each() 的查询通常会将“ORDER BY rowid”视为无效操作。
    5. 提升查询规划器在表达式经过常量传播优化修改后依然能够使用索引表达式的能力。 （参见论坛讨论 0a539c7。）
  2. 添加内置的 unhex() SQL 函数。
  3. 将 base64 和 base85 应用定义函数作为扩展，并在CLI中包含该扩展。
  4. 新增 sqlite3_stmt_scanstatus_v2() 接口。（仅在使用SQLITE_ENABLE_STMT_SCANSTATUS编译时可用。）
  5. 使用 sqlite3_deserialize() 创建的内存数据库现在将其文件名报告为空字符串，而非“x”。
  6. CLI 的更改：
    1. 添加新的 base64() 与 base85() SQL 函数。
    2. 在使用SQLITE_ENABLE_STMT_SCANSTATUS编译时，使用新的sqlite3_stmt_scanstatus_v2()接口增强EXPLAIN QUERY PLAN输出。
    3. “.scanstats est”命令在分析报告中提供查询规划器估算值。
    4. 续行提示会指出当前输入是否处于字符串字面量、标识符字面量、注释、触发器定义等上下文。
    5. 增强 --safe 命令行选项以禁止危险的 SQL 函数。
    6. CLI 构建默认禁用双引号字符串异常。遗留用例可在运行时通过 “.dbconfig dqs_dml on”与“.dbconfig dqs_ddl on”命令重新启用该异常。
  7. 增强PRAGMA integrity_check命令，使其能够检测表中字符串与索引中的相同字符串在内容等价但字节不完全相同时的情况。
  8. 增强carray 表值函数，使其能够绑定一个 BLOB 对象数组。
  9. 新增sqlite3_is_interrupted()接口。
  10. 对sqlite3_prepare()及类似函数的长时间调用现在会触发进度处理程序回调并响应sqlite3_interrupt()。
  11. sqlite3_vtab_in_first()与sqlite3_vtab_in_next()函数经过增强， 能够可靠地检测是否在没有为多值 IN 处理使用sqlite3_vtab_in()所选参数上调用。 这种情况下会返回 SQLITE_ERROR，而非 SQLITE_MISUSE。
  12. 解析器现在会忽略 IN 运算符右侧子查询外包的多余括号，使 SQLite 在此方面与 PostgreSQL 行为一致。 之前，SQLite 会将该子查询视为带有隐式“LIMIT 1”的表达式。
  13. 为sqlite3_file_control() API 添加了SQLITE_FCNTL_RESET_CACHE选项。
  14. Makefile 改进：
    1. 新的 makefile 目标“sqlite3r.c”构建一个包含恢复扩展的融合源。
    2. 新增 makefile 目标“devtest”与“releasetest”，分别用于在提交前运行快速开发测试及进行完整发布测试。
  15. 其他性能增强。

此补丁发布（版本 3.41.1，2023-03-10）的变更：

  1. 提供编译时选项 -DHAVE_LOG2=0 与 -DHAVE_LOG10=0，以便 SQLite 可以在缺失标准库函数 log2() 与 log10() 的系统上编译。
  2. 确保在“CREATE TABLE t1 AS SELECT CAST(7 AS INT) AS x;”中，列 t1.x 的数据类型仍为 INT 而非 NUM，以保持历史兼容性。
  3. 增强PRAGMA integrity_check，使其能检测索引记录末尾出现的多余字节。
  4. 修复用户社区报告的各种晦涩错误。详情请参阅 变更时间线。
