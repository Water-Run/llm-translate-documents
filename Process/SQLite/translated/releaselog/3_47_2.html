SQLite 3.47.2 发布于 2024-12-07

源自版本 3.47.0（2024-10-21）的先前更改：

  1. 允许在 RAISE 函数 的第二个参数中使用任意表达式。
  2. 如果 ->> 操作符 的右侧为负，则从右侧开始计数以访问数组元素。
  3. 修复在很少使用的 unix-dotfile VFS 中回滚热日志文件时的问题。
  4. 即使使用尚未注册的非标准分词器，也现在可以删除 FTS5 表。
  5. 修复 group_concat() 聚合函数，使其在收到唯一输入值且该值为空字符串时返回空字符串而非 NULL。
  6. 增强 generate_series() 表值函数，使其能够识别并利用其输出值的约束。
  7. 预更新钩子现在可以识别 ALTER TABLE ADD COLUMN 添加的列具有非空默认值的情况。
  8. 性能优化：
    1. 改进与 IN 运算符 相关的子查询的重用，尤其是在由于 谓词下推 而导致 IN 运算符被复制时。
    2. 在 IN 运算符 右侧的子查询上使用 Bloom 过滤器，在可能提升性能的情况下。
    3. 确保类似 "SELECT func(a) FROM tab GROUP BY 1" 的查询每行仅调用一次 func() 函数。
    4. 不再尝试在因在其他已分析的索引中使用而已知为非选择性的列上创建 自动索引。
    5. 对查询规划器进行调整，使其能为包含大量维度表的星型查询生成更优计划。
    6. 新增 “order-by-subquery” 优化，当子查询的 ORDER BY 子句自然产生所需排序时，尝试在外层查询中禁用排序操作。
    7. “indexed-subtype-expr” 优化努力使用属于索引的一部分的表达式，而不是根据表值重新计算表达式，只要查询规划器能证明该表达式的 子类型 永远不会被使用。
    8. 各种编码调整以实现更快的运行时间。
  9. SQLite 相关命令行程序的增强：
    1. 新增实验性 sqlite3_rsync 程序。
    2. 在 CLI 中新增扩展函数 median()、percentile()、percentile_cont() 和 percentile_disc()。
    3. 在 CLI 中新增 .www dot-command。
    4. sqlite3_analyzer 实用程序现在提供 WITHOUT ROWID 表的统计细分。
    5. sqldiff 实用程序在其第二个参数不存在时避免创建空数据库。
  10. 增强 sqlite_dbpage 表值函数，使得可以使用 INSERT 来增加或减少数据库文件的大小。
  11. SQLite 不再使用 “long double” 数据类型，因为硬件对 long double 的支持日益减少且 long double 给某些编译工具链带来挑战。当需要扩展精度时，SQLite 改用 Dekker 算法。
  12. SQLite 的 TCL 接口 支持 TCL9。虽然无明确保障，但 TCL 8.5 及更高版本很可能仍可正常使用。鼓励用户升级到 TCL9。
  13. JavaScript/WASM：
    1. 修复 JavaScript “opfs” VFS 中导致损坏的错误。
    2. 修正 “opfs” VFS 中对 “mode=ro” 的处理。
    3. 绕过一些特定浏览器的 OPFS 异常行为。
  14. FTS5 更改：
    1. 新增 fts5_tokenizer_v2 API 和 locale=1 选项，用于创建可利用自定义语言环境感知分词器的 fts5 表。
    2. 新增 contentless_unindexed=1 选项，用于创建无内容的 fts5 表，将任何 UNINDEXED 列的值持久存储在数据库中。
    3. 即使 FTS5 表使用的自定义分词器实现不可用，也允许删除该表。

源自版本 3.47.1（2024-11-25）的先前更改：

  1. 修复 makefile，使其在 “install” 目标中再次尊重 DESTDIR。
  2. 为 VFS 添加 SQLITE_IOCAP_SUBPAGE_READ 能力，以绕过在某些非标准 VFS 上因在 3.45.0 版本中将 SQLITE_DIRECT_OVERFLOW_READ 设为默认值而产生的问题。
  3. 修复新 sqlite3_rsync.exe 工具在 Windows 上的行尾问题。
  4. 修复 3.47.0 发布新增的查询优化导致某些晦涩 IN 查询返回错误结果的问题。
  5. 其他一些小错误修复。

本次补丁版本 3.47.2（2024-12-07）的更改：

  1. 修复 SQLite 中的文本到浮点转换问题，该问题会导致对于任意指数 NNN，介于 '1.8446744073709550592eNNN' 与 '1.8446744073709551609eNNN' 之间的数值被错误呈现。换句话说，某些前 16 位有效数字为 '1844674407370955' 的数值文本可能会被转换成错误的浮点值。详见 论坛帖子 569a7209179a7f5e。该问题仅在 x64 与 i386 硬件上出现，并在 3.47.0 中引入。
  2. 其他一些小错误修复。
