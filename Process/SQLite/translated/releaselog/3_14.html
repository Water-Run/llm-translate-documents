SQLite 3.14 发行于 2016-08-08


    用自制派庆祝 SQLite "π 发布"。
  1. 新增对 WITHOUT ROWID 虚表 的支持。
  2. 改进查询规划器，使得 OR 优化 可以用于虚表，即使一个或多个析取式使用 LIKE、GLOB、REGEXP、MATCH 操作符。
  3. 新增 CSV 虚表，用于读取符合 RFC 4180 格式的逗号分隔值文件。
  4. 新增 carray() 表值函数 扩展。
  5. 通过扩展入口点的新SQLITE_OK_LOAD_PERMANENTLY 返回码启用持久可加载扩展。
  6. 在sqlite3_db_status() 中新增 SQLITE_DBSTATUS_CACHE_USED_SHARED 选项。
  7. 新增vfsstat.c 可加载扩展 —— 一个测量 I/O 的 VFS 垫片，并附带同名虚表以访问测量结果。
  8. 改进了处理同时包含 ORDER BY 和 LIMIT 的查询的算法，使得仅最内层循环自然按正确顺序生成行。
  9. 增强 Lemon 解析器生成器，使其生成更快的解析器。
  10. PRAGMA compile_options 命令现在尝试显示生成该库的编译器版本号。
  11. 增强 PRAGMA table_info，使其提供关于同名虚表的信息。
  12. 新增与“unix-none”VFS 类似的“win32-none”VFS，其行为与默认“win32”VFS 相同，但忽略所有文件锁。
  13. 在合适的情况下，查询规划器使用对部分索引的全表扫描，而不是主表的全表扫描。
  14. 允许表值函数出现在IN 操作符的右侧。
  15. 新增 dbhash.exe 命令行工具。
  16. 新增两个 C 语言接口：sqlite3_expanded_sql() 和sqlite3_trace_v2()。这两个新接口取代了现在已弃用的sqlite3_trace() 与sqlite3_profile() 函数。
  17. 在json1 扩展中新增 json_quote() SQL 函数。
  18. 在重新解析模式时禁用授权器回调。
  19. 新增SQLITE_ENABLE_UNKNOWN_SQL_FUNCTION 编译时选项，并在构建命令行 shell时默认启用该选项。

    错误修复：

  20. 修复ALTER TABLE 命令，以便在向旧版文件格式数据库添加列时不会破坏降序索引。票据f68bf68513a1c15f
  21. 修复在传递性 WHERE 子句引用不存在的排序序列时可能发生的空指针解引用/崩溃。票据e8d439c77685eca6。
  22. 改进包含 WHERE 子句的索引扫描的成本估算，该 WHERE 子句可以使用索引中的列部分或全部计算，且无需查表。这修复了随着 3.12.0 版本引入 ORDER BY LIMIT 优化后某些晦涩查询出现的性能回退。
