2023-11-22 发布的 SQLite 3.44.1

来自 3.44.0 版本 (2023-11-01) 的更新：

  1. 聚合函数 现在可以在其最后一个参数之后包含 ORDER BY 子句。函数的参数按照指定顺序处理。 这对于 string_agg() 和 json_group_array() 等函数尤其重要。
  2. 增加对与 PostgreSQL、SQLServer 和 MySQL 兼容的 concat() 和 concat_ws() 标量 SQL 函数的支持。
  3. 增加对与 PostgreSQL 和 SQLServer 兼容的 string_agg() 聚合 SQL 函数的支持。
  4. strftime() SQL 函数 添加了新的转换字母：%e %F %I %k %l %p %P %R %T %u
  5. 添加了新的 C 语言 API：sqlite3_get_clientdata() 和 sqlite3_set_clientdata()。
  6. 与 CREATE TABLE 相关的许多错误现在在运行 CREATE TABLE 语句本身时就会触发，而不再延迟到第一次实际使用该表时。
  7. PRAGMA integrity_check 命令现在使用新的 xIntegrity 方法 验证各种内置 虚拟表 的内容一致性。 这适用于 FTS3、FTS4、FTS5、RTREE 以及 GEOPOLY 扩展。
  8. SQLITE_DBCONFIG_DEFENSIVE 设置现在可以阻止 PRAGMA writable_schema 被启用。 之前 writable_schema 可能被开启，但实际并不能让架构可写。现在它根本无法被开启。
  9. 将内置的 FTS3、FTS4、FTS5、RTREE 和 GEOPOLY 虚拟表标记为 SQLITE_VTAB_INNOCUOUS， 以便它们可以在高安全部署环境中的触发器内使用。
  10. PRAGMA case_sensitive_like 语句现已弃用，因为在架构包含 LIKE 运算符时使用它可能会导致 PRAGMA integrity_check 报告数据库损坏。
  11. SQLITE_USE_SEH（结构化异常处理）在使用 Microsoft C 编译器构建 SQLite 时现在默认启用。可以使用 -DSQLITE_OMIT_SEH 禁用它。
  12. 查询规划器优化：
    1. 在部分索引扫描中，如果 WHERE 子句为某个表列隐含一个常量值，则将该表列的出现替换为该常量。这会增加部分索引作为覆盖索引的可能性。
    2. 禁用视图扫描优化（在 3.40.0 版 - 项目 3d 中引入），因为它导致了多项性能回退。取而代之的是，将 DISTINCT 子查询的估计行数缩小 8 倍。
  13. SQLite 现在在运行时检测底层硬件是否支持比 double 更高精度的“long double”，并根据检测结果使用合适的浮点例程。
  14. Windows 的 CLI 现在在支持的平台上默认对输入和输出使用 UTF-8。可以使用 --no-utf8 选项禁用 UTF-8 支持。

此特定补丁发布版本 3.44.1 (2023-11-22) 的更改：

  1. 将 CLI 更改为在 Windows 控制台 I/O 上使用 UTF-16。这使旧 Windows7 机器上的 Unicode 文本能够正确显示。
  2. 其他一些晦涩的错误修复。
