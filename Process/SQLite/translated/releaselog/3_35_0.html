SQLite 发行 3.35.0 于 2021-03-12

  1. 添加了内置的 SQL 数学函数()。（需要-DSQLITE_ENABLE_MATH_FUNCTIONS 编译时选项。）
  2. 添加了对ALTER TABLE DROP COLUMN的支持。
  3. 泛化了UPSERT：
    1. 允许多个按顺序计算的 ON CONFLICT 子句，
    2. 最终的 ON CONFLICT 子句可以省略冲突目标，但仍然可以使用 DO UPDATE。
  4. 为DELETE、INSERT和UPDATE语句添加了对RETURNING子句的支持。
  5. 当数据库包含非常大的 TEXT 或 BLOB 值时，运行VACUUM时使用更少内存。不再需要一次性将整个 TEXT 或 BLOB 保存在内存中。
  6. 在指定公共表表达式时，为MATERIALIZED和NOT MATERIALIZED提示添加支持。默认行为以前为 NOT MATERIALIZED，但现在对重复使用的 CTE 改为 MATERIALIZED。
  7. SQLITE_DBCONFIG_ENABLE_TRIGGER和SQLITE_DBCONFIG_ENABLE_VIEW设置已修改为仅控制主数据库模式或附加数据库模式中的触发器和视图，而不控制 TEMP 模式。TEMP 触发器和视图始终允许。
  8. 查询规划器/优化器改进：
    1. 对最小/最大优化进行了增强，使其在与 IN 运算符和上一版本的 OP_SeekScan 优化协作时表现更好。
    2. 尝试在 WHERE 子句中将 EXISTS 运算符视为 IN 运算符来处理，在该转换有效且可能提高性能的情况下进行。
    3. 即使父查询是连接，也允许将 UNION ALL 子查询展开。
    4. 如适用，在 WHERE 子句中对 IS NOT NULL 表达式使用索引，即使 STAT4 已禁用。
    5. 如果“x”是具有“NOT NULL”约束且不参与外部连接的列，则形如“x IS NULL”或“x IS NOT NULL”的表达式可能被转换为简单的 FALSE 或 TRUE。
    6. 如果 UPDATE 语句未修改与外键相关的任何列，则避免检查外键约束。
    7. 允许 WHERE 条款被下推到包含窗口函数的子查询，只要该 WHERE 条款完全由常量和所有窗口函数的 PARTITION BY 子句中出现的表达式副本组成。
  9. CLI 增强：
    1. 增强“.stats”命令以接受新参数“stmt”和“vmstep”，分别仅显示准备语句统计和虚拟机步数。
    2. 添加“.filectrl data_version”命令。
    3. 增强“.once”和“.output”命令，如果目标参数以“|”开头（表示输出重定向到管道），则参数无需加引号。
  10. 错误修复：
    1. 修复在处理具有相关 WHERE 子句和“HAVING 0”子句的语法错误的 SELECT 语句时可能发生的 NULL 指针解引用。（也在 3.34.1 补丁版中修复。）
    2. 修复版本 3.33.0 的IN 运算符优化中的错误，该错误可能导致不正确的结果。
    3. 修复当模式以“%”结尾并且存在“ESCAPE '_'”子句时LIKE 运算符返回不正确结果的问题。
