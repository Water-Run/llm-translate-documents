SQLite 3.18.2 发行于 2017-06-17

在版本 3.18.0 (2017-03-30) 之前的更改：

  1. 添加了 PRAGMA optimize 命令
  2. 通过 sqlite_source_id() SQL 函数、sqlite3_sourceid() C API 以及 SQLITE_SOURCE_ID 宏返回的 SQLite 版本标识符 现在是一个 64 位的 SHA3-256 哈希，而不是 40 位的 SHA1 哈希。
  3. 向 JSON1 扩展 添加了 json_patch() SQL 函数。
  4. 增强了 LIKE 优化，使其可以处理左侧为任意表达式，只要右侧的 LIKE 模式 不以数字或减号开头。
  5. 添加了 sqlite3_set_last_insert_rowid() 接口，并在 FTS3、FTS4 和 FTS5 扩展中使用新的接口， 以确保 sqlite3_last_insert_rowid() 接口始终返回合理的值。
  6. 增强了 PRAGMA integrity_check 和 PRAGMA quick_check，使它们可以验证 CHECK 约束。
  7. 改进了连接的查询计划，提前检测空表并及时停止，避免不必要的工作。
  8. 增强了 sqlite3_mprintf() 系列接口以及 printf SQL 函数， 如果在 "%" 与 "d" 之间使用 "," 格式修饰符（例如 "%,d"），会在整数的千位处插入逗号分隔符。
  9. 添加了 -DSQLITE_MAX_MEMORY=N 编译时选项。
  10. 向 命令行 shell 添加了 .sha3sum 点命令和 .selftest 点命令
  11. 开始执行 SQLITE_LIMIT_VDBE_OP。例如，在接受来自不受信用户 SQL 查询的系统中， 可以利用此限制来防止过大的预编译语句。
  12. 各种性能改进。

    错误修复：

  13. 确保带有排序序列的索引表达式能够正确处理。 此修复针对工单 eb703ba7b50c1a5。
  14. 修复 日期和时间函数 中 "'start of ...'" 修饰符的一个 bug。 工单 6097cb92745327a1
  15. 修复了复杂递归触发器中可能导致段错误的潜在问题，该问题由 3.15.0 版本中作为性能优化引入的 OP_Once 操作码的缺陷引起。 工单 06796225f59c057c
  16. 在 RBU 扩展 中增加了额外的同步操作，以避免在断电后可能出现的损坏。
  17. sqlite3_trace_v2() 对嵌套 SQL 语句的输出现在应始终以 “--” 注释标记开头。

在版本 3.18.1 (2017-06-16) 之前的更改：

  1. 修复了与 auto_vacuum 相关的一个错误，该错误可能导致数据库损坏。此错误在 3.16.0 版本 (2017-01-02) 中引入。 工单 fda22108。

该特定补丁版本 3.18.2 (2017-06-17) 的更改：

  1. 修复了在 WHERE 子句中使用 IN 运算符时可能导致重复输出行的错误。 工单 61fe9745。
