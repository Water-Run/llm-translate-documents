SQLite 3.43.0 于 2023-08-24 发布

  1. 新增对无内容删除 FTS5 索引的支持。这是FTS5全文搜索索引的一种变体，它在允许删除记录的同时省略存储正在被编入索引的内容。
  2. 对日期和时间函数的增强：
    1. 新增形式为 ±YYYY-MM-DD HH:MM:SS.SSS 的时间偏移修改符。
    2. 新增timediff() SQL 函数。
  3. 新增octet_length(X) SQL 函数。
  4. 新增sqlite3_stmt_explain() API。
  5. 查询规划器增强：
    1. 将 LEFT JOIN 强度降低优化泛化，使其也适用于 RIGHT 和 FULL JOIN，并将其重命名为OUTER JOIN 强度降低。
    2. 加强OUTER JOIN 强度降低优化中的定理证明器，使其返回更少的误报。
  6. 对十进制扩展的增强：
    1. 新增函数 decimal_pow2(N)，在 -20000 到 +20000 的整数 N 范围内返回 2 的 N 次幂。
    2. 新增函数 decimal_exp(X)，其行为类似于 decimal(X)，但返回的结果采用指数表示法，并在末尾附加 "e+NN"。
    3. 当 X 为浮点值时，decimal(X) 函数现在会将该值完全展开为其精确的十进制等价形式。
  7. 对JSON 处理的性能增强使某些大型 JSON 字符串处理的性能提高了两倍。
  8. 新的 makefile 目标 "verify-source" 用于检查源树中是否存在非预期更改。（仅适用于规范源代码——不适用于预编译整合包。）
  9. 新增SQLITE_USE_SEH编译时选项，在使用作为WAL 模式处理一部分的内存映射shm 文件时启用 Windows 上的结构化异常处理。使用 Makefile.msc 在 Windows 上构建时默认启用此选项。
  10. unix 的VFS现在假设 nanosleep() 系统调用可用，除非使用 -DHAVE_NANOSLEEP=0 编译。
