SQLite 3.19.1 发布于 2017-05-24

从 3.19.0 版本（2017-05-22）起的先前更改：

  1. 对于查询中引用但未提取任何列的每个表，在调用 SQLITE_READ authorizer callback 时会传入一个空字符串作为列名，仅调用一次。
  2. 在对表达式使用索引时，尽量直接使用索引中已有的表达式值，而不是加载原始列再重新计算表达式。
  3. 增强 flattening optimization，使其能够展平成 LEFT JOIN 右侧的视图。
  4. 在 command-line shell 提供的 .dump 输出中，对于字符串中内嵌的换行符和回车符，使用 replace() 代替 char() 进行转义。
  5. 避免在 UPDATE 语句中，对那些未触及外键约束列的更新执行不必要的外键处理。
  6. 在使用索引的 DISTINCT 查询中，若存在合适的索引，则尽量直接用索引跳到下一个不同的条目，而不是逐行遍历。
  7. 在修改无关表时，避免对 sqlite3_blob 句柄执行不必要的失效操作。
  8. 将 HAVING 子句中仅使用 GROUP BY 子句中列的条件转移到 WHERE 子句中以加快处理。
  9. 如果同一个查询中同一个 VIEW 出现多次，则重用相同的物化结果。
  10. 增强 PRAGMA integrity_check，使其能够识别具有相同 rowid 的两行或多行的表。
  11. 增强 FTS5 查询语法，使 column filters 可以应用于任意表达式。
  12. 增强 json_extract() 函数，使其缓存并重用 JSON 输入文本的解析结果。
  13. 添加了 anycollseq.c loadable extension，使通用 SQLite 数据库连接能够读取包含未知和/或应用特定的 collating sequences 的模式。

    错误修复：

  14. 修复 REPLACE 中的一个问题，该问题可能导致数据库损坏并出现两个或更多具有相同 rowid 的行。修复了票据 f68dc596c4e6018d。
  15. 修复 PRAGMA integrity_check 中的问题，该问题导致后续的 VACUUM 表现欠佳。
  16. 修复 PRAGMA foreign_key_check 命令，使其可以正确处理 WITHOUT ROWID 表上的外键。
  17. 修复 b-tree 逻辑中的一个错误，该错误可能导致 IN 操作符查询返回错误的重复结果。票据 61fe9745。
  18. 在 JSON 中禁止数字常量带前导零。修复票据 b93be8729a895a528e2。
  19. 在 JSON 的字符串中禁止控制字符。修复票据 6c9b5514077fed34551。
  20. 限制 JSON 对象和数组的递归深度，以避免递归下降解析器中栈使用过多。修复票据 981329adeef51011052。

此补丁版本（3.19.1，2017-05-24）中的更改：

  1. 修复 LEFT JOIN flattening optimization 中的一个错误。票据 cad1ab4cb7b0fc。
  2. 移除一个多余的分号，该分号正在为旧版 MSVC 带来问题。
