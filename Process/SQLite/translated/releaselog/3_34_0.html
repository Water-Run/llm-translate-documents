SQLite 3.34.0 发布于 2020-12-01

  1. 添加了 sqlite3_txn_state() 接口，用于报告数据库连接的当前事务状态。
  2. 增强了 递归公共表表达式，支持像 SQL Server 那样使用两个或更多递归项，从而使得 面向图的查询 更容易编写且执行更快。
  3. 改进了 CHECK 约束 失败时的错误消息。
  4. CLI 增强功能：
    1. .read 点命令现在除了文件名外，还接受管道。
    2. 为 .dump 点命令添加了 --data-only 和 --nosys 选项。
    3. 为 .schema 点命令添加了 --nosys 选项。
    4. .import 点命令现在能正确处理表名引用。
    5. generate_series(START,END,STEP) 表值函数扩展现已内置于 CLI。
    6. .databases 点命令现在显示每个数据库文件的状态，该状态由 sqlite3_db_readonly() 和 sqlite3_txn_state() 决定。
    7. 新增 --tabs 命令行选项，用于设置 .mode tabs。
    8. --init 选项在其参数指定的文件无法打开时报告错误。--init 选项现在也遵循 --bail 选项。
  5. 查询规划器改进：
    1. 改进了运行 DISTINCT 运算符的成本估算。
    2. 当使用多列索引执行 UPDATE 或 DELETE 时，如果索引的前几列只有少数对索引查找有用，则延后主表查找，直到所有 WHERE 子句约束都被评估完毕，以防这些约束可以通过索引中未使用的后续项覆盖，从而避免不必要的主表查找。
    3. 新的 OP_SeekScan 操作码用于提升在后续列被 IN 运算符约束时，多列索引查找的性能。
  6. 即使一个或多个附加的数据库文件为只读，BEGIN IMMEDIATE 和 BEGIN EXCLUSIVE 命令现在也能正常工作。
  7. 增强了 FTS5，以支持 三元组索引。
  8. 在有数百个连接同时访问同一数据库文件的情况下，提升了 WAL 模式 锁原语的性能。
  9. 增强了 carray() 表值函数，加入了一个单参数形式，通过辅助的 sqlite3_carray_bind() 接口绑定。
  10. substr() SQL 函数现在也可以被调用为 “substring()”，以兼容 SQL Server。
  11. 语法图 现在以 Pikchr 脚本实现，并渲染为 SVG，以提高可读性和维护便利性。
