SQLite 发布 3.31.1 于 2020-01-27

来自 3.31.0 版本（2020-01-22）的早期更改：

  1. 添加对 生成列 的支持。
  2. 添加 sqlite3_hard_heap_limit64() 接口以及对应的 PRAGMA hard_heap_limit 命令。
  3. 增强 function_list pragma 以显示每个函数的参数数量、函数类型（标量、聚合、窗口）以及函数属性标志 SQLITE_DETERMINISTIC、SQLITE_DIRECTONLY、 SQLITE_INNOCUOUS 和/或 SQLITE_SUBTYPE。
  4. 为 DBSTAT 虚拟表 添加 聚合模式 功能。
  5. 在 sqlite3_open_v2() 中添加 SQLITE_OPEN_NOFOLLOW 选项， 防止 SQLite 打开符号链接。
  6. 为 JSON 函数路径参数 添加了 "#-N" 数组表示法。
  7. 添加 SQLITE_DBCONFIG_TRUSTED_SCHEMA 连接设置， 该设置也可通过新的 trusted_schema pragma 以及 编译时使用 -DSQLITE_TRUSTED_SCHEMA 选项控制。
  8. 添加适用于专业扩展的 API：sqlite3_filename_database()、sqlite3_filename_journal() 和 sqlite3_filename_wal()。
  9. 添加 sqlite3_uri_key() 接口。
  10. 增强 sqlite3_uri_parameter() 函数，使其在数据库文件名之外也能处理回滚日志或 WAL 文件名。
  11. 为 应用程序自定义 SQL 函数 提供标记能力， 可增加新属性 SQLITE_INNOCUOUS 或 SQLITE_DIRECTONLY。
  12. 为 sqlite3_vtab_config() 添加新动词，使虚拟表的 xConnect 方法 能将虚拟表声明为 SQLITE_VTAB_INNOCUOUS 或 SQLITE_VTAB_DIRECTONLY。
  13. 加快对 sqlite3_interrupt() 的响应。
  14. 添加实现 RFC-4122 UUID 处理函数的 uuid.c 扩展模块。
  15. 旁路内存分配器 已增强以支持两个独立的内存池，每个池中可用不同大小的分配。 这样可以覆盖更多的内存分配，同时将每个连接的堆内存使用量从 120KB 降低到 48KB。
  16. legacy_file_format pragma 已停用，现成为空操作。 取而代之的是 SQLITE_DBCONFIG_LEGACY_FILE_FORMAT 选项， 可通过 sqlite3_db_config() 使用。 legacy_file_format pragma 被停用，是因为（1）它很少有用，（2）在同时具有生成列和降序索引的模式中与 VACUUM 不兼容。 工单 6484e6ce678fffab

本补丁版本 3.31.1（2020-01-27）中的更改：

  1. 还原仅用于内部的 SQLite 数据结构的数据布局。 不应有任何使用 SQLite 的应用引用内部 SQLite 数据结构，但仍有部分应用这么做， 并且 3.30.0 中对某个此类数据结构的更改导致一个流行且广泛部署的应用无法运行。 在 SQLite 中暂时还原此更改，可为不按规矩来的应用开发者争取修复代码的时间。
  2. 修复 sqlite3ext.h 头文件中的拼写错误，避免 sqlite3_stmt_isexplain() 和 sqlite3_value_frombind() 接口 在 运行时可加载扩展 中无法被调用。
