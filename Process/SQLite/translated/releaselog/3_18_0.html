SQLite 发行 3.18.0 于 2017-03-30

  1. 新增 PRAGMA optimize 命令
  2. 通过 sqlite_source_id() SQL 函数返回的 SQLite 版本标识、sqlite3_sourceid() C API 返回的值以及 SQLITE_SOURCE_ID 宏中的值现在全部采用 64 位数的 SHA3-256 散列，替代原来的 40 位数 SHA1 散列。
  3. 向 JSON1 扩展 添加了 json_patch() SQL 函数。
  4. 增强 LIKE 优化，使其在左侧是任意表达式的情况下仍能正常工作，只要右侧的 LIKE 模式不是以数字或减号开头。
  5. 新增 sqlite3_set_last_insert_rowid() 接口，并在 FTS3、FTS4 与 FTS5 扩展中使用该接口，以确保 sqlite3_last_insert_rowid() 接口始终返回合理的值。
  6. 增强 PRAGMA integrity_check 与 PRAGMA quick_check，使其能验证 CHECK 约束。
  7. 增强联接的查询计划，让它们能尽早检测空表并在不做多余工作的情况下停止。
  8. 增强 sqlite3_mprintf() 系列接口与 printf SQL 函数，在 "%" 与 "d" 之间使用 "," 格式修饰符（例如 "%,d"）时，会在整数的千位处插入千位分隔符。
  9. 新增 -DSQLITE_MAX_MEMORY=N 编译时选项。
  10. 在 命令行 shell 中新增 .sha3sum 句点命令 与 .selftest 句点命令
  11. 开始强制执行 SQLITE_LIMIT_VDBE_OP。例如，在接受不可信用户 SQL 查询的系统中，可以利用该限制防止过大的预编译语句。
  12. 各种性能改进。

    错误修复：

  13. 确保带有排序序列的索引表达式被正确处理。修复票据 eb703ba7b50c1a5。
  14. 修复日期与时间函数中 "start of ..." 修饰符的一个错误。票据 6097cb92745327a1
  15. 修复复杂递归触发器中一个可能导致段错误的问题，该问题源于 3.15.0 版本性能优化中引入的 OP_Once 操作码的错误。票据 06796225f59c057c
  16. 在 RBU 扩展 中增加额外的同步操作，以避免停电后可能发生的数据损坏。
  17. 嵌套 SQL 语句的 sqlite3_trace_v2() 输出现在应始终以 "--" 注释标记开头。
