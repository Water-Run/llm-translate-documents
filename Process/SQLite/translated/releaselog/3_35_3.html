SQLite 3.35.3 发行于 2021-03-26

来自版本 3.35.0（2021-03-12）的先前更改：

  1. 添加了内置 SQL 数学函数()。（需要-DSQLITE_ENABLE_MATH_FUNCTIONS编译时选项。）
  2. 为ALTER TABLE DROP COLUMN增加了支持。
  3. 泛化UPSERT：
    1. 允许多个按顺序评估的 ON CONFLICT 子句，
    2. 最后一个 ON CONFLICT 子句可以省略冲突目标，却仍然使用 DO UPDATE。
  4. 在DELETE、INSERT和UPDATE语句中增加了对RETURNING子句的支持。
  5. 在对包含非常大的 TEXT 或 BLOB 值的数据库运行VACUUM时使用更少内存。不再需要一次性将整个 TEXT 或 BLOB 保持在内存中。
  6. 在指定公共表表达式时增加对MATERIALIZED和NOT MATERIALIZED提示的支持。默认行为原本为 NOT MATERIALIZED，但现已改为对被多次使用的 CTE 采用 MATERIALIZED。
  7. SQLITE_DBCONFIG_ENABLE_TRIGGER和SQLITE_DBCONFIG_ENABLE_VIEW设置被修改，使其只控制主数据库模式或附加数据库模式中的触发器和视图，而不控制 TEMP 模式中的。TEMP 模式中的触发器和视图始终允许。
  8. 查询规划器/优化器改进：
    1. 对min/max 优化的增强，使其在与 IN 运算符和上一版本的 OP_SeekScan 优化一起使用时表现更好。
    2. 尝试将 WHERE 子句中的 EXISTS 运算符处理为 IN 运算符，只要这种转换有效且有望提高性能。
    3. 允许即使父查询是联接，也可以对 UNION ALL 子查询进行扁平化。
    4. 在 WHERE 子句中对 IS NOT NULL 表达式（即使 STAT4 被禁用）也能适当使用索引。
    5. 如果 "x" 是具有 "NOT NULL" 约束且未参与外连接的列，则形式为 "x IS NULL" 或 "x IS NOT NULL" 的表达式可能被简化为 FALSE 或 TRUE。
    6. 如果 UPDATE 语句未修改与外键相关的任何列，则避免检查外键约束。
    7. 允许将 WHERE 项下推到包含窗口函数的子查询中，只要该 WHERE 项完全由常量以及出现在子查询中所有窗口函数 PARTITION BY 子句中的表达式拷贝组成。
  9. 命令行界面（CLI）增强：
    1. 增强 ".stats" 命令以接受新参数 "stmt" 和 "vmstep"，分别只显示准备语句统计信息或仅显示虚拟机步骤计数。
    2. 添加 ".filectrl data_version" 命令。
    3. 增强 ".once" 和 ".output" 命令，使得当目标参数以 "|" 开头（表示输出重定向到管道）时，该参数无需加引号。
  10. 错误修复：
    1. 修复在处理语法错误的 SELECT 语句（具有相关 WHERE 子句和 "HAVING 0" 子句）时可能发生的 NULL 指针取消引用。（也在 3.34.1 补丁版中修复。）
    2. 修复版本 3.33.0 中 IN 运算符优化的一个缺陷，该缺陷可能导致错误结果。
    3. 修复当模式以 "%" 结尾且存在 "ESCAPE '_'" 子句时，LIKE 运算符可能返回错误结果的问题。

来自版本 3.35.1（2021-03-15）的先前更改：

  1. 修复在新 DROP COLUMN 功能中对带索引且在索引定义中带引号的列使用时的一个错误。
  2. 改善CLI中.dump命令的内置文档。

来自版本 3.35.2（2021-03-17）的先前更改：

  1. 修复appendvfs.c扩展在 3.35.0 版本中引入的问题。
  2. 确保无参数的日期/时间函数（会生成依赖于当前时间的结果）被视为非确定性函数。参考工单2c6c8689fb5f3d2f。
  3. 修复sqldiff 工具在处理虚拟表定义中的异常空白字符时的问题。
  4. 限制在 3.35.0 版本第 8c 条中描述的新 UNION ALL 优化，使其不会尝试创建过多的新子查询。详情见论坛帖子 140a67d3d2。

本次补丁版本 3.35.3（2021-03-26）中的更改：

  1. 增强字节码引擎的 OP_OpenDup 操作码，使其即使在复制的游标本身来自 OP_OpenDup 时也能正常工作。修复工单 bb8a9fd4a9b7fce5。该问题仅由于最近的 MATERIALIZED 提示增强而暴露。
  2. 在实现材料化相关的公共表表达式时，为每个使用场景分别材料化，这是正确性的必要条件。此修复解决了因 MATERIALIZED 提示增强引入的问题。
  3. 修复 unixVFS的文件名归一化器中的问题。
  4. 修复CLI中“box”输出模式，使其能够处理返回一行或多行零列（例如 PRAGMA incremental_vacuum）的语句。参考论坛帖子 afbbcb5b72。
  5. 改进由于错误公共表表达式生成的错误消息。参考论坛帖子 aa5a0431c99e。
  6. 修复一些不正确的 assert() 语句。
  7. 修正SELECT 语句语法图，确保正确显示 FROM 子句语法。参考论坛帖子 9ed02582fe。
  8. 修复 EBCDIC 字符分类器，使其将换行符识别为空白字符。参考论坛帖子 58540ce22dcd。
  9. 改进在实现（不受支持的）wholenumber 虚拟表扩展时的xBestIndex 方法，使其更有效地说服查询规划器避免尝试物化具有无限行的表。参考论坛帖子 b52a020ce4。
