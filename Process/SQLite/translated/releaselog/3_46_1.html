SQLite 3.46.1 发布于 2024-08-13

从 3.46.0 版本（2024-05-23）以来的主要变更：

  1. 从多个方面增强 PRAGMA optimize，以便更易使用：
    1. PRAGMA optimize 自动实施临时 分析限制，以防止大型数据库运行时间过长。
    2. 添加了新的 0x10000 位掩码选项，可检查所有表的更新。
    3. 自动重新分析没有 sqlite_stat1 条目的表。
  2. 对 日期和时间函数的增强：
    1. strftime() SQL 函数现在支持 %G、%g、%U 和 %V。
    2. 新的 'ceiling' 和 'floor' 修饰符控制在按整数月和/或年偏移日期时解析 模糊日期所使用的算法。
    3. 当 SQLite 已知时间已经处于 UTC 或本地时间时，'utc' 和 'localtime' 修饰符现在分别成为无操作。
  3. 在 数字字面量的数字之间添加对下划线 ("_") 字符的支持。
  4. 添加 json_pretty() SQL 函数。
  5. 查询规划器改进：
    1. “VALUES 作为协程” 优化使得 VALUES 子句中包含数千行的 INSERT 语句在大约一半的时间内解析并执行，且使用大约一半的内存。
    2. 允许对类似 “SELECT count(DISTINCT col) FROM ...” 的查询使用索引，即使索引记录并不比表记录更小。
    3. 增强对以下情况的识别：由于所有参数都是常量，SQL 函数的结果也是常量。
    4. 增强 WHERE 子句下推优化，使其能够下推包含非关联子查询的 WHERE 子句项。
  6. 如果 SQL 解析器堆栈溢出，则从堆中分配额外的内存，而不是报告“解析器堆栈溢出”错误。
  7. JSON 改进：
    1. 允许 JSON5 字符串字面量中包含 ASCII 控制字符。
    2. 修复 -> 和 ->> 运算符，以便在右侧操作数是看起来像整数的字符串时仍然将其视为字符串，因为 PostgreSQL 的行为也是如此。
  8. 允许将大型十六进制字面量用作表列的 DEFAULT 值。

本次 3.46.1 补丁版本（2024-08-13）中的变更：

  1. 解析 FTS5 中的 tokenize= 参数时提升了健壮性。 论坛帖子 171bcc2bcd。
  2. 增强查询规划器中 覆盖索引 的预测。 添加对覆盖索引过度预测的早期检测，这样 sqlite3_prepare() 会返回错误，而不是仅生成错误字节码。 论坛帖子 e60e4c295d22f8ce。
  3. 不要让 VALUES 子句 中的项数受 SQLITE_LIMIT_COMPOUND_SELECT 的限制，即使 VALUES 子句包含由于 双引号字符串字面量 而看起来像变量的元素。
  4. 修复 window 函数 版本的 group_concat()，使其在有一个或多个空字符串输入时返回空字符串。
  5. 在 FTS5 安全删除模式 中，修复关于损坏索引的误报完整性检查报告。
  6. ALTER TABLE 中的语法错误现在应始终返回 SQLITE_ERROR。在某些情况下，它们以前会返回 SQLITE_INTERNAL。
  7. JavaScript/WASM：
    1. 修复 JavaScript “opfs” VFS 中导致损坏的错误。
    2. 处理几个特定浏览器的 OPFS 奇怪行为。
  8. 其他小修复。
