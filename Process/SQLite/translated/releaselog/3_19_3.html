SQLite 发布 3.19.3 于 2017-06-08

来自版本 3.19.0 (2017-05-22) 的之前更改：

  1. 对于 SQLITE_READ authorizer 回调，在从中未提取任何列的查询所引用的每个表上，该回调会以列名为空字符串的方式被调用一次。
  2. 当在表达式上使用索引时，尽量重用索引中已有的表达式值，而不是加载原始列并重新计算表达式。
  3. 增强 扁平化优化，以便能够扁平化 LEFT JOIN 右侧的视图。
  4. 在 命令行 shell 的 .dump 输出中，用 replace() 替代 char() 来转义嵌入字符串中的换行符和回车符。
  5. 在 UPDATE 语句中，如果未触及受外键约束的列，则避免不必要的外键处理。
  6. 在使用索引的 DISTINCT 查询中，当存在合适的索引时，尝试通过索引跳到下一个不重复的条目，而不是逐行遍历。
  7. 在修改不相关表时，避免不必要地使 sqlite3_blob 句柄失效。
  8. 将只使用 GROUP BY 子句中列的 HAVING 条件转移到 WHERE 子句，以更快地处理。
  9. 如果一个 VIEW 在同一个查询中出现多次，应重用其相同的物化结果。
  10. 增强 PRAGMA integrity_check，以便识别具有两个或更多相同 rowid 的表。
  11. 增强 FTS5 查询语法，使 列过滤器 可以应用于任意表达式。
  12. 增强 json_extract() 函数，以缓存并重用对 JSON 输入文本的解析。
  13. 添加 anycollseq.c 可加载扩展，允许通用 SQLite 数据库连接读取包含未知和/或应用特定 排序序列 的架构。

    错误修复：

  14. 修复 REPLACE 中的问题，该问题可能导致数据库损坏并包含两个或多个相同 rowid 的行。修复票据 f68dc596c4e6018d。
  15. 修复 PRAGMA integrity_check 中的问题，该问题曾导致随后的 VACUUM 表现欠佳。
  16. 修复 PRAGMA foreign_key_check 命令，使其能与 WITHOUT ROWID 表上的外键正确搭配使用。
  17. 修复 B 树逻辑中的一个错误，该错误可能导致 IN 运算符查询返回错误的重复结果。票据 61fe9745
  18. 在 JSON 中禁止数字常量的前导零。修复票据 b93be8729a895a528e2。
  19. 在 JSON 的字符串中禁止控制字符。修复票据 6c9b5514077fed34551。
  20. 限制 JSON 对象和数组的递归深度，以避免递归下降解析器中堆栈使用过多。修复票据 981329adeef51011052。

来自版本 3.19.1 (2017-05-24) 的之前更改：

  1. 修复 LEFT JOIN 扁平化优化 中的一个错误。票据 cad1ab4cb7b0fc。
  2. 删除了导致早期 MSVC 版本出现问题的多余分号。

来自版本 3.19.2 (2017-05-25) 的之前更改：

  1. 修复 LEFT JOIN 扁平化优化 中更多错误。票据 7fde638e94287d2c。

此补丁版本 3.19.3 (2017-06-08) 的更改：

  1. 修复与 auto_vacuum 相关的一个错误，该错误可能导致数据库损坏。此错误在 版本 3.16.0 (2017-01-02) 中引入。票据 fda22108。
