SQLite 3.40.0 发布于 2022-11-16

  1. 添加支持将 SQLite 编译为 WASM 并在网页浏览器中运行。注意：WASM 构建及其接口被视为“测试版”，如有需要可能会有小范围更改。我们预计在下一版本最终确定该接口。
  2. 添加 恢复扩展，可能能够从损坏的数据库文件中恢复部分内容。
  3. 查询规划器增强：
    1. 识别在拥有超过 63 列的表上使用的 覆盖索引，其中查询中使用了第 63 列之后的列和/或索引引用了这些列。
    2. 在可行时提取包含在 表达式索引 中的表达式值，而不是重新计算表达式。
    3. NOT NULL 和 IS NULL 运算符（及其等效项）避免从磁盘加载大型字符串和 BLOB 值的内容。
    4. 避免将对其进行完整扫描的视图物化一次。按照计算结果使用并丢弃视图的行。
    5. 允许在聚合查询中将 LEFT JOIN 右侧操作数的子查询展开。
  4. 添加名为 sqlite3_filename 的新 typedef，用于表示数据库文件名。多个接口已修改为使用该 typedef 代替 “char*”。该接口更改应当完全向后兼容，尽管在重建某些旧应用时可能会引发（无害的）编译器警告。
  5. 添加 sqlite3_value_encoding() 接口。
  6. 安全增强：SQLITE_DBCONFIG_DEFENSIVE 增强以禁止更改 schema_version。schema_version 在防御模式下变为只读。
  7. 增强 PRAGMA integrity_check 语句：
    1. 非 STRICT 表中具有 TEXT 亲和力的列不应包含数值。
    2. 非 STRICT 表中具有 NUMERIC 亲和力的列不应包含可以转换为数字的 TEXT 值。
    3. 验证 WITHOUT ROWID 表的行顺序是否正确。
  8. 增强 VACUUM INTO 语句，使其遵循 PRAGMA synchronous 设置。
  9. 增强 sqlite3_strglob() 和 sqlite3_strlike() API，使其能够接受字符串参数的 NULL 指针并仍生成合理结果。
  10. 提供新的 SQLITE_MAX_ALLOCATION_SIZE 编译时选项，用于限制内存分配大小。
  11. 将 SQLite 内置伪随机数生成器（PRNG）使用的算法从 RC4 更改为 Chacha20。
  12. 允许两个或更多索引使用相同名称，只要它们位于不同的架构中。
  13. 各种性能优化使典型工作负载下的 CPU 周期使用减少约 1%。
