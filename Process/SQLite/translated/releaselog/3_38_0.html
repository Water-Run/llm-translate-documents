SQLite 版本 3.38.0 于 2022-02-22 发布

  1. 添加了 -> 和 ->> 运算符，以更轻松地处理 JSON。 新运算符与 MySQL 和 PostgreSQL 兼容。
  2. JSON 函数现在成为内置函数。不再需要 使用 -DSQLITE_ENABLE_JSON1 编译时选项来启用 JSON 支持。JSON 默认开启。使用新选项 -DSQLITE_OMIT_JSON 编译时选项禁用 JSON 接口。
  3. 对 日期和时间函数 的增强：
    1. 添加了 unixepoch() 函数。
    2. 添加了 auto 修改器 与 julianday 修改器。
  4. 为了更好的兼容性，将 printf() SQL 函数 重命名为 format()。 原始的 printf() 名称保留为向后兼容的别名。
  5. 添加了 sqlite3_error_offset() 接口，有时可以帮助将 SQL 错误定位到输入 SQL 文本的具体字符，以便应用程序提供更好的错误消息。
  6. 对 虚拟表 接口的增强如下：
    1. 添加了 sqlite3_vtab_distinct() 接口。
    2. 添加了 sqlite3_vtab_rhs_value() 接口。
    3. 添加了新的运算符类型 SQLITE_INDEX_CONSTRAINT_LIMIT 与 SQLITE_INDEX_CONSTRAINT_OFFSET。
    4. 添加了 sqlite3_vtab_in() 接口（及相关接口），以便 虚拟表能够一次性处理 IN 运算符 的约束， 而非单独处理 IN 运算符右侧的每一个值。
  7. 命令行界面 的增强：
    1. 列对齐输出模式 增强后能够正确处理嵌入文本中的制表符 与换行符。
    2. 在 列对齐输出模式 中添加了类似 "--wrap N"、"--wordwrap on" 与 "--quote" 的选项。
    3. 添加了 .mode qbox 别名。
    4. .import 命令 自动消除列名歧义。
    5. 使用新的 sqlite3_error_offset() 接口提供更佳的 错误消息。
  8. 查询规划器的增强：
    1. 使用布隆过滤器加速大型分析型查询。
    2. 使用平衡归并树评估带有 ORDER BY 子句的 UNION 或 UNION ALL 复合 SELECT 语句。
  9. ALTER TABLE 语句在 PRAGMA writable_schema=ON 时会默默忽略 sqlite_schema 表 中无法解析的条目。
