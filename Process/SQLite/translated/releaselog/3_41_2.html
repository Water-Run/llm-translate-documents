SQLite 发布 3.41.2 于 2023-03-22

此前自版本 3.41.0 (2023-02-21) 的变更：

  1. 查询规划器改进：
    1. 在包含 GROUP BY 子句的聚合查询中使用索引表达式。
    2. 查询规划器现在更能识别何时某索引为覆盖索引并据此调整预测运行时间。
    3. 查询规划器在使用协程而非物化子查询和视图方面更为积极。
    4. 针对内建的表值函数json_tree()与json_each()的查询现在通常将 “ORDER BY rowid” 视为无操作。
    5. 加强查询规划器在表达式经常量传播优化修改后仍能使用索引表达式的能力。 （参见论坛帖 0a539c7。）
  2. 添加内建 SQL 函数 unhex()。
  3. 将 base64 与 base85 应用定义函数作为扩展，并将该扩展包含在命令行工具中。
  4. 增加 sqlite3_stmt_scanstatus_v2() 接口。（只有在使用 SQLITE_ENABLE_STMT_SCANSTATUS 编译 SQLite 时才可用。）
  5. 使用 sqlite3_deserialize() 创建的内存数据库现在报告其文件名为空字符串，而非 'x'。
  6. 对命令行工具的变更：
    1. 增加新的 base64() 与 base85() SQL 函数
    2. 在使用SQLITE_ENABLE_STMT_SCANSTATUS 编译时， 使用新的 sqlite3_stmt_scanstatus_v2() 接口增强EXPLAIN QUERY PLAN 输出。
    3. “.scanstats est” 命令在分析中提供查询规划器估值。
    4. 续行提示会指示当前输入是否处于字符串字面量、标识符字面量、注释、触发器定义等之内。
    5. 增强 --safe 命令行选项以禁止危险的 SQL 函数。
    6. CLI 构建默认禁用了双引号字符串的反常行为。历史兼容用例可通过运行时命令 “.dbconfig dqs_dml on” 与 “.dbconfig dqs_ddl on” 来重新启用。
  7. 增强 PRAGMA integrity_check 命令，使其在表中的文本字符串与索引中相同但非字节完全一致时能够检测出来。
  8. 增强carray 表值函数，使其能够绑定一组 BLOB 对象。
  9. 新增sqlite3_is_interrupted() 接口。
  10. 对 sqlite3_prepare() 等调用耗时较长的情况，现在会调用进度处理器回调并响应sqlite3_interrupt()。
  11. 增强 sqlite3_vtab_in_first() 与 sqlite3_vtab_in_next() 函数，使其可靠地检测是否在未为多值 IN 处理通过 sqlite3_vtab_in() 选择的参数上调用。 在此情况下它们返回 SQLITE_ERROR 而非 SQLITE_MISUSE。
  12. 解析器现在忽略 IN 运算符右侧子查询周围多余的括号，使 SQLite 此方面行为与 PostgreSQL 保持一致。 之前 SQLite 将该子查询视为带隐含 “LIMIT 1” 的表达式。
  13. 为sqlite3_file_control() API 添加了SQLITE_FCNTL_RESET_CACHE 选项。
  14. Makefile 改进：
    1. 新增 makefile 目标 “sqlite3r.c”，构建包含恢复扩展的合并源。
    2. 新增目标 “devtest” 与 “releasetest”，分别用于进行提交前的快速开发测试与完整的发布测试。
  15. 其他性能增强。

此前自版本 3.41.1 (2023-03-10) 的变更：

  1. 提供编译期选项 -DHAVE_LOG2=0 与 -DHAVE_LOG10=0，以让缺少标准库函数 log2() 与 log10() 的系统能够编译 SQLite。
  2. 确保在执行 “CREATE TABLE t1 AS SELECT CAST(7 AS INT) AS x;” 时，列 t1.x 的数据类型仍为 INT 而非 NUM，以保持历史兼容。
  3. 增强 PRAGMA integrity_check，使其能够检测索引记录末尾出现的多余字节。
  4. 修复用户社区报告的各类模糊错误。详情请见变更时间线。

此补丁版本 3.41.2 (2023-03-22) 的变更：

  1. 多处修复了在以下情况下对内存缓冲区末尾发生越界读取（注意：是读取而非写入）的问题：
    1. 在使用非标准编译选项 SQLITE_ENABLE_STAT4 处理损坏数据库文件时。
    2. 在命令行工具中，当sqlite3_error_offset() 返回了超出范围的值时（另见下文对 sqlite3_error_offset() 的修复）。
    3. 在恢复扩展中。
    4. 在FTS3 处理中损坏数据库文件时。
  2. 修复sqlite3_error_offset()，使其在报告与生成列相关的错误时不返回超出范围的值。
  3. 查询优化器中多处修复，解决会导致针对奇怪的模糊器生成查询输出错误结果的问题。
  4. 将页面缓存对象中的引用计数器大小增加至 64 位，以确保计数器永不溢出。
  5. 修复补丁版本 3.41.1 中一个造成性能回退的错误修复。
  6. 修复若干不正确的 assert() 语句。
