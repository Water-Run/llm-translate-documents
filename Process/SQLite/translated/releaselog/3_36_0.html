SQLite 3.36.0 发布于 2021-06-18

  1. 改进了EXPLAIN QUERY PLAN的输出，使其更容易理解。
  2. 将标记开始处的字节序标记视为与空白符一样跳过。
  3. 尝试访问VIEW或子查询的rowid时会引发错误。 以前，VIEW的rowid是不确定的，且通常为NULL。 提供了 -DSQLITE_ALLOW_ROWID_IN_VIEW 编译期选项，以便需要旧行为的应用程序恢复旧行为。
  4. sqlite3_deserialize() 和 sqlite3_serialize() 接口现在默认启用。 不再需要 -DSQLITE_ENABLE_DESERIALIZE 编译期选项。 取而代之的是一个新的 -DSQLITE_OMIT_DESERIALIZE 编译期选项，用于省略这些接口。
  5. “memdb” VFS 现在允许在同一进程中多个数据库连接共享相同的内存数据库，只要数据库名称以 “/” 开头。
  6. 取消了 EXISTS 转 IN 的优化（参见 SQLite 3.35.0 变更日志中的第 8b 条）， 因为发现它在更多情况下是拖慢查询而不是加速。
  7. 改进了常量传播优化，使其可在非连接查询上工作。
  8. REGEXP 扩展现在包含在CLI构建中。
