SQLite 3.12.2 发行于 2016-04-18

来自版本 3.12.0（2016-03-29）的先前更改：

    可能具有破坏性的更改：

  1. 将 SQLITE_DEFAULT_PAGE_SIZE 从 1024 提升到 4096。 将 SQLITE_DEFAULT_CACHE_SIZE 改为 -2000， 以便默认情况下仍然使用相同数量的缓存内存。 有关更多信息，请参阅 版本 3.12.0 页面大小更改 的应用说明。

    性能增强：

  2. 对 Lemon 解析器生成器 进行了增强， 使其生成更小且更快的 SQL 解析器。
  3. 仅在两个或更多附加数据库都被修改、PRAGMA synchronous 未设置为 OFF， 且没有将journal_mode 设置为 OFF、MEMORY 或 WAL 的情况下 创建 主日志 文件。
  4. 只有在 语句日志 文件大小超过阈值时才创建。 否则日志保留在内存中，不会发生任何 I/O。该阈值可以在编译时通过 SQLITE_STMTJRNL_SPILL 配置，或在启动时通过 sqlite3_config(SQLITE_CONFIG_STMTJRNL_SPILL) 设置。
  5. 即使 xBestIndex 方法未将虚拟表 IN 运算符左侧列的 sqlite3_index_constraint_usage.omit 标志设为 1， 查询规划器也能够优化 虚拟表 上的 IN 运算符。
  6. 查询规划器现在能更好地优化在三表或更多表联接中对 虚拟表 的访问， 当虚拟表的约束分布在联接中的另一个或多个表上时。
  7. 更高效地处理 应用定义的 SQL 函数，尤其是在应用定义了数百或数千 个自定义函数的情况下。
  8. 查询规划器在估算 ORDER BY 的成本时会考虑 LIMIT 子句。
  9. 配置脚本（在 Unix 上）会自动检测 pread() 和 pwrite()， 并在可用时将编译时选项设置为使用这些操作系统接口。
  10. 减少保存模式所需的内存量。
  11. 其他各种微优化以提升性能并减少内存使用。

    新功能：

  12. 向 sqlite3_db_config() 添加了 SQLITE_DBCONFIG_ENABLE_FTS3_TOKENIZER 选项， 允许在运行时启用或禁用具有两个参数的 fts3_tokenizer() SQL 函数。
  13. 向 RBU 扩展添加了 sqlite3rbu_bp_progress() 接口。
  14. PRAGMA defer_foreign_keys=ON 语句现在也会禁用外键上的 RESTRICT 操作。
  15. 添加了 sqlite3_system_errno() 接口。
  16. 添加了 SQLITE_DEFAULT_SYNCHRONOUS 和 SQLITE_DEFAULT_WAL_SYNCHRONOUS 编译时选项。 SQLITE_DEFAULT_SYNCHRONOUS 编译选项替代了 SQLITE_EXTRA_DURABLE 选项，后者不再受支持。
  17. 在 命令行 shell 中增强了 ".stats" 命令，以便在可用时显示从 /proc 获取的更多 I/O 性能信息。

    错误修复：

  18. 确保来自单个语句中的多个触发器的 sqlite3_set_auxdata() 值不会相互干扰。 工单 dc9b1c91。
  19. 修复针对形式为 “x IN (SELECT...)” 的表达式的代码生成器错误， 当右侧的 SELECT 语句是关联子查询时会导致问题。 工单 5e3c886796e5512e。
  20. 修复与 sqlite3_db_readonly() 接口相关的无害 TSAN 警告。

来自版本 3.12.1（2016-04-08）的先前更改：

  1. 修复版本 3.12.0 引入的边界条件错误， 该错误可能在大量使用 SAVEPOINT 时导致崩溃。 工单 7f7f8026eda38。
  2. 修复 视图，使其在可能的情况下继承所基于表的列数据类型。
  3. 修复查询规划器，使 IS 和 IS NULL 运算符能够在 LEFT OUTER JOIN 上驱动索引。

此补丁版本 3.12.2（2016-04-18）中的更改：

  1. 修复版本 3.12.0 和 3.12.1 中的向后兼容性问题： 声明为 "INTEGER" PRIMARY KEY（在数据类型关键字周围带有引号）的列 未被识别为 INTEGER PRIMARY KEY，导致数据库文件不兼容。 工单 7d7525cb01b68
  2. 修复一个自 版本 3.9.0 起就存在的错误， 当开启 PRAGMA reverse_unordered_selects 时， DELETE 操作可能会遗漏行。 工单 a306e56ff68b8fa5
  3. 修复代码生成器中的错误，当两个或更多 虚拟表 联接且联接外层循环中使用的虚拟表具有 IN 运算符 约束时可能导致结果不正确。
  4. 在确定用于排序大量数据的缓存大小时正确解释负数的 "PRAGMA cache_size" 值。
