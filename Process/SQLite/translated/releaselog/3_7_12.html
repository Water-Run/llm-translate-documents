SQLite 3.7.12 版发布于 2012-05-14

  1. 为 sqlite3_db_status() 添加 SQLITE_DBSTATUS_CACHE_WRITE 选项。
  2. 优化 typeof() 与 length() SQL 函数，使它们避免从磁盘不必要地读取数据库内容。
  3. 添加 FTS4 “merge” 命令、FTS4 “automerge” 命令 以及 FTS4 “integrity-check” 命令。
  4. 报告失败的特定 CHECK 约束名称。
  5. 在命令行 shell 中，如果 ".output" 命令的参数首字符是 "|"，则使用 popen() 而不是 fopen()。
  6. 在 Windows 的 VFS 中利用 OVERLAPPED，以避免一些系统调用，从而提升性能。
  7. 当 AND 运算符的一侧或另一侧恒为假时，进行更积极的优化。
  8. 提升那些 WHERE 子句中包含可以全部建立索引的多个 OR 连接项的查询性能。
  9. 添加 SQLITE_RTREE_INT_ONLY 编译时选项，以强制 R*Tree 扩展模块 在存储与计算中使用整数而非浮点数值。
  10. 增强 PRAGMA integrity_check 命令，在处理多千兆字节数据库时使用更少内存。
  11. 在 test_quota.c 附加模块中新增接口。
  12. 在命令行 shell 中添加 ".trace" 点命令。
  13. 允许递归调用虚拟表构造器。
  14. 改善复合查询中 ORDER BY 子句的优化。
  15. 优化嵌套在聚合查询中的聚合子查询。
  16. 错误修复：修复 RELEASE 命令，使其不会取消待处理查询。这解决了 3.7.11 中引入的问题。
  17. 错误修复：只有当结果集的子集满足 并且其中没有列可以为 NULL 的 UNIQUE 约束时，才不将 DISTINCT 视为多余并丢弃。工单 385a5b56b9。
  18. 错误修复：除非与 UNIQUE 索引的项相同且这些项也都非 NULL，否则不要优化掉 ORDER BY 子句。工单 2a5629202f。
  19. SQLITE_SOURCE_ID： "2012-05-14 01:41:23 8654aa9540fe9fd210899d83d17f3f407096c004"
  20. sqlite3.c 的 SHA1：57e2104a0f7b3f528e7f6b7a8e553e2357ccd2e1
