SQLite 发行版 3.50.1 于 2025-06-06

3.50.0 版本（2025-05-29）之前的更改：

  1. 新增 sqlite3_setlk_timeout() 接口，用于在支持阻塞锁的构建中为阻塞锁设置与 sqlite3_busy_timeout() 不同的单独超时。
  2. 在 上一个版本 中新增的 SQLITE_DBCONFIG_ENABLE_COMMENTS 约束略作放宽，当从已有的 sqlite_schema 表读取架构时总是允许注释。注释只会在新的 SQL 中被阻止。
  3. 新增 SQL 函数：
    1. unistr()
    2. unistr_quote()
  4. 对于内置 printf() 中的 %Q 和 %q 转换（涵盖 sqlite3_mprintf() API 以及 format() SQL 函数 等），alternate-form-1 标志（“#”）会将 控制字符 转换成适用于 unistr() 的反斜杠转义序列。
  5. CLI 增强功能：
    1. 避免直接输出大多数 控制字符。
    2. .dump 命令 的输出使用新的 unistr() SQL 函数对特殊字符进行编码，除非将 --escape 模式设为关闭。
    3. ".schema --indent" 命令 的输出在显示复杂的部分索引时格式更佳。
  6. 对 sqlite3_rsync 的增强：
    1. 取消了要求数据库必须处于 WAL 模式的限制。
    2. 同步协议得到增强，当双方初始状态非常相似时可减少网络带宽使用。
    3. sqlite3_rsync 程序在 Mac 上现在可以运行，无需指定远程端 sqlite3_rsync 可执行文件的完整路径，只要将该可执行文件安装在以下目录之一即可：$HOME/bin:/usr/local/bin:/opt/homebrew/bin。
  7. JSON 函数的更改：
    1. 修复错误：强制执行 JSON5 限制，即“\0”转义不能跟随数字。
    2. 修复错误：当 json_group_object(LABEL,VALUE) 的 LABEL 参数为 NULL 时，对象中的该元素会被省略。
    3. 优化： 如果 jsonb_set() 或 jsonb_replace() 函数在大型 JSONB 对象的内部做出更改，它们会尽量保持 JSONB 对象的大小不变，并尽量减少对内部字节的修改。这样可减少 I/O，因为 SQLite 只需写入包含更改字节的页面，而不是所有相邻页面。
  8. 改进在 Cygwin、MinGW 以及类似环境（包括 Termux）上的构建支持。
  9. 文档和源代码注释中的错字修复。
  10. 各种性能改进。
  11. JavaScript/WASM：
    1. 修复 OPFS SAHPool VFS 中长期存在的文件名摘要计算错误。在该 VFS 中由 3.50.0+ 创建的数据库不能被旧版本的 VFS 读取，但 3.50.0 可以与旧版本创建的现有数据库向后兼容。

此次特定修补版本 3.50.1（2025-06-06）中的更改：

  1. 修复由于 3.50.0 版本中加入的新优化暴露出的 jsonb_set() 及类似函数的旧有错误。
  2. 修复在使用 -DSQLITE_DEFAULT_MEMSTATUS=0 构建时可能出现的貌似无害的 ASAN 警告。
  3. 修复 sqlite3_rsync 中的一个减一错误，该错误可能导致后续页未从复制数据库传输。
  4. 查询计划优化：允许 LEFT JOIN 的右侧被 扁平化，即使它是一个虚表。
  5. 修复 sqlite3_setlk_timeout()，使其在打开快照事务以及在被另一个运行恢复的进程阻塞时使用阻塞锁。
  6. 其他在 3.50.0 发布后报告的次要修复。
