2024-01-15 发布的 SQLite 3.45.0

  1. 为 应用程序定义的 SQL 函数 添加了 SQLITE_RESULT_SUBTYPE 属性。 所有调用 sqlite3_result_subtype() 的应用程序定义的 SQL 函数 都必须使用这个新属性进行注册。否则调用 sqlite3_result_subtype() 时可能会被视为无操作。使用 -DSQLITE_STRICT_SUBTYPE=1 编译可以在试图调用 sqlite3_result_subtype() 但函数未注册为 SQLITE_RESULT_SUBTYPE 时触发 SQL 错误。 对于所有使用子类型的应用程序来说，在编译时启用 -DSQLITE_STRICT_SUBTYPE=1 是一项推荐的选项。
  2. 对 JSON SQL 函数 的增强：
    1. 所有 JSON 函数都已重写为使用一种名为 JSONB 的全新内部解析树格式。 这种新解析树格式可序列化，因此可以存储在数据库中，从而避免每次使用 JSON 值时都重新解析。
    2. 生成 JSON 的新版本函数会产生二进制 JSONB 而非 JSON 文本。
    3. json_valid() 函数添加了可选的第二个参数，用于指定第一个参数何时被视为“格式良好”。
  3. 为 FTS5 虚拟表添加了 FTS5 tokendata 选项。
  4. SQLITE_DIRECT_OVERFLOW_READ 优化现在默认启用。 若要禁用它，请在编译时使用 -DSQLITE_DIRECT_OVERFLOW_READ=0。
  5. 查询规划器改进：
    1. 不再允许传递约束优化误导查询规划器在存在更优等值约束时使用范围约束。 (论坛帖子 2568d1f6e6)
    2. 查询规划器现在更好地忽略 ANALYZE 识别出的低质量索引。 (论坛帖子 6f0958b03b)
  6. 将 SQLITE_MAX_PAGE_COUNT 的默认值从 1073741824 提升至 4294967294。
  7. 对 命令行界面（CLI） 的增强：
    1. 改善在 Windows 上显示 UTF-8 内容的效果。
    2. 自动检测 ".dump" 脚本的回放，并自动调整如 ".dbconfig defensive off" 和 ".dbconfig dqs_dll on" 等设置。
