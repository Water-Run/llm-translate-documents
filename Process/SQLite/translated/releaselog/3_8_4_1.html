SQLite 3.8.4.1 版本于 2014-03-11 发布

来自版本 3.8.4（2014-03-10）的先前更改：

  1. 通过代码优化和重构以提升性能。
  2. 在命令行 shell 中新增“.clone”和“.save”命令。
  3. 更新命令行 shell 的横幅，以在用户正在使用临时内存数据库时提醒新手。
  4. 修复命令行 shell 中对 editline 的支持。
  5. 通过 SQLITE_TESTCTRL_VDBE_COVERAGE 操作符的 sqlite3_test_control() 为 VDBE 程序添加覆盖率测试支持。
  6. 更新 _FILE_OFFSET_BITS 宏，以便在 QNX 上再次能够构建。
  7. 将 SrcList.nSrc 的数据类型从 u8 更改为 int，以规避 AIX 上 C 编译器的问题。
  8. 使 Cygwin 上的扩展加载功能正常工作。
  9. 错误修复：修复 char() SQL 函数，使其在无参数调用时返回空字符串，而不是“内存不足”错误。
  10. 错误修复：DISTINCT 现在识别 zeroblob 与全部为 0x00 的 blob 是相同的。工单 [fccbde530a]
  11. 错误修复：对于 WHERE 子句中包含 IS NOT NULL 项并且还包含 OR 项且在启用 SQLITE_ENABLE_STAT4 时编译的查询，确保计算出正确答案。工单 [4c86b126f2]
  12. 错误修复：确保在普通表与 WITHOUT ROWID 表之间的连接中正确解析“rowid”列。工单 [c34d0557f7]
  13. 错误修复：确保在用于实现包含 ORDER BY 子句的复合 SELECT 语句的并发协程中不会重复使用相同的临时寄存器，因为这样会导致错误结果。工单 [8c63ff0eca]
  14. 错误修复：确保“ORDER BY random()”子句不会被优化掉。工单 [65bdeb9739]
  15. 错误修复：修复可能出现在 TRIGGER 中包含的子查询语句中的名称解析错误。工单 [4ef7e3cfca]
  16. 错误修复：修复形式为“DEFAULT(-(-9223372036854775808))”的列默认值表达式，使其能够正确工作，并将列初始化为大约等于 +9223372036854775808.0 的浮点值。

此补丁版本 3.8.4.1（2014-03-11）的更改：

  1. 绕过一个在某些 Microsoft Visual Studio 配置中导致构建失败的 C 预处理宏冲突。
  2. 在计算skip-scan 优化的成本时，考虑到需要多个查找。
  3. SQLITE_SOURCE_ID："2014-03-11 15:27:36 018d317b1257ce68a92908b05c9c7cf1494050d0"
  4. sqlite3.c 的 SHA1：d5cd1535053a50aa8633725e3595740b33709ac5
