SQLite 发布 3.8.11.1 于 2015-07-29

来自 3.8.11 版本（2015-07-27）的先前更改：

  1. 添加了实验性的 RBU 扩展。注意此扩展仍在实验阶段，可能会以不兼容的方式发生变更。
  2. 添加了实验性的 FTS5 扩展。注意此扩展仍在实验阶段，可能会以不兼容的方式发生变更。
  3. 添加了 sqlite3_value_dup() 和 sqlite3_value_free() 接口。
  4. 增强 spellfix1 扩展以支持 ON CONFLICT 子句。
  5. IS 操作符 现在可以驱动索引。
  6. 增强查询优化器，以允许对由协程实现的 FROM 子句子查询执行 自动索引。
  7. 禁止在 公共表表达式 中使用 “rowid”。
  8. 添加 PRAGMA cell_size_check 命令，以便更早更好地检测数据库文件损坏。
  9. 在 FTS3 中的 matchinfo() 函数里添加了 matchinfo 'b' 标志。
  10. 改进数据库文件的模糊测试，并修复了发现的问题。
  11. 添加 fuzzcheck 测试程序，并在 “make test” 时使用 SQL 与数据库测试用例自动运行该程序。
  12. 添加 SQLITE_MUTEX_STATIC_VFS1 静态互斥锁，并在 Windows 的 VFS 中使用之。
  13. sqlite3_profile() 回调（由 sqlite3_reset() 或 sqlite3_finalize() 触发）将被调用，用于那些未执行完成的语句。
  14. 增强页缓存，使其可以预分配一块内存用于初始页缓存行。默认预分配设置为 100 页。在常见工作负载下可提升约 5% 的性能。
  15. 各种微优化使得在相同 CPU 周期数下的工作量增加 22.3%。SQLite 现在比 3.8.0 版本运行速度快一倍，比 3.3.9 版本快三倍。 （此数据基于在 Ubuntu 14.04 x64 上使用 gcc 4.8.2 和 -Os 对 speedtest1.c 工作负载使用 cachegrind 所测得，性能视环境而异。）
  16. 添加了 sqlite3_result_zeroblob64() 和 sqlite3_bind_zeroblob64() 接口。

    重要的错误修复：

  17. 修复 CREATE TABLE AS，以确保 TEXT 类型的列永远不会保存 INT 值。工单 f2ad7de056ab1dc9200
  18. 修复 CREATE TABLE AS，以避免在右侧的 SELECT 语句因错误中止时在 sqlite_master 表 中留下 NULL 项。工单 873cae2b6e25b
  19. 修复 skip-scan 优化，使其在对 WITHOUT ROWID 表使用 OR 优化 时仍能正确工作。工单 8fd39115d8f46
  20. 修复 sqlite3_memory_used() 与 sqlite3_memory_highwater() 接口，使其能够真正提供 64 位结果。

此特定补丁版本 3.8.11.1（2015-07-29）中的更改：

  1. 恢复 PRAGMA cache_size 的一个未记录副作用：如果数据库尚未被访问，则强制解析数据库模式。
  2. 修复了在 WITHOUT ROWID 表上存在已久的问题，该问题在 3.8.11 发布数小时后由 sqlite3_changes() 报告。
  3. SQLITE_SOURCE_ID：“2015-07-29 20:00:57 cf538e2783e468bbc25e7cb2a9ee64d3e0e80b2f”
  4. sqlite3.c 的 SHA1：3be71d99121fe5b17f057011025bcf84e7cc6c84
