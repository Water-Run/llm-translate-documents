SQLite 3.25.1 版本于 2018-09-18 发布

从 3.25.0 版本（2018-09-15）起的先前更改：

  1. 添加对 窗口函数 的支持
  2. 增强 ALTER TABLE 命令：
    1. 新增支持使用 ALTER TABLE table RENAME COLUMN oldname TO newname 重命名表中的列。
    2. 修复表重命名功能，使其在 触发器 和 视图 中对重命名表的引用也能更新。
  3. 查询优化器改进：
    1. 避免在聚合查询中对不在聚合函数内且不属于 GROUP BY 子句的列执行不必要的加载。
    2. IN 提前退出优化：在对多列索引进行查找且在非最左列上使用 IN 运算符时，如果第一个 IN 值没有匹配的行，则在继续下一个 IN 值之前先检查是否存在匹配右侧列的行。
    3. 利用传递性尝试在 WHERE 子句中传播常量值。例如，将 “a=99 AND b=a” 转换为 “a=99 AND b=99”。
  4. 在 unix VFS 中的每个 inode 上使用独立的互斥锁，而不是在它们之间共享单个锁，以在多线程环境中略微提高并发性。
  5. 增强 PRAGMA integrity_check 命令，以便更好地检测页空闲列表中的问题。
  6. 在 命令行 shell 的 ".dump" 命令中将无限大输出为 1e999。
  7. 新增 SQLITE_FCNTL_DATA_VERSION 文件控制。
  8. 新增 Geopoly 模块

    错误修复：

  9. ORDER BY LIMIT 优化可能会在预编译语句的字节码中导致无限循环；在非常隐蔽的情况下，由查询优化器中的多个小缺陷共同引发。修复票据 9936b2fa443fec03ff25。
  10. 在 UPSERT 中重新安排约束检查顺序时，确保对插入内容的亲和性转换在任何约束检查之前完成。修复票据 79cad5e4b2e219dd197242e9e。
  11. 避免在 ".eqp full" 逻辑关闭后再次使用 CLI 的 ".stats on" 命令的预编译语句。修复票据 7be932dfa60a8a6b3b26bcf76。
  12. LIKE 优化在左侧操作数具有数字亲和性且右侧模式为 "/%" 或以 ESCAPE 字符开头时会生成错误的字节码并因此返回错误结果。修复票据 c94369cae9b561b1f996d0054b。

在此特定补丁发布（3.25.1 版，2018-09-18）中的更改：

  1. 3.25.0 版本在 ALTER TABLE 中添加的额外有效性检查在被修改的表具有更新虚拟表的触发器时有时会触发误报，从而导致 ALTER TABLE 回滚并使模式保持不变。票据 b41031ea2b537237。
  2. 3.25.0 版本为 ORDER BY LIMIT 优化相关的字节码无限循环所做的修复在某些涉及窗口函数的查询中无效，需要额外的修正。票据 510cde277783b5fb
