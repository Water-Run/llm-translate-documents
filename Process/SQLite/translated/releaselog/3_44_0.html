SQLite 3.44.0 版本于 2023-11-01 发布

  1. 聚合函数现在可以在其最后一个参数之后包含 ORDER BY 子句。函数的参数按指定顺序处理。这对于像 string_agg() 和 json_group_array() 这样的函数尤为重要。
  2. 增加了对 concat() 和 concat_ws() 标量 SQL 函数的支持，使其兼容 PostgreSQL、SQLServer 和 MySQL。
  3. 增加了对 string_agg() 聚合 SQL 函数的支持，使其兼容 PostgreSQL 和 SQLServer。
  4. 在 strftime() SQL 函数 上新增了转换字母：%e %F %I %k %l %p %P %R %T %u
  5. 增加了 C 语言 API：sqlite3_get_clientdata() 与 sqlite3_set_clientdata()。
  6. 与 CREATE TABLE 相关的许多错误现在会在执行 CREATE TABLE 语句时立即抛出，而不是推迟到第一次真正使用该表时才报告。
  7. PRAGMA integrity_check 命令现在使用新的 xIntegrity 方法 验证各种内建 虚拟表 中内容的一致性。这适用于 FTS3、FTS4、FTS5、RTREE 和 GEOPOLY 扩展。
  8. SQLITE_DBCONFIG_DEFENSIVE 设置现在阻止 PRAGMA writable_schema 被开启。此前 writable_schema 可以被开启，但实际上不会使模式可写；现在它根本无法开启。
  9. 将内建的 FTS3、FTS4、FTS5、RTREE 和 GEOPOLY 虚拟表标记为 SQLITE_VTAB_INNOCUOUS，以便它们可以在高安全部署中的触发器内部使用。
  10. PRAGMA case_sensitive_like 语句已弃用，因为当模式包含 LIKE 操作符时使用它会导致 PRAGMA integrity_check 报告数据库损坏。
  11. SQLITE_USE_SEH（结构化异常处理）在使用 Microsoft C 编译器构建 SQLite 时现在默认启用。可以通过 -DSQLITE_OMIT_SEH 来禁用。
  12. 查询规划优化：
    1. 在部分索引扫描中，如果 WHERE 子句为某个表列暗示了一个常量值，就用该常量替换该列的出现。这提高了部分索引成为覆盖索引的可能性。
    2. 禁用视图扫描优化（在 版本 3.40.0 的第 3d 项添加），因为它导致多个性能回退。取而代之的是将 DISTINCT 子查询的估计行数减少 8 倍。
  13. SQLite 现在在运行时检测底层硬件是否支持比 double 精度更高的 long double，并根据检测结果选择合适的浮点例程。
  14. Windows 版 CLI 在支持的平台上现在默认对输入和输出使用 UTF-8。可以使用 --no-utf8 选项来禁用 UTF-8 支持。
