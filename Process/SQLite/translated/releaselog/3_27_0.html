SQLite 3.27.0 发布于 2019-02-07

  1. 添加了VACUUM INTO命令
  2. 如果使用了双引号字符串字面量，则在错误日志上发出 SQLITE_WARNING 消息。
  3. sqlite3_normalized_sql() 接口现在可以用于任何通过sqlite3_prepare_v2()或 sqlite3_prepare_v3() 创建的预处理语句。使用sqlite3_normalized_sql()时， 已无需再通过sqlite3_prepare_v3() 配合 SQLITE_PREPARE_NORMALIZE。
  4. 为FTS3和FTS5增加了 remove_diacritics=2 选项。
  5. 为sqlite3_prepare_v3()添加了 SQLITE_PREPARE_NO_VTAB选项。 使用此选项可防止对影子表的循环引用 导致资源泄漏。
  6. 对sqlite3_deserialize()接口的增强：
    1. 增加 SQLITE_FCNTL_SIZE_LIMIT 文件控制选项，用于设置 sqlite3_deserialize 创建的内存数据库大小的上限。默认上限为 1GiB，或 由 sqlite3_config(SQLITE_CONFIG_MEMDB_MAXSIZE) 和/或 SQLITE_MEMDB_DEFAULT_MAXSIZE 指定的值。
    2. 遵循SQLITE_DESERIALIZE_READONLY标志，该标志此前仅在文档中描述，实际为无操作。
    3. 增强TCL 接口的 "deserialize" 命令，新增 "--maxsize N" 和 "--readonly BOOLEAN" 选项。
  7. 对命令行界面的增强，主要为了支持SQLite库本身的测试和调试：
    1. 增加对 ".open --hexdb" 的支持。 用于生成 "hexdb" 文本的 "dbtotxt" 工具 已被加入源码树。
    2. 对 ".open --deserialize" 增加 "--maxsize N" 选项支持。
    3. 增加 "--memtrace" 命令行选项，用于显示所有内存分配和释放。
    4. 在启用 SQLITE_DEBUG 的构建中增加 ".eqp trace" 选项， 可一次性启用带缩进的字节码程序列表以及 PRAGMA vdbe_trace。
    5. 增加 ".progress" 命令，用于访问 sqlite3_progress_handler() 接口。
    6. 为 ".backup" 命令增加 "--async" 选项。
    7. 为 ".trace" 命令增加 "--expanded"、"--normalized"、"--plain"、"--profile"、 "--row"、"--stmt" 和 "--close" 选项。
  8. 提高了对恶意 SQL（针对恶意破坏的数据库）执行时的鲁棒性。

    错误修复：

  9. 避免在 IN 操作符上使用部分索引进行表扫描。 工单 1d958d90596593a774。
  10. 修复查询扁平化器，使其能处理包含使用 窗口函数的子查询的查询。 工单 709fcd17810f65f717
  11. 确保 ALTER TABLE 修改视图和触发器中 WITH 子句内嵌的表名和列名。
  12. 修复解析器错误，允许在表值函数周围使用括号。
  13. 修复在表达式索引上执行 OR 优化时出现的问题。 工单 d96eba87698a428c1d。
  14. 修复LEFT JOIN 强度降低优化中的问题，该优化因为 IS NOT NULL 运算符而被不当应用。 工单 5948e09b8c415bc45d。
  15. 修复REPLACE 命令，以防止它在 NOT NULL 列具有 NULL 默认值时仍然将 NULL 值悄然写入。 工单 e6f1f2e34dceeb1ed6
  16. 修复窗口函数在 关联子查询中使用时的问题。 工单 d0866b26f83e9c55e3
  17. 修复ALTER TABLE RENAME COLUMN 命令，使其能在具有冗余 UNIQUE 约束的表上正确工作。 工单 bc8d94f0fbd633fd9a
  18. 修复一个导致将zeroblob 值插入 使用表达式索引的表时被截断的错误。 工单 bb4bdb9f7f654b0bb9
