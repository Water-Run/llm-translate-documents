SQLite 3.32.1 发布于 2020-05-25

来自版本 3.32.0（2020-05-22）的先行更改：

  1. 通过 近似 ANALYZE 和 PRAGMA analysis_limit 命令添加了支持。
  2. 添加了 字节码虚拟表。
  3. 将 校验和 VFS shim 添加到源树中可运行时加载的扩展集合。
  4. 新增 iif() SQL 函数。
  5. INSERT 与 UPDATE 语句现在在计算 CHECK 约束 之前始终应用 列亲和性。该错误修复理论上可能会为具有非正统 CHECK 约束（要求 INSERT 的输入类型与声明的列类型不同）的旧数据库带来问题。更多信息请参见票证 86ba67afafded936。
  6. 添加了 sqlite3_create_filename()、sqlite3_free_filename() 以及 sqlite3_database_file_object() 接口，以更好地支持 VFS shim 实现。
  7. 将 参数数量的默认上限 从 999 提高到 32766。
  8. 为 UINT 排序序列 作为可选 加载扩展 添加代码。
  9. 对 CLI 的增强：
    1. 为 .import 命令添加选项：--csv、--ascii、--skip
    2. .dump 命令现在接受多个 LIKE 模式参数，并输出所有匹配表的并集。
    3. 在调试构建中添加 .oom 命令
    4. 为 .excel、.output 与 .once 命令添加 --bom 选项。
    5. 增强 .filectrl 命令以支持 --schema 选项。
    6. 自动加载 UINT 排序序列 扩展。
  10. LIKE 运算符的 ESCAPE 子句现在覆盖通配符字符，使行为与 PostgreSQL 保持一致。

此补丁版本 3.32.1（2020-05-25）的更改：

  1. 修复两个长期存在的漏洞，这些漏洞允许恶意 SQL 语句使运行 SQLite 的进程崩溃。这些漏洞在 3.32.0 发布约 24 小时后由第三方公布，但并非特定于 3.32.0 发布。
  2. 其他一些编译器警告修复等。
