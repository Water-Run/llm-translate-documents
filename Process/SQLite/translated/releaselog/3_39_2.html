SQLite 发布 3.39.2 于 2022-07-21

从 3.39.0 版本（2022-06-25）起的先前更改：

  1. 增加（久违的）对于 RIGHT 和 FULL OUTER JOIN 的支持。
  2. 新增二元比较运算符 IS NOT DISTINCT FROM 与 IS DISTINCT FROM， 它们分别与 IS 和 IS NOT 等价，以兼容 PostgreSQL 与 SQL 标准。
  3. 为 sqlite3_vtab_distinct() 接口新增一个返回码（值为 “3”），以表示同时具有 DISTINCT 和 ORDER BY 子句的查询。
  4. 新增了 sqlite3_db_name() 接口。
  5. Unix 操作系统接口在打开数据库文件之前会解析文件名中的所有符号链接， 以生成规范名。 如果在 sqlite3_open_v2() 或类似函数中使用了 SQLITE_OPEN_NOFOLLOW 标志， 则一旦路径中的任一元素是符号链接，打开操作就会失败。
  6. 推迟视图的物化直到确实需要，以避免当物化最终未被使用时的额外开销。
  7. HAVING 子句 现在允许用于任何聚合查询， 即使这些查询不包含 GROUP BY 子句。
  8. 诸多 微优化 总体上减少了约 2.3% 的 CPU 周期。

从 3.39.1 版本（2022-07-13）起的先前更改：

  1. 修复一个在使用包含复合 SELECT 的视图（其中只有一个分支含有 RIGHT JOIN）且该视图不位于包含该视图的查询的第一个 FROM 子句项时产生的错误结果。 forum post 174afeae5734d42d。
  2. 修复一些无害的编译器警告。
  3. 修复 ALTER TABLE RENAME 的一个长期问题， 该问题仅在 sqlite3_limit(SQLITE_LIMIT_SQL_LENGTH) 被设置为非常小的值时才会出现。
  4. 修复 FTS3 的一个长期问题， 该问题仅在启用了 SQLITE_ENABLE_FTS3_PARENTHESIS 编译时选项时才会显现。
  5. 修复构建流程，使其在同时启用 SQLITE_DEBUG 与 SQLITE_OMIT_WINDOWFUNC 编译时选项时也能正常工作。
  6. 修复 REGEXP 扩展的初始前缀优化， 使其即使前缀包含需要三字节 UTF8 编码的字符也能正确运行。
  7. 增强 sqlite_stmt 虚拟表，使其缓存全部输出。

此特定补丁版本 3.39.2（2022-07-21）的更改：

  1. 修复与在存在 LEFT JOIN 时重新排列 FROM 子句项顺序相关的查询规划器性能回归。
  2. 修复 CVE-2022-35737、Chromium 缺陷 1343348 与 1345947、 forum post 3607259d3c 以及内部测试发现的其他小问题。
