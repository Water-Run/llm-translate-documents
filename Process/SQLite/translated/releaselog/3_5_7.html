SQLite 3.5.7 版本发布于 2008-03-17

  1. 修复复合 SELECT 的寄存器分配中的一个 bug（工单 #2927），该问题由 3.5.5 版本中新 VM 代码引入。
  2. ALTER TABLE 使用双引号而非单引号来引用文件名。
  3. 在 UPDATE 或 DELETE 语句中，使用 WHERE 子句缩小物化视图的大小。（优化）
  4. 如果外层查询是聚合且内层查询包含 ORDER BY，则不要应用扁平化优化。（工单 #2943）
  5. 附加的 OS/2 更新
  6. 新增实验性的以 2 的幂为基础的首适配内存分配器。
  7. 移除代码中所有 sprintf() 的实例
  8. 在日期字符串末尾接受 “Z” 作为 Zulu 时区
  9. 修复 LIKE 优化器中的一个 bug，当第一个通配符前的最后一个字符为大写的 “Z” 时会触发该 bug
  10. 新增用于跟踪哪些页已写入日志的 “bitvec” 对象。提高速度并减少内存消耗，尤其是针对大型数据库文件。
  11. 再次让 Mac OS X 上的 SQLITE_ENABLE_LOCKING_STYLE 宏正常工作。
  12. 将语句日志存储在临时文件目录中，而不是与数据库文件同目录。
  13. 对 configure 脚本做了大量改进与清理

单页版的 SQLite 版本完整列表 与 版本编年史 也都可用。 每次提交的详细历史可在 SQLite 版本控制站点 获取。
