SQLite 3.51.0 版本于 2025-11-04 发布

  1. sqlite3.h 中的新宏：
    1. SQLITE_SCM_BRANCH → 源代码所取自的分支名称。
    2. SQLITE_SCM_TAGS → 源代码提交上的标签，空格分隔。
    3. SQLITE_SCM_DATETIME → 源代码提交的 ISO-8601 日期和时间。
  2. 两种新的 JSON 函数，jsonb_each() 和 jsonb_tree() 的功能与现有的 json_each() 和 json_tree() 函数相同，不同之处在于当 "type" 为 'array' 或 'object' 时，它们为 "value" 列返回 JSONB。
  3. carray 与 percentile 扩展现在已被内置进 源码聚合文件，但默认情况下被禁用，必须在编译时分别使用 -DSQLITE_ENABLE_CARRAY 和/或 -DSQLITE_ENABLE_PERCENTILE 选项启用。
  4. 对 TCL 接口 的增强：
    1. 为 eval 命令添加 -asdict 标志，使其将行数据设置为字典而非数组。
    2. 现在用户定义函数可以使用 break 来返回 SQL NULL。
  5. CLI 的增强：
    1. 将 ".timer" 的精度提高到微秒。
    2. 改进 "box" 和 "column" 格式模式以兼容双宽字符。
    3. ".imposter" 命令提供只读的 冒充表，可与 VACUUM 配合使用，无需 --unsafe-testing 选项。
    4. 为 CLI 命令行选项和 .open 命令 添加 --ifexists 选项。
    5. 将 ".width" 命令设置的列宽限制为 30,000 或以下，没有必要设置更宽的列，而支持更宽列会给不良分子提供机会。
  6. 性能提升：
    1. 提交只读事务时使用更少的 CPU 周期。
    2. 及早检测由于一个或多个表不含行而导致无行返回的连接。
    3. 如果标量子查询的结果未改变整体表达式的结果，则避免对其求值。
    4. 当使用带有较大 :y 的 "BETWEEN :x FOLLOWING AND :y FOLLOWING" 时，加速窗口函数查询。
  7. 添加了 PRAGMA wal_checkpoint=NOOP; 命令以及 SQLITE_CHECKPOINT_NOOP 参数至 sqlite3_wal_checkpoint_v2()。
  8. 为扩展添加了 sqlite3_set_errmsg() 接口。
  9. 添加了 sqlite3_db_status64() 接口，其行为与现有的sqlite3_db_status() 接口相同，只是返回 64 位结果。
  10. 向 sqlite3_db_status() 和 sqlite3_db_status64() 接口添加了 SQLITE_DBSTATUS_TEMPBUF_SPILL 选项。
  11. 在 会话扩展 中添加了 sqlite3changeset_apply_v3() 接口。
  12. 对于 内置的 printf() 和 format() SQL 函数，如果省略了 '+' 标志且存在 "#" 标志且所有显示数字均为 '0'，则省略负浮点数的前导 '-'。使用 '%#f' 或类似格式可避免输出 '-0.00'，而改为仅显示 '0.00'。
  13. 改善了 FTS5 生成的错误信息。
  14. 对计算列强制执行 STRICT 类型。
  15. 增强了对 VxWorks 的支持。
  16. JavaScript/WASM 现在支持 64 位 WASM。官方版本仍为 32 位，但创建自己的 64 位构建现在只需运行 "make"。
  17. 增强了对因应用程序 通过 close() 破坏 Posix 顾问锁 导致的数据库损坏的抵抗力。
