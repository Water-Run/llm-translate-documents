SQLite 3.22.0 发布于 2018-01-22

  1. sqlite3_trace_v2() 的输出现在会显示触发器内运行的每一条 SQL 语句。
  2. 即使应用程序对数据库及其所在目录没有写权限，只要该目录中存在 -shm 和 -wal 文件，也可以从 WAL 模式 数据库中读取内容。
  3. 在 R-Tree 扩展 中新增了 rtreecheck() 标量 SQL 函数。
  4. 新增了sqlite3_vtab_nochange() 和sqlite3_value_nochange() 接口，以帮助虚拟表实现优化 UPDATE 操作。
  5. 新增了 sqlite3_vtab_collation() 接口。
  6. FTS5 中新增对 "^" 初始符号语法的支持。
  7. 新扩展：
    1. Zipfile 虚拟表 可以读取并写入 ZIP 存档。
    2. 在 fileio.c 扩展中添加了 fsdir(PATH) 表值函数，用于列出目录中的文件。
    3. 用于自省和估算数据库中 btree 大小的同名虚拟表 sqlite_btreeinfo。
    4. Append VFS 是一个VFS shim，允许将 SQLite 数据库追加到其他文件。例如，可以将数据库追加到一个可执行文件，然后该可执行文件打开并读取数据库。
  8. 查询优化器增强：
    1. 使用索引快速计算 aggregate min() 或 max() 的优化扩展到了表达式索引。
    2. 是否将 FROM 子句的子查询实现为协程或使用查询扁平化，现在会考虑外部查询的结果集是否“复杂”（如果包含函数或表达式子查询）。复杂结果集会倾向于使用协程。
    3. 优化器避免使用带有未知排序函数的索引。
    4. 即使 LEFT JOIN 不是查询中最右侧的连接，优化器也会省略未使用的 LEFT JOIN。
  9. 其它性能优化：
    1. 更小更快的文本转换浮点数子程序 sqlite3AtoF()。
    2. Lemon 解析器生成器 生成更快的解析器。
    3. 使用 strcspn() C 库函数来加速 LIKE 和 GLOB 操作符。
  10. 命令行 shell 的改进：
    1. ".schema" 命令显示虚拟表的结构。
    2. 通过 .archive 命令 增加了读写SQLite Archive 文件的支持。
    3. 增加了实验性的 .expert 命令
    4. 添加了 ".eqp trigger" 版本的 ".eqp" 命令
    5. 强化了 ".lint fkey-indexes" 命令，使其可以用于WITHOUT ROWID 表。
    6. 如果 shell 的文件名参数是 ZIP 存档而不是 SQLite 数据库，则 shell 会自动使用 Zipfile 虚拟表 打开该 ZIP 存档。
    7. 增加了 edit() SQL 函数。
    8. 添加了 .excel 命令，以简化将数据库内容导出到电子表格。
    9. 当命令行或 .open 命令使用 --append 标志时，数据库会通过 Append VFS 打开。
  11. 增强 SQLITE_ENABLE_UPDATE_DELETE_LIMIT 编译时选项，使其对WITHOUT ROWID 表也有效。
  12. 提供返回记录 X 在数据库文件中起始字节偏移量的 SQL 函数 sqlite_offset(X)，前提是编译时启用了 -DSQLITE_ENABLE_OFFSET_SQL_FUNC。
  13. Bug 修复：
    1. 在 WHERE 子句中使用 OR 操作的 UPDATE 会产生无限循环。问题在 3.17.0 引入，并在一年后通过邮件列表报告。工单 47b2581aa9bfecec。
    2. 使用 skip-ahead-distinct 优化时查询结果不正确。工单 ef9318757b152e3a。
    3. 在带有 ORDER BY DESC 的连接查询中结果不正确。工单 123c9ba32130a6c9。
    4. CREATE TABLE AS 与简单 SELECT 之间的结果集列名不一致。工单 3b4450072511e621
    5. 对表达式上的索引执行 REPLACE 时出现断言失败。工单 dc3f932f5a147771
    6. 在常量索引上执行 IN 操作时出现断言失败。工单 aa98619ad08ddcab
