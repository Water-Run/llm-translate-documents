SQLite 发布 3.38.5 于 2022-05-06

从版本 3.38.0（2022-02-22）以来的变更：

  1. 添加了 -> 和 ->> 操作符，以更便捷地处理 JSON。 新的操作符与 MySQL 和 PostgreSQL 兼容。
  2. JSON 函数现已成为内置函数。不再需要使用 -DSQLITE_ENABLE_JSON1 编译时选项来启用 JSON 支持。 JSON 默认开启。使用新的 -DSQLITE_OMIT_JSON 编译时选项可禁用 JSON 接口。
  3. 对 日期和时间函数 的增强：
    1. 添加了 unixepoch() 函数。
    2. 添加了 auto 修饰符 和 julianday 修饰符。
  4. 将 printf() SQL 函数 重命名为 format() 以提升兼容性。 原始的 printf() 名称作为别名保留以兼容旧版本。
  5. 添加了 sqlite3_error_offset() 接口， 有助于将 SQL 错误定位到输入 SQL 文本中的具体字符，从而让应用程序提供更好的错误提示。
  6. 对 虚拟表 接口的增强如下：
    1. 添加了 sqlite3_vtab_distinct() 接口。
    2. 添加了 sqlite3_vtab_rhs_value() 接口。
    3. 添加了新的操作符类型 SQLITE_INDEX_CONSTRAINT_LIMIT 和 SQLITE_INDEX_CONSTRAINT_OFFSET。
    4. 添加了 sqlite3_vtab_in() 接口（及相关接口），使虚拟表能够一次性处理 IN 操作符 约束，而不必单独处理 IN 操作符右侧的每个值。
  7. 命令行界面（CLI） 的增强：
    1. 列式输出模式 得到增强，可正确处理文本中嵌入的制表符和换行符。
    2. 在 列式输出模式 中添加了诸如 "--wrap N"、“--wordwrap on”及“--quote”之类的选项。
    3. 添加了 .mode qbox 别名。
    4. .import 命令 自动消除了列名歧义。
    5. 使用新的 sqlite3_error_offset() 接口以提供更好的错误提示。
  8. 查询优化器的增强：
    1. 使用布隆过滤器加速大型分析查询。
    2. 使用平衡归并树来评估含 ORDER BY 子句的 UNION 或 UNION ALL 复合 SELECT 语句。
  9. ALTER TABLE 语句进行了更改， 在启用 PRAGMA writable_schema=ON 时，静默忽略无法解析的 sqlite_schema 表 条目。

从版本 3.38.1（2022-03-12）以来的变更：

  1. 修复了新的布隆过滤器优化可能导致某些冷门查询返回错误结果的问题。
  2. 修复了 localtime 修饰符， 使得 日期和时间函数 保留小数秒部分。
  3. 修复了 sqlite_offset SQL 函数， 即使在参数是虚拟列或视图的列等极端情况下也能正常工作。
  4. 修复了 行值 IN 操作符 约束在 虚拟表 上的问题， 即使虚拟表实现依赖字节码来过滤不满足约束的行也能正常工作。
  5. 对 assert() 语句、测试用例和文档进行了其他小修复。 详情请参阅 源码时间线。

从版本 3.38.2（2022-03-26）以来的变更：

  1. 修复了用户发现的布隆过滤器优化问题， 在对 RIGHT 表中某列为 NULL 的 LEFT JOIN 并带 WHERE 约束时可能返回错误结果。 详情见 论坛贴 031e262a89b6a9d2。
  2. 其他小补丁。详情请参阅 时间线。

从版本 3.38.3（2022-04-27）以来的变更：

  1. 修复了查询优化器在优化自动索引和布隆过滤器构造时过于激进的问题， 错误地使用 ON 子句中的条件来限制自动索引或布隆过滤器的大小，导致输出中缺失行。 详情见 论坛贴 0d3200f4f3bcd3a3。
  2. 其他小补丁。详情请参阅 时间线。

从版本 3.38.4（2022-05-04）以来的变更：

  1. 修复了在 3.38.0 版本引入的布隆过滤器拉下优化中的字节码问题， 该问题会在遇到 NULL 键时让字节码引擎进入无限循环。 详情见 论坛贴 2482b32700384a0f。
  2. 其他小补丁。详情请参阅 时间线。

在本次补丁版本 3.38.5（2022-05-06）中的变更：

  1. 修复了 3.38.4 版本命令行界面 中的一个失误。
