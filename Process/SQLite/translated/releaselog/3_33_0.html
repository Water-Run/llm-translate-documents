SQLite 3.33.0 版本于 2020-08-14 发布

  1. 支持遵循 PostgreSQL 语法的 UPDATE FROM。
  2. 将数据库文件的最大大小提升至 281 TB。
  3. 扩展了 PRAGMA integrity_check 语句，使其可选择仅验证单个表及其索引，而不必检查整个数据库文件。
  4. 为执行任意精度十进制运算新增了 decimal extension。
  5. 为处理 IEEE 754 binary64 数字增强了 ieee754 extension。
  6. CLI 增强：
    1. 为 输出模式新增了四种模式：“box”、“json”、“markdown”以及“table”。
    2. “column” 输出模式会自动扩展列以容纳最长的输出行，并且如果此前未设置过则会自动开启 “.header”。
    3. “quote” 输出模式尊重 “.separator”。
    4. decimal extension 与 ieee754 extension 已内嵌至 CLI。
  7. 查询规划器改进：
    1. 为使用 INDEXED BY 的查询新增查找全索引扫描查询计划的能力，此前这类查询会因“no query solution”而失败。
    2. 更加准确地检测缺失、不完整或有问题的 sqlite_stat1 数据，并在这些错误信息存在的情况下仍生成良好的查询计划。
    3. 改善了像 “SELECT min(x) FROM t WHERE y IN (?,?,?)” 的查询性能，前提是 t(x,y) 上有索引。
  8. 在 WAL 模式 下，若写入器崩溃并将 shm 文件 留在不一致状态，后续事务现在即使存在活动的只读事务也能恢复该 shm 文件。此增强之前，在这种情况下恢复 shm 文件会导致 SQLITE_PROTOCOL 错误。
