SQLite 发行版 3.23.0 于 2018-04-02

  1. 当使用 SQLITE_ENABLE_DESERIALIZE 编译时选项时，添加sqlite3_serialize() 和 sqlite3_deserialize() 接口。
  2. 将 TRUE 和 FALSE 识别为常量。（为了兼容性，如果存在名为“true”或“false”的列，则这些标识符指代列而非布尔常量。）
  3. 支持 IS TRUE、IS FALSE、IS NOT TRUE 和 IS NOT FALSE 运算符。
  4. 为 sqlite3_db_status() 添加 SQLITE_DBSTATUS_CACHE_SPILL 选项，用于报告已发生的缓存溢出次数。
  5. 内置的 printf 实现上的“alternate-form-2”（“!”）标志现在使字符串替换时以字符而非字节测量宽度和精度。
  6. 如果 虚表 实现中的 xColumn 方法使用 sqlite3_result_error() 返回错误信息，则优先采用该错误信息，而不是内部生成的信息。
  7. CLI 添加了 -A 命令行选项，以便更容易地管理 SQLite Archive 文件。
  8. 在 Zipfile 虚表 中增加对 INSERT OR REPLACE、INSERT OR IGNORE 与 UPDATE OR REPLACE 的支持。
  9. 强化 sqlite3changeset_apply() 接口，使其能够抵御来自故意损坏的 changeset 对象的攻击。
  10. 添加 sqlite3_normalize() 扩展函数。
  11. 查询优化器增强：
    1. 改进omit-left-join 优化，以便在右表是 UNIQUE 但不一定非 NULL 的情况下也能奏效。
    2. 改进push-down 优化，使其适用于更多的 LEFT JOIN。
    3. 添加LEFT JOIN 强度降低优化，当 WHERE 子句中存在能阻止 LEFT JOIN 额外全 NULL 行出现在输出集合中的条件时，将 LEFT JOIN 转换为普通 JOIN。
    4. 在使用行号小于最大值的 AUTOINCREMENT 表进行更新时，避免对 sqlite_sequence 表执行不必要写入。
  12. 错误修复：
    1. 修复解析器以接受有效的 row value 语法。工单 7310e2fb3d046a5
    2. 修复查询规划器，使其在 WHERE 子句的子表达式中考虑表值函数参数的依赖性。工单 80177f0c226ff54
    3. 修复复杂 OR 连接 WHERE 与 STAT4 产生的不正确结果。工单 ec32177c99ccac2
    4. 修复因自动数据类型转换导致的 表达式索引 可能损坏的问题。工单 343634942dd54ab
    5. FTS4 中的断言错误。工单 d6ec09eccf68cfc
    6. row values 中小于运算符的不正确结果。工单 f484b65f3d62305
    7. 始终将非零浮点值解释为 TRUE，即使其整数部分为零。工单 36fae083b450e3a
    8. 修复 fsdir(PATH) 表值函数（位于 fileio.c 扩展）的问题：如果将 fsdir() 表作为连接的内表使用，会导致段错误。该问题在邮件列表上报告，并由提交 7ce4e71c1b7251be 修复。
    9. 当 sqlite_master 表损坏以致 sqlite_sequence 表根页确实为 btree 索引页时，发出错误而非触发断言错误或空指针解引用。提交 525deb7a67fbd647
    10. 修复 ANALYZE 命令，使其计算名称以“sqlite”开头的表的统计信息。提交 0249d9aecf69948d
  13. 对 OSSFuzz 发现的问题的附加修复：
    1. 修复针对损坏数据库文件的 VACUUM 操作可能出现的无限循环。提交 27754b74ddf64
    2. 禁止在触发器和视图的 WITH 子句 中使用参数。提交 b918d4b4e546d
    3. 修复 row value 处理中的潜在内存泄漏。提交 2df6bbf1b8ca8
    4. 提升 replace() SQL 函数 在对兆字节级字符串进行大量替换时的性能，以避免 OSSFuzz 测试超时。提交 fab2c2b07b5d3
    5. 当 sqlite_master 表包含 CREATE TABLE AS 语句时提供适当错误信息。此前这会导致断言错误或空指针解引用。该问题由 OSSFuzz 在 GDAL 项目中发现。提交 d75e67654aa96
    6. 删除不正确的 assert() 语句。提交 823779d31eb09cda
    7. 修复在 LIKE 优化 中对 INTEGER PRIMARY KEY 的问题。提交 b850dd159918af56
