SQLite 版本 3.49.0 发布于 2025-02-06

  1. 对查询规划器的增强：
    1. 改进 查询时索引 优化，使其可用于WITHOUT ROWID 表。
    2. 为大型星形查询联接提供更好的查询计划。这修复了SQLite论坛上报告的三种不同性能回退。
    3. 当两个或多个查询具有相同的估计成本时，优先选择每行字节数较少的查询。
  2. 增强iif() SQL 函数，使其可以接受任意多（且不少于两个）的参数。
  3. 增强会话扩展，使其适用于使用生成列的数据库。
  4. 省略未正确实现且从未正常工作的 SQLITE_USE_STDIO_FOR_CONSOLE 编译期选项，改为添加 SQLITE_USE_W32_FOR_CONSOLE_IO 编译期选项。该选项仅适用于命令行工具，而不影响 SQLite 核心。它使控制台 I/O 使用 Win32 API 替代 stdio。该选项只影响 Windows 构建。
  5. 为sqlite3_db_config() 添加三个新选项，默认均为“启用”。
    1. SQLITE_DBCONFIG_ENABLE_ATTACH_CREATE
    2. SQLITE_DBCONFIG_ENABLE_ATTACH_WRITE
    3. SQLITE_DBCONFIG_ENABLE_COMMENTS
  6. 将用于预编译汇总压缩包的 configure 脚本中的Autotools替换为Autosetup。Canonical 源代码的 configure 脚本在上一个（3.48.0）版本中已经改为 Autosetup。只有汇总压缩包中的主 SQLite configure 脚本被更改。汇总压缩包的TEA子目录仍然使用依赖 Autotools 的（已弃用）配置脚本。
  7. 针对 3.48.0 版本中出现的问题，进行各类小补丁和修复。
