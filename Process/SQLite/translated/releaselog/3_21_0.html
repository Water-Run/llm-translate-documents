2017-10-24 发布的 SQLite 3.21.0

  1. 在可用时利用 F2FS 文件系统的原子写能力， 可以大幅降低事务开销。目前这需要 SQLITE_ENABLE_BATCH_ATOMIC_WRITE 编译时选项。
  2. 允许 ATTACH 和 DETACH 命令在事务内部运行。
  3. 如果 PRIMARY KEY 仅包含一列，则允许 WITHOUT ROWID 虚拟表 可写。
  4. 在 WAL 重置中写入头部后发生的 "fsync()" 现在使用检查点的同步设置。 这意味着如果在 mac 上启用了 PRAGMA checkpoint_fullfsync， 则会使用 "fullfsync"。
  5. sqlite3_sourceid() 函数尝试检测源代码是否 与版本控制中的内容不同，如果有修改，则在版本哈希的最后四个字符上显示 "alt1" 或 "alt2"。目的是检测意外或疏忽的编辑。 该机制可以被伪造。
  6. 改进了在右侧包含聚合查询的 CREATE TABLE AS 语句中列名的去引号处理。
  7. unix VFS 发出的 "stat()" 系统调用减少。
  8. 增强了 LIKE 优化，使其可以与 ESCAPE 子句一起工作。
  9. 增强了 PRAGMA integrity_check 和 PRAGMA quick_check，以检测以前遗漏的隐蔽行损坏。 同时更新这两个 pragma，使其在遇到 记录 损坏时返回错误文本，而不是 SQLITE_CORRUPT。
  10. 查询规划器现在更倾向于使用 协程 实现 FROM 子查询，而不是使用 查询扁平化 优化。协程对子查询的支持不能再被禁用。
  11. 传递 !=、IS、IS NOT、NOT NULL 以及 IS NULL 约束的信息到虚拟表的 xBestIndex 方法。
  12. 强化了 CSV 虚拟表，在最终换行符缺失时也接受最后一行输入。
  13. 删除了很少使用的 "scratch" 内存分配器。改用 SQLITE_CONFIG_SMALL_MALLOC 配置，提示 SQLite 尽量避免大内存分配。
  14. 将 群集虚拟表 添加到现有的并集虚拟表扩展中。
  15. 添加了 sqlite_dbpage 虚拟表， 用于直接访问数据库文件的页面。 源代码已内置在 合并源 中， 可通过 -DSQLITE_ENABLE_DBPAGE_VTAB 编译时选项激活。
  16. 添加了一种新的 fts5vocab 虚拟表类型 - “instance” - 它在最低层级提供对 FTS5 全文索引的直接访问。
  17. 在 Windows VFS 中删除了 rand_s() 的调用，因为它在某些旧款笔记本的 Firefox 中引发问题。
  18. src/shell.c 源代码 现在不再受版本控制。该文件现由构建过程生成。
  19. 各种 微优化 将 CPU 使用率降低约 2.1%。
  20. 错误修复：
    1. 修复 OSSFuzz 发现的错误 assert() 语句。 票据 cb91bf4290c211d
    2. 修复 sqlite3_result_pointer() 中一个隐蔽的内存泄漏。 票据 7486aa54b968e9b
    3. 通过延迟模式重置直到查询规划器完成运行，避免可能的释放后使用错误。 票据 be436a7f4587ce5
    4. 仅在 COLLATE 正确时使用表达式索引优化 ORDER BY 或 GROUP BY。 票据 e20dd54ab0e4383
    5. 修复在表达式索引中表达式为常量时出现的断言错误。 票据 aa98619ad08ddca
    6. 修复在 PRAGMA reverse_unordered_selects 后可能出现的断言错误。 票据 cb91bf4290c211d
    7. 修复使用表值函数的 IN 或 EXISTS 子查询可能引发的段错误。 票据 b899b6042f97f5
    8. 修复编译某个复杂公共表表达式时可能发生的整数溢出问题。 这是 OSSFuzz 发现的另一问题。提交 6ee8cb6ae5。
    9. 修复查询损坏数据库文件时可能出现的越界读取， 该问题由 Google Project Zero 的 Natalie Silvanovich 发现。 提交 04925dee41a21f。
