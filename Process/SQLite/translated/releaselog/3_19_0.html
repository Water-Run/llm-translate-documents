SQLite 3.19.0 发布于 2017-05-22

  1. 对每个在查询中引用但未提取列的表，SQLITE_READ 授权回调会以空字符串作为列名调用一次。
  2. 在使用表达式索引时，尝试利用索引中已经存在的表达式值，而不是加载原始列并重新计算表达式。
  3. 加强扁平化优化，使其能够扁平化 LEFT JOIN 右侧的视图。
  4. 在命令行 shell的 .dump 输出中，使用replace()而不是char()来转义嵌入的换行符和回车符。
  5. 避免在 UPDATE 语句中对未涉及外键约束列的情况执行不必要的外键处理。
  6. 在使用索引的 DISTINCT 查询中，当存在合适索引时，尝试通过索引跳转到下一个不同的条目，而不是逐行遍历。
  7. 在对无关表进行更改时，避免不必要地使sqlite3_blob句柄失效。
  8. 将 HAVING 子句中仅使用 GROUP BY 子句所列列的条件移至 WHERE 子句，以加快处理速度。
  9. 如果某个视图在同一查询中出现多次，则复用相同的视图物化。
  10. 增强PRAGMA integrity_check，使其能识别具有两个或更多相同rowid行的表。
  11. 增强FTS5查询语法，使列过滤器可应用于任意表达式。
  12. 增强json_extract()函数，以缓存并重用 JSON 输入文本的解析。
  13. 新增anycollseq.c可加载扩展，使通用 SQLite 数据库连接能够读取包含未知和/或应用特定排序序列的架构。

    Bug 修复：

  14. 修复REPLACE中的问题，该问题可能导致数据库损坏，包含两个或更多相同rowid的行。修复票据f68dc596c4e6018d。
  15. 修复PRAGMA integrity_check中的问题，该问题导致后续的VACUUM行为不佳。
  16. 修复PRAGMA foreign_key_check命令，使其在WITHOUT ROWID表上的外键工作正常。
  17. 修复 B 树逻辑中的错误，该错误可能导致 IN 运算符查询返回错误的重复结果。票据61fe9745
  18. 禁止 JSON 中数值常量的前导零。修复票据b93be8729a895a528e2。
  19. 禁止 JSON 字符串内部出现控制字符。修复票据6c9b5514077fed34551。
  20. 限制 JSON 对象和数组的递归深度，以避免递归下降解析器中过度的堆栈使用。修复票据981329adeef51011052。
