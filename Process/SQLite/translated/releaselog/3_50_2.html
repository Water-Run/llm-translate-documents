SQLite 3.50.2 发行于 2025-06-28

来自 3.50.0 版本（2025-05-29）的前期更改：

  1. 添加 sqlite3_setlk_timeout() 接口，在支持阻塞锁的构建中为阻塞锁设置一个独立于 sqlite3_busy_timeout() 的超时。
  2. 稍微放宽了 SQLITE_DBCONFIG_ENABLE_COMMENTS 约束（在上一个版本中添加），以便在从现有的 sqlite_schema 表读取模式时始终允许注释。注释仅在新 SQL 中被阻止。
  3. 新增 SQL 函数：
    1. unistr()
    2. unistr_quote()
  4. 对于内置 printf() 中的 %Q 和 %q 转换（涵盖 sqlite3_mprintf() API 和 format() SQL 函数 等），alternate-form-1 标志（“#”）会将 控制字符 转换成适用于 unistr() 的反斜杠转义形式。
  5. 命令行工具增强：
    1. 避免直接输出大多数 控制字符。
    2. .dump 命令 的输出利用新的 unistr() SQL 函数对特殊字符进行编码，除非 --escape 模式被关闭。
    3. ".schema --indent" 命令 输出在复杂部分索引的格式上有所改善。
  6. 对 sqlite3_rsync 的增强：
    1. 不再要求数据库必须处于 WAL 模式。
    2. 当双方起始非常相似时，同步协议经过加强以减少网络带宽的使用。
    3. 只要将 sqlite3_rsync 可执行文件安装在以下目录之一：$HOME/bin:/usr/local/bin:/opt/homebrew/bin，就无需在远端指定 sqlite3_rsync 可执行文件的完整路径，程序在 Mac 上即可运行。
  7. JSON 函数的更改：
    1. 错误修复：强制执行 JSON5 限制，即 "\0" 转义后不得跟数字。
    2. 错误修复：当 json_group_object(LABEL,VALUE) 的 LABEL 参数为 NULL 时，在生成的对象中省略该元素。
    3. 优化：如果 jsonb_set() 或 jsonb_replace() 函数在大型 JSONB 对象的内部进行更改，它们会尽量保持 JSONB 对象大小不变，并尽可能减少内部修改的字节数。这有助于减少 I/O，因为 SQLite 只需写入包含已更改字节的页面，而无需写入所有相邻页面。
  8. 增强了在 Cygwin、MinGW 及类似环境以及 Termux 上的构建支持。
  9. 修复文档和源代码注释中的错别字。
  10. 各种性能改进。
  11. JavaScript/WASM：
    1. 修复在 OPFS SAHPool VFS 中一直存在的文件名摘要计算错误。使用 3.50.0 及以上版本在该 VFS 中创建的数据库不能被旧版本读取，但 3.50.0 可以向后兼容地与旧版本创建的数据库一起工作。

来自 3.50.1 版本（2025-06-06）的前期更改：

  1. 修复在版本 3.50.0 中引入的新优化中暴露出来的 jsonb_set() 及类似函数的长期存在的错误。
  2. 修复在使用 -DSQLITE_DEFAULT_MEMSTATUS=0 构建时可能出现的一个看似无害的 ASAN 警告。
  3. 修复 sqlite3_rsync 中的一个 off-by-one 错误，该错误可能导致复制数据库的最后一页未被传输。
  4. 查询优化器优化：即使右侧是虚拟表，也允许将 LEFT JOIN 的右侧进行 展平。
  5. 修复 sqlite3_setlk_timeout()，以便在打开快照事务或被另一个正在恢复的进程阻塞时使用阻塞锁。
  6. 其他在 3.50.0 发行后报告的次要修复。

该补丁版本 3.50.2（2025-06-28）中的更改：

  1. 修复 concat_ws() SQL 函数，使其在连接中包含空字符串。 论坛帖 52503ac21d。
  2. 修复用于 CLI 的 file-io 扩展，使其可以使用 MinGW 编译链构建。
  3. 如果在将脏页已经溢出到 wal 文件后回滚保存点，则避免向 wal 文件写入没有校验和的帧。 论坛帖 b490f726db。
  4. 修复 Bitvec 对象，以防数据库接近最大大小 60 页时发生堆栈溢出。
  5. 修复包含 BLOB 值的 fts5 表上的 UPDATE 问题。
  6. 修复 RIGHT JOIN 上传递 IS 约束的问题。
  7. 如果查询中的聚合项数量超过最大列数，则提前抛出错误，以避免之后出现断言错误。
  8. 确保 sqlite3_setlk_timeout() 持有数据库互斥锁。
  9. 修复 API 文档中的错字。
