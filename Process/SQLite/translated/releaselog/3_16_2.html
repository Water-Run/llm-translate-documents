SQLite 3.16.2 版本发布于 2017-01-06

来自版本 3.16.0（2017-01-02）的先前更改：

  1. 使用的 CPU 周期减少 9%。(有关此性能提升的计算方式，请参见 CPU 性能测量 报告。)
  2. 为 PRAGMA 函数 增加了实验性支持。
  3. 向 sqlite3_db_config() 添加了 SQLITE_DBCONFIG_NO_CKPT_ON_CLOSE 选项。
  4. 增强 日期和时间函数，使“unixepoch”修饰符在所有支持的日期范围内都能生效。
  5. 将 lookaside 内存分配器 的默认配置从每个 128 字节 500 个槽更改为每个 512 字节 125 个槽。
  6. 增强 “WHERE x NOT NULL” 局部索引，使得当 “x” 列出现在 LIKE 或 GLOB 操作符中时仍然可用。
  7. 增强 sqlite3_interrupt()，使其会中断正在进行的 checkpoint 操作。
  8. 加快 LIKE 和 GLOB 匹配算法在模式包含多个通配符时的速度。
  9. 添加 SQLITE_FCNTL_WIN32_GET_HANDLE 文件控制操作码。
  10. 在 命令行 Shell 中添加 “.mode quote”。
  11. 在 命令行 Shell 中添加 “.lint fkey-indexes”。
  12. 在 命令行 Shell 中添加 .imposter 点命令。
  13. 作为 可加载扩展 添加 remember(V,PTR) SQL 函数。
  14. 将 SQLITE_OMIT_BUILTIN_TEST 编译时选项重命名为 SQLITE_UNTESTABLE，以更好地反映使用它的含义。

    错误修复：

  15. 修复查询优化器中一个长期存在的错误，该错误会导致在左表是子查询且连接约束来自左侧子查询的裸列名的 LEFT JOIN 中产生不正确的结果。工单 2df0107b。
  16. 在查询优化器中正确处理整数字面量 -0x8000000000000000。

来自版本 3.16.1（2017-01-03）的先前更改：

  1. 修复一个关于在 触发器 中使用 行值 的错误（参见工单 8c9458e7），该错误出现在 3.15.0 版本中，但直到 3.16.0 发布后不久才被报告。

在此补丁版本 3.16.2（2017-01-06）中的更改：

  1. 修复针对缺少辅助索引的 WITHOUT ROWID 表的 REPLACE 语句，以便其在触发器和外键存在时能够正确工作。该问题由 3.16.0 中增加的性能优化引入。工单 30027b613b4
  2. 修复 sqlite3_value_text() 接口，使其能够将 zeroblob() 生成的内容正确转换为全 0x00 字符的字符串。这是由 OSS-Fuzz 在 3.16.1 发布后发现的长期存在的问题。
  3. 修复字节码生成器，以处理 FROM 子句中本身是 UNION ALL 的子查询，其中 UNION ALL 的一侧是包含 ORDER BY 的视图。这是一个长期存在的问题，在发布 3.16.1 后被发现。见工单 190c2507。
  4. 调整 sqlite3_column_count() API，使其在返回 PRAGMA 语句时更频繁地与之前版本一致，以尽量减少可能以意想不到的方式使用该接口的应用程序的影响。
