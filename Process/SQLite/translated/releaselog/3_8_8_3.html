SQLite 3.8.8.3 版本发布于 2015-02-25

版本 3.8.8（2015-01-16）之前的更改：

    新功能：

  1. 添加了 PRAGMA data_version 命令，可用于确定数据库文件是否已经被其他进程修改。
  2. 为 sqlite3_wal_checkpoint_v2() 接口添加了 SQLITE_CHECKPOINT_TRUNCATE 选项，并对 PRAGMA wal_checkpoint 进行了相应增强。
  3. 添加了仅在启用了 SQLITE_ENABLE_STMT_SCANSTATUS 后才可用的 sqlite3_stmt_scanstatus() 接口。
  4. sqlite3_table_column_metadata() 增强后可以在 WITHOUT ROWID 表上正确工作，并在列名参数为 NULL 时检查表是否存在。该接口现在默认包含在构建中，无需额外启用 SQLITE_ENABLE_COLUMN_METADATA 编译选项。
  5. 添加了 SQLITE_ENABLE_API_ARMOR 编译时选项。
  6. 添加了 SQLITE_REVERSE_UNORDERED_SELECTS 编译时选项。
  7. 添加了 SQLITE_SORTER_PMASZ 编译时选项和 SQLITE_CONFIG_PMASZ 启动时选项。
  8. 为 sqlite3_config() 添加了 SQLITE_CONFIG_PCACHE_HDRSZ 选项，使应用程序更容易确定用于 SQLITE_CONFIG_PAGECACHE 的合适内存量。
  9. VALUES 子句 中行数不再受 SQLITE_LIMIT_COMPOUND_SELECT 的限制。
  10. 添加了实现 eval() SQL 函数的 eval.c 可加载扩展，能够递归地评估 SQL。

    性能提升：

  11. 减少平衡 B 树时涉及的 memcpy() 操作次数，整体性能提高约 3.2%。
  12. 改进了 跳跃扫描优化 的成本估算。
  13. 自动索引 优化现在可以在适当情况下生成 部分索引。

    错误修复：

  14. 在截断日志文件后立即调用 fsync()，以确保在使用 “PRAGMA journal_mode=TRUNCATE” 时断电后的持久性。
  15. 查询优化器现在识别出 LEFT JOIN 的右表中的任意列都可能为 NULL，即使该列有 NOT NULL 约束。在这种情况下避免尝试优化掉 NULL 测试。修复问题 6f2222d550f5b0ee7ed。
  16. 确保即使 DISTINCT 运算符是通过降序索引实现的，ORDER BY 仍然会将行按升序排序。修复问题 c5ea805691bfc4204b1cb9e。
  17. 修复在 共享缓存模式 下多线程运行时打开和关闭连接时可能出现的数据竞争。
  18. 修复 american fuzzy lop 发现的某些难以触发的崩溃错误。问题编号 a59ae93ee990a55。
  19. 解决了 GCC 优化器错误（MacOS 10.7 上的 gcc 4.2.1）导致 R-Tree 扩展 在使用 -O3 编译时计算结果错误的问题。

    其他更改：

  20. 除非使用 -DHAVE_STRCHRNULL 编译时选项显式启用，否则禁用 strchrnul() C 库例程。
  21. 提升了 likelihood()、likely() 和 unlikely() SQL 提示函数的有效性和准确性。

版本 3.8.8.1（2015-01-20）之前的更改：

  1. 修复了自 3.8.4 版本以来的排序逻辑错误；在包含 ORDER BY 子句、LIMIT 子句且结果集中列数约为 60 个或更多时，查询输出可能出现顺序错误的问题。问题编号 f97c4637102a3ae72b79。

版本 3.8.8.2（2015-01-30）之前的更改：

  1. 增强了 sqlite3_wal_checkpoint_v2(TRUNCATE) 接口，即使无需执行检查点操作也会截断 WAL 文件。

本次补丁版本 3.8.8.3（2015-02-25）中的更改：

  1. 修复了一个错误（问题 2326c258d02ead33），当部分索引的限定约束出现在 LEFT JOIN 的 ON 子句中时可能导致结果错误。
  2. 为 Unix 平台上的 命令行 shell 添加了可链接使用 "linenoise" 命令行编辑库的能力。
  3. SQLITE_SOURCE_ID：“2015-02-25 13:29:11 9d6c1880fb75660bbabd693175579529785f8a6b”
