SQLite 发布 3.37.2 于 2022-01-06

版本 3.37.0（2021-11-27）之前的更改：

  1. STRICT 表 提供了一种规范的数据类型管理方式，适合偏好此类做法的开发者。
  2. 添加包含 CHECK 约束 或包含 NOT NULL 约束 的 生成列 的列时，ALTER TABLE ADD COLUMN 现在会将新的约束与数据库中已有的行进行验证，只有在不违反任何约束的情况下才会继续。
  3. 新增 PRAGMA table_list 语句。
  4. CLI 的增强：
    1. 新增 .connection 命令，使 CLI 能够同时保持多个数据库连接处于打开状态。
    2. 新增 --safe 命令行选项，该选项禁用可能会对命令行指定的单个数据库文件以外产生副作用的 点命令 和 SQL 语句。
    3. 在读取跨越多行的 SQL 语句时，性能得到改进。
  5. 新增 sqlite3_autovacuum_pages() 接口。
  6. sqlite3_deserialize() 无法且从未用于 TEMP 数据库。此限制现在已在文档中注明。
  7. 查询优化器现在会在子查询和视图上省略 ORDER BY 子句，只要去除这些子句不会改变查询语义。
  8. generate_series 表值函数扩展的首个参数（“START”）现为必需。此更改旨在演示如何为表值函数编写必需参数。通过编译时选项 -DZERO_ARGUMENT_GENERATE_SERIES 可以保留旧行为。
  9. 新增 sqlite3_changes64() 和 sqlite3_total_changes64() 接口。
  10. 为 sqlite3_open_v2() 添加了 SQLITE_OPEN_EXRESCODE 标志选项。
  11. 使用更少内存来容纳数据库模式。

版本 3.37.1（2021-12-30）之前的更改：

  1. 修复了由UPSERT 增强在版本 3.35.0中引入的一个错误，该错误可能会为某些晦涩但有效的 SQL 生成错误的字节码，甚至可能导致空指针解除引用。
  2. 修复了在读取损坏数据库文件时，FTS5 中可能发生的越界读取。
  3. 提升了 CLI 中 --safe 选项的稳健性。
  4. 对 assert() 语句和测试用例进行了其他一些小修复。

此补丁发布版本 3.37.2（2022-01-06）的更改：

  1. 修复了在版本 3.35.0（2021-03-12）中引入的一个错误，当在 PRAGMA temp_store=MEMORY 模式下回滚 SAVEPOINT 且随后进行了其他更改并最终提交外层事务时，该错误可能导致数据库损坏。提交 73c2b50211d3ae26
  2. 修复了 ON DELETE CASCADE 与 ON UPDATE CASCADE 的一个长期问题：在局部 DDL 更改后用于实现级联更改的字节码缓存未被重置。提交 5232c9777fe4fb13。
  3. 其他一些对生产构建无影响的小修复。
