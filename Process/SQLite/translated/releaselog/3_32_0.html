SQLite Release 3.32.0 On 2020-05-22

  1. 添加了对 近似 ANALYZE 使用 PRAGMA analysis_limit 命令的支持。
  2. 添加了 字节码虚拟表。
  3. 将 checksum VFS shim 添加到源代码树中的运行时可加载扩展集合中。
  4. 添加了 iif() SQL 函数。
  5. INSERT 和 UPDATE 语句现在在计算 CHECK 约束 之前始终应用 列亲和性。此 Bug 修复理论上可能会对那些 CHECK 约束要求 INSERT 的输入类型与声明的列类型不同的不按常规编写的旧数据库造成问题。更多信息请参见票据 86ba67afafded936。
  6. 添加了 sqlite3_create_filename()、sqlite3_free_filename() 和 sqlite3_database_file_object() 接口，以更好地支持 VFS shim 实现。
  7. 将 参数 数量的 默认上限 从 999 提高到 32766。
  8. 将 UINT 校对顺序 作为一个可选的 可加载扩展 添加到代码中。
  9. 对 CLI 的增强：
    1. 为 .import 命令添加选项：--csv、--ascii、--skip。
    2. .dump 命令现在接受多个 LIKE 模式参数并输出所有匹配表的并集。
    3. 在调试构建中添加 .oom 命令。
    4. 为 .excel、.output 和 .once 命令添加 --bom 选项。
    5. 增强 .filectrl 命令以支持 --schema 选项。
    6. 自动加载 UINT 校对顺序 扩展。
  10. LIKE 操作符的 ESCAPE 子句现在会覆盖通配符字符，使其行为与 PostgreSQL 保持一致。
