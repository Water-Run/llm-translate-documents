SQLite 3.39.1 发行于 2022-07-13

来自版本 3.39.0 (2022-06-25) 的早前更改：

  1. 添加（早该实现的）对 RIGHT and FULL OUTER JOIN 的支持。
  2. 添加新的二元比较操作符 IS NOT DISTINCT FROM 和 IS DISTINCT FROM，它们分别与 IS 和 IS NOT 等价，以兼容 PostgreSQL 和 SQL 标准。
  3. 为 sqlite3_vtab_distinct() 接口添加新的返回码（值为 “3”），用于指示同时包含 DISTINCT 和 ORDER BY 子句的查询。
  4. 添加了 sqlite3_db_name() 接口。
  5. unix OS 接口在打开数据库文件前，会将文件名中的所有符号链接解析为规范名称。如果在 sqlite3_open_v2() 或类似接口中使用了 SQLITE_OPEN_NOFOLLOW 标志，则路径中的任一元素若为符号链接，打开操作将失败。
  6. 延迟视图的物化直至真正需要，从而避免在物化最终未被使用时进行不必要的工作。
  7. SELECT 语句 的 HAVING 子句 现在允许出现在任何聚合查询中，即使该查询没有 GROUP BY 子句。
  8. 许多 微优化 累计将 CPU 周期减少约 2.3%。

此特定补丁版本 3.39.1 (2022-07-13) 的更改：

  1. 修复使用包含复合 SELECT 的视图时产生的错误结果，该视图的其中一个分支仅包含 RIGHT JOIN 且该视图不是包含它的查询的第一个 FROM 子句项。论坛帖子 174afeae5734d42d。
  2. 修复一些无害的编译器警告。
  3. 修复 ALTER TABLE RENAME 的一个长期问题，仅在将 sqlite3_limit(SQLITE_LIMIT_SQL_LENGTH) 设为非常小的值时才会出现。
  4. 修复 FTS3 的一个长期问题，该问题仅在启用 SQLITE_ENABLE_FTS3_PARENTHESIS 编译时才会出现。
  5. 修复构建，使其在同时提供 SQLITE_DEBUG 与 SQLITE_OMIT_WINDOWFUNC 编译选项时仍能正常工作。
  6. 修复 REGEXP 扩展的初始前缀优化，使其即使在前缀包含需要 3 字节 UTF8 编码的字符时也能正确工作。
  7. 增强 sqlite_stmt 虚拟表，以缓冲其所有输出。
