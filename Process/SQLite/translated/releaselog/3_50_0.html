SQLite 3.50.0 发布于 2025-05-29

  1. 添加 sqlite3_setlk_timeout() 接口，该接口为支持阻塞锁的构建设置一个独立的超时， 与 sqlite3_busy_timeout() 区别开来，用于阻塞锁。
  2. SQLITE_DBCONFIG_ENABLE_COMMENTS 约束（在 上一个版本 中添加）稍作放宽，以便从预先存在的 sqlite_schema 表中读取模式时始终允许注释。只有在新的 SQL 中会阻止注释。
  3. 新的 SQL 函数：
    1. unistr()
    2. unistr_quote()
  4. 对于内置的 printf() 中的 %Q 与 %q 转换（涵盖 sqlite3_mprintf() API、format() SQL 函数 及类似功能）， 备用格式标志 1 (“#”) 会将 控制字符 转换为适用于 unistr() 的反斜杠转义序列。
  5. 命令行界面增强：
    1. 避免直接输出大多数 控制字符。
    2. .dump 命令 的输出使用新的 unistr() SQL 函数对特殊字符进行编码，除非 --escape 模式设为 off。
    3. 从 ".schema --indent" 命令 输出复杂局部索引时格式更佳。
  6. 对 sqlite3_rsync 的增强：
    1. 取消对数据库必须处于 WAL 模式的要求。
    2. 在双方初始非常相似的情况下，优化同步协议以使用更少的网络带宽。
    3. sqlite3_rsync 程序现在在 Mac 上运行时，无需为远程端指定 sqlite3_rsync 可执行文件的完整路径， 只要将可执行文件安装在这些目录中的任意一个：$HOME/bin:/usr/local/bin:/opt/homebrew/bin。
  7. JSON 函数的更改：
    1. 错误修复：强制执行 JSON5 规则，禁止 “\0” 转义后接数字。
    2. 错误修复：当 json_group_object(LABEL,VALUE) 的 LABEL 参数为 NULL 时，结果对象中省略该元素。
    3. 优化： 如果 jsonb_set() 或 jsonb_replace() 函数在大型 JSONB 对象的内部进行更改，它们努力保持 JSONB 对象的大小不变，并尽量少修改内部字节。 这有助于减少 I/O，因为 SQLite 只需要写入包含被更改字节的页面，而不是所有周围的页面。
  8. 更好地支持 Cygwin、MinGW 及类似环境的构建，以及 Termux。
  9. 文档和源代码注释中的错别字修复。
  10. 各类性能改进。
  11. JavaScript/WASM：
    1. 修复 OPFS SAHPool VFS 中长期存在的文件名摘要计算错误。由 3.50.0+ 创建的数据库 无法被旧版本的 VFS 读取，但 3.50.0 可以向后兼容地处理旧版本创建的现有数据库。
