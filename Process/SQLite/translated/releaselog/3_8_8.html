SQLite 3.8.8 版本发布于 2015-01-16

    新特性：

  1. 新增 PRAGMA data_version 命令，可用于确定数据库文件是否已被其他进程修改。
  2. 在 sqlite3_wal_checkpoint_v2() 接口中新增 SQLITE_CHECKPOINT_TRUNCATE 选项，并对 PRAGMA wal_checkpoint 做出相应增强。
  3. 新增 sqlite3_stmt_scanstatus() 接口，仅在编译时启用 SQLITE_ENABLE_STMT_SCANSTATUS 时可用。
  4. sqlite3_table_column_metadata() 被增强后可在 WITHOUT ROWID 表上正确工作，并且在列名参数为 NULL 时会检查表是否存在。该接口现在也默认包含在构建中，无需再使用 SQLITE_ENABLE_COLUMN_METADATA 编译选项。
  5. 新增 SQLITE_ENABLE_API_ARMOR 编译选项。
  6. 新增 SQLITE_REVERSE_UNORDERED_SELECTS 编译选项。
  7. 新增 SQLITE_SORTER_PMASZ 编译选项以及 SQLITE_CONFIG_PMASZ 启动时选项。
  8. 在 sqlite3_config() 中新增 SQLITE_CONFIG_PCACHE_HDRSZ 选项，便于应用程序确定用于 SQLITE_CONFIG_PAGECACHE 的合适内存量。
  9. VALUES 子句 中的行数不再受到 SQLITE_LIMIT_COMPOUND_SELECT 的限制。
  10. 新增 eval.c 可加载扩展，实现了会递归执行 SQL 的 eval() SQL 函数。

    性能增强：

  11. 减少平衡 B 树时涉及的 memcpy() 操作次数，整体性能提升约 3.2%。
  12. 改进了 跳跃扫描优化 的成本估算。
  13. 自动索引 优化现在可以在合适时生成 部分索引。

    错误修复：

  14. 在设置 “PRAGMA journal_mode=TRUNCATE” 后，截断日志文件后立即调用 fsync() 以确保停电后的持久性。
  15. 查询规划器现在认识到 LEFT JOIN 右侧表中的任意列都可能为 NULL，即便该列定义了 NOT NULL 约束。在这些情况下避免尝试优化掉 NULL 检查。修复了工单 6f2222d550f5b0ee7ed。
  16. 确保 ORDER BY 即便在使用降序索引实现 DISTINCT 操作时也能按升序排列行。修复了工单 c5ea805691bfc4204b1cb9e。
  17. 修复在多个线程的 共享缓存模式 下运行时，某些线程打开和关闭连接可能发生的竞态条件。
  18. 修复 american fuzzy lop 发现的晦涩崩溃问题。对应工单 a59ae93ee990a55。
  19. 规避 GCC 优化器的一个错误（适用于 MacOS 10.7 上的 gcc 4.2.1），该错误在使用 -O3 编译时导致 R-Tree 扩展 计算出错误结果。

    其他变更：

  20. 除非通过 -DHAVE_STRCHRNULL 编译选项显式启用，否则禁用 strchrnul() C 库例程。
  21. 提高 likelihood()、likely() 和 unlikely() SQL 提示函数的效果与准确性。
  22. SQLITE_SOURCE_ID: "2015-01-16 12:08:06 7d68a42face3ab14ed88407d4331872f5b243fdf"
  23. sqlite3.c 的 SHA1: 91aea4cc722371d58aae3d22e94d2a4165276905
