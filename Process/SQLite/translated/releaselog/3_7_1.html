SQLite 3.7.1 版本发布 - 2010-08-23

  1. 在 sqlite3_db_status() 接口中新增命令 SQLITE_DBSTATUS_SCHEMA_USED 和 SQLITE_DBSTATUS_STMT_USED，以报告连接用于保存模式和预处理语句所占用的内存量。
  2. 将数据库页面的最大大小从 32KiB 提升至 64KiB。
  3. 即便右侧字符串不包含通配符，也会使用 LIKE 优化。
  4. 在 unix 和 windows 上的 sqlite3_file_control() 接口中新增 SQLITE_FCNTL_CHUNK_SIZE 控制语义，从而使数据库文件以较大块增长，以减少磁盘碎片。
  5. 修复了查询优化器中的一个错误，避免在某些复杂连接上相对于 3.6.23.1 的性能回退。
  6. 修复了 OS/2 后端中的一个拼写错误。
  7. 重构了分页器模块。
  8. 编译时选项 SQLITE_MAX_PAGE_SIZE 现已被悄无声息地忽略。页面的最大大小现已固定为 65536 字节。

在同一页面上还提供了 SQLite 发布完整列表 以及 年表。每次提交的详细历史记录可在 SQLite 版本控制站点 查阅。
