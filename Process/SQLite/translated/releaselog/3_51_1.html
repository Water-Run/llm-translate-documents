SQLite 版本 3.51.1 于 2025-11-28 发布

从版本 3.51.0（2025-11-04）以来的先前更改：

  1. sqlite3.h 中的新宏：
    1. SQLITE_SCM_BRANCH → 源代码所属分支的名称。
    2. SQLITE_SCM_TAGS → 源代码提交上的标签，空格分隔。
    3. SQLITE_SCM_DATETIME → 源代码提交的 ISO-8601 日期和时间。
  2. 两个新的 JSON 函数，jsonb_each() 和 jsonb_tree()， 与现有的 json_each() 和 json_tree() 函数工作方式相同， 但当 "type" 为 'array' 或 'object' 时，“value”列返回 JSONB。
  3. carray 和 percentile 扩展现已内嵌 在汇编源中，但默认被禁用，必须在编译时 使用 -DSQLITE_ENABLE_CARRAY 和/或 -DSQLITE_ENABLE_PERCENTILE 选项分别启用。
  4. 对TCL 接口的增强：
    1. 为 eval 命令新增 -asdict 标志，以便其将行数据设置为字典而非数组。
    2. 现在用户定义函数可以通过 break 返回 SQL NULL。
  5. CLI 增强：
    1. 将 ".timer" 的精度提升至微秒。
    2. 改进 "box" 和 "column" 格式模式以处理双宽字符。
    3. ".imposter" 命令提供只读的 冒名顶替表， 可与 VACUUM 配合使用，并且不需要 --unsafe-testing 选项。
    4. 为 CLI 命令行选项和 .open 命令 添加 --ifexists 选项。
    5. 将 ".width" 命令设置的列宽限制为 30,000 及以下，没有充分理由支持更宽的列，而支持更宽的列可能被恶意利用。
  6. 性能提升：
    1. 提交只读事务时使用更少的 CPU 周期。
    2. 提前检测由于某些表为空而导致的无结果连接。
    3. 如果标量子查询的结果不会改变整个表达式的结果，则避免对其进行评估。
    4. 使用具有较大 :y 的 "BETWEEN :x FOLLOWING AND :y FOLLOWING" 时，窗口函数查询更快。
  7. 添加 PRAGMA wal_checkpoint=NOOP; 命令以及 SQLITE_CHECKPOINT_NOOP 参数供 sqlite3_wal_checkpoint_v2() 使用。
  8. 添加供扩展使用的 sqlite3_set_errmsg() API。
  9. 添加 sqlite3_db_status64() API，其用法与现有的 sqlite3_db_status() 相同，只是返回 64 位结果。
  10. 为 sqlite3_db_status() 和 sqlite3_db_status64() 接口添加 SQLITE_DBSTATUS_TEMPBUF_SPILL 选项。
  11. 在会话扩展中添加 sqlite3changeset_apply_v3() 接口。
  12. 对于内置的 printf() 以及 format() SQL 函数， 如果省略 '+' 标志且带有 "#" 标志且所有显示的数字均为 '0'，则省略负浮点数的前导 '-'。 请使用 '%#f' 或类似格式以避免输出 '-0.00'，而仅显示 '0.00'。
  13. 改进 FTS5 生成的错误信息。
  14. 对计算列强制 STRICT 类型。
  15. 改进对 VxWorks 的支持。
  16. JavaScript/WASM 现在支持 64 位 WASM。官方构建仍为 32 位，但现在只需运行 "make" 即可自行创建 64 位构建。
  17. 增强数据库对因应用程序 通过 close() 破坏 Posix 顾问锁 导致的损坏的抗性。

特定补丁版本 3.51.1（2025-11-28）的更改：

  1. 修复由于 3.51.0 版本第 6b 项的优化导致的嵌套 EXISTS 查询错误结果。
  2. 修复在 3.51.0 版本中新优化所暴露的 fts5vocab 虚拟表的潜在缺陷。
