SQLite 发行 3.15.0 于 2016-10-14

  1. 添加了对 行值 的支持。
  2. 允许在 部分索引 的 WHERE 子句中使用 确定性 SQL 函数。
  3. 在 unix VFS 上添加了 "modeof=filename" URI 参数
  4. 添加了对 SQLITE_DBCONFIG_MAINDBNAME 的支持。
  5. 新增对 ATTACH 数据库执行 VACUUM 的能力。
  6. 对 命令行 Shell 的增强：
    1. 添加了 ".testcase" 和 ".check" 点命令。
    2. 在 ".open" 点命令中新增 --new 选项，使其在打开之前清除数据库中的任何先前内容。
  7. 增强了 fts5vocab 虚拟表，以高效处理 "ORDER BY term"。
  8. 各种微观优化在常见工作负载下减少了超过 7% 的 CPU 使用率。本次版本的大多数优化集中在前端 (sqlite3_prepare_v2())。

    错误修复：

  9. 乘法运算符现在可以在所有边缘情况下正确检测 64 位整数溢出，并提升为浮点数。修复工单 1ec41379c9c1e400。
  10. 在列作为 IN 运算符 左侧使用时，正确处理具有冗余唯一索引的列。修复工单 0eab1ac759。
  11. 在 表达式索引 的范围查询中跳过 NULL 条目。 修复工单 4baa46491212947。
  12. 确保 sqlite_sequence 表中的 AUTOINCREMENT 计数器在对 "INSERT ... SELECT" 语句执行“Xfer 优化”时被初始化。 修复工单 7b3328086a5c116c。
  13. 确保 ORDER BY LIMIT 优化 (来自检入 559733b09e) 能与 INTEGER PRIMARY KEY 上的 IN 运算符配合使用。修复工单 96c1454c
