2021-03-15 发布的 SQLite 3.35.1

来自版本 3.35.0 (2021-03-12) 的先前更改：

  1. 添加了 内置 SQL 数学函数()。 (需要 -DSQLITE_ENABLE_MATH_FUNCTIONS 编译时选项。)
  2. 增加了对 ALTER TABLE DROP COLUMN 的支持。
  3. 推广 UPSERT：
    1. 允许按顺序评估的多个 ON CONFLICT 子句。
    2. 最后的 ON CONFLICT 子句可以省略冲突目标，但仍然使用 DO UPDATE。
  4. 为 DELETE、INSERT 和 UPDATE 语句添加了对 RETURNING 子句的支持。
  5. 在包含非常大的 TEXT 或 BLOB 值的数据库上运行 VACUUM 时使用更少内存。 不再需要一次性将整个 TEXT 或 BLOB 保存在内存中。
  6. 在指定 公用表表达式 时为 MATERIALIZED 和 NOT MATERIALIZED 提示添加支持。 默认行为此前为 NOT MATERIALIZED，但现已改为对多次使用的 CTE 使用 MATERIALIZED。
  7. SQLITE_DBCONFIG_ENABLE_TRIGGER 和 SQLITE_DBCONFIG_ENABLE_VIEW 设置现已修改，仅控制主数据库模式或附加数据库模式中的触发器和视图，不再控制 TEMP 模式中的内容。 TEMP 模式中的触发器和视图始终允许。
  8. 查询规划器/优化器改进：
    1. 强化 最小/最大优化，使其更好地与 IN 运算符和上一个版本的 OP_SeekScan 优化配合。
    2. 尝试将 WHERE 子句中的 EXISTS 运算符视为 IN 运算符进行处理，在这种转换有效且可能提升性能的情况下。
    3. 即使父查询是连接，也允许将 UNION ALL 子查询 展平。
    4. 在 WHERE 子句中的 IS NOT NULL 表达式上(如适用)使用索引，即使 STAT4 被禁用。
    5. 如果 "x" 是具有 "NOT NULL" 约束且未参与外连接的列，那么形如 "x IS NULL" 或 "x IS NOT NULL" 的表达式 可能被直接简化为 FALSE 或 TRUE。
    6. 如果 UPDATE 语句未修改与外键关联的任何列，则避免检查外键约束。
    7. 允许将由常量和子查询中所有窗口函数 PARTITION BY 子句中出现的表达式组成的 WHERE 条件 下推 至包含窗口函数的子查询。
  9. CLI 增强：
    1. 增强 ".stats" 命令以接受新参数 "stmt" 和 "vmstep"，分别仅显示准备语句统计信息和仅显示虚拟机步计数。
    2. 添加 ".filectrl data_version" 命令。
    3. 增强 ".once" 和 ".output" 命令，使得如果目标参数以 "|" 开头（表示输出重定向到管道），则参数无需加引号。
  10. Bug 修复：
    1. 修复在处理带有相关的 WHERE 子句且包含 "HAVING 0" 子句的语法错误 SELECT 语句时可能发生的 NULL 指针解引用。（在 3.34.1 补丁版本中也已修复。）
    2. 修复版本 3.33.0 的 IN 运算符优化 中的一个可能导致错误结果的 bug。
    3. 如果模式以 "%" 结尾且存在 "ESCAPE '_'" 子句，则修复 LIKE 运算符 返回错误答案的问题。

此补丁版本 3.35.1 (2021-03-15) 的更改：

  1. 修复在新 DROP COLUMN 功能中，当作用于建立索引且索引定义中引用了列名时可能出现的 bug。
  2. 改进 CLI 中 .dump 命令的内置文档。
