SQLite 发布 3.8.4 于 2014-03-10

  1. 代码优化和重构以提升性能。
  2. 向命令行 shell 添加 “.clone” 和 “.save” 命令。
  3. 更新命令行 shell 横幅以提醒新手用户正在使用临时内存数据库。
  4. 修复命令行 shell 中的 editline 支持。
  5. 通过 SQLITE_TESTCTRL_VDBE_COVERAGE 命令和 sqlite3_test_control() 支持 VDBE 程序的覆盖测试。
  6. 更新 _FILE_OFFSET_BITS 宏以使构建在 QNX 上再次可用。
  7. 将 SrcList.nSrc 的数据类型从 u8 更改为 int 以解决 AIX 上 C 编译器的问题。
  8. 在 Cygwin 上使扩展加载正常工作。
  9. 错误修复：修复 char() SQL 函数，使其在无参数调用时返回空字符串而不是“内存不足”错误。
  10. 错误修复：DISTINCT 现在识别 zeroblob 与全 0x00 字节的 blob 是同一个东西。 问题单 [fccbde530a]
  11. 错误修复：在 WHERE 子句包含 IS NOT NULL 以及 OR 项且使用 SQLITE_ENABLE_STAT4 编译时，对应查询返回正确结果。 问题单 [4c86b126f2]
  12. 错误修复：确保在普通表与 WITHOUT ROWID 表的联接中“rowid”列正确解析。 问题单 [c34d0557f7]
  13. 错误修复：确保用于实现包含 ORDER BY 子句的复合 SELECT 语句的并发协程不会重复使用相同的临时寄存器，因为这样可能导致错误答案。 问题单 [8c63ff0eca]
  14. 错误修复：确保 “ORDER BY random()” 子句不会被优化掉。 问题单 [65bdeb9739]
  15. 错误修复：修复可能出现在 TRIGGER 内部的子查询语句中的名称解析错误。 问题单 [4ef7e3cfca]
  16. 错误修复：修复形式为 “DEFAULT(-(-9223372036854775808))” 的列默认值表达式，使其正确工作，将列初始化为约等于 +9223372036854775808.0 的浮点值。
  17. SQLITE_SOURCE_ID: "2014-03-10 12:20:37 530a1ee7dc2435f80960ce4710a3c2d2bfaaccc5"
  18. sqlite3.c 的 SHA1：b0c22e5f15f5ba2afd017ecd990ea507918afe1c
