SQLite 发布 3.8.8.2 于 2015-01-30

来自版本 3.8.8（2015-01-16）的先前更改：

    新特性：

  1. 新增了 PRAGMA data_version 命令，可用于确定数据库文件是否已被其他进程修改。
  2. 在 sqlite3_wal_checkpoint_v2() 接口中新增了 SQLITE_CHECKPOINT_TRUNCATE 选项，并针对 PRAGMA wal_checkpoint 做了相应增强。
  3. 新增了 sqlite3_stmt_scanstatus() 接口，仅在编译时启用 SQLITE_ENABLE_STMT_SCANSTATUS 时可用。
  4. sqlite3_table_column_metadata() 得到增强，可在 WITHOUT ROWID 表上正确运行，并在列名参数为 NULL 时检查表是否存在。该接口现在默认包含在构建中，无需再启用 SQLITE_ENABLE_COLUMN_METADATA 编译选项。
  5. 新增了 SQLITE_ENABLE_API_ARMOR 编译选项。
  6. 新增了 SQLITE_REVERSE_UNORDERED_SELECTS 编译选项。
  7. 新增了 SQLITE_SORTER_PMASZ 编译选项以及 SQLITE_CONFIG_PMASZ 启动时选项。
  8. 在 sqlite3_config() 中新增了 SQLITE_CONFIG_PCACHE_HDRSZ 选项，使应用程序更容易确定与 SQLITE_CONFIG_PAGECACHE 相配的内存量。
  9. VALUES 子句 中的行数不再受 SQLITE_LIMIT_COMPOUND_SELECT 的限制。
  10. 新增了 eval.c 可加载扩展，实现了一个将递归评估 SQL 的 eval() 函数。

    性能提升：

  11. 减少在平衡 B 树时涉及的 memcpy() 操作次数，整体性能提升 3.2%。
  12. 改进了 跳过扫描优化 的成本估算。
  13. 自动索引 优化现在能够在合适时生成 部分索引。

    错误修复：

  14. 在使用 “PRAGMA journal_mode=TRUNCATE” 时，截断日志文件后立即调用 fsync()，以确保断电后的持久性。
  15. 查询规划器现在认识到 LEFT JOIN 右侧表中的任意列即便定义了 NOT NULL 约束也可以为 NULL，避免在这些情况下尝试优化掉 NULL 测试。修复票据 6f2222d550f5b0ee7ed。
  16. 确保即使 DISTINCT 操作符通过降序索引实现，ORDER BY 仍然以升序排序输出。修复票据 c5ea805691bfc4204b1cb9e。
  17. 修复在 共享缓存模式 下使用大量线程运行时某些线程打开关闭连接时可能发生的数据竞争问题。
  18. 修复 american fuzzy lop 发现的隐晦崩溃漏洞。票据 a59ae93ee990a55。
  19. 在 GCC 4.2.1 for macOS 10.7 的优化器存在错误时，R 树扩展在使用 -O3 编译时会计算错误结果，此问题已通过规避优化器 Bug 解决。

    其他更改：

  20. 除非通过 -DHAVE_STRCHRNULL 编译选项特别启用，否则禁用 C 库函数 strchrnul()。
  21. 提高了 likelihood()、likely() 与 unlikely() SQL 提示函数的效果与准确性。

来自版本 3.8.8.1（2015-01-20）的先前更改：

  1. 修复了自 3.8.4 版本以来存在的排序逻辑错误，在包含 ORDER BY、LIMIT 且结果集列数约为 60 或更多的查询中可能导致输出顺序错误。票据 f97c4637102a3ae72b79。
