SQLite 3.35.4 发布于 2021-04-02

3.35.0 版本前的更改（2021-03-12）：

  1. 新增内置 SQL 数学函数()。（需要-DSQLITE_ENABLE_MATH_FUNCTIONS 编译时选项。）
  2. 新增对ALTER TABLE DROP COLUMN 的支持。
  3. 将UPSERT 扩展：
    1. 允许多个按顺序评估的 ON CONFLICT 子句，
    2. 最后一个 ON CONFLICT 子句可以省略冲突目标但仍然使用 DO UPDATE。
  4. 在DELETE、INSERT 和UPDATE 语句中新增对RETURNING 子句的支持。
  5. 在包含极大 TEXT 或 BLOB 值的数据库上运行VACUUM 时减少内存使用。现在无需一次性将整个 TEXT 或 BLOB 保留在内存中。
  6. 在指定公共表表达式时新增对MATERIALIZED 与NOT MATERIALIZED 提示的支持。此行为之前默认为 NOT MATERIALIZED，现在对多次使用的 CTE 改为 MATERIALIZED。
  7. SQLITE_DBCONFIG_ENABLE_TRIGGER 与SQLITE_DBCONFIG_ENABLE_VIEW 设置被修改为仅控制主数据库架构或附加数据库架构中的触发器与视图，而不再控制 TEMP 架构。TEMP 触发器与视图始终允许。
  8. 查询规划器/优化器改进：
    1. 对最小/最大优化进行了增强，使其在处理 IN 运算符和前一版本的 OP_SeekScan 优化时表现更佳。
    2. 在 WHERE 子句中尽可能将 EXISTS 运算符视为 IN 运算符处理，前提是该转换有效且有望提升性能。
    3. 即使父查询为联接，也允许对 UNION ALL 子查询进行扁平化。
    4. 在 WHERE 子句中的 IS NOT NULL 表达式上（如适用）使用索引，即使 STAT4 被禁用。
    5. 形式为 “x IS NULL” 或 “x IS NOT NULL” 的表达式如果 “x” 是具有 “NOT NULL” 约束且未参与外部连接的列，则可能被简化为 FALSE 或 TRUE。
    6. 如果 UPDATE 语句未修改任何与外键相关的列，则跳过外键约束检查。
    7. 允许 WHERE 项被推送到包含窗口函数的子查询中，只要该 WHERE 项完全由常量以及在子查询中所有窗口函数的 PARTITION BY 子句中出现的表达式副本组成。
  9. 命令行界面（CLI） 增强：
    1. 增强 ".stats" 命令以接受新参数 "stmt" 和 "vmstep"，分别仅显示准备语句统计信息和虚拟机步数。
    2. 新增 ".filectrl data_version" 命令。
    3. 增强 ".once" 与 ".output" 命令，使得当目标参数以 "|" 开头（表示输出重定向到管道）时，无需对参数进行引号包裹。
  10. 错误修复：
    1. 修复在处理带有关联 WHERE 子句且包含 "HAVING 0" 子句的语法错误的 SELECT 语句时可能导致的空指针解引用。（也在 3.34.1 补丁中修复。）
    2. 修复3.33.0 版本 IN 运算符优化中的 bug，该 bug 可能导致返回错误结果。
    3. 修复当 LIKE 模式以 "%" 结尾且包含 "ESCAPE '_'" 子句时产生的错误结果。

3.35.1 版本前的更改（2021-03-15）：

  1. 修复新 DROP COLUMN 功能在用于有索引且在索引定义中被引用的列时出现的错误。
  2. 改进CLI 中.dump 命令的内置文档。

3.35.2 版本前的更改（2021-03-17）：

  1. 修复在 3.35.0 版本中引入的appendvfs.c 扩展中的问题。
  2. 确保无参数的日期/时间函数（其输出依赖当前时间）被视为非确定性函数。票据2c6c8689fb5f3d2f
  3. 修复sqldiff 工具在处理虚表 定义中出现非常规空白字符时的问题。
  4. 限制第 3.35.0 版本第 8c 项描述的新 UNION ALL 优化，以免尝试生成过多新子查询。详情参见论坛帖子 140a67d3d2。

3.35.3 版本前的更改（2021-03-26）：

  1. 强化字节码引擎的 OP_OpenDup 操作码，使其即使被复制的游标本身来自 OP_OpenDup 也能正常工作。修复票据 bb8a9fd4a9b7fce5。该问题因最近的 MATERIALIZED 提示增强才暴露。
  2. 在物化关联公共表表达式时，为每个使用场景分别物化，以确保正确性。此修复解决了 MATERIALIZED 提示增强引入的问题。
  3. 修复 unixVFS 的文件名归一化器中的问题。
  4. 修复CLI 中的"box" 输出模式，使其能与返回零列（例如PRAGMA incremental_vacuum）或多行的语句一起使用。参见论坛帖子 afbbcb5b72。
  5. 改进由有问题的公共表表达式生成的错误信息。参见论坛帖子 aa5a0431c99e。
  6. 修复部分错误的 assert() 语句。
  7. 修复SELECT 语句语法图，使 FROM 子句语法正确显示。参见论坛帖子 9ed02582fe。
  8. 修复 EBCDIC 字符分类器，使其将换行视为空白字符。参见论坛帖子 58540ce22dcd。
  9. 提升在（不受支持的）wholenumber 虚表 扩展实现中的xBestIndex 方法，使其更有效地说服查询规划器避免物化行数无限的表。参见论坛帖子 b52a020ce4。

本次补丁版本 3.35.4（2021-04-02）的更改：

  1. 修复由上述第 8b 项识别出的查询规划器优化缺陷。票据de7db14784a08053。
  2. 修复新RETURNING 语法中的缺陷。票据132994c8b1063bfb。
  3. 修复新的RETURNING 功能，使其在 RETURNING 子句中的某一项引用未知表时抛出错误，而不是静默忽略。
  4. 修复与聚合函数处理相关的断言，该断言被下推优化错误触发。
