SQLite 3.3.9 版本于 2007-01-04 发布

  1. 修复 pager.c 中的错误，该错误可能在两个进程同时尝试恢复热日志时导致数据库损坏
  2. 添加了 sqlite3_prepare_v2() API。
  3. 命令行外壳中的 ".dump" 命令再次显示索引、触发器和视图。
  4. 更改 table_info pragma，使其在没有默认值时返回 NULL
  5. 支持 win95 文件名中的非 ASCII 字符
  6. 查询优化器增强：
    1. 优化器在使用索引满足按整数主键排序的 ORDER BY 子句方面做得更好
    2. 使用索引满足 WHERE 子句中的 IS NULL 操作符
    3. 修复一个错误，该错误导致优化器错过 OR 优化机会
    4. 即使存在 LEFT 连接，优化器在 FROM 子句中重新排列表的自由度也更大。
  7. 在 WinCE 上添加了扩展加载支持
  8. 允许在表定义的 DEFAULT 子句中指定约束名称
  9. 在命令行外壳中添加了 ".bail" 命令
  10. 使命令行外壳的 CSV（逗号分隔值）输出更符合公认做法
  11. 添加了实验性的 FTS2 模块
  12. 使用 sqlite3_mprintf() 替代 strdup() 以避免 libc 依赖
  13. VACUUM 在官方 TEMP 文件夹中使用临时文件，而不是与原始数据库相同的目录
  14. Windows 上临时文件名的前缀从 "sqlite" 更改为 "etilqs"。

一个 SQLite 发布的完整列表 和一个 时间线 都可以在同一页面上获取。每次提交的详细历史可在 SQLite 版本控制站点 上查看。
