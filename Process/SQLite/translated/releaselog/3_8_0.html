SQLite 3.8.0 发行于 2013-08-26

  1. 添加对部分索引的支持
  2. 切换到下一代查询规划器以获得更快、更优的查询计划。
  3. EXPLAIN QUERY PLAN的输出不再显示联接中每个循环所生成的行数估算。
  4. 添加了FTS4 notindexed 选项，允许在 FTS4 表中使用未建索引的列。
  5. 向sqlite3_stmt_status()添加了SQLITE_STMTSTATUS_VM_STEP选项。
  6. 添加了cache_spill pragma。
  7. 添加了query_only pragma。
  8. 添加了defer_foreign_keys pragma以及sqlite3_db_status(db, SQLITE_DBSTATUS_DEFERRED_FKS,...) C 语言接口。
  9. 在源代码树的 ext/misc 子目录中，作为可加载扩展添加了“percentile()”函数。
  10. 添加了SQLITE_ALLOW_URI_AUTHORITY编译时选项。
  11. 添加sqlite3_cancel_auto_extension(X)接口。
  12. 没有 FROM 子句的运行中的 SELECT 语句（或任何不从数据库文件读取或写入的语句）不会阻止读取事务关闭。
  13. 添加SQLITE_DEFAULT_AUTOMATIC_INDEX编译时选项。将其设置为 0 可默认禁用自动索引。
  14. 当查询规划器使用自动索引时，在SQLITE_CONFIG_LOG上发出SQLITE_WARNING_AUTOINDEX警告。
  15. 添加了SQLITE_FTS3_MAX_EXPR_DEPTH编译时选项。
  16. 为 next_char() 扩展 SQL 函数添加了定义排序序列的可选第五个参数。
  17. 在 WAL 模式下，当读取事务无法升级为写入事务，因为读取是针对较旧快照时，会返回SQLITE_BUSY_SNAPSHOT扩展错误码。
  18. 增强了 sqlite3_analyzer 实用程序，除了整体大小外，还为表的每个索引分别提供大小信息。
  19. 如果应用定义的 SQL 函数是在不访问任何数据库表的 SELECT 语句中调用的，则允许 SQL 语句在该函数实现中自由地打开和关闭读取事务。
  20. 除非使用了 HAVE_POSIX_FALLOCATE 编译时选项，否则在所有（Unix）系统上禁用 posix_fallocate() 的使用。
  21. 更新命令行 shell中的“.import”命令以支持多行字段、正确的 RFC-4180 引号，并在输入文本不严格符合 RFC-4180 时发出警告或错误信息。
  22. 错误修复：在FTS4的unicode61分词器中，将所有私有码点视为标识符符号。
  23. 错误修复：ORDER BY 子句中的裸标识符更紧密地绑定输出列名，但表达式中的标识符更紧密地绑定输入列名。然而，GROUP BY 子句中的标识符始终优先输出列名。
  24. 错误修复：通过迁移到NGQP修复了传统查询优化器中的多个问题。
  25. SQLITE_SOURCE_ID: “2013-08-26 04:50:08 f64cd21e2e23ed7cff48f7dafa5e76adde9321c2”
  26. sqlite3.c 的 SHA1：b7347f4b4c2a840e6ba12040093d606bd16ea21e
