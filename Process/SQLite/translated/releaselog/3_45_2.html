SQLite 3.45.2 版本发布于 2024-03-12

3.45.0 版本（2024-01-15）之前的更改：

  1. 为 SQLITE_RESULT_SUBTYPE 属性添加了对 应用程序定义的 SQL 函数 的支持。 所有调用 sqlite3_result_subtype() 的应用程序定义 SQL 函数必须使用此新属性进行注册。 未能这样做可能导致对 sqlite3_result_subtype() 的调用被视为无操作。 使用 -DSQLITE_STRICT_SUBTYPE=1 编译可以在非 SQLITE_RESULT_SUBTYPE 的函数尝试调用 sqlite3_result_subtype() 时引发 SQL 错误。 对于使用子类型的每个应用程序，推荐在编译时启用 -DSQLITE_STRICT_SUBTYPE=1。
  2. 对 JSON SQL 函数 的增强：
    1. 所有 JSON 函数都重写为使用一种名为 JSONB 的新内部解析树格式。 该新解析树格式可序列化，因此可以存储在数据库中，以避免每次使用 JSON 值时重复解析。
    2. 新版本的生成 JSON 的函数返回二进制 JSONB，而非 JSON 文本。
    3. json_valid() 函数增加了一个可选的第二个参数， 指定第一个参数“格式良好”的含义。
  3. 为 FTS5 虚拟表添加了 FTS5 tokendata 选项。
  4. 默认启用 SQLITE_DIRECT_OVERFLOW_READ 优化。 可使用 -DSQLITE_DIRECT_OVERFLOW_READ=0 在编译时禁用。
  5. 查询优化器改进：
    1. 不再允许传递约束优化将查询优化器误导为使用范围约束，而实际上可以使用更优的等值约束。 （论坛帖子 2568d1f6e6。）
    2. 查询优化器现在更好地忽略 ANALYZE 识别为低质量的索引。 （论坛帖子 6f0958b03b。）
  6. 将 SQLITE_MAX_PAGE_COUNT 的默认值从 1073741824 提高到 4294967294。
  7. 对 命令行界面（CLI） 的增强：
    1. 改善在 Windows 上显示 UTF-8 内容的表现
    2. 自动检测“.dump”脚本的回放，并相应调整设置，例如“.dbconfig defensive off”和 “.dbconfig dqs_dll on”。

3.45.1 版本（2024-01-30）之前的更改：

  1. 恢复了 JSON BLOB 输入错误，并承诺在随后的版本中继续支持该异常，以保持向后兼容。
  2. 修复了 PRAGMA integrity_check 命令，使其可以在包含 FTS3 和 FTS5 表的只读数据库上运行。 该问题由 3.44.0 版本 引入，但直到 3.45.0 发布后才被发现。
  3. 修复与处理损坏的 JSONB 输入有关的问题：
    1. 防止在将损坏的 JSONB 转换为文本时出现指数时间运行。
    2. 修复在转换损坏的 JSONB 为文本时可能越界读取 JSONB 二进制数据末尾一个字节的情况。
    3. 使用 jfuzz 加强测试，以防止未来出现类似 JSONB 问题。
  4. 修复一个长期存在的错误，在访问经过巧妙破坏的数据库并使用内存映射数据库时， 可能会在内存映射段末尾读取多字节。
  5. 修复一个长期存在的错误，在某类特意设计以压迫查询优化器但其实无意义的 SQL 语句中， 字节码生成错误可能导致 字节码引擎 的 NULL 指针解引用。

此补丁版本 3.45.2（2024-03-12）的更改：

  1. 修复了 UPSERT 中的一个错误，该错误由 3.35.0 版本（2021-03-12）中的 3a 项增强引入， 可能导致索引与其表不同步。 论坛帖 919c6579c8。
  2. 缩小了在 3.35.0 版本（2021-03-12）中的第 8e 项 添加的 NOT NULL 强度降低优化的应用范围。 该优化在某些上下文中尝试执行时无效，导致查询结果错误。 论坛帖 440f2a2f17。
  3. 自上一个补丁版本以来出现的其他细微修正和编译器警告修复。 详情请参阅 时间线。
