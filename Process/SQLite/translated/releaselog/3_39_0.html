SQLite 发布 3.39.0 于 2022-06-25

  1. 添加（早该添加的）对 RIGHT 和 FULL OUTER JOIN 的支持。
  2. 添加新的二元比较运算符 IS NOT DISTINCT FROM 与 IS DISTINCT FROM， 在兼容 PostgreSQL 与 SQL 标准的同时，分别等价于 IS 与 IS NOT。
  3. 为 sqlite3_vtab_distinct() 接口添加新的返回码（值 “3”）， 用以表示同时包含 DISTINCT 与 ORDER BY 子句的查询。
  4. 添加了 sqlite3_db_name() 接口。
  5. unix 操作系统接口在打开数据库之前，会解析数据库文件名中的所有符号链接，创建标准化的数据库名。 如果在 sqlite3_open_v2() 或类似函数中使用了 SQLITE_OPEN_NOFOLLOW 标志， 那么只要路径中任意一个元素是符号链接，打开操作就会失败。
  6. 延迟视图的物化，直到真正需要物化为止，从而避免在最终未使用物化结果时进行不必要的工作。
  7. 现在在任何聚合查询上都允许使用 HAVING 子句，即便这些查询没有 GROUP BY 子句。
  8. 许多 微优化 共同减少了约 2.3% 的 CPU 周期。
