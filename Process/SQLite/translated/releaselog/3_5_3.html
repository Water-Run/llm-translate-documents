2007年11月27日发布的 SQLite 3.5.3

  1. 将网站和文档文件从源代码树中移出，放入独立的 CM 系统。
  2. 修复 INSERT INTO ... SELECT ... 语句中，当 SELECT 是复合语句时的老问题。
  3. 修复 BEFORE 触发器中使用 RAISE(IGNORE) 时的长期存在的错误。
  4. 修复 ~ 运算符的运算符优先级。
  5. 在 Win32 上，尝试删除不存在的文件时不再返回错误。
  6. 允许对排序序列名称使用引号。
  7. 修改 TCL 接口以使用sqlite3_prepare_v2()。
  8. 修复 malloc() 失败后可能出现的多个错误。
  9. sqlite3_step() 在参数为 NULL 时返回SQLITE_MISUSE，而不是崩溃。
  10. FTS3 现在完全使用 SQLite 的内存分配器。FTS3 的汇编代码现在可以附加到 SQLite 的汇编代码中，生成同时包含两者的超级汇编。
  11. 当有合适的索引且优化器认为使用索引可能更有利时，DISTINCT 关键字现在有时会使用 INDEX。

SQLite 各版本完整列表（在单页展示）和时间顺序记录亦可查阅。每次提交的详细历史可在 SQLite 版本控制网站 查看。
