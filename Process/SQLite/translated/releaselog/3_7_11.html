SQLite Release 3.7.11 On 2012-03-20

  1. 增强 INSERT 语法，允许通过 VALUES 子句插入多行数据。
  2. 增强 CREATE VIRTUAL TABLE 命令以支持 IF NOT EXISTS 子句。
  3. 添加 sqlite3_stricmp() 接口，作为 sqlite3_strnicmp() 的对应功能。
  4. 添加 sqlite3_db_readonly() 接口。
  5. 添加 SQLITE_FCNTL_PRAGMA 文件控制，使 VFS 实现 能够添加新的 PRAGMA 语句或覆盖内置 PRAGMA。
  6. 形式为：“SELECT max(x), y FROM table”的查询将返回包含最大 x 值的行上的 y 值。
  7. 添加对 FTS4 languageid 选项 的支持。
  8. 记录了对 FTS4 content 选项 的支持。该功能实际上自 3.7.9 版本 以来就已存在于代码中，但现在才被认为是正式支持的。
  9. 未完成的语句不再阻塞 ROLLBACK。相反，该语句在 ROLLBACK 之后的下一次访问时将返回 SQLITE_ABORT。
  10. 改进 命令行 shell 对 CSV 输入的处理。
  11. 修复在 3.7.10 版本 中引入的一个 bug，该 bug 可能导致在 WHERE 子句中由 OR 连接的可索引项出现时，LEFT JOIN 被错误地转换为 INNER JOIN。
  12. SQLITE_SOURCE_ID： "2012-03-20 11:35:50 00bb9c9ce4f465e6ac321ced2a9d0062dc364669"
  13. sqlite3.c 的 SHA1：d460d7eda3a9dccd291aed2a9fda868b9b120a10

在一个页面中还提供了 SQLite 发行版本的完整列表 以及一份 年代记。 每次提交的详细历史可在 SQLite 版本控制站点 上查看。
