SQLite 发布 3.4.0 于 2007-06-18

  1. 修复一个漏洞：当显式事务中途发生 SQLITE_BUSY 错误且该事务后来被提交时，可能导致数据库损坏。问题编号 #2409。
  2. 修复一个漏洞：当自动清理模式开启且在事务内部因缓存溢出后执行 CREATE TABLE 或 CREATE INDEX 语句紧接 malloc() 失败时，可能导致数据库损坏。详情请参见编号 #2418。
  3. 为 SQLite 可处理的对象大小和数量添加了明确的 上限。这一改动可能会给极限使用 SQLite 的应用带来兼容性问题，这也是当前版本为 3.4.0 而非 3.3.18 的原因。
  4. 添加了对 增量 BLOB I/O 的支持。
  5. 添加了 sqlite3_bind_zeroblob() API 与 zeroblob() SQL 函数。
  6. 添加了对 增量 Vacuum 的支持。
  7. 添加了 SQLITE_MIXED_ENDIAN_64BIT_FLOAT 编译期选项，以支持具有怪异字节序的 ARM7 处理器。
  8. 移除了核心库中所有 sprintf() 与 strcpy() 的使用。
  9. 为全文搜索扩展添加了对 国际化组件 Unicode (ICU) 的支持。
  10. 在 Windows 操作系统驱动中，如果尝试获取排他锁失败，则重新获取共享锁。问题编号 #2354。
  11. 修复 REPLACE() 函数：当第二个参数为空字符串时应返回 NULL。问题编号 #2324。
  12. 记录 sqlite3_column_blob() 及相关 API 中类型转换的风险；修复不必要的类型转换。问题编号 #2321。
  13. TRIM() 函数国际化。问题编号 #2323。
  14. 在可能重叠的内存区域之间移动数据时改用 memmove() 替代 memcpy()。问题编号 #2334。
  15. 修复优化器中的一个 bug：复合 SELECT 中包含 ORDER BY 与 LIMIT 时，子查询处理出错。问题编号 #2339。
  16. 确保 sqlite3_snprintf() 接口在缓冲区大小小于 1 时不会进行终止零填充。问题编号 #2341。
  17. 修复内置 printf 逻辑，使其在浮点 NaN 时打印 “NaN” 而非 “Inf”。问题编号 #2345。
  18. 将 BLOB 转为 TEXT 时，使用主数据库的文本编码。问题编号 #2349。
  19. 将整数转换为 NUMERIC 时尽量保留全部精度（如果可能）。问题编号 #2364。
  20. 修复对 UTF16 码点 0xE000 的处理缺陷。
  21. 在查询优化器中匹配 WHERE 约束到索引时，考虑显式的 collate 子句。问题编号 #2391。
  22. 修复查询优化器，使其能正确处理 LEFT JOIN 的 ON 子句中的常量表达式。问题编号 #2403。
  23. 修复查询优化器，使其能正确处理与 NULL 的 rowid 比较。问题编号 #2404。
  24. 修复可能因恶意 SQL 语句导致的多种潜在段错误。

也提供了一个 SQLite 发行版本完整列表 页面，以及一个 时间轴。 每次提交的详细历史记录可在 SQLite 版本控制站点 查阅。
