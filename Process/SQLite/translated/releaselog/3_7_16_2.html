SQLite 3.7.16.2 版本发布于 2013-04-12

  1. 新增 PRAGMA foreign_key_check 命令。
  2. 为所有 SQLITE_CONSTRAINT 错误新增扩展错误码。
  3. 新增 SQLITE_READONLY_ROLLBACK 扩展错误码，用于在数据库因为需要回滚恢复但为只读时无法打开的情况。
  4. 新增 SQL 函数 unicode(A) 和 char(X1,...,XN)。
  5. 对 PRAGMA incremental_vacuum 进行了性能提升，特别是在空闲页数量超过 freelist 单个主干页容纳量的情况下。
  6. 改进包含聚合 min() 或 max() 的查询的优化。
  7. 增强虚拟表，使其在 WHERE 子句包含 IN 运算符时也有可能使用索引。
  8. 即使索引的前置项在 WHERE 子句中被 IN 运算符约束，也允许使用索引进行排序。
  9. 增强 PRAGMA table_info 命令，使 “pk” 列为递增整数，以显示主键列的顺序。
  10. 增强查询优化器以利用可传递的连接约束。
  11. 查询优化器的性能提升。
  12. 允许 PRAGMA integrity_check 的错误消息长度超过 20000 字节。
  13. 改进深层嵌套查询的名称解析。
  14. 新增 test_regexp.c 模块，演示如何实现 REGEXP 运算符。
  15. 改进 RTREE 扩展的错误消息。
  16. 增强 命令行 shell，使 .exit 命令的非零参数可立即退出而不清理关闭数据库连接。
  17. 为命令行 shell 中点命令的非法布尔参数提供更清晰的错误消息。
  18. 改进 “外键不匹配” 错误消息，显示两个相关表的名称。
  19. 移除 unix VFS 中所有对 umask() 的调用。
  20. 新增 PRAGMA vdbe_addoptrace 和 PRAGMA vdbe_debug 命令。
  21. 在比较非零终止字符串时改用 strncmp() 或等效函数而不是 memcmp()。
  22. 更新 cygwin 接口以移除已弃用的 API 调用。
  23. 增强 spellfix1 扩展，使其可通过在 “command” 字段插入像 'edit_cost_table=TABLE' 的字符串来在运行时修改编辑距离费用表。
  24. Bug 修复：解决一个长期存在的问题，该问题可能导致在三表或更多表的连接中，当在两个或多个位置将 INTEGER 字段与 TEXT 字段比较时产生不正确的查询结果。 工单 fc7bd6358f
  25. Bug 修复：若视图的 16 位引用计数因查询过于复杂而溢出则发出错误消息。
  26. Bug 修复：避免在深度嵌套的 UNION ALL 查询中的 LIMIT 和 OFFSET 子句泄漏内存。
  27. Bug 修复：在运行 PRAGMA table_info、index_list、index_info 和 foreign_key_list 之前确保架构是最新的。

  1. 修复在 版本 3.7.15 中引入的 ORDER BY 优化器错误，该错误有时会在排序步骤实际上需要时错误地将其优化掉。 工单 a179fe7465
  2. 修复 CAST 表达式 中一个长期存在的 Bug，该 Bug 会将最高有效字节非零的 UTF16 字符识别为数字。 工单 689137afb6da41。
  3. 修复 FTS3 的 NEAR 运算符在应用于子字段时的 Bug。 工单 38b1ae018f。
  4. 修复存储引擎中的一个长期 Bug，该 Bug 极少情况下会错误地报告 SQLITE_CORRUPT，但其他时候无害。 工单 6bfb98dfc0c。
  5. 已移除 SQLITE_OMIT_MERGE_SORT 选项。合并排序器现为 SQLite 的必需组件。
  6. 修复源码注释中的大量拼写错误。

  1. 修复自版本 3.7.13 起存在的一个 Bug：当两个或更多进程尝试同时访问同一数据库文件且随后第三个进程在提交阶段崩溃时，Windows 上可能导致数据库损坏。 详见工单 7ff3120e4f。
