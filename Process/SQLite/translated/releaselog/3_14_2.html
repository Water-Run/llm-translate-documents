SQLite 3.14.2 发布于 2016-09-12

来自 3.14 版（2016-08-08）的先前更改：


    庆祝 SQLite "π 版本"，附上一块自制派。
  1. 增加了对 WITHOUT ROWID 虚拟表 的支持。
  2. 改进查询优化器，使 OR 优化 即使在一个或多个析取项使用 LIKE、GLOB、REGEXP、MATCH 操作符的情况下，也可以在 虚拟表 上使用。
  3. 新增 CSV 虚拟表，用于读取 RFC 4180 格式的逗号分隔值文件。
  4. 新增 carray() 表值函数 扩展。
  5. 通过扩展入口点使用新的 SQLITE_OK_LOAD_PERMANENTLY 返回码启用了 持久可加载扩展。
  6. 在 SQLITE_DBSTATUS_CACHE_USED_SHARED 选项中新增 sqlite3_db_status()。
  7. 增加 vfsstat.c 可加载扩展 —— 一个测量 I/O 的 VFS 适配层，以及一个提供这些测量数据访问的 同名虚拟表。
  8. 改进了在同时具有 ORDER BY 和 LIMIT 且仅最内层循环自然按正确顺序生成行的查询中的执行算法。
  9. 增强 Lemon 语法分析器生成器，使其生成更高效的解析器。
  10. PRAGMA compile_options 命令现在尝试显示生成库的编译器版本号。
  11. 增强 PRAGMA table_info，使其提供关于 同名虚拟表 的信息。
  12. 新增 “win32-none” VFS，与 “unix-none” VFS 类似，其行为与默认 “win32” VFS 一致，只是忽略所有文件锁。
  13. 在合适的情况下，查询优化器会对 部分索引 执行全表扫描，而不是对主表执行全表扫描。
  14. 允许 表值函数 出现在 IN 操作符 的右侧。
  15. 新增 dbhash.exe 命令行工具。
  16. 新增两个 C 接口：sqlite3_expanded_sql() 和 sqlite3_trace_v2()。这些新接口取代了现已弃用的 sqlite3_trace() 和 sqlite3_profile() 的功能。
  17. 在 json1 扩展 中新增 json_quote() SQL 函数。
  18. 在重新解析架构时禁用 authorizer 回调。
  19. 新增 SQLITE_ENABLE_UNKNOWN_SQL_FUNCTION 编译时选项，并在构建 命令行 shell 时默认启用该选项。

    错误修复：

  20. 修复 ALTER TABLE 命令，使其在向 传统文件格式 数据库添加列时不会损坏 降序索引。问题单 f68bf68513a1c15f
  21. 修复一个当传递 WHERE 子句引用不存在的排序序列时可能发生的空指针解引用/崩溃。问题单 e8d439c77685eca6。
  22. 改进了包含 WHERE 子句的索引扫描的成本估算，该 WHERE 子句可以利用索引列部分或全部评估且无需执行表查找。这修复了在 3.12.0 版 引入 ORDER BY LIMIT 优化后某些不常见查询出现的性能回退。

来自 3.14.1 版（2016-08-11）的先前更改：

  1. 对页缓存“截断”操作的性能增强，在具有较大页缓存的系统上将 COMMIT 时间减少了数十毫秒。
  2. 修复 sqldiff 的 --rbu 选项。

3.14.2 补丁版本（2016-09-12）中的更改：

  1. 改进在 winsqlite3.dll 中使用 STDCALL 调用约定的支持。
  2. 修复 sqlite3_trace_v2() 接口，使其在回调或掩码参数为零时按文档规定禁用。
  3. 修复注释错误，并在使用 -DSQLITE_ENABLE_EXPLAIN_COMMENTS 编译时选项时改进 EXPLAIN 列表生成的注释。
  4. 修复 命令行 shell 中的 ".read" 命令，使其能够识别输入不是交互式的。
  5. 修正在 IN 操作符 右侧的 SELECT 的亲和性计算。问题单 199df4168c。
  6. 除非查询计划确实使用最内层的 IN 操作符循环，否则 ORDER BY LIMIT 优化无效。问题单 0c4df46116e90f92。
  7. 修复一个内部代码生成器问题，该问题导致某些 DELETE 操作无效。问题单 ef360601
