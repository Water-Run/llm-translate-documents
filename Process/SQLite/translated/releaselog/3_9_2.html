SQLite 3.9.2 版本发布（2015-11-02）

在 3.9.0 版本（2015-10-14）之后的更改：

    政策变更：

  1. SQLite 的 版本编号惯例已改为采用 新兴的 语义化版本控制 标准。

    新特性与增强：

  2. 在源码树和 合并文件中增加了 json1 扩展 模块。 使用 SQLITE_ENABLE_JSON1 编译时选项打开支持。
  3. 在 合并文件中加入了 全文搜索版本 5（FTS5）， 通过 SQLITE_ENABLE_FTS5 启用。FTS5 将在至少一个版本周期内被视为“实验性”（可能出现不兼容的更改）。
  4. CREATE VIEW 语句现在在视图名称之后接受可选的列名列表。
  5. 增加了对 表达式索引 的支持。
  6. 允许在 SELECT 语句的 FROM 子句中使用 表值函数。
  7. 增加了对 命名虚拟表 的支持。
  8. 创建 VIEW 时现在可以引用未定义的表和函数。 当该 VIEW 在查询中使用时，才报告缺失的表和函数。
  9. 增加了用于 json1 扩展 的 sqlite3_value_subtype() 和 sqlite3_result_subtype() 接口。
  10. 查询规划器现在可以使用在 WHERE 子句中包含 AND 连接项的 部分索引。
  11. sqlite3_analyzer.exe 工具更新为报告每个 btree 的深度，并显示索引与 WITHOUT ROWID 表的平均度数。
  12. 增强了 dbstat 虚拟表，使其可以作为表值函数，参数为要分析的模式。

    其他更改：

  13. 已将已废弃且 8 年未记录的 sqlite3_memory_alarm() 接口更改为无操作。

    重要修复：

  14. 修复了 SQLite 加密扩展 中的一个关键错误，该错误在 执行 VACUUM 命令改变加密随机数长度时可能导致数据库无法读取且无法恢复。
  15. 在 sqlite3_initialize() 实现中添加了内存屏障，以帮助确保线程安全。
  16. 修正 OR 优化，以始终忽略未使用索引的子计划。
  17. 不再对来源于 LEFT JOIN 的 ON 或 USING 子句项应用 WHERE 子句下推优化。 修复工单 c2a19d81652f40568c。

在 3.9.1 版本（2015-10-16）之后的更改：

  1. 修复 json1 扩展，使其不会将 ASCII 换页符识别为空白字符，以符合 RFC-7159。 修复工单 57eec374ae1d0a1d
  2. 增加了一些 #ifdef 和构建脚本更改，以解决 3.9.0 发布后出现的编译问题。

该补丁版本 3.9.2（2015-11-02）的更改：

  1. 修复了模式解析器，使其能够将某些（晦涩且格式不佳的）CREATE TABLE 语句 与旧版本具有相同的解释方式。修复工单 ac661962a2aeab3c331
  2. 修复了查询规划器问题，该问题在相关标量子查询的 FROM 子句中使用 自动索引 时可能导致错误结果。修复工单 8a2adec1。
  3. SQLITE_SOURCE_ID：“2015-11-02 18:31:45 bda77dda9697c463c3d0704014d51627fceee328”
  4. sqlite3.c 的 SHA1：1c4013876f50bbaa3e6f0f98e0147c76287684c1
