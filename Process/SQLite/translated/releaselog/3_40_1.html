SQLite 3.40.1 版本发布于 2022-12-28

自 3.40.0 版本 (2022-11-16) 以来的更改：

  1. 添加对将 SQLite 编译为 WASM 并在网络浏览器中运行的支持。注意：WASM 构建及其接口被视为“测试版”，如有必要可能会做出小幅更改。我们预计将在下一次发布中最终确定接口。
  2. 添加可能能够从损坏数据库文件中恢复部分内容的 恢复扩展。
  3. 查询规划器增强：
    1. 在列数超过 63 的表上识别 覆盖索引，当查询中使用或索引引用超出第 63 列的列时也会生效。
    2. 在可行的情况下提取 表达式索引 中表达式所包含的值，而不是重新计算表达式。
    3. NOT NULL 与 IS NULL 操作符（及其等价形式）在判断大型字符串和 BLOB 值时避免从磁盘加载其内容。
    4. 避免对执行全表扫描的视图进行一次性实体化。在视图行被计算出来时立即使用并丢弃。
    5. 允许将作为聚合查询中 LEFT JOIN 右操作数的子查询进行扁平化。
  4. 新增一个名为 sqlite3_filename 的 typedef，用于表示数据库文件名。多个接口已修改为使用新 typedef 替代 “char*”。该接口更改应与旧版本完全兼容，尽管在重建某些遗留应用时可能会引发（无害的）编译器警告。
  5. 添加 sqlite3_value_encoding() 接口。
  6. 安全性增强：SQLITE_DBCONFIG_DEFENSIVE 增强为禁止更改 schema_version。在防御模式下 schema_version 变为只读。
  7. 对 PRAGMA integrity_check 语句的增强：
    1. 非 STRICT 表中具有 TEXT 亲和力的列不应包含数值。
    2. 非 STRICT 表中具有 NUMERIC 亲和力的列不应包含可转换为数字的 TEXT 值。
    3. 验证 WITHOUT ROWID 表的行是否按正确顺序排列。
  8. 增强 VACUUM INTO 语句，使其遵循 PRAGMA synchronous 设置。
  9. 增强 sqlite3_strglob() 与 sqlite3_strlike() API，使它们能够接受 NULL 指针作为字符串参数并仍产生合理结果。
  10. 提供新的编译时选项 SQLITE_MAX_ALLOCATION_SIZE 以限制内存分配大小。
  11. 将 SQLite 内置伪随机数生成器（PRNG）的算法从 RC4 改为 Chacha20。
  12. 允许两个或多个索引使用相同名称，只要它们位于不同的模式中。
  13. 各种性能优化使典型工作负载下的 CPU 周期减少约 1%。

此补丁版本 3.40.1 (2022-12-28) 的更改：

  1. 修复 --safe 命令行选项 以使 CLI 正确禁止使用如 writefile() 之类可能导致有害副作用的 SQL 函数。
  2. 修复 memsys5 替代内存分配器中的潜在无限循环。该漏洞由 3.39.0 版本中的性能优化引入。
  3. 各种其他不明显的修复。
