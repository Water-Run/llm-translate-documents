SQLite 3.39.4 版发布于 2022-09-29

来自版本 3.39.0（2022-06-25）的先前更改：

  1. 添加（早该实现的）对 RIGHT 以及 FULL OUTER JOIN 的支持。
  2. 新增二元比较运算符 IS NOT DISTINCT FROM 与 IS DISTINCT FROM， 分别等价于 IS 与 IS NOT，以兼容 PostgreSQL 及 SQL 标准。
  3. sqlite3_vtab_distinct() 接口新增返回码（值为“3”）， 用以表示查询同时包含 DISTINCT 和 ORDER BY 子句。
  4. 添加了 sqlite3_db_name() 接口。
  5. Unix 操作系统接口在打开数据库之前，会将数据库文件名中的所有符号链接解析为规范名称。 若在 sqlite3_open_v2() 或类似接口中使用 SQLITE_OPEN_NOFOLLOW 标志，则只要路径中的任意组成部分是符号链接，打开操作就会失败。
  6. 将视图的物化延迟到真正需要时，若物化最终未被使用，可避免不必要的工作。
  7. 当前的 SELECT 语句 中的 HAVING 子句 现在可适用于所有聚合查询， 即使查询没有 GROUP BY 子句。
  8. 多项 微观优化 累计减少约 2.3% 的 CPU 周期消耗。

来自版本 3.39.1（2022-07-13）的先前更改：

  1. 修复在使用包含复合 SELECT 的视图时，如果其中只有一个分支包含 RIGHT JOIN 且该视图并非查询中 FROM 子句的首项时造成的错误结果。 详见 论坛帖子 174afeae5734d42d。
  2. 修复若干无害的编译器警告。
  3. 修复 ALTER TABLE RENAME 的一个长期问题，该问题仅在将 sqlite3_limit(SQLITE_LIMIT_SQL_LENGTH) 设置得非常小时才会出现。
  4. 修复发表于 FTS3 中一个长期存在的问题，该问题仅在启用了 SQLITE_ENABLE_FTS3_PARENTHESIS 编译时选项时才会出现。
  5. 修复构建过程，使其在同时开启 SQLITE_DEBUG 与 SQLITE_OMIT_WINDOWFUNC 这两个编译时选项时仍然可用。
  6. 修复 REGEXP 扩展的前缀优化，使其即便前缀包含需使用 3 字节 UTF8 编码的字符也能正确运行。
  7. 增强 sqlite_stmt 虚拟表，使其缓冲所有输出。

来自版本 3.39.2（2022-07-21）的先前更改：

  1. 修复查询规划器在存在 LEFT JOIN 时重新排列 FROM 子句项顺序所引入的性能回归。
  2. 应用针对 CVE-2022-35737、Chromium 漏洞 1343348 和 1345947（详见 论坛帖子 3607259d3c）以及内部测试中发现的其他若干小问题的修复。

来自版本 3.39.3（2022-09-05）的先前更改：

  1. 针对影响两行或以上数据库行且使用可能中止的 SQL 函数的 DML 语句，使用语句日志。详见 论坛帖 9b9e4716c0d7bbd1。
  2. 使用互斥锁来保护 PRAGMA temp_store_directory 与 PRAGMA data_store_directory 语句，即便它们已被弃用并标注为非线程安全。详见 论坛帖 719a11e1314d1c70。
  3. 其他错误与警告修复。详情请参见 时间线。

此补丁版本 3.39.4（2022-09-29）的更改：

  1. 修复 Windows 平台的构建，使其在启用 -DSQLITE_OMIT_AUTOINIT 时仍能正常运行。
  2. 修复 btree 平衡器中一个长期存在的问题，在极少数情况下如果应用使用 应用自定义页面缓存，可能导致数据库损坏。
  3. 增强 SQLITE_DBCONFIG_DEFENSIVE， 使其在触发器主体中的某条或多条语句写入 影子表 时拒绝 CREATE TRIGGER 语句。
  4. 修复 FTS3 中某次内存分配的大小计算可能出现的整数溢出。
  5. 修复 sqlite3_set_auxdata() 接口在 ICU 扩展 中的误用。
