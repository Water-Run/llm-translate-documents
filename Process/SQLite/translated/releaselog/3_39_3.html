SQLite 3.39.3 版本发布于 2022-09-05

自版本 3.39.0（2022-06-25）以来的先前更改：

  1. 为 RIGHT 和 FULL OUTER JOIN 添加（久违的）支持。
  2. 新增二元比较运算符 IS NOT DISTINCT FROM 和 IS DISTINCT FROM，它们分别等价于 IS 和 IS NOT，以兼容 PostgreSQL 与 SQL 标准。
  3. 从 sqlite3_vtab_distinct() 接口新增一个返回码（值为“3”），用于表示同时具有 DISTINCT 和 ORDER BY 子句的查询。
  4. 新增 sqlite3_db_name() 接口。
  5. Unix 操作系统接口在打开数据库文件之前会解析所有符号链接，以创建数据库的规范名。如果在 sqlite3_open_v2() 或类似接口中使用 SQLITE_OPEN_NOFOLLOW 标志，则如果路径中的任何元素是符号链接，打开操作将失败。
  6. 推迟视图的物化，直到真正需要物化为止，从而避免如果最终不使用而进行无谓工作。
  7. SELECT 语句 的 HAVING 子句现在可以用于任何聚合查询，即便该查询没有 GROUP BY 子句。
  8. 多项 微观优化 共计减少约 2.3% 的 CPU 周期。

自版本 3.39.1（2022-07-13）以来的先前更改：

  1. 修复在包含复合 SELECT 的视图中仅一个分支包含 RIGHT JOIN 且该视图不是包含它的查询的第一个 FROM 子句项时的错误结果。论坛帖子 174afeae5734d42d。
  2. 修复一些无害的编译器警告。
  3. 修复 ALTER TABLE RENAME 的长期问题，该问题仅在 sqlite3_limit(SQLITE_LIMIT_SQL_LENGTH) 设置为非常小的值时才会出现。
  4. 修复 FTS3 中的长期问题，该问题仅在启用了 SQLITE_ENABLE_FTS3_PARENTHESIS 编译时选项时才会出现。
  5. 修复当同时提供 SQLITE_DEBUG 和 SQLITE_OMIT_WINDOWFUNC 编译时选项时的构建问题。
  6. 修复 REGEXP 扩展的前缀优化，使其即使前缀包含需要 3 字节 UTF8 编码的字符也能正确工作。
  7. 增强 sqlite_stmt 虚拟表，使其缓存所有输出。

自版本 3.39.2（2022-07-21）以来的先前更改：

  1. 修复在存在 LEFT JOIN 时重新排列 FROM 子句项顺序所导致的查询规划器性能回退。
  2. 应用 CVE-2022-35737、Chromium 漏洞 1343348 和 1345947、论坛帖子 3607259d3c 以及内部测试中发现的其他次要问题的修复。

该补丁版本 3.39.3（2022-09-05）中的更改：

  1. 如果 DML 语句影响两个或更多数据库行，并且该语句使用可能导致中止的 SQL 函数，则对该语句使用语句日志。详见 论坛主题 9b9e4716c0d7bbd1。
  2. 使用互斥量来保护 PRAGMA temp_store_directory 和 PRAGMA data_store_directory 语句，即使它们已弃用且文档中指出不保证线程安全。详见 论坛帖子 719a11e1314d1c70。
  3. 其他缺陷和警告修复。详见时间线。
