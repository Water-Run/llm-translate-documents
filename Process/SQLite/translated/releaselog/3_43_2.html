SQLite Release 3.43.2 On 2023-10-10

来自版本 3.43.0（2023-08-24）的先前更改：

  1. 增加对 无内容删除的 FTS5 索引 的支持。这是 FTS5 全文搜索索引的一种变体，它省略了被索引内容的存储，同时仍允许删除记录。
  2. 对 日期与时间函数 的增强：
    1. 添加了新的 时间偏移修饰符，形式为 ±YYYY-MM-DD HH:MM:SS.SSS。
    2. 添加了 timediff() SQL 函数。
  3. 添加了 octet_length(X) SQL 函数。
  4. 添加了 sqlite3_stmt_explain() API。
  5. 查询规划器的增强：
    1. 将 LEFT JOIN 强度削减优化泛化，使其也适用于 RIGHT 和 FULL JOIN，并将其重新命名为 OUTER JOIN 强度削减。
    2. 增强 OUTER JOIN 强度削减 优化中的定理证明器，使其返回更少的误报。
  6. 对 十进制扩展 的增强：
    1. 新函数 decimal_pow2(N) 返回 N-th 次幂的 2，其中整数 N 在 -20000 到 +20000 之间。
    2. 新函数 decimal_exp(X) 的行为类似 decimal(X)，但结果以指数表示法返回，并在末尾带有 "e+NN"。
    3. 如果 X 是浮点值，则 decimal(X) 函数现在会将该值完全展开为其精确的十进制等价物。
  7. JSON 处理 的性能增强，在某些针对大型 JSON 字符串的处理上提升了 2 倍性能。
  8. 新的 makefile 目标 “verify-source” 检查源树中是否存在非预期的更改。（仅适用于 规范源代码 —— 不适用于 预编译的 amalgamation 压缩包。）
  9. 添加了 SQLITE_USE_SEH 编译时选项，当在 Windows 上使用 Makefile.msc 构建且处理 WAL 模式 中的内存映射 shm 文件 时启用结构化异常处理。该选项在 Windows 上默认启用。
  10. Unix 的 VFS 现在假设 nanosleep() 系统调用是可用的，除非使用 -DHAVE_NANOSLEEP=0 编译。

来自版本 3.43.1（2023-09-11）的先前更改：

  1. 修复 sum()、avg() 和 total() 聚合函数处理无穷大的回归。
  2. 修复了在参数直接来自 json_remove() 时发生的 json_array_length() 函数错误。
  3. 修复了在子查询为复合查询且其中一个分支为 DISTINCT 而另一个不是时的省略未使用子查询列优化（该优化在版本 3.42.0 中引入）。
  4. 其他小修复。

本次补丁版本 3.43.2（2023-10-10）中的更改：

  1. 修复了几个鲜为人知的 UAF 错误以及一个鲜为人知的内存泄漏。
  2. 在 CLI 中省略使用标准库中的 sprintf() 函数，因为该函数在某些平台上会产生警告。
  3. 避免将 double 转换为 unsigned long long 整数，因为某些平台不能正确执行此类转换。
