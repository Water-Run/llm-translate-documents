SQLite 3.8.0.1 发布于 2013-08-29

  1. 添加对部分索引的支持
  2. 切换到下一代查询规划器以获得更快更优的查询计划。
  3. EXPLAIN QUERY PLAN的输出不再显示连接中每个循环生成的行数估计值。
  4. 新增FTS4 notindexed 选项，允许 FTS4 表中的非索引列。
  5. 新增SQLITE_STMTSTATUS_VM_STEP选项到sqlite3_stmt_status()。
  6. 新增cache_spill pragma。
  7. 新增query_only pragma。
  8. 新增defer_foreign_keys pragma以及sqlite3_db_status(db, SQLITE_DBSTATUS_DEFERRED_FKS,...) 的 C 语言接口。
  9. 在源码树的 ext/misc 子目录中新增以可加载扩展形式提供的 "percentile()" 函数。
  10. 新增SQLITE_ALLOW_URI_AUTHORITY编译时选项。
  11. 新增sqlite3_cancel_auto_extension(X)接口。
  12. 运行中的 SELECT 语句如果缺少 FROM 子句（或任何永远不从数据库文件读取或写入的语句）将不会阻止读取事务关闭。
  13. 新增SQLITE_DEFAULT_AUTOMATIC_INDEX编译时选项。将此选项设置为 0 默认禁用自动索引。
  14. 每当查询规划器使用自动索引时，在SQLITE_CONFIG_LOG上发出SQLITE_WARNING_AUTOINDEX警告。
  15. 新增SQLITE_FTS3_MAX_EXPR_DEPTH编译时选项。
  16. 为 next_char() 扩展 SQL 函数新增可选的第五个参数，以定义排序序列。
  17. 在 WAL 模式下，当读取事务无法升级为写入事务因为读取的是旧快照时，会返回SQLITE_BUSY_SNAPSHOT扩展错误代码。
  18. 增强 sqlite3_analyzer 实用程序，除了聚合大小，还单独提供表中每个索引的大小信息。
  19. 允许从应用自定义 SQL 函数的实现中运行的 SQL 语句自由地打开和关闭读取事务，只要该函数由不访问任何数据库表的 SELECT 语句调用。
  20. 除非使用 HAVE_POSIX_FALLOCATE 编译时选项，否则在所有（Unix）系统上禁用 posix_fallocate() 的使用。
  21. 更新命令行 shell中的 ".import" 命令以支持多行字段并正确处理 RFC-4180 引用，如果输入文本不严格符合 RFC-4180 标准则发出警告或错误消息。
  22. 错误修复：在FTS4的unicode61分词器中，将所有私有代码点视为标识符符号。
  23. 错误修复：ORDER BY 子句中的裸标识符更紧密绑定输出列名，但表达式中的标识符更紧密绑定输入列名。GROUP BY 子句中的标识符始终优先输出列名。
  24. 错误修复：通过迁移到NGQP，修复了旧查询优化器中的多个问题。

  1. 修复一个 off-by-one 错误，该错误导致命令行 shell 在 CRLF 结尾的 CSV 输入行末尾错误地读取带引号的空字符串。
  2. 修复一个查询规划器错误，该错误涉及带 BETWEEN 或 LIKE/GLOB 约束的 LEFT JOIN，然后右侧进行另一个带 OR 约束的 INNER JOIN。
  3. 修复一个查询规划器错误，该错误可能在查询具有超过四列的 UNIQUE 或 PRIMARY KEY 约束的表时导致段错误。

  5. sqlite3.c 的 SHA1：99906bf63e6cef63d6f3d7f8526ac4a70e76559e
