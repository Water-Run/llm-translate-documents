SQLite 3.38.3 版本发布于 2022-04-27

3.38.0 版本（2022-02-22）之前的变更：

  1. 新增 -> 与 ->> 运算符，以便更方便地处理 JSON。 新运算符与 MySQL 和 PostgreSQL 兼容。
  2. JSON 函数现在为内置函数。 无需再使用 -DSQLITE_ENABLE_JSON1 编译时选项即可启用 JSON 支持。 JSON 默认启用。 可使用新的 -DSQLITE_OMIT_JSON 编译时选项来禁用 JSON 接口。
  3. 对 日期与时间函数 的增强：
    1. 新增 unixepoch() 函数。
    2. 新增 auto 修饰符 与 julianday 修饰符。
  4. 将 printf() SQL 函数 重命名为更兼容的 format()。 为保持向后兼容，原始 printf() 名称仍保留为别名。
  5. 添加 sqlite3_error_offset() 接口，有助于将 SQL 错误定位到输入 SQL 文本的具体字符位置， 以便应用程序提供更友好的错误提示。
  6. 对 虚拟表 的接口增强如下：
    1. 添加 sqlite3_vtab_distinct() 接口。
    2. 添加 sqlite3_vtab_rhs_value() 接口。
    3. 添加新的操作符类型 SQLITE_INDEX_CONSTRAINT_LIMIT 与 SQLITE_INDEX_CONSTRAINT_OFFSET。
    4. 添加 sqlite3_vtab_in() 接口（及相关），使虚拟表 能一次性处理 IN 操作符 约束， 而非针对 IN 操作符右侧的每个值分别处理。
  7. 命令行界面 的增强：
    1. 列模式输出 增强后能正确处理文本中的制表符与换行符。
    2. 为 列模式输出 添加了 "--wrap N"、"--wordwrap on" 与 "--quote" 等选项。
    3. 添加了 .mode qbox 别名。
    4. .import 命令 自动消歧列名。
    5. 使用新的 sqlite3_error_offset() 接口以提供更准确的错误提示。
  8. 查询优化器增强：
    1. 使用布隆过滤器加速大型分析查询。
    2. 对带有 ORDER BY 子句的 UNION 或 UNION ALL 复合 SELECT 语句， 使用平衡归并树进行评估。
  9. ALTER TABLE 语句在 PRAGMA writable_schema=ON 时， 对无法解析的 sqlite_schema 表 条目改为静默忽略。

3.38.1 版本（2022-03-12）之前的变更：

  1. 修复新的布隆过滤器优化在一些冷门查询中可能导致错误结果的问题。
  2. 修复 localtime 修饰符， 使得 日期与时间函数 能保留小数秒部分。
  3. 修复 sqlite_offset SQL 函数， 即使参数为虚拟列或视图的列，也能在边缘情况下正确工作。
  4. 修复 行值 IN 操作符 在 虚拟表 上的约束， 即使虚拟表实现依赖字节码来过滤不满足约束的行，也能正常工作。
  5. 对 assert() 语句、测试用例及文档进行其它小修复。 详见 源码时间线。

3.38.2 版本（2022-03-26）之前的变更：

  1. 修复用户发现的布隆过滤器优化问题，该问题在执行带 WHERE 子句约束的 LEFT JOIN 时可能导致错误结果， 该约束指示 LEFT JOIN 右表的某个列为 NULL。详见 讨论串 031e262a89b6a9d2。
  2. 其他小补丁。详见 时间线。

该次补丁发布（3.38.3 版本，2022-04-27）的变更：

  1. 修复查询优化器在优化自动索引与布隆过滤器构造时过于激进的问题， 使用不恰当的 ON 子句项来限制自动索引或布隆过滤器的大小， 导致输出结果缺失行。 讨论串 0d3200f4f3bcd3a3。
  2. 其他小补丁。详见 时间线。
