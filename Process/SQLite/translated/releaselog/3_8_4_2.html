SQLite 3.8.4.2 版本于 2014-03-26 发布

在版本 3.8.4（2014-03-10）之前的更改：

  1. 对代码进行优化和重构以提升性能。
  2. 向命令行 shell 添加 ".clone" 与 ".save" 命令。
  3. 更新命令行 shell 的横幅，以便在使用临时内存数据库时提醒新手用户。
  4. 修复命令行 shell 中 editline 的支持。
  5. 通过 SQLITE_TESTCTRL_VDBE_COVERAGE 语义的 sqlite3_test_control() 为 VDBE 程序增加覆盖率测试支持。
  6. 更新 _FILE_OFFSET_BITS 宏以便在 QNX 上再次成功构建。
  7. 将 SrcList.nSrc 的数据类型从 u8 改为 int，以规避 AIX 上 C 编译器的问题。
  8. 使 Cygwin 上的扩展加载机制正常工作。
  9. 错误修复：修复 char() SQL 函数，使其在无参数时返回空字符串，而不是“内存不足”错误。
  10. 错误修复：DISTINCT 现在识别 zeroblob 与全部为 0x00 字节的 blob 相同。 Ticket [fccbde530a]
  11. 错误修复：对于 WHERE 子句中包含 IS NOT NULL 术语且同时包含 OR 术语、并在编译时启用了 SQLITE_ENABLE_STAT4 的查询，确保计算出正确答案。 Ticket [4c86b126f2]
  12. 错误修复：确保在普通表与 WITHOUT ROWID 表之间的连接中能正确解析 "rowid" 列。 Ticket [c34d0557f7]
  13. 错误修复：确保在实现包含 ORDER BY 子句的复合 SELECT 语句的并发协程中不会重复使用相同的临时寄存器，因为那样会导致错误的结果。 Ticket [8c63ff0eca]
  14. 错误修复：确保不会优化掉 "ORDER BY random()" 子句。 Ticket [65bdeb9739]
  15. 错误修复：解决触发器内部子查询中可能发生的名称解析错误。 Ticket [4ef7e3cfca]
  16. 错误修复：修复形如 "DEFAULT(-(-9223372036854775808))" 的列默认值表达式，使其正确工作，将该列初始化为约等于 +9223372036854775808.0 的浮点值。

在版本 3.8.4.1（2014-03-11）之前的更改：

  1. 规避 C 预处理器宏冲突，该冲突会导致某些使用 Microsoft Visual Studio 的配置构建失败。
  2. 在计算 跳跃扫描优化 的成本时，考虑到需要多次查找。

此补丁发布（版本 3.8.4.2，2014-03-26）中的更改：

  1. 修复在尝试搜索损坏的数据库文件时可能导致的缓冲区溢出读取。
  2. SQLITE_SOURCE_ID：“2014-03-26 18:51:19 02ea166372bdb2ef9d8dfbb05e78a97609673a8e”
  3. sqlite3.c 的 SHA1：4685ca86c2ea0649ed9f59a500013e90b3fe6d03
