SQLite Release 3.3.3 On 2006-01-31

  1. 在 CREATE INDEX 上移除对 ON CONFLICT 子句的支持——它从 未能正确工作，因此这不应引发任何向后兼容性问题。
  2. 授权回调现在会收到 ALTER TABLE ADD COLUMN 命令的通知
  3. 对 TEMP 数据库模式进行任何更改后，所有准备语句 均失效，必须使用新的 sqlite3_prepare() 调用重新创建
  4. 其他一些小型错误修复，以便为 3.3 版本的首个稳定发行做好准备

在单页中也同时提供了SQLite 发布完整列表和时间线。 有关每次检入的详细历史可在 SQLite 版本控制站点获取。
