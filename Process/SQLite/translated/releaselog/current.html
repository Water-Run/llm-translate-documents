2026-01-09 发布的 SQLite 3.51.2

来自版本 3.51.0（2025-11-04）的先前更改：

  1. sqlite3.h 中的新宏：
    1. SQLITE_SCM_BRANCH → 源代码所在分支的名称。
    2. SQLITE_SCM_TAGS → 源代码提交中标签的空格分隔列表。
    3. SQLITE_SCM_DATETIME → 源代码提交的 ISO-8601 日期和时间。
  2. 两个新的 JSON 函数，jsonb_each() 和 jsonb_tree() 的工作方式与现有的 json_each() 和 json_tree() 相同，只是在 “type” 为 ‘array’ 或 ‘object’ 时其 “value” 列返回 JSONB。
  3. carray 和 percentile 扩展现已内置于 amalgamation 中， 但默认情况下被禁用，需要在编译时分别通过 -DSQLITE_ENABLE_CARRAY 与/或 -DSQLITE_ENABLE_PERCENTILE 选项启用。
  4. 对TCL 接口的改进：
    1. 为 eval 命令添加 -asdict 标志， 使其将行数据设置为字典而非数组。
    2. 现在用户定义函数可以通过 break 返回一个 SQL NULL。
  5. CLI 改进：
    1. 将 “.timer” 的精度提高到微秒级。
    2. 增强 “box” 与 “column” 格式模式以处理双宽字符。
    3. “.imposter” 命令提供只读 imposter 表， 与 VACUUM 配合使用且无需 --unsafe-testing 选项。
    4. 为 CLI 命令行选项及 .open 命令 添加 --ifexists 选项。
    5. 将 “.width” 命令设置的列宽限制为 30000 以内， 因为没有充分理由支持更宽的列，但支持更宽的列会给恶意行为者可乘之机。
  6. 性能增强：
    1. 提交读取事务时使用更少的 CPU 周期。
    2. 提前检测由于某个表为空而导致的无行连接。
    3. 避免评估标量子查询，若该子查询结果不会改变整体表达式结果。
    4. 当使用带大 :y 的 “BETWEEN :x FOLLOWING AND :y FOLLOWING” 时窗口函数查询更快。
  7. 添加 PRAGMA wal_checkpoint=NOOP; 命令及 SQLITE_CHECKPOINT_NOOP 参数供 sqlite3_wal_checkpoint_v2() 使用。
  8. 添加用于扩展的 sqlite3_set_errmsg() API。
  9. 添加 sqlite3_db_status64() API，其工作方式与现有的 sqlite3_db_status() 相同，但返回 64 位结果。
  10. 向 sqlite3_db_status() 和 sqlite3_db_status64() 接口添加 SQLITE_DBSTATUS_TEMPBUF_SPILL 选项。
  11. 在session 扩展中添加 sqlite3changeset_apply_v3() 接口。
  12. 对于内置 printf() 与 format() SQL 函数， 若省略了 '+' 标志且存在 "#" 标志并且所有显示数字均为 '0'，则省略负浮点数的前导 ‘-’。 使用 '%#f' 等避免输出像 '-0.00'，而是直接显示 '0.00'。
  13. FTS5 生成的错误消息更清晰。
  14. 在计算列上强制实行 STRICT 类型。
  15. 改善对 VxWorks 的支持。
  16. JavaScript/WASM 现支持 64 位 WASM。官方构建仍为 32 位，但现在通过运行 “make” 也能轻松生成自己的 64 位构建。
  17. 增强防止因应用程序 使用 close() 破坏 Posix 建议锁 导致的数据库损坏。

来自版本 3.51.1（2025-11-28）的先前更改：

  1. 修复由于 3.51.0 发布中第 6b 项的优化导致的嵌套 EXISTS 查询结果错误。
  2. 修复 fts5vocab 虚拟表的潜在错误，该错误由 3.51.0 发布中的新优化暴露。

此补丁版本 3.51.2（2026-01-09）的更改：

  1. 修复上文第 17 项中新破坏 Posix 锁检测逻辑中的一个难以察觉的死锁。
  2. 修复在上述第 6b 优化中新增的 EXISTS 到 JOIN 优化中的多个问题。
  3. 其他小型错误修复。
