SQLite 发布 3.23.1 于 2018-04-10

从版本 3.23.0（2018-04-02）起的更改：

  1. 在使用 SQLITE_ENABLE_DESERIALIZE 编译时选项的情况下，添加 sqlite3_serialize() 和 sqlite3_deserialize() 接口。
  2. 将 TRUE 和 FALSE 识别为常量。（为兼容性考虑，如果存在名为“true”或“false”的列，则这些标识符仍然指向列，而非布尔常量。）
  3. 支持运算符 IS TRUE、IS FALSE、IS NOT TRUE 和 IS NOT FALSE。
  4. 在 sqlite3_db_status() 中添加 SQLITE_DBSTATUS_CACHE_SPILL 选项，用于报告缓存溢出次数。
  5. 内置 printf 实现中的“alternate-form-2”（“!”）标志现在使字符串替换以字符而非字节来测量宽度和精度。
  6. 如果 xColumn 方法在 虚拟表 实现中使用 sqlite3_result_error() 返回错误信息，则优先展示该错误，而非内部生成的消息。
  7. 为 CLI 添加 -A 命令行选项，以便更轻松地管理 SQLite Archive 文件。
  8. 在 Zipfile 虚拟表 中添加对 INSERT OR REPLACE、INSERT OR IGNORE 和 UPDATE OR REPLACE 的支持。
  9. 加强 sqlite3changeset_apply() 接口，使其能够抵御经过故意破坏的 changeset 对象攻击。
  10. 添加了 sqlite3_normalize() 扩展函数。
  11. 查询优化器改进：
    1. 改进 omit-left-join 优化，使其在右表具有 UNIQUE 但不一定为 NOT NULL 的情况下也能正常工作。
    2. 改进 push-down 优化，使其适用于更多 LEFT JOIN。
    3. 添加 LEFT JOIN 力度削弱优化，当 WHERE 子句中存在会阻止 LEFT JOIN 额外全 NULL 行出现的条件时，将 LEFT JOIN 转换为普通 JOIN。
    4. 当 AUTOINCREMENT 表使用小于当前最大值的 rowid 更新时，避免对 sqlite_sequence 表进行不必要的写入。
  12. Bug 修复：
    1. 修复解析器以接受有效的 row value 语法。 Ticket 7310e2fb3d046a5
    2. 修复查询规划器，使其在 WHERE 子句的子表达式中考虑表值函数参数之间的依赖关系。 Ticket 80177f0c226ff54
    3. 修复在复杂 OR 连接的 WHERE 与 STAT4 组合中产生的不正确结果。 Ticket ec32177c99ccac2
    4. 修复由于自动数据类型转换导致的 表达式索引 潜在损坏。 Ticket 343634942dd54ab
    5. FTS4 中的断言错误。 Ticket d6ec09eccf68cfc
    6. row value 中小于运算符返回不正确结果。 Ticket f484b65f3d62305
    7. 即使浮点值的整数部分为零，也始终将非零浮点值解释为 TRUE。 Ticket 36fae083b450e3a
    8. 修复 fsdir(PATH) 表值函数（fileio.c 扩展）的问题，当 fsdir() 表作为连接的内部表时会导致段错误。问题在邮件列表中报告，并由 check-in 7ce4e71c1b7251be 修复。
    9. 当 sqlite_master 表损坏以致 sqlite_sequence 根页实际上是 btree 索引页时，改为报错，而非断言失败或空指针引用。Check-in 525deb7a67fbd647
    10. 修复 ANALYZE 命令，使其能对名称以“sqlite”开头的表计算统计信息。Check-in 0249d9aecf69948d
  13. 由 OSSFuzz 发现问题的额外修复：
    1. 修复在损坏的数据库文件上执行 VACUUM 时可能出现的无限循环。 Check-in 27754b74ddf64
    2. 禁止在触发器和视图的 WITH 子句中使用 参数。 Check-in b918d4b4e546d
    3. 修复 row value 处理中的潜在内存泄漏。 Check-in 2df6bbf1b8ca8
    4. 提升 replace() SQL 函数在对兆级字符串执行大量替换时的性能，以避免 OSSFuzz 测试超时。 Check-in fab2c2b07b5d3
    5. 当 sqlite_master 表包含 CREATE TABLE AS 语句时提供适当错误信息；此前会导致断言失败或空指针引用。该问题由 OSSFuzz 在 GDAL 项目中发现。Check-in d75e67654aa96
    6. 移除不正确的 assert() 语句。Check-in 823779d31eb09cda。
    7. 修复在 INTEGER PRIMARY KEY 上使用 LIKE 优化时的问题。Check-in b850dd159918af56

此补丁发布（版本 3.23.1，2018-04-10）中的更改：

  1. 修复 LEFT JOIN 力度削弱优化 中的两个问题。 Tickets 1e39b966ae9ee739 与 fac496b61722daf2。
  2. 修复 FTS5 xBestIndex 方法的异常行为。Ticket 2b8aed9f7c9e61e8。
  3. 修复对未初始化虚拟机寄存器的无害引用。Ticket 093420fc0eb7cba7。
  4. 修复 CLI，使其可在 -DSQLITE_UNTESTABLE 下构建。
  5. 修复 eval.c 扩展，使其可与 PRAGMA empty_result_callbacks=ON 一起使用。
  6. 修复 generate_series 虚拟表，在任一约束为 NULL 时正确返回零行。
  7. 解析器性能增强。
