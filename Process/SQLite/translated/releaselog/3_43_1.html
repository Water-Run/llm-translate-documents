SQLite 3.43.1 版本发布 于 2023-09-11

版本 3.43.0（2023-08-24）之前的更改：

  1. 添加对无内容删除 FTS5 索引的支持。这是一种FTS5全文搜索索引，它省略了正在建立索引的内容的存储，同时仍允许删除记录。
  2. 对日期和时间函数的增强：
    1. 添加了格式为 ±YYYY-MM-DD HH:MM:SS.SSS 的新时间偏移修饰符。
    2. 添加了timediff() SQL 函数。
  3. 添加了octet_length(X) SQL 函数。
  4. 添加了sqlite3_stmt_explain() API。
  5. 查询规划器的增强：
    1. 将 LEFT JOIN 强度降低优化泛化，使其也适用于 RIGHT 和 FULL JOIN，并重命名为OUTER JOIN 强度降低。
    2. 增强了OUTER JOIN 强度降低优化中的定理证明，从而减少其返回的假阴性。
  6. 十进制扩展的增强：
    1. 新函数 decimal_pow2(N) 返回 N 在 -20000 到 +20000 之间的 2 的 N 次幂。
    2. 新函数 decimal_exp(X) 的行为类似于 decimal(X)，但它以指数表示法（末尾带有 "e+NN"）返回结果。
    3. 如果 X 是浮点值，则 decimal(X) 函数现在会将该值完全展开为其精确的十进制等价形式。
  7. 对JSON 处理的性能增强，使得在大型 JSON 字符串上的某些处理速度提升 2 倍。
  8. 新的 makefile 目标 "verify-source" 用于检查源码树中是否存在非预期的更改。（仅适用于规范源代码——不适用于预编译 amalgamation 压缩包。）
  9. 添加了SQLITE_USE_SEH编译时选项，在使用包含于WAL 模式处理一部分的内存映射shm 文件时启用 Windows 上的结构化异常处理。使用 Makefile.msc 在 Windows 上构建时默认启用该选项。
  10. unix 平台的VFS现在假定 nanosleep() 系统调用可用，除非在编译时使用 -DHAVE_NANOSLEEP=0。

本补丁版本 3.43.1（2023-09-11）的更改：

  1. 修复了sum()、avg()和total()聚合函数处理无穷大的回归。
  2. 修复了json_array_length()函数在参数直接来源于json_remove()时出现的错误。
  3. 修复了 omit-unused-subquery-columns 优化（在 3.42.0 版本中引入），使其在子查询为复合查询且其中一个分支为 DISTINCT 而另一个不是时也能正常工作。
  4. 其它小修复。
