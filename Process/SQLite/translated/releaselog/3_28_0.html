SQLite 3.28.0 发布于 2019-04-16

  1. 增强了窗口函数：
    1. 添加对EXCLUDE 子句的支持。
    2. 添加对窗口链式调用的支持。
    3. 添加对GROUPS 窗口帧的支持。
    4. 在 RANGE 窗口帧中添加对“<expr> PRECEDING”和“<expr> FOLLOWING”边界的支持。
  2. 新增sqlite3_stmt_isexplain(S)接口，用于判断预处理语句是否为EXPLAIN。
  3. 增强了VACUUM INTO，使其可用于只读数据库。
  4. 新的查询优化：
    1. 在存在 ESCAPE 关键字且PRAGMA case_sensitive_like开启时，启用LIKE 优化。
    2. 在由部分索引驱动的查询中，避免对部分索引 WHERE 子句中命名的约束进行不必要的测试，因为我们知道该约束始终为真。
  5. TCL 接口的增强：
    1. 在function 方法中新增 -returntype 选项。
    2. 新增bind_fallback 方法。
  6. CLI 的增强：
    1. 添加对绑定参数和.parameter 命令的支持。
    2. 修复readfile()函数，使其在读取空文件时返回空 BLOB，而不是抛出内存不足错误。
    3. 修复writefile()函数，使其在为新文件路径创建目录时应用 umask 权限，而不是与文件相同的权限。
    4. 更改.archive 命令中的--update 选项，使其跳过已在归档中且未更改的文件。新增 --insert 选项，其行为类似于旧版 --update。
  7. 新增可以创建、应用和解析Fossil DVCS 文件增量格式的fossildelta.c扩展，该格式由RBU 扩展使用。
  8. 为sqlite3_db_config()接口新增SQLITE_DBCONFIG_WRITABLE_SCHEMA动词，其功能与PRAGMA writable_schema相同且无需使用 SQL 解析器。
  9. 新增sqlite3_value_frombind() API，用于判断 SQL 函数的参数是否来自绑定参数。
  10. 对fts3_tokenizer()的安全性与兼容性增强：
    1. 除非通过sqlite3_db_config(SQLITE_DBCONFIG_ENABLE_FTS3_TOKENIZER)设置启用旧版应用定义的 FTS3 分词器接口，或除非fts3_tokenizer()的第一个参数是绑定参数，否则该函数始终返回 NULL。
    2. 如果第二个参数是绑定参数，则即使未设置sqlite3_db_config(SQLITE_DBCONFIG_ENABLE_FTS3_TOKENIZER)，fts3_tokenizer()的两个参数版本仍可接受指向分词器方法对象的指针。
  11. 增强了对损坏数据库文件的健壮性。
  12. 各种性能增强。
  13. 建立了官方 SQLite 源代码树的 Git 镜像。SQLite 的权威源码使用Fossil DVCS在https://sqlite.org/src维护。Git 镜像可在https://github.com/sqlite/sqlite查看。
