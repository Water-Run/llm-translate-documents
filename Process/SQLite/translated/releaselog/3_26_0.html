SQLite 3.26.0 版本发布于 2018-12-01

  1. 优化：当对具有表达式索引的表执行UPDATE操作时，如果表达式索引不引用所更新表的任何列，则不会更新该表达式索引。
  2. 允许虚拟表实现中的xBestIndex()方法返回SQLITE_CONSTRAINT，以表明建议的查询计划不可用，不应再继续考虑。
  3. 添加了SQLITE_DBCONFIG_DEFENSIVE选项，该选项禁止使用普通 SQL 创建损坏的数据库文件。
  4. 在启用SQLITE_DBCONFIG_DEFENSIVE选项时，新增对只读shadow 表的支持。
  5. 添加了PRAGMA legacy_alter_table命令，启用后可使ALTER TABLE命令在兼容性方面表现得像旧版本（3.25.0 之前）的 SQLite。
  6. 添加了 PRAGMA table_xinfo，其功能与PRAGMA table_info相同，但还显示虚拟表中的隐藏列。
  7. 新增了explain 虚拟表，作为运行时可加载扩展。
  8. 向查询优化器添加了限制计数器，以防止某些病态 SQL 输入导致过度的sqlite3_prepare()时间。
  9. 添加了在启用 SQLITE_ENABLE_NORMALIZE 编译时支持的sqlite3_normalized_sql()接口。
  10. 增强触发器，使其可以使用定义在触发器所在模式之外的模式中的表值函数。
  11. 对命令行工具的增强：
    1. 改进了 “.help” 命令。
    2. 如果存在 SQLITE_HISTORY 环境变量，则其指定命令行编辑历史文件的名称。
    3. 与打开新数据库关联的 --deserialize 选项会将数据库文件读入内存，并通过sqlite3_deserialize() API 访问，从而简化了在不修改磁盘文件的情况下对数据库进行测试的过程。
  12. 对geopoly扩展的增强：
    1. 始终使用二进制格式存储多边形，速度更快且占用空间更少。
    2. 添加了geopoly_regular()函数。
    3. 添加了geopoly_ccw()函数。
  13. 对session扩展的增强：
    1. 添加了SQLITE_CHANGESETAPPLY_INVERT标志。
    2. 添加了sqlite3changeset_start_v2()接口及SQLITE_CHANGESETSTART_INVERT标志。
    3. 添加了changesetfuzz.c测试用例生成工具。
