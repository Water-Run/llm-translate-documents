SQLite Release 3.49.2 On 2025-05-07

来自版本 3.49.0（2025-02-06）的先前更改：

  1. 对查询规划器的增强：
    1. 改进查询时索引优化，使其可用于WITHOUT ROWID表。
    2. 为大型星型查询连接生成更优查询计划。此修复了解决在SQLite论坛上报告的三个不同性能回归问题。
    3. 当两个或多个查询估计成本相同时，优先采用每行字节数较少的查询。
  2. 增强iif() SQL 函数，使其可以接受任意数量（大于等于二个）的参数。
  3. 增强会话扩展，使其可在使用生成列的数据库上工作。
  4. 省略未正确实现且从未正常工作的编译时选项 SQLITE_USE_STDIO_FOR_CONSOLE，改为添加 SQLITE_USE_W32_FOR_CONSOLE_IO 编译时选项。该选项仅适用于如CLI的命令行工具，不影响 SQLite 内核。它使控制台输入输出使用 Win32 API 而非 stdio，仅影响 Windows 构建。
  5. 为sqlite3_db_config()新增三个选项，默认为“开启”：
    1. SQLITE_DBCONFIG_ENABLE_ATTACH_CREATE
    2. SQLITE_DBCONFIG_ENABLE_ATTACH_WRITE
    3. SQLITE_DBCONFIG_ENABLE_COMMENTS
  6. 将Autotools替换为Autosetup，用于预编译汇编代码包中的配置脚本。标准源码的配置脚本在上一版本（3.48.0）中已改为 Autosetup。只有汇编代码包中的主 SQLite 配置脚本发生了更改。汇编包中由TEA子目录使用的（已弃用）配置脚本仍然依赖 Autotools。
  7. 修复 3.48.0 版本中出现的各种小补丁和问题。

来自版本 3.49.1（2025-02-18）的先前更改：

  1. 改进 Makefile 和配置脚本的可移植性。
  2. 修复concat_ws()函数（在3.44.0 版本中引入）中的一个错误，当分隔符字符串大于两兆字节时可能导致内存错误，具体表现为写出已分配空间末尾。
  3. 增强SQLITE_DBCONFIG_LOOKASIDE接口，使其在误用时更具健壮性。

本次补丁版本 3.49.2（2025-05-07）中的更改：

  1. 修复 3.40.0 版本的 NOT NULL 优化中的一个错误（在3.40.0 更改日志的第 3c 项中提及），避免滥用时导致内存错误。
  2. 修复计数视图优化，避免在 DISTINCT 查询中给出错误结果。
  3. 修复一个可能导致错误结果的情况：当某表的 UNIQUE 约束包含 PRIMARY KEY 列并且 IN 操作符使用该 UNIQUE 约束时。
  4. 修复 generate_series() 扩展函数中的一些晦涩问题。
  5. 对 configure/make 进行了渐进式改进。
