SQLite 3.51.2 版本发布（2026-01-09）

来自版本 3.51.0（2025-11-04）的先前更改：

  1. sqlite3.h 中新增宏：
    1. SQLITE_SCM_BRANCH → 源代码所在分支的名称。
    2. SQLITE_SCM_TAGS → 以空格分隔的源代码提交标签列表。
    3. SQLITE_SCM_DATETIME → 源代码提交的 ISO-8601 日期和时间。
  2. 新增两个 JSON 函数，jsonb_each() 与 jsonb_tree()， 它们与现有的 json_each() 和 json_tree() 函数的行为相同，不同之处在于当 “type” 为 “array” 或 “object” 时， “value” 列返回 JSONB。
  3. carray 与 percentile 扩展现已 内嵌入汇编源，但默认状态下被禁用，必须在编译阶段通过 -DSQLITE_ENABLE_CARRAY 和/或 -DSQLITE_ENABLE_PERCENTILE 选项分别启用。
  4. 对 TCL 接口 的增强：
    1. 为 eval 命令增加 -asdict 标志，以便将行数据设置为字典而非数组。
    2. 用户定义函数现在可以通过 break 来返回 SQL NULL。
  5. CLI 的增强：
    1. 将 “.timer” 的精度提高到微秒。
    2. “box” 和 “column” 格式模式增强，以支持双宽字符。
    3. “.imposter” 命令提供只读的仿冒表，它们可以与VACUUM 一起使用，并且无需 --unsafe-testing 选项。
    4. 在命令行选项与 .open 命令 中增加 --ifexists 选项。
    5. 将 “.width” 命令设置的列宽限制为 30,000 或以下，因为没有充分理由允许更宽的列，而支持更宽的列会给不法分子提供可乘之机。
  6. 性能提升：
    1. 在提交只读事务时使用更少的 CPU 周期。
    2. 提前检测由于一个或多个表不含行而导致的返回零行的连接。
    3. 如果标量子查询的结果不会改变整体表达式的结果，则避免对其求值。
    4. 在使用 "BETWEEN :x FOLLOWING AND :y FOLLOWING" 且 :y 很大时，加速窗口函数查询。
  7. 新增 PRAGMA wal_checkpoint=NOOP; 命令与 SQLITE_CHECKPOINT_NOOP 参数供 sqlite3_wal_checkpoint_v2() 使用。
  8. 新增供扩展使用的 sqlite3_set_errmsg() API。
  9. 新增 sqlite3_db_status64() API，其行为与现有的 sqlite3_db_status() API 相同，只是返回 64 位结果。
  10. 在 sqlite3_db_status() 与 sqlite3_db_status64() 接口中增加 SQLITE_DBSTATUS_TEMPBUF_SPILL 选项。
  11. 在会话扩展中新增 sqlite3changeset_apply_v3() 接口。
  12. 对于内置的 printf() 与 SQL 函数 format()，当省略 “+” 标志并且存在 “#” 标志且所有显示的数字均为 “0” 时， 负浮点数的前导 “-” 将被省略。可使用 '%#f' 或类似方式避免产生类似 '-0.00' 的输出，而直接显示 '0.00'。
  13. 改进 FTS5 生成的错误消息。
  14. 对计算列强制实行 STRICT 类型。
  15. 增强对 VxWorks 的支持。
  16. JavaScript/WASM 现已支持 64 位 WASM。官方构建仍为 32 位，但通过运行 “make” 创建自己的 64 位构建变得同样简单。
  17. 增强对因应用程序通过 close() 破坏 Posix 咨询锁 而导致的数据库损坏的抵抗力。

来自版本 3.51.1（2025-11-28）的先前更改：

  1. 修复由 3.51.0 版本中的第 6b 项优化导致的嵌套 EXISTS 查询返回错误结果的问题。
  2. 修复在 3.51.0 版本新增优化暴露的 fts5vocab 虚拟表中的潜在错误。

此补丁版本 3.51.2（2026-01-09）的变更：

  1. 修复第 17 项中新增的破损 Posix 锁检测逻辑中的一个难以察觉的死锁。
  2. 修复在第 6b 项优化中新增的 EXISTS 到 JOIN 优化的多个问题。
  3. 其他一些细微的错误修复。
