SQLite 3.38.4 发布 (2022-05-04)

来自 3.38.0 版本 (2022-02-22) 的早期变更：

  1. 新增 -> 和 ->> 操作符 以便更容易处理 JSON。 新增的操作符与 MySQL 和 PostgreSQL 兼容。
  2. JSON 函数现在是内置的。不再需要使用 -DSQLITE_ENABLE_JSON1 编译时选项来启用 JSON 支持。 JSON 默认启用。可使用新的 -DSQLITE_OMIT_JSON 编译时选项禁用 JSON 接口。
  3. 对 日期和时间函数 的增强：
    1. 新增 unixepoch() 函数。
    2. 新增 auto 修饰符 和 julianday 修饰符。
  4. 为了更好的兼容性，将 printf() SQL 函数 重命名为 format()。 原始的 printf() 名称作为别名保留以保持向后兼容。
  5. 新增 sqlite3_error_offset() 接口， 有助于将 SQL 错误定位到输入 SQL 文本中的特定字符， 以便应用程序能提供更友好的错误信息。
  6. 对 虚拟表 接口的增强如下：
    1. 新增 sqlite3_vtab_distinct() 接口。
    2. 新增 sqlite3_vtab_rhs_value() 接口。
    3. 新增操作符类型 SQLITE_INDEX_CONSTRAINT_LIMIT 和 SQLITE_INDEX_CONSTRAINT_OFFSET。
    4. 新增 sqlite3_vtab_in() 接口（及相关接口）， 使虚拟表能够一次性处理 IN 操作符 约束， 而不是单独处理 IN 操作符右侧的每个值。
  7. 命令行界面 的增强：
    1. 列式输出模式 现在可以正确处理文本中嵌入的制表符和换行符。
    2. 为 列式输出模式 新增 "--wrap N"、"--wordwrap on" 和 "--quote" 等选项。
    3. 新增 .mode qbox 别名。
    4. .import 命令 自动消除列名歧义。
    5. 使用新的 sqlite3_error_offset() 接口来提供更好的错误信息。
  8. 查询规划器的增强：
    1. 使用布隆过滤器来加速大型分析查询。
    2. 使用平衡归并树来评估带有 ORDER BY 子句的 UNION 或 UNION ALL 复合 SELECT 语句。
  9. ALTER TABLE 语句已修改为在 PRAGMA writable_schema=ON 时静默忽略 sqlite_schema 表 中无法解析的条目。

来自 3.38.1 版本 (2022-03-12) 的早期变更：

  1. 修复了新的布隆过滤器优化可能导致某些怪异查询返回错误结果的问题。
  2. 修复 localtime 修饰符 使 日期和时间函数 保留分数秒部分。
  3. 修复 sqlite_offset SQL 函数， 使其在诸如参数为虚拟列或视图列等边缘情况下也能正常工作。
  4. 修复 行值 IN 操作符 对 虚拟表 的约束，使其即便在虚拟表实现依赖字节码过滤不满足约束的行时也能正确工作。
  5. 对 assert() 语句、测试用例和文档的其他小修补。 详情请参阅 源码时间线。

来自 3.38.2 版本 (2022-03-26) 的早期变更：

  1. 修复用户发现的新的布隆过滤器优化问题： 当对右侧表的某列在 LEFT JOIN 中被 WHERE 条件约束为 NULL 时， 可能导致错误结果。详情见 论坛帖子 031e262a89b6a9d2。
  2. 其他小补丁。详情见 时间线。

来自 3.38.3 版本 (2022-04-27) 的早期变更：

  1. 修复查询规划器在构建自动索引和布隆过滤器时过度激进的问题， 错误地使用 ON 子句项来限制自动索引或布隆过滤器的大小， 结果导致输出缺失行。 论坛帖子 0d3200f4f3bcd3a3。
  2. 其他小补丁。详情见 时间线。

本次补丁版本 3.38.4 (2022-05-04) 的变更：

  1. 修复了 3.38.0 版本中新增的布隆过滤器下推优化的字节码问题： 当下推优化遇到 NULL 键时，字节码中的一个错误会导致字节码引擎进入无限循环。 论坛帖子 2482b32700384a0f。
  2. 其他小补丁。详情见 时间线。
