SQLite 版本 3.45.1 于 2024-01-30 发布

来自版本 3.45.0（2024-01-15）的先前更改：

  1. 为 SQLITE_RESULT_SUBTYPE 属性添加对 应用程序定义的 SQL 函数 的支持。所有调用 sqlite3_result_subtype() 的应用程序定义 SQL 函数必须使用这个新属性进行注册。否则，调用 sqlite3_result_subtype() 时可能无效。使用-DSQLITE_STRICT_SUBTYPE=1 编译可以在非 SQLITE_RESULT_SUBTYPE 的函数试图调用 sqlite3_result_subtype() 时引发 SQL 错误。对于使用子类型的每个应用程序，建议在编译时添加-DSQLITE_STRICT_SUBTYPE=1 选项。
  2. 对 JSON SQL 函数 进行增强：
    1. 所有 JSON 函数都已改写为使用一种名为 JSONB 的新内部解析树格式。该解析树格式可序列化，因此可以存储在数据库中，从而避免每次使用 JSON 值时重复解析。
    2. 新版本的 JSON 生成函数会生成二进制 JSONB，而不是 JSON 文本。
    3. json_valid() 函数增加了一个可选的第二个参数，用于指定第一个参数被视为“格式良好”的含义。
  3. 将 FTS5 tokendata 选项 添加到 FTS5 虚拟表。
  4. SQLITE_DIRECT_OVERFLOW_READ 优化现在默认启用。编译时使用 -DSQLITE_DIRECT_OVERFLOW_READ=0 可以禁用它。
  5. 查询规划器改进：
    1. 不要让传递约束优化在有更好的等值约束可用时误导查询规划器去使用范围约束（论坛帖子 2568d1f6e6。）
    2. 查询规划器现在更有效地忽略 ANALYZE 识别为低质量的索引。（论坛帖子 6f0958b03b。）
  6. 将 SQLITE_MAX_PAGE_COUNT 的默认值从 1073741824 提高到 4294967294。
  7. 对 CLI 的增强：
    1. 改进 Windows 上 UTF-8 内容的显示。
    2. 自动检测 ".dump" 脚本的回放，并相应调整 ".dbconfig defensive off" 和 ".dbconfig dqs_dll on" 等设置。

本次补丁发布的更改，版本 3.45.1（2024-01-30）：

  1. 恢复 JSON BLOB 输入错误，并承诺在后续版本中继续支持该异常，以保持向后兼容。
  2. 修复 PRAGMA integrity_check 命令，使其在包含 FTS3 和 FTS5 表的只读数据库中也能正常工作。这个问题在 3.44.0 版本 中引入，但直到 3.45.0 发布后才被发现。
  3. 修复与处理损坏的 JSONB 输入相关的问题：
    1. 在将损坏的 JSONB 转换为文本时防止指数级运行时间。
    2. 修复在将损坏的 JSONB 转换为文本时可能读取 JSONB blob 末尾之后一个字节的问题。
    3. 使用 jfuzz 进行了增强测试，以防止未来出现类似的 JSONB 问题。
  4. 修复一个长期存在的错误，在使用 内存映射数据库 访问经过精心破坏的数据库时，可能会读到内存映射段末尾之后的几个字节。
  5. 修复一个长期存在的错误，由于为一类专门用于考验查询规划器但本质无意义的 SQL 语句生成了错误的字节码，字节码引擎 中可能发生空指针解引用。
