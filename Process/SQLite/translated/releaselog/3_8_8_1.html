SQLite 3.8.8.1 发行于 2015-01-20

来自版本 3.8.8（2015-01-16）的先前更改：

    新功能：

  1. 添加了 PRAGMA data_version 命令，可用于确定数据库文件是否已被其他进程修改。
  2. 向 sqlite3_wal_checkpoint_v2() 接口添加了 SQLITE_CHECKPOINT_TRUNCATE 选项，并对 PRAGMA wal_checkpoint 进行了相应增强。
  3. 添加了 sqlite3_stmt_scanstatus() 接口，仅在编译时启用了 SQLITE_ENABLE_STMT_SCANSTATUS 时可用。
  4. sqlite3_table_column_metadata() 已增强，可在 WITHOUT ROWID 表上正确工作，并在列名称参数为 NULL 时检查表是否存在。该接口现在默认包含在构建中，无需 SQLITE_ENABLE_COLUMN_METADATA 编译时选项。
  5. 添加了 SQLITE_ENABLE_API_ARMOR 编译时选项。
  6. 添加了 SQLITE_REVERSE_UNORDERED_SELECTS 编译时选项。
  7. 添加了 SQLITE_SORTER_PMASZ 编译时选项和 SQLITE_CONFIG_PMASZ 启动时选项。
  8. 为 sqlite3_config() 添加了 SQLITE_CONFIG_PCACHE_HDRSZ 选项，使应用程序更容易确定与 SQLITE_CONFIG_PAGECACHE 一起使用的适当内存量。
  9. VALUES 子句 中的行数不再受 SQLITE_LIMIT_COMPOUND_SELECT 的限制。
  10. 添加了 eval.c 可加载扩展，该扩展实现了 eval() SQL 函数，可递归评估 SQL。

    性能增强：

  11. 减少平衡 b 树时所涉及的 memcpy() 操作数量，整体性能提升约 3.2%。
  12. 改进了 skip-scan 优化 的成本估算。
  13. 自动索引 优化现在可以在适当时生成 部分索引。

    错误修复：

  14. 在电源中断后使用 "PRAGMA journal_mode=TRUNCATE" 时，通过在截断日志文件后立即调用 fsync() 来确保持久性。
  15. 查询规划器现在会识别左连接右表中的任何列均可以为 NULL，即使该列具有 NOT NULL 约束，在这些情况下避免尝试优化去除 NULL 测试。修复工单 6f2222d550f5b0ee7ed。
  16. 即使在使用降序索引实现 DISTINCT 运算符的情况下，也确保 ORDER BY 将行按升序排列。修复工单 c5ea805691bfc4204b1cb9e。
  17. 修复在 共享缓存模式 下运行大量线程时，某些线程打开和关闭连接时可能发生的数据竞争。
  18. 修复 American Fuzzy Lop 发现的隐蔽崩溃错误。工单 a59ae93ee990a55。
  19. 解决 GCC 优化器错误（适用于 MacOS 10.7 上的 gcc 4.2.1），该错误在使用 -O3 编译时导致 R-Tree 扩展 计算出错误结果。

    其他更改：

  20. 禁用对 strchrnul() C 库例程的使用，除非使用 -DHAVE_STRCHRNULL 编译时选项明确启用。
  21. 提高 likelihood()、likely() 和 unlikely() SQL 提示函数的有效性和准确性。

此特定补丁发行版（版本 3.8.8.1（2015-01-20））中的更改：
