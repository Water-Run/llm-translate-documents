SQLite 3.10.0 发行于 2016-01-06

    总体改进：

  1. 为LIKE、GLOB和REGEXP运算符在虚拟表上增加了支持。
  2. 为sqlite3_index_info增加了colUsed 字段，供sqlite3_module.xBestIndex方法使用。
  3. 增强了PRAGMA cache_spill语句，以接受一个 32 位整数参数作为禁止缓存溢出的阈值。
  4. 在 Unix 平台上，如果打开的是数据库文件的符号链接，则对应的日志文件将基于实际文件名，而不是符号链接名。
  5. 为sqldiff添加了“--transaction”选项。
  6. 添加了sqlite3_db_cacheflush()接口。
  7. 添加了sqlite3_strlike()接口。
  8. 使用内存映射 I/O时，将数据库文件以只读方式映射，以防应用程序中溢出的指针或数组意外修改数据库文件。
  9. 添加了实验性的sqlite3_snapshot_get()、sqlite3_snapshot_open()和sqlite3_snapshot_free()接口。它们可能在后续版本中更改或移除。
  10. 增强了'utc' 修改器在日期和时间函数中的表现：如果已知日期/时间已经为 UTC，则该修改器变为无操作。（这不构成兼容性中断，因为此行为长期以来在该情况下都被记录为“未定义”。）
  11. 在json扩展中增加了json_group_array()和json_group_object() SQL 函数。
  12. 添加了SQLITE_LIKE_DOESNT_MATCH_BLOBS编译时选项。
  13. 进行了大量小规模性能优化。

    可移植性增强：

  14. 修复了 HP/UX 上 HP C 编译器优化器的符号扩展错误。(详情)

    命令行 shell 的增强：

  15. 添加了“.changes ON|OFF”和“.vfsinfo”点命令。
  16. 在 Windows 的cmd.exe中运行时，在 MBCS 与 UTF8 之间进行转换。

    Makefile 的增强：

  17. 为各个由 autoconf 生成的 configure 脚本添加了 --enable-editline 和 --enable-static-shell 选项。
  18. 在 makefile 中完全省略了“awk”的使用，以便 MSVC 用户更容易构建。

    重要修复：

  19. 修复了不一致的整数与浮点比较操作：如果在包含大量整数与大小相近浮点值的表列上创建索引，可能导致索引损坏。相关工单 38a97a87a6。
  20. 修复了在格式错误的公用表表达式上查询规划器中可能出现的无限循环。
  21. 在sqldiff工具中修复了各种错误。
