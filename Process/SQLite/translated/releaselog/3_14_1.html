SQLite 3.14.1 发布于 2016-08-11

来自版本 3.14（2016-08-08）前的更改：


    庆祝 SQLite “π 发布”，配以自家烘焙的派。
  1. 新增对 WITHOUT ROWID 虚拟表 的支持。
  2. 改进查询规划器，使得 OR 优化 可以在 虚拟表 上使用，即使一个或多个析取项使用了 LIKE、GLOB、REGEXP、MATCH 运算符。
  3. 新增 CSV 虚拟表，用于读取符合 RFC 4180 格式的逗号分隔值文件。
  4. 新增 carray() 表值函数 扩展。
  5. 利用扩展入口点中的新返回码 SQLITE_OK_LOAD_PERMANENTLY 启用 持久化可加载扩展。
  6. 在 sqlite3_db_status() 中添加了 SQLITE_DBSTATUS_CACHE_USED_SHARED 选项。
  7. 添加了 vfsstat.c 可加载扩展，这是一个 VFS 适配层，用于测量 I/O，并配合提供访问测量结果的 同名虚拟表。
  8. 改进了在同时使用 ORDER BY 和 LIMIT 的查询中的算法，在只有最里层循环天然按正确顺序生成行的情况下也能更好运行。
  9. 增强了 Lemon 解析器生成器，以生成更快速的解析器。
  10. PRAGMA compile_options 命令现在尝试显示生成库的编译器版本号。
  11. 增强 PRAGMA table_info 以提供有关 同名虚拟表 的信息。
  12. 新增与“unix-none” VFS 类似的 “win32-none” VFS，其工作方式与默认的 “win32” VFS 相同，只是忽略所有文件锁。
  13. 在合适的情况下，查询规划器改为完全扫描 部分索引 而不是主表。
  14. 允许 表值函数 出现在 IN 操作符 的右侧。
  15. 创建 dbhash.exe 命令行实用程序。
  16. 新增两个 C 语言接口：sqlite3_expanded_sql() 和 sqlite3_trace_v2()。这些新接口涵盖了现在已弃用的 sqlite3_trace() 和 sqlite3_profile() 的功能。
  17. 在 json1 扩展 中添加了 json_quote() SQL 函数。
  18. 在重新解析架构时禁用 授权回调。
  19. 添加了 SQLITE_ENABLE_UNKNOWN_SQL_FUNCTION 编译时选项，并在构建 命令行 shell 时默认启用该选项。

    错误修复：

  20. 修复 ALTER TABLE 命令，在向 旧版文件格式 数据库添加列时不会破坏 降序索引。问题单 f68bf68513a1c15f
  21. 修复在传递 WHERE 子句引用不存在的排序序列时可能产生的 NULL 指针解引用/崩溃。问题单 e8d439c77685eca6。
  22. 改进了包含 WHERE 子句的索引扫描的成本估算，该 WHERE 子句可以仅使用索引中的列进行部分或完全求值，而无需访问主表。这修复了在引入 3.12.0 版本 的 ORDER BY LIMIT 优化后部分复杂查询出现的性能回退。

本次特定补丁发行版本 3.14.1（2016-08-11）的更改：

  1. 针对页面缓存“截断”操作的性能增强，在拥有大型 页面缓存 的系统上将 COMMIT 时间缩短数十毫秒。
  2. 修复 sqldiff 的 --rbu 选项。
