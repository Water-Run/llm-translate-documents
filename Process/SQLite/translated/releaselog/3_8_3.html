SQLite 3.8.3 版发布于 2014-02-03

  1. 新增对公用表表达式和WITH 子句的支持。
  2. 新增printf() SQL 函数。
  3. 在sqlite3_create_function()及相关接口的第四个参数中，新增可选位SQLITE_DETERMINISTIC，使应用程序可以创建在常量参数时可以从内层循环中抽取出去的新函数。
  4. 新增SQLITE_READONLY_DBMOVED错误代码，在事务开始时返回，以表示底层数据库文件已被重命名或移出 SQLite 之外。
  5. 允许在ATTACH的文件名参数中使用任意表达式，包括函数调用和子查询。
  6. 允许在任何SELECT语句有效的位置使用VALUES 子句。
  7. 当用 N==0 调用sqlite3_randomness(N,P)时重新播种伪随机数生成器。在 unix 上 fork() 后自动重新播种。
  8. 增强spellfix1虚拟表，使其可以通过 rowid 高效搜索。
  9. 性能增强。
  10. 在运行EXPLAIN时，改进 VDBE 字节码显示中的注释。
  11. 在Lemon 解析生成器中新增 "%token_class" 指令，并利用它简化语法。
  12. 更改Lemon的源代码，避免调用 OpenBSD 认为危险的 C 标准库函数（例如 sprintf）。
  13. 错误修复：在命令行 shell的 CSV 导入功能中，当转义双引号出现在 CRLF 行尾时，不再结束字段。
  14. SQLITE_SOURCE_ID： "2014-02-03 13:52:03 e816dd924619db5f766de6df74ea2194f3e3b538"
  15. sqlite3.c 的 SHA1：98a07da78f71b0275e8d9c510486877adc31dbee
