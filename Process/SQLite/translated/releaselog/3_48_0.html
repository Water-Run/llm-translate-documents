SQLite 3.48.0 版本发布于 2025-01-14

  1. 重构用于从标准源构建SQLite的“configure”脚本，以修复错误、提升性能并增强代码的可维护性。
    • 此更改不影响 sqlite3-autoconf-NNNNNNN.tar.gz “集成包”中的“configure”脚本，仅影响标准源。集成包的构建系统保持不变。如果你使用的是集成包，则此更新日志条目与你无关。
    • 这里的关键创新是现在使用Autosetup替代GNU Autoconf。这看起来像是重大变化，但实际上只是实现细节不同。./configure 脚本的编码方式完全不同，但应与以前一样工作。
    • 新的 configure 的一个优势是无需再安装TCL即可构建大多数 SQLite 目标。TCL 仍然是运行测试或构建依赖 TCL 的辅助程序（如sqlite3_analyzer）所必需的，但对大多数常见目标则不是必需。因此，从本版本开始，唯一的构建依赖项就是 C 编译器以及“make”或“nmake”。
  2. 改进了针对覆盖索引的EXPLAIN QUERY PLAN 输出。
  3. 允许iif() SQL 函数的两个参数版本，同时允许if()作为iif()的替代拼写。
  4. 为CLI添加“.dbtotxt”命令。
  5. 为sqlite3_io_methods对象的 xDeviceCharacteristics 方法添加SQLITE_IOCAP_SUBPAGE_READ属性。
  6. 将SQLITE_PREPARE_DONT_LOG选项添加至sqlite3_prepare_v3()，该选项可防止在 SQL 语句格式错误时向错误日志发送警告信息。这使得可以使用 sqlite3_prepare_v3() 来进行 SQL 的测试编译，以验证其有效性而不会污染错误日志。
  7. 将SQLITE_LIMIT_LENGTH允许的最小值从 1 提高到 30。
  8. 添加了SQLITE_FCNTL_NULL_IO文件控制。
  9. 通过insttoken配置选项和fts5_insttoken() SQL 函数，将 FTS5 辅助 API xInstToken() 扩展为可与前缀查询一起使用。
  10. 将SQL 函数的最大参数数从 127 增加到 1000。
  11. 移除了 SQLITE_USER_AUTHENTICATION 的残留痕迹。
  12. 若干模糊的错误修复。
