SQLite 3.3.14 于 2007-04-02 发布

  1. 修复了一个可能导致段错误的 bug（工单 #2273），当 IN 运算符用于双列索引的一项且 IN 运算符的右侧包含 NULL 时会发生。
  2. 新增一个用于确定底层介质扇区大小的操作系统接口方法： sqlite3OsSectorSize().
  3. 针对形式为 INSERT INTO table1 SELECT * FROM table2 的语句采用了更快并且减少碎片的新算法。VACUUM 使用此类语句，因而运行更快且更好地整理碎片。
  4. 通过减少磁盘 I/O 实现性能提升：
    1. 删除行时不读取溢出链的最后一页——只需将该页添加到空闲链表。
    2. 不再将正在删除的页面存储在回滚日志中。
    3. 不读取从空闲链表中提取页面的（无意义的）内容。
    4. 除非其他进程更改底层数据库文件，否则不要刷新页面缓存（从而避免缓存重新填充）。
    5. 在专用访问模式提交事务或提交 TEMP 数据库时，截断回滚日志而不是删除它。
  5. 通过 “PRAGMA locking_mode=EXCLUSIVE” 添加了对专用访问模式的支持
  6. 分页器中的大型缓冲区改为使用堆空间而不是栈空间——这对栈空间有限的嵌入式平台非常有用。
  7. 新增 makefile 目标 “sqlite3.c”，用于构建将核心 SQLite 库 C 代码汇总至单一文件的合并体。
  8. 使库在使用 GCC 选项 “-fstrict-aliasing” 编译时也能正常工作。
  9. 移除了残留的 SQLITE_PROTOCOL 错误。
  10. 改善了测试覆盖率，修复其他一些小错误，堵住内存泄露，并在部分位置对代码进行了重构或提出改进建议，以便更易阅读。

在一个页面上也提供了完整的 SQLite 发布列表和时间顺序记载。每一次提交的详细历史可在 SQLite 版本控制站点查看。
