SQLite 3.35.2 版本发布于 2021-03-17

从版本 3.35.0（2021-03-12）继承的更改：

  1. 添加了 内置 SQL 数学函数()。 （需要 -DSQLITE_ENABLE_MATH_FUNCTIONS 编译期选项。）
  2. 添加了对 ALTER TABLE DROP COLUMN 的支持。
  3. 推广 UPSERT：
    1. 允许多个按顺序求值的 ON CONFLICT 子句，
    2. 最终的 ON CONFLICT 子句可以省略冲突目标， 但仍然可以使用 DO UPDATE。
  4. 为 DELETE、INSERT 和 UPDATE 语句添加了对 RETURNING 子句的支持。
  5. 在对包含非常大的 TEXT 或 BLOB 值的数据库执行 VACUUM 时使用更少的内存。现在不再需要一次性将整个 TEXT 或 BLOB 保持在内存中。
  6. 在指定 公共表表达式 时，为 MATERIALIZED 和 NOT MATERIALIZED 提示添加支持。默认行为原先是 NOT MATERIALIZED，现在改为对重复使用的 CTE 采用 MATERIALIZED。
  7. SQLITE_DBCONFIG_ENABLE_TRIGGER 和 SQLITE_DBCONFIG_ENABLE_VIEW 设置已修改为仅控制主数据库模式或附加数据库模式中的触发器和视图，不再控制 TEMP 模式。TEMP 触发器和视图始终允许。
  8. 查询规划器/优化器改进：
    1. 对 min/max 优化 进行了增强，以便与 IN 操作符和上一版本的 OP_SeekScan 优化更好配合。
    2. 如果可能，并可能提高性能，则在 WHERE 子句中尝试将 EXISTS 操作符视为 IN 操作符进行处理。
    3. 即使父查询是连接，也允许将 UNION ALL 子查询进行 扁平化。
    4. 在 WHERE 子句中的 IS NOT NULL 表达式（即使 STAT4 被禁用）也会在合适时使用索引。
    5. 像 "x IS NULL" 或 "x IS NOT NULL" 这样的表达式，如果 "x" 是具有 "NOT NULL" 约束且未参与外连接的列，可能被转换为简单的 FALSE 或 TRUE。
    6. 如果 UPDATE 语句未修改与外键相关联的任何列，则避免检查外键约束。
    7. 如果 WHERE 术语完全由常量和子查询中所有窗口函数 PARTITION BY 子句中的表达式副本构成，则允许将其 下推 到包含窗口函数的子查询中。
  9. CLI 增强：
    1. 增强 ".stats" 命令以接受新参数 "stmt" 和 "vmstep"，分别仅显示准备语句统计信息和虚拟机步骤计数。
    2. 添加 ".filectrl data_version" 命令。
    3. 增强 ".once" 和 ".output" 命令，当目标参数以 "|" 开头（表示输出重定向到管道）时，该参数无需加引号。
  10. Bug 修复：
    1. 修复处理带有相关 WHERE 子句和 "HAVING 0" 子句的语法错误 SELECT 语句时可能出现的 NULL 指针解引用问题。（在 3.34.1 修补版本中也已修复。）
    2. 修复版本 3.33.0 中的 IN 操作符优化 的一个 bug，该问题可能导致错误结果。
    3. 修复如果模式以 "%" 结尾并带有 "ESCAPE '_'" 子句时 LIKE 操作符 返回错误结果的问题。

从版本 3.35.1（2021-03-15）继承的更改：

  1. 修复 一个 bug，该 bug 出现在新 DROP COLUMN 功能应用于有索引且在索引定义中加引号的列时。
  2. 改进 CLI 中 .dump 命令的内置文档。

此次特定补丁版本 3.35.2（2021-03-17）中的更改：

  1. 修复在版本 3.35.0 中引入的 appendvfs.c 扩展中的一个问题。
  2. 确保不带参数的日期/时间函数（其响应取决于当前时间）被视为 非确定性函数。问题追踪 2c6c8689fb5f3d2f
  3. 修复 sqldiff 实用程序程序在处理 虚拟表 定义中的特殊空白字符时出现的问题。
  4. 限制 3.35.0 发布中第 8c 条描述的新 UNION ALL 优化，避免创建过多新子查询。 详情请参见 论坛帖子 140a67d3d2。
