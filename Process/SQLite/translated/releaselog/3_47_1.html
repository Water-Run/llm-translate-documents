SQLite 3.47.1 版本发布于 2024-11-25

版本 3.47.0 (2024-10-21) 的先前更改：

  1. 允许在 RAISE 函数 的第二个参数中使用任意表达式。
  2. 如果 ->> 操作符 的右侧操作数为负，则从右侧开始计数访问数组元素。
  3. 修复在罕见使用的 unix-dotfile VFS 中回滚热日志文件时的问题。
  4. 即便 FTS5 表使用尚未注册的非标准分词器，也可以删除它们。
  5. 修复 group_concat() 聚合函数，使其在接收到单个为空字符串的输入时返回空字符串而非 NULL。
  6. 增强 generate_series() 表值函数，使其能够识别并使用对其输出值的约束。
  7. 预更新钩子现在可以识别 ALTER TABLE ADD COLUMN 添加的列具有非空默认值的情况。
  8. 性能优化：
    1. 改进与 IN 操作符 关联的子查询的重用，尤其是在由于 谓词下推 而导致 IN 操作符被复制时。
    2. 在 IN 操作符 右侧的子查询上使用布隆过滤器，以提高在适当情况下的性能。
    3. 确保类似 "SELECT func(a) FROM tab GROUP BY 1" 的查询每行仅调用 func() 函数一次。
    4. 不再尝试在因在其他已分析的索引中使用而被认为非选择性的列上创建 自动索引。
    5. 对查询规划器进行调整，使其为包含大量维度表的星型查询生成更优的执行计划。
    6. 新增“order-by-subquery”优化，当子查询的 ORDER BY 子句天然提供所需顺序时，尝试禁用外层查询中的排序操作。
    7. “indexed-subtype-expr”优化力求使用索引中已有的表达式，而不是基于表值重新计算，只要查询规划器能证明该表达式的 子类型 永远不会被使用。
    8. 各类代码调整以加快运行时性能。
  9. SQLite 相关命令行程序的增强：
    1. 添加实验性的 sqlite3_rsync 程序。
    2. 为 CLI 添加扩展函数 median()、percentile()、percentile_cont() 和 percentile_disc()。
    3. 将 CLI 中添加 .www 点命令。
    4. sqlite3_analyzer 工具现在提供 WITHOUT ROWID 表统计的细分。
    5. sqldiff 工具在其第二个参数不存在时避免创建空数据库。
  10. 增强 sqlite_dbpage 表值函数，使 INSERT 可用于增大或缩小数据库文件的大小。
  11. SQLite 不再使用 “long double” 数据类型，因为对 long double 的硬件支持越来越少，而且 long double 会给某些编译器工具链带来挑战。在需要扩展精度时，SQLite 改用 Dekker 的算法。
  12. SQLite 的 TCL 接口 支持 TCL9。虽然不能保证，但 TCL 8.5 及其以后的版本很可能仍然可用。建议用户升级到 TCL9。
  13. JavaScript/WASM：
    1. 修复 JavaScript “opfs” VFS 中导致损坏的错误。
    2. 修正 “opfs” VFS 的 “mode=ro” 处理。
    3. 解决一些浏览器特定的 OPFS 异常行为。
  14. FTS5 更改：
    1. 添加 fts5_tokenizer_v2 API 与 locale=1 选项，以创建支持自定义语言环境感知分词器的 FTS5 表。
    2. 添加 contentless_unindexed=1 选项，以创建无内容的 FTS5 表，该表会将任何 UNINDEXED 列的值持久存储在数据库中。
    3. 允许即使使用的自定义分词器实现不可用，也能删除 FTS5 表。

本小版本更新（3.47.1 版本，2024-11-25）的更改：

  1. 修复 makefile，使其在 “install” 目标上再次尊重 DESTDIR。
  2. 向 VFS 添加 SQLITE_IOCAP_SUBPAGE_READ 能力，以解决在 3.45.0 版本将 SQLITE_DIRECT_OVERFLOW_READ 设为默认时某些非标准 VFS 出现的问题。
  3. 修复 Windows 上新的 sqlite3_rsync.exe 实用程序中的行结束符问题。
  4. 修复 3.47.0 版本新增的查询优化导致的某些晦涩 IN 查询返回错误结果的问题。
  5. 其他一些次要错误修复。
