SQLite 3.7.10 发行于 2012-01-16

  1. 默认的schema 格式编号从 1 更改为 4。除非执行PRAGMA legacy_file_format=ON语句，否则新创建的数据库文件将无法被 3.3.0 (2006-01-10) 之前的 SQLite 版本读取。这也意味着降序索引默认开启。
  2. SQLITE_CONFIG_PCACHE和SQLITE_CONFIG_GETPCACHE配置参数及其对应的 sqlite3_pcache_methods 结构已被弃用。它们由新的sqlite3_pcache_methods2结构以及SQLITE_CONFIG_PCACHE2和SQLITE_CONFIG_GETPCACHE2配置参数替代。
  3. 在 VFS 接口中新增powersafe overwrite属性。提供SQLITE_IOCAP_POWERSAFE_OVERWRITE I/O 能力、SQLITE_POWERSAFE_OVERWRITE 编译期选项，以及用于URI 文件名的 "psow=BOOLEAN" 查询参数。
  4. 新增sqlite3_db_release_memory() 接口和shrink_memory pragma。
  5. 新增sqlite3_db_filename() 接口。
  6. 新增sqlite3_stmt_busy() 接口。
  7. 新增sqlite3_uri_boolean()和sqlite3_uri_int64()接口。
  8. 如果PRAGMA cache_size的参数为负数 N，则表示无论页面大小如何，页缓存都将使用大约 -1024*N 字节的内存。
  9. 增强默认内存分配器，使其在 Windows 上利用 _msize()，在 Mac 上利用 malloc_size()，在 Linux 上利用 malloc_usable_size()。
  10. 增强查询优化器，以支持对 rowid 的范围约束索引查询。
  11. 增强查询优化器的扁平化逻辑，使得即使复合语句为连接，也可以将 UNION ALL 复合体向上提升，以替换简单的包装 SELECT。
  12. 增强查询优化器，使得只要目标表最初为空，xfer 优化就可以与 INTEGER PRIMARY KEY ON CONFLICT 一起使用。
  13. 增强 Windows VFS，使得可通过 xSetSystemCall 接口覆盖所有系统调用。
  14. 更新 "unix-dotfile" VFS，改用 mkdir() 和 rmdir() 锁定目录，而不是使用 open() 和 unlink() 锁定文件。
  15. 扩展 test_quota.c，使其支持具有配额限制的类 stdio 接口。
  16. 更改 unix VFS，以容忍 read() 系统调用返回少于请求字节数的情况。
  17. 更改 unix 和 Windows 两种VFS，将扇区大小报告为 4096，而不是旧的默认 512。
  18. 在TCL 接口中，为用于创建新数据库连接对象的 "sqlite3" TCL 命令增加 -uri 选项。
  19. 新增SQLITE_TESTCTRL_EXPLAIN_STMT测试控制选项，并配合SQLITE_ENABLE_TREE_EXPLAIN编译期选项，使命令行 shell能够显示其处理的 SQL 语句的 ASCII 艺术解析树，以便调试与分析。
  20. 错误修复： 在重新启动 WAL 时添加额外的 xSync，以防止断电后极不可能但理论上存在的数据库损坏。 工单 ff5be73dee。
  21. 错误修复： 更改 VDBE，使得所有寄存器都初始化为 Invalid，而不是 NULL。 工单 7bbfb7d442
  22. 错误修复： 修复可能由 32 位整数溢出引发的问题。 工单 ac00f496b7e2
