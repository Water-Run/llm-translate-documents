SQLite 3.50.3 发布于 2025-07-17

从 3.50.0 版本（2025-05-29）起的变更：

  1. 新增 sqlite3_setlk_timeout() 接口，在支持阻塞锁的构建中为阻塞锁设置与 sqlite3_busy_timeout() 不同的独立超时。
  2. SQLITE_DBCONFIG_ENABLE_COMMENTS 约束（在上一发布中引入）被略微放宽，现在在读取现有的 sqlite_schema 表时始终允许注释。新 SQL 中仍然禁止注释。
  3. 新的 SQL 函数：
    1. unistr()
    2. unistr_quote()
  4. 对于内置 printf()（涵盖 sqlite3_mprintf() 接口、format() SQL 函数等）的 %Q 和 %q 转换，使用 可选形式 1 标志（“#”）时，会将 控制字符 转换为适用于 unistr() 的反斜杠转义形式。
  5. CLI 增强：
    1. 避免直接输出大多数 控制字符。
    2. .dump 命令 的输出使用新的 unistr() SQL 函数对特殊字符进行编码，除非将 --escape 模式关闭。
    3. ".schema --indent" 命令对复杂部分索引的输出格式进行了改进。
  6. sqlite3_rsync 的增强：
    1. 不再要求数据库处于 WAL 模式。
    2. 同步协议经过增强，当双方初始状态非常相似时可减少网络带宽消耗。
    3. sqlite3_rsync 程序在 Mac 上运行时无需在远程端指定 sqlite3_rsync 可执行文件的完整路径，只要在以下目录之一安装该可执行文件即可：$HOME/bin:/usr/local/bin:/opt/homebrew/bin
  7. JSON 函数的改进：
    1. 错误修复：强制执行 JSON5 规范，禁止“\0”转义后跟随数字。
    2. 错误修复：当 json_group_object(LABEL,VALUE) 的 LABEL 参数为 NULL 时，结果对象中省略该元素。
    3. 优化：如果 jsonb_set() 或 jsonb_replace() 在大型 JSONB 对象内部进行更改，它们会尽量保持 JSONB 大小不变并仅修改内部尽可能少的字节。这有助于减少 I/O，因为 SQLite 可以只写入包含更改字节的页面而不是周围所有页面。
  8. 改善了在 Cygwin、MinGW 及类似环境以及 Termux 上的构建支持。
  9. 文档和源码注释的拼写修复。
  10. 各种性能改进。
  11. JavaScript/WASM：
    1. 修复了 OPFS SAHPool VFS 中长期存在的文件名摘要计算错误。在此 VFS 中由 3.50.0+ 创建的数据库无法由旧版本读取，但 3.50.0 可以向后兼容旧版本创建的数据库。

从 3.50.1 版本（2025-06-06）起的变更：

  1. 修复了在 3.50.0 中新增优化暴露出来的 jsonb_set() 及类似函数中的长期错误。
  2. 修复在使用 -DSQLITE_DEFAULT_MEMSTATUS=0 构建时可能出现的看似无害的 ASAN 警告。
  3. 修复 sqlite3_rsync 中的一个减一错误，避免复制数据库时最后一页未被传输。
  4. 查询规划器优化：允许 LEFT JOIN 的右侧被 展开，即使它是虚拟表。
  5. 修复 sqlite3_setlk_timeout()，在打开快照事务或被另一个运行恢复的进程阻塞时使用阻塞锁。
  6. 其他在 3.50.0 发布后报告的小修复。

从 3.50.2 版本（2025-06-28）起的变更：

  1. 修复 concat_ws() SQL 函数，使其在拼接时包含空字符串。 论坛帖子 52503ac21d。
  2. 修复 CLI 使用的文件 I/O 扩展，使其可在 MinGW 编译链下构建。
  3. 避免在回滚保存点且脏页已溢出到 wal 文件后写入无校验和帧。 论坛帖子 b490f726db。
  4. 修复 Bitvec 对象在数据库接近最大大小 60 页时发生的栈溢出。
  5. 修复包含 BLOB 值的 fts5 表上的 UPDATE 问题。
  6. 修复 RIGHT JOIN 上的传递性 IS 约束问题。
  7. 若查询中的聚合项数量超过最大列数，则提前报错以避免后续断言故障。
  8. 确保 sqlite3_setlk_timeout() 持有数据库互斥锁。
  9. 修复 API 文档中的拼写错误。

本次补丁发布（3.50.3 版本，2025-07-17）的变更：

  1. 修复在针对以非常特定方式故意损坏的 FTS5 索引执行查询时可能出现的内存错误。
  2. 修复解析器，使其在 CREATE TRIGGER 语句的所有位置都忽略 SQL 注释。此修复解决了在 3.49.0 版本引入 SQLITE_DBCONFIG_ENABLE_COMMENTS 功能时引入的问题。
  3. 修复由于 AND 运算符过度优化导致的错误答案。 论坛帖子 f4878de3e。
  4. 修复若干 makefile 问题和文档拼写错误。
