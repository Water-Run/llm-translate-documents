SQLite 3.12.0 版本 发布于 2016-03-29

    可能引起重大影响的变更：

  1. SQLITE_DEFAULT_PAGE_SIZE 从 1024 增加到 4096。
    SQLITE_DEFAULT_CACHE_SIZE 从 2000 改为 -2000，以便默认使用相同量的缓存内存。
    有关详情，请参阅 3.12.0 版本页面大小变更 的应用说明。

    性能增强：

  2. 对 Lemon 解析器生成器 的增强，使其生成更小更快的 SQL 解析器。
  3. 仅在两个或更多附加数据库都被修改、其 PRAGMA synchronous 未设为 OFF，且journal_mode 未设为 OFF、MEMORY 或 WAL 时，才创建 master journal 文件。
  4. 仅在 statement journal 文件的大小超过阈值时创建，否则日志保存在内存中，不发生任何 I/O。该阈值可以在编译时使用 SQLITE_STMTJRNL_SPILL 配置，或在启动时通过 sqlite3_config(SQLITE_CONFIG_STMTJRNL_SPILL) 配置。
  5. 即使 xBestIndex 方法未在 IN 运算符左侧的虚拟表列上设置 sqlite3_index_constraint_usage.omit 标志，查询规划器仍然可以优化虚拟表上的 IN 运算符。
  6. 在包含三个或更多表的连接中，如果对虚拟表的约束分布在其他两个或更多表上，查询规划器现在能更好地优化对虚拟表的访问。
  7. 对应用自定义 SQL 函数提供更高效的处理，特别是当应用定义了数百或数千个自定义函数时。
  8. 查询规划器在估算 ORDER BY 的成本时会考虑 LIMIT 子句。
  9. 配置脚本（在 Unix 上）会自动检测 pread() 和 pwrite()，并在可用时设置编译时选项以使用这些操作系统接口。
  10. 减少保存模式所需的内存量。
  11. 其他杂项微优化，以提升性能并减少内存使用。

    新功能：

  12. 为 sqlite3_db_config() 添加了 SQLITE_DBCONFIG_ENABLE_FTS3_TOKENIZER 选项，可在运行时启用或禁用具有两个参数版本的 fts3_tokenizer() SQL 函数。
  13. 向 RBU 扩展添加了 sqlite3rbu_bp_progress() 接口。
  14. PRAGMA defer_foreign_keys=ON 语句现在也会禁止外键上的 RESTRICT 操作。
  15. 增加了 sqlite3_system_errno() 接口。
  16. 新增了 SQLITE_DEFAULT_SYNCHRONOUS 和 SQLITE_DEFAULT_WAL_SYNCHRONOUS 编译时选项。SQLITE_DEFAULT_SYNCHRONOUS 编译选项取代了已不再支持的 SQLITE_EXTRA_DURABLE 选项。
  17. 增强了 命令行 shell 中的“.stats”命令（在可用的情况下）以显示更多来自 /proc 的 I/O 性能信息。

    错误修复：

  18. 确保单条语句内多个触发器的 sqlite3_set_auxdata() 值不会互相干扰。 票号 dc9b1c91。
  19. 修复形式为“x IN (SELECT...)”的表达式的代码生成器，当右侧的 SELECT 语句是相关子查询时的处理。 票号 5e3c886796e5512e。
  20. 修复与 sqlite3_db_readonly() 接口相关的无害 TSAN 警告。

    哈希值：

  21. SQLITE_SOURCE_ID: "2016-03-29 10:14:15 e9bb4cf40f4971974a74468ef922bdee481c988b"
