SQLite 3.46.0 发布（2024-05-23）

  1. 通过多种方式增强 PRAGMA optimize，使其 更易使用：
    1. PRAGMA optimize 会自动实施临时 分析限制，以防止在大型数据库上运行时间过长。
    2. 添加了新的 0x10000 位掩码选项，以检查所有表的更新。
    3. 自动重新分析没有 sqlite_stat1 条目的表。
  2. 对日期和时间函数的增强：
    1. strftime() SQL 函数现在支持 %G、%g、%U 和 %V。
    2. 新的 'ceiling' 和 'floor' 修饰符控制在按整数月和/或年平移日期时用于解析 模糊日期的算法。
    3. 'utc' 和 'localtime' 修饰符现在在 SQLite 已知 时间分别已处于 UTC 或本地时间时将不再起作用。
  3. 在 数字字面值中的数字之间添加对下划线 ("_") 字符的支持。
  4. 添加 json_pretty() SQL 函数。
  5. 查询规划器改进：
    1. “VALUES 作为协程”优化使得 VALUES 子句中包含数千行的 INSERT 语句在约一半的时间内 解析并运行，并使用约一半的内存。
    2. 允许在类似 “SELECT count(DISTINCT col) FROM ...” 的查询中使用索引，即使索引记录不比表记录更小。
    3. 改进了识别 SQL 函数值恒定的情况，因为其所有参数都恒定。
    4. 增强了 WHERE 子句下推优化，使其能够下推包含无关联子查询的 WHERE 子句项。
  6. 如果 SQL 解析器堆栈溢出，会从堆中分配更多内存，而不是报告“解析器堆栈溢出”错误。
  7. JSON 更改：
    1. 允许 JSON5 字符串字面值中包含 ASCII 控制字符。
    2. 修复 -> 和 ->> 操作符，使得当右操作数是看起来像整数的字符串时仍然将其视为字符串， 因为 PostgreSQL 也是这样做的。
  8. 允许将大型十六进制字面值用作表列的 DEFAULT 值。
