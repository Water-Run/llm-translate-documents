SQLite 3.11.0 版本发布于 2016-02-15

    常规改进：

  1. 增强了 WAL 模式，使其在事务大于 cache_size 时仍能高效运行。
  2. 新增了 FTS5 detail 选项。
  3. 在 PRAGMA synchronous 中新增了“EXTRA”选项，在 DELETE 模式下 unlink 回滚日志时同步其所在目录，以提升耐久性。标志为 SQLITE_EXTRA_DURABLE 的编译期选项会默认启用 PRAGMA synchronous=EXTRA。
  4. 增强了 查询规划器，使其能够在 OR 优化 中使用 覆盖索引。
  5. 在 UPDATE 语句中，对于未改变的列，避免重新计算 NOT NULL 和 CHECK 约束。
  6. 进行多项微优化，使得库比上一版本更快。

    命令行外壳的增强：

  7. 默认情况下，外壳现在处于“自动解释”模式。EXPLAIN 命令的输出会自动格式化。
  8. 新增了 ".vfslist" 点命令。
  9. 标准构建中默认开启 SQLITE_ENABLE_EXPLAIN_COMMENTS 编译期选项。

    TCL 接口增强：

  10. 如果使用 "-uri 1" 选项打开数据库连接，则 "backup" 和 "restore" 命令会遵循 URI 文件名。
  11. 在 "sqlite3" 命令中新增 "-sourceid" 选项。

    Makefile 改进：

  12. 改进 configure 脚本中对 pthreads 的检测。
  13. 增加从 合并代码包 进行 MSVC Windows 构建的能力。

    错误修复

  14. 修复了协程之间错误共享 VDBE 临时寄存器而在某些罕见情况下导致查询结果不正确的问题。对应工单 d06a25c84454a。
  15. 修复了 sqlite3_result_subtype() 接口中的一个问题，该问题在某些罕见情况下会影响 json1 扩展。对应工单 f45ac567eaa9f9。
  16. 对 JSON 字符串中的控制字符进行转义。对应工单 ad2559db380abf8。
  17. 只要未定义 SQLITE_OMIT_DEPRECATED，就重新启用内置 unix VFS 中的 xCurrentTime 和 xGetLastError 方法。

    向后兼容性：

  18. 由于持续的安全顾虑，罕用且鲜为人知的 fts3_tokenizer() 的两个参数版本默认处于禁用状态，除非 SQLite 在编译时启用了 SQLITE_ENABLE_FTS3_TOKENIZER。
