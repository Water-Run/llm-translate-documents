SQLite Release 3.20.1 On 2017-08-24

自 3.20.0 版本（2017-08-01）以来的更改：

  1. 更新某些由 sqlite3_errmsg() 返回的错误消息文本。
  2. 添加新的 指针传递接口。
  3. 为了利用新的 指针传递接口 提供的更高安全性，对某些扩展进行了向后不兼容的修改：
    1. 扩展 FTS5 → 需要 sqlite3_bind_pointer() 来查找 fts5_api 指针。
    2. carray(PTR,N) → 需要 sqlite3_bind_pointer() 来设置 PTR 参数。
    3. remember(V,PTR) → 需要 sqlite3_bind_pointer() 来设置 PTR 参数。
  4. 新增 SQLITE_STMT 虚表 扩展。
  5. 新增 COMPLETION 扩展 —— 旨在为交互式用户界面提供 Tab 补全建议。此项仍在开发中，未来发布将持续增强。
  6. 新增 UNION 虚表 扩展。
  7. 内置的 日期和时间函数 已增强，可在 CHECK 约束、表达式索引 和 部分索引 的 WHERE 子句中使用，前提是它们不使用“now”、“localtime”或“utc”关键字。了解更多信息。
  8. 新增带额外“prepFlags”参数的 sqlite3_prepare_v3() 和 sqlite3_prepare16_v3() 接口。
  9. 为 sqlite3_prepare_v3() 提供 SQLITE_PREPARE_PERSISTENT 标志，并利用它限制 旁路内存 被 FTS3、FTS5 和 R 树扩展 滥用。
  10. 添加 PRAGMA secure_delete=FAST 命令。当 secure_delete 设置为 FAST 时，只要不增加 I/O 量，就会用零覆盖旧内容。已删除内容可能仍留在 空闲页面列表 上，但会从所有 B 树页面中清除。
  11. 命令行 shell 的增强功能：
    1. 为 readline 和 linenoise 添加对 COMPLETION 扩展 的 Tab 补全支持。
    2. 添加 ".cd" 命令。
    3. 增强 ".schema" 命令以显示所有附加数据库的模式。
    4. 增强 ".tables" 命令，使其在名称不为 “main” 时显示所有附加数据库的模式名称。
    5. ".import" 命令忽略开头的 UTF-8 BOM。
    6. 为 ".dump" 命令添加 "--newlines" 选项，以便直接输出 U+000a 和 U+000d 字符，而不是通过 replace() 函数进行转义。
  12. 查询规划器增强：
    1. 为 OR 扫描中的每个 OR 条件生成单独循环时，将任何恒定的 WHERE 表达式移出循环，类似于顶层循环所做的处理。
    2. 查询规划器检查绑定参数的值，以帮助判断部分索引是否可用。
    3. 在两个估计成本相同的计划之间做决定时，偏向选择不使用排序器的方案。
    4. 延后评估包含相关子查询的 WHERE 约束，期望它们甚至不需要被评估。
    5. 对位于 LEFT JOIN 右侧、读取 虚表 数据的子查询，不要使用 扁平化优化，因为这会阻止查询规划器在子查询结果上创建 自动索引，从而影响查询性能。
  13. 为 sqlite3_stmt_status() 接口新增 SQLITE_STMTSTATUS_REPREPARE、SQLITE_STMTSTATUS_RUN 和 SQLITE_STMTSTATUS_MEMUSED 选项。
  14. 为 PRAGMA integrity_check、PRAGMA quick_check 和 PRAGMA foreign_key_check 提供 PRAGMA 函数。
  15. 为 TCL 接口 eval 方法 添加 -withoutnulls 选项。
  16. 增强 sqlite3_analyzer.exe 工具，使其显示 B 树页面上元数据的字节数。
  17. SQLITE_DBCONFIG_ENABLE_QPSG 运行时选项与 SQLITE_ENABLE_QPSG 编译时选项启用了 查询规划器稳定性保证。另见工单 892fc34f173e99d8。
  18. 各种优化带来约 2% 的 CPU 周期 降低。

    修复的错误：

  19. 修复了使用 扁平化优化 的查询在调用 sqlite3_column_name() 时的表现，使其与未使用该优化的查询以及 PostgreSQL、MySQL 和 SQLServer 保持一致。工单 de3403bf5ae。
  20. 修复查询规划器在 WHERE 子句使用 IS 运算符 时仍在 LEFT JOIN 的右表上使用 自动索引 的问题。修复工单 ce68383bf6aba。
  21. 确保查询规划器知道在 扁平化 的 LEFT JOIN 中，即使某列标记为 “NOT NULL”，该列仍然可能为 NULL。修复工单 892fc34f173e99d8。
  22. 修复在带有 附加数据库 的连接上运行 PRAGMA integrity_check 时偶发的误报。工单 a4e06e75a9ab61a12。
  23. 修复一个由 OSSFuzz 发现的 bug，在使用某些可疑的 CREATE TABLE 声明时会导致断言错误。工单 bc115541132dad136。
