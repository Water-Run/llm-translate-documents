SQLite 3.8.11 版本发布于 2015-07-27

  1. 添加了实验性的 RBU 扩展。请注意此扩展仍处于实验阶段，可能会以不兼容的方式更改。
  2. 添加了实验性的 FTS5 扩展。请注意此扩展仍处于实验阶段，可能会以不兼容的方式更改。
  3. 添加了 sqlite3_value_dup() 和 sqlite3_value_free() 接口。
  4. 增强了 spellfix1 扩展以支持 ON CONFLICT 子句。
  5. IS 操作符 现在可以驱动索引。
  6. 增强查询规划器，允许对由协程实现的 FROM 子句子查询执行 自动索引。
  7. 禁止在 公共表表达式 中使用“rowid”。
  8. 添加了 PRAGMA cell_size_check 命令，以更好且更早地检测数据库文件损坏。
  9. 为 FTS3 中的 matchinfo() 函数添加了 matchinfo 'b' 标志。
  10. 改进了数据库文件的模糊测试，并修复发现的问题。
  11. 添加了 fuzzcheck 测试程序，并在执行 “make test” 时自动使用 SQL 和数据库测试用例运行此程序。
  12. 添加了 SQLITE_MUTEX_STATIC_VFS1 静态互斥锁，并在 Windows VFS 中使用它。
  13. 对于未运行完成的语句，在 sqlite3_reset() 或 sqlite3_finalize() 调用时触发 sqlite3_profile() 回调。
  14. 增强页面缓存，使其可预分配一块用于初始页面缓存行的内存。默认预分配设置为 100 页。在常见负载下可带来约 5% 的性能提升。
  15. 各种微优化使得相同 CPU 周期完成 22.3% 更多工作。SQLite 现在运行速度是 3.8.0 版本 的两倍，是 3.3.9 版本 的三倍。（在 Ubuntu 14.04 x64 上使用 gcc 4.8.2 和 -Os，通过 cachegrind 对 speedtest1.c 负载测试。实际性能可能会有所不同。）
  16. 添加了 sqlite3_result_zeroblob64() 和 sqlite3_bind_zeroblob64() 接口。

    重要错误修复：

  17. 修复 CREATE TABLE AS，使得 TEXT 类型的列不再可能保存 INT 值。问题编号 f2ad7de056ab1dc9200
  18. 修复 CREATE TABLE AS，使其在右侧的 SELECT 语句因错误终止时，不会在 sqlite_master 表 中留下 NULL 条目。问题编号 873cae2b6e25b
  19. 修复 跳过扫描优化，使其在对 WITHOUT ROWID 表使用 OR 优化 时依然能正确工作。问题编号 8fd39115d8f46
  20. 修复 sqlite3_memory_used() 和 sqlite3_memory_highwater() 接口，使其确实返回 64 位结果。
