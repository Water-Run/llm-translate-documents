SQLite 发布版 3.35.5 于 2021-04-19

来自版本 3.35.0 (2021-03-12) 的前期更改：

  1. 新增内置 SQL 数学函数()。(-DSQLITE_ENABLE_MATH_FUNCTIONS 编译时选项要求。)
  2. 新增对ALTER TABLE DROP COLUMN 的支持。
  3. 通用UPSERT：
    1. 允许按顺序评估多个 ON CONFLICT 子句，
    2. 最终的 ON CONFLICT 子句可以省略冲突目标，并仍可使用 DO UPDATE。
  4. 在DELETE、INSERT与UPDATE 语句中新增对RETURNING 子句的支持。
  5. 在包含非常大的 TEXT 或 BLOB 值的数据库上运行VACUUM 时使用更少内存。如今不再需要一次性在内存中保存整个 TEXT 或 BLOB。
  6. 指定公共表表达式时新增对MATERIALIZED与NOT MATERIALIZED 提示的支持。默认行为原先为 NOT MATERIALIZED，但现在对于使用超过一次的 CTE 改为 MATERIALIZED。
  7. SQLITE_DBCONFIG_ENABLE_TRIGGER 与 SQLITE_DBCONFIG_ENABLE_VIEW 设置仅控制主数据库模式或附加数据库模式中的触发器与视图，而不再控制 TEMP 模式。TEMP 触发器与视图始终允许。
  8. 查询规划器/优化器改进：
    1. 对最小/最大优化的增强，使其在与 IN 运算符以及上一版本的 OP_SeekScan 优化配合时表现更佳。
    2. 尝试将 WHERE 子句中的 EXISTS 运算符视为 IN 运算符来处理，在这种转换有效且可能提升性能的情况下。
    3. 即使父查询为连接，仍允许将 UNION ALL 子查询展平。
    4. 即使禁用 STAT4，也在 WHERE 子句中适当使用索引来处理 IS NOT NULL 表达式。
    5. 形如 “x IS NULL” 或 “x IS NOT NULL” 的表达式，若 “x” 是具有 “NOT NULL” 约束且未参与外连接的列，则可能直接转换为 FALSE 或 TRUE。
    6. 如果 UPDATE 语句未修改与外键相关的任何列，则避免检查外键约束。
    7. 允许 WHERE 条件下推到包含窗口函数的子查询，只要该 WHERE 条件完全由常量与子查询中所有窗口函数 PARTITION BY 子句中出现的表达式的副本组成。
  9. 命令行界面 (CLI) 增强：
    1. 增强 ".stats" 命令以接受新参数 "stmt" 与 "vmstep"，分别仅显示预处理语句统计与虚拟机步骤计数。
    2. 添加 ".filectrl data_version" 命令。
    3. 增强 ".once" 与 ".output" 命令，当目标参数以 "|" 开头（表示输出重定向至管道）时，可无需引号。
  10. Bug 修复：
    1. 修复在处理语法错误且具有相关 WHERE 子句与 “HAVING 0” 的 SELECT 语句时可能发生的 NULL 指针解引用。（在 3.34.1 补丁版中亦已修复。）
    2. 修复版本 3.33.0 的 IN 运算符优化存在的错误，该错误可能导致不正确结果。
    3. 修复在模式以 "%" 结尾且包含 "ESCAPE '_'" 子句时 LIKE 运算符 可能产生的错误结果。

来自版本 3.35.1 (2021-03-15) 的前期更改：

  1. 修复新 DROP COLUMN 功能在应用于具索引且在索引定义中带引号的列时的错误。
  2. 改善命令行界面中.dump 命令的内置文档。

来自版本 3.35.2 (2021-03-17) 的前期更改：

  1. 修复appendvfs.c 扩展在 3.35.0 版本中引入的问题。
  2. 确保无参数的日期/时间函数（其响应依赖当前时间）被视为非确定性函数。工单 2c6c8689fb5f3d2f
  3. 修复sqldiff 实用程序在处理带有不寻常空白字符的虚拟表 定义时的问题。
  4. 将 3.35.0 发布中条目 8c 描述的新 UNION ALL 优化限制在不会产生过多新子查询的程度。详情见论坛帖子 140a67d3d2。

来自版本 3.35.3 (2021-03-26) 的前期更改：

  1. 增强字节码引擎的 OP_OpenDup 操作码，使其在被复制的游标本身来自 OP_OpenDup 时仍可正常工作。修复工单 bb8a9fd4a9b7fce5。该问题是因为最近的 MATERIALIZED 提示增强才显现出来。
  2. 对于相关的公共表表达式，在每个使用场景中分别物化，因为这是正确性的要求。此项修复是 MATERIALIZED 提示增强引入的问题。
  3. 修复 unix VFS 文件名规范器中的问题。
  4. 修复CLI 中“box” 输出模式，使其可与返回一行或多行零列（例如PRAGMA incremental_vacuum）的语句配合使用。详见论坛帖子 afbbcb5b72。
  5. 改善由错误的公共表表达式生成的错误消息。详见论坛帖子 aa5a0431c99e。
  6. 修复若干不正确的 assert() 语句。
  7. 修复SELECT 语句语法图中 FROM 子句的语法显示，使其正确。 论坛帖子 9ed02582fe。
  8. 修复 EBCDIC 字符分类器，使其将换行视为空白。 论坛帖子 58540ce22dcd。
  9. 改进xBestIndex 方法以及(不受支持的)wholenumber 虚拟表 扩展的实现，使其更好地说服查询规划器避免尝试物化具有无限行的表。 论坛帖子 b52a020ce4。

来自版本 3.35.4 (2021-04-02) 的前期更改：

  1. 修复条目 8b 所述查询规划优化中的缺陷。工单 de7db14784a08053。
  2. 修复新RETURNING 语法中的缺陷。工单 132994c8b1063bfb。
  3. 修复新的RETURNING 功能，使其在 RETURNING 子句中的术语引用未知表时抛出错误，而不是静默忽略。
  4. 修复与聚合函数处理相关的断言，该断言被下推优化错误触发。

此具体补丁版本 3.35.5 (2021-04-19) 的更改：

  1. 修复新 ALTER TABLE DROP COLUMN 功能中的缺陷，该缺陷可能损坏数据库文件。
  2. 修复一个可能导致查询结果不正确的晦涩查询优化器问题。
