SQLite 3.8.0.2 于 2013-09-03 发布

  1. 新增对部分索引的支持
  2. 切换到下一代查询优化器，以获得更快更优的查询计划。
  3. EXPLAIN QUERY PLAN输出不再显示连接中各循环生成的行数估计。
  4. 新增FTS4 notindexed 选项，允许在 FTS4 表中使用非索引列。
  5. 为sqlite3_stmt_status()添加SQLITE_STMTSTATUS_VM_STEP选项。
  6. 新增cache_spill pragma。
  7. 新增query_only pragma。
  8. 新增defer_foreign_keys pragma以及sqlite3_db_status(db, SQLITE_DBSTATUS_DEFERRED_FKS,...) C 语言接口。
  9. 在源代码树的 ext/misc 子目录中新增percentile()函数作为可加载扩展。
  10. 新增SQLITE_ALLOW_URI_AUTHORITY编译时选项。
  11. 添加sqlite3_cancel_auto_extension(X)接口。
  12. 正在运行且缺少 FROM 子句的 SELECT 语句（或任何不从数据库文件读写的语句）不会阻止读事务关闭。
  13. 新增SQLITE_DEFAULT_AUTOMATIC_INDEX编译时选项。将其设置为 0 可默认禁用自动索引。
  14. 当查询优化器使用自动索引时，在SQLITE_CONFIG_LOG上发出SQLITE_WARNING_AUTOINDEX警告。
  15. 新增SQLITE_FTS3_MAX_EXPR_DEPTH编译时选项。
  16. 为 next_char() 扩展 SQL 函数添加可选的第五个参数以定义排序序列。
  17. 在 WAL 模式下，如果由于读取的是旧快照而无法将读事务升级为写事务，则返回SQLITE_BUSY_SNAPSHOT扩展错误代码。
  18. 增强了 sqlite3_analyzer 工具，除了总大小外还可分别提供表中每个索引的大小信息。
  19. 若应用定义的 SQL 函数由不访问任何数据库表的 SELECT 语句调用，则该函数实现内部运行的 SQL 语句可自由打开和关闭读事务。
  20. 除非使用了 HAVE_POSIX_FALLOCATE 编译时选项，否则在所有（Unix）系统上禁用 posix_fallocate()。
  21. 更新命令行 shell中的“.import”命令以支持多行字段、正确的 RFC-4180 引号，并在输入文本不严格符合 RFC-4180 时发出警告或错误。
  22. 错误修复：在FTS4的unicode61分词器中，将所有私有编码点视为标识符符号。
  23. 错误修复：ORDER BY 子句中的裸标识符更紧密地绑定到输出列名，而表达式中的标识符更倾向于绑定到输入列名。GROUP BY 子句中的标识符始终优先选择输出列名。
  24. 错误修复：通过迁移到NGQP修复了旧版查询优化器中的多个问题。

  1. 修复了一个减一错误，该错误导致命令行 shell 错误读取以 CRLF 终止行末的带引号空字符串。
  2. 修复了一个查询优化器错误，该错误涉及带有 BETWEEN 或 LIKE/GLOB 约束的 LEFT JOIN，其右侧又带有包含 OR 约束的 INNER JOIN。
  3. 修复了一个查询优化器错误，该错误在查询具有超过四列的 UNIQUE 或 PRIMARY KEY 约束的表时可能导致段错误。

  1. 修复了尝试省略未使用 LEFT JOIN 的优化中的一个错误。
