SQLite 3.13.0 发行于 2016-05-18

  1. 尽可能推迟与 TEMP 文件相关的 I/O，希望最终可以完全避免这些 I/O。
  2. 将 会话 扩展合并到主干。
  3. 向 命令行 shell 添加“.auth ON|OFF”命令。
  4. 向 命令行 shell 的“.schema”和“.fullschema”命令添加“--indent”选项，以启用格式化输出。
  5. 向 命令行 shell 添加“.eqp full”选项，对每个执行的语句同时运行 EXPLAIN 与 EXPLAIN QUERY PLAN。
  6. 改进 Windows 上 命令行 shell 的 Unicode 文件名处理。
  7. 增强对应用程序对 sqlite_stat1 表进行不完整或错误修改所导致的荒谬查询规划器决策的抵抗能力。
  8. 添加 sqlite3_db_config(db,SQLITE_DBCONFIG_ENABLE_LOAD_EXTENSION) 接口，允许在禁用 load_extension() SQL 函数以保障安全的同时启用 sqlite3_load_extension() C 接口。
  9. 更改 Unix 上的 临时目录搜索算法，允许具有写和执行权限但无读取权限的目录作为临时目录，并对“.”回退目录采用同样标准。

    错误修复：

  10. 修复多行一次性 DELETE 优化中的一个问题，当 WHERE 子句中有自引用子查询时会计算出错误的结果。修复工单 dc6ebeda9396087
  11. 修复 DELETE 在表为 rowid 表 且具有 INTEGER PRIMARY KEY 时可能发生的段错误，尤其是在 WHERE 子句包含 OR 并且表中存在一个或多个可触发 OR 优化的索引但这些索引仅引用 INTEGER PRIMARY KEY 以外的列。工单 16c9801ceba49。
  12. 在检查 WHERE 子句下推优化 时，验证复合内部 SELECT 的所有项都是非聚合项，而不仅仅是最后一项。修复工单 f7f8c97e97597。
  13. 修复 Windows 中的一个锁竞争条件，当两个或更多进程同时尝试恢复相同的 热日志 时可能发生该问题。

    哈希值：

  14. SQLITE_SOURCE_ID: "2016-05-18 10:57:30 fc49f556e48970561d7ab6a2f24fdd7d9eb81ff2"
  15. sqlite3.c 的 SHA1: 9b9171b1e6ce7a980e6b714e9c0d9112657ad552

    回传至 3.12.2 补丁版（2016-04-18）的错误修复：

  16. 修复 3.12.0 与 3.12.1 中的一个向后兼容性问题：将列声明为 "INTEGER" PRIMARY KEY（数据类型关键词带引号）时无法被识别为 INTEGER PRIMARY KEY，最终导致数据库文件不兼容。工单 7d7525cb01b68
  17. 修复一个自 3.9.0 版本 起存在的 bug：当启用 PRAGMA reverse_unordered_selects 时，DELETE 操作可能漏删行。工单 a306e56ff68b8fa5
  18. 修复代码生成器中的一个 bug：在连接两个或多个 虚拟表 且用于外循环的虚拟表具有 IN 运算符 约束时，可能生成错误结果。
  19. 确定用于排序大量数据的缓存大小时正确解释负值的 PRAGMA cache_size。

    回传至 3.12.1 补丁版（2016-04-08）的错误修复：

  20. 修复 3.12.0 引入的边界条件错误，在大量使用 SAVEPOINT 时可能导致崩溃。工单 7f7f8026eda38。
  21. 修复 视图 的一个问题，使其在可能的情况下继承所定义表的列数据类型。
  22. 修复查询规划器，使 IS 与 IS NULL 运算符能够驱动左外连接上的索引。
