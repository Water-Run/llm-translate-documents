SQLite 3.10.2 版本于 2016-01-20 发布

自版本 3.10.0（2016-01-06）以来的先前更改：

    一般改进：

  1. 在虚拟表上增加了对 LIKE、GLOB 和 REGEXP 运算符的支持。
  2. 在 sqlite3_index_info 中增加了 colUsed 字段，以供 sqlite3_module.xBestIndex 方法使用。
  3. 增强 PRAGMA cache_spill 语句，使其接受一个 32 位整数参数，该参数表示禁止缓存溢出的阈值以下的值。
  4. 在 unix 上，如果打开的是数据库文件的符号链接，则相关的日志文件基于实际文件名，而不是符号链接名。
  5. 为 sqldiff 增加了 “--transaction” 选项。
  6. 增加了 sqlite3_db_cacheflush() 接口。
  7. 增加了 sqlite3_strlike() 接口。
  8. 使用 内存映射 I/O 时，将数据库文件映射为只读，以防应用中的野指针或数组越界意外修改数据库文件。
  9. 增加实验性的 sqlite3_snapshot_get()、sqlite3_snapshot_open() 和 sqlite3_snapshot_free() 接口。这些接口可能在后续版本中更改或移除。
  10. 增强 日期和时间函数 中的 'utc' 修饰符，使其在已知日期/时间已经为 UTC 时成为无操作。（这不是兼容性破坏，因为此情况的行为长期以来已记录为“未定义”。）
  11. 在 json 扩展中增加 json_group_array() 和 json_group_object() SQL 函数。
  12. 增加 SQLITE_LIKE_DOESNT_MATCH_BLOBS 编译时选项。
  13. 多项小的性能优化。

    可移植性增强：

  14. 规避 HP/UX 上 HP C 编译器优化器中的符号扩展错误。(详情)

    对命令行 shell的增强：

  15. 增加了 “.changes ON|OFF” 和 “.vfsinfo” 点命令。
  16. 在 Windows 的 cmd.exe 中运行时实现 MBCS 与 UTF8 之间的转换。

    对 makefile 的增强：

  17. 为各个 autoconf 生成的 configure 脚本增加 --enable-editline 和 --enable-static-shell 选项。
  18. 在 makefile 中移除所有对 "awk" 的使用，以便 MSVC 用户更容易构建。

    重要修复：

  19. 修复整数与浮点数不一致的比较操作；若索引建立在同时包含大整数和相近大小浮点值的表列上，该问题可能导致索引损坏。工单 38a97a87a6。
  20. 修复查询规划器中的无限循环，该问题可能在格式错误的 公共表表达式 上发生。
  21. 在 sqldiff 工具中修复了多个错误。

自版本 3.10.1（2016-01-14）以来的先前更改：

    新功能：

  1. 增加 SQLITE_FCNTL_JOURNAL_POINTER 文件控制。

    错误修复：

  2. 修复查询规划器中存在 16 个月之久的错误；当标量子查询尝试使用 块排序 优化时，该错误可能产生错误结果。工单 cb3aa0641d9a4。

此补丁版本 3.10.2（2016-01-20）中的更改：

    关键错误修复：

  1. 版本 3.10.0 在 LIKE 运算符中引入了一个大小写折叠错误，已由本补丁修复。工单 80369eddd5c94。

    其他杂项错误修复：

  2. 修复在使用 -DSQLITE_HAS_CODEC 编译 SQLite 时可能发生的使用后释放错误。
  3. 修复构建，以便在使用 -DSQLITE_OMIT_WAL 时可以工作。
  4. 修复合并版本的 configure 脚本，使 --readline 选项在 Raspberry Pi 上再次可用。

    哈希：

  5. SQLITE_SOURCE_ID：“2016-01-20 15:27:19 17efb4209f97fb4971656086b138599a91a75ff9”
  6. sqlite3.c 的 SHA1：f7088b19d97cd7a1c805ee95c696abd54f01de4f
