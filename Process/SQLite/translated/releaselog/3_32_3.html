SQLite 3.32.3 版本发布于 2020-06-18

来自版本 3.32.0（2020-05-22）的先前更改：

  1. 添加了使用 近似 ANALYZE 的支持，借助 PRAGMA analysis_limit 命令。
  2. 添加了 字节码虚拟表。
  3. 将 checksum VFS shim 添加到源树中包含的可在运行时加载的扩展集合。
  4. 添加了 iif() SQL 函数。
  5. INSERT 和 UPDATE 语句现在在计算 CHECK 约束 之前始终应用 列亲和性。这个错误修复理论上可能会对那些具有不寻常 CHECK 约束的旧数据库造成问题，这些约束要求 INSERT 的输入类型不同于声明的列类型。详情请参阅工单 86ba67afafded936。
  6. 添加了 sqlite3_create_filename()、sqlite3_free_filename() 以及 sqlite3_database_file_object() 接口，以更好地支持 VFS shim 实现。
  7. 将默认参数数量的 上限 从 999 提高到 32766。
  8. 为 UINT 排序序列 作为可选的 可加载扩展 添加了代码。
  9. 对 命令行界面 的增强：
    1. 为 .import 命令添加了选项：--csv、--ascii、--skip
    2. .dump 命令现在接受多个 LIKE 模式参数，并输出所有匹配表的并集。
    3. 在调试构建中添加 .oom 命令
    4. 为 .excel、.output 和 .once 命令添加 --bom 选项。
    5. 增强 .filectrl 命令以支持 --schema 选项。
    6. UINT 排序序列 扩展会自动加载
  10. LIKE 运算符的 ESCAPE 子句现在会覆盖通配符字符，使其行为与 PostgreSQL 保持一致。

来自版本 3.32.1（2020-05-25）的先前更改：

  1. 修复两个长期存在的错误，这些错误允许恶意 SQL 语句使正在运行 SQLite 的进程崩溃。这些错误在 3.32.0 发布后约 24 小时由第三方公布，但并不限于 3.32.0 版本。
  2. 其他较小的编译器警告修复等。

来自版本 3.32.2（2020-06-04）的先前更改：

  1. 修复字节码引擎中的一个长期存在的错误，该错误可能导致 COMMIT 命令报告成功，而实际上提交失败。工单 810dc8038872e212
