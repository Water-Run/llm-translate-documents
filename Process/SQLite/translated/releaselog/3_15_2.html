SQLite Release 3.15.2 On 2016-11-28

来自版本 3.15.0（2016-10-14）的先前更改：

  1. 添加对 行值 的支持。
  2. 允许在 部分索引 的 WHERE 子句中使用 确定性 SQL 函数。
  3. 在 unix VFS 上添加了 "modeof=filename" URI 参数
  4. 添加对 SQLITE_DBCONFIG_MAINDBNAME 的支持。
  5. 增加对 VACUUM 一个 ATTACH 的数据库的能力。
  6. 对 命令行 shell 的增强：
    1. 添加 ".testcase" 和 ".check" 点命令。
    2. 在 ".open" 点命令中添加 --new 选项，使在打开之前清除数据库中的任何先前内容。
  7. 增强 fts5vocab 虚拟表以高效处理 "ORDER BY term"。
  8. 各种微优化在常见工作负载中将 CPU 使用率降低了 7% 以上。本次发布的大多数优化集中在前端 (sqlite3_prepare_v2())。

    错误修复：

  9. 乘法运算符现在在所有边缘情况下都能正确检测 64 位整数溢出并提升为浮点数。修复工单 1ec41379c9c1e400。
  10. 当这些列在 IN 运算符 的左侧使用时，含有冗余唯一索引的列的处理现在正确。修复工单 0eab1ac759。
  11. 在 表达式索引 的范围查询中跳过 NULL 条目。 修复工单 4baa46491212947。
  12. 确保 sqlite_sequence 表中的 AUTOINCREMENT 计数器在对 "INSERT ... SELECT" 语句执行 "Xfer 优化" 时被初始化。修复工单 7b3328086a5c116c。
  13. 确保 ORDER BY LIMIT 优化 （来自检查点 559733b09e） 能与 INTEGER PRIMARY KEY 上的 IN 运算符协同工作。修复工单 96c1454c

来自版本 3.15.1（2016-11-04）的先前更改：

  1. 添加 SQLITE_FCNTL_WIN32_GET_HANDLE 文件控制操作码。
  2. 修复 VACUUM 命令，使其将多余内容写入磁盘而不是全部保存在内存中，否则较大的数据库文件可能会导致内存不足错误。此修复解决了版本 3.15.0 引入的问题。
  3. 修复自 3.8.0（2013-08-26）以来存在的一个情况：LEFT JOIN 的 ON 子句中的 OR 连接项可能导致错误结果。工单 34a579141b2c5ac。
  4. 修复在 LEFT JOIN 的 ON 子句中使用 行值 可能导致错误结果的情况。工单 fef4bb4bd9185ec8f。

此特定补丁版本 3.15.2（2016-11-28）中的更改：

  1. 对版本 3.15.0 中引入的 行值 逻辑进行了多项错误修复。
  2. 修复在 ATTACH/DETACH 后因恶意构造的语法错误导致的 NULL 指针解引用。工单 2f1b168ab4d4844。
  3. 修复在内置 instr() 函数中发生内存不足情况后可能出现的崩溃。
  4. 在 JSON 扩展 中，修复 JSON 验证器以便正确拒绝字符串中无效的反斜杠转义。
