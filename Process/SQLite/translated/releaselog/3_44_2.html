SQLite 3.44.2 版本发布 于 2023-11-24

来自动作 3.44.0 版本（2023-11-01）之前的更改：

  1. 聚合函数 现在可以在最后一个参数之后包含 ORDER BY 子句。函数的参数会按照指定顺序进行处理。对于像 string_agg() 和 json_group_array() 这样的函数，这一点非常重要。
  2. 添加对 concat() 和 concat_ws() 标量 SQL 函数的支持，兼容 PostgreSQL、SQLServer 和 MySQL。
  3. 添加对 string_agg() 聚合 SQL 函数的支持，兼容 PostgreSQL 和 SQLServer。
  4. strftime() SQL 函数 新增转换字母：%e %F %I %k %l %p %P %R %T %u
  5. 添加新的 C 语言 API：sqlite3_get_clientdata() 和 sqlite3_set_clientdata()。
  6. 与 CREATE TABLE 关联的许多错误现在在执行 CREATE TABLE 语句时立即抛出，而不再延迟到实际第一次使用该表时。
  7. PRAGMA integrity_check 命令现在通过新的 xIntegrity 方法 检查各种内置 虚拟表 中内容的一致性。这适用于 FTS3、FTS4、FTS5、RTREE 和 GEOPOLY 扩展。
  8. SQLITE_DBCONFIG_DEFENSIVE 设置现在阻止 PRAGMA writable_schema 被启用。此前 writable_schema 可以被打开，但实际上不会让模式可写。现在它根本无法打开。
  9. 将内置的 FTS3、FTS4、FTS5、RTREE 和 GEOPOLY 虚拟表标记为 SQLITE_VTAB_INNOCUOUS，以便它们可以在高安全性部署中的触发器内部使用。
  10. PRAGMA case_sensitive_like 语句已弃用，因为在模式包含 LIKE 运算符的情况下使用该语句可能会导致 PRAGMA integrity_check 报告数据库损坏。
  11. SQLITE_USE_SEH（结构化异常处理）现在在使用 Microsoft C 编译器构建 SQLite 时默认启用。可以通过 -DSQLITE_OMIT_SEH 来禁用。
  12. 查询规划器优化：
    1. 在部分索引扫描中，如果 WHERE 子句为某个表列推导出常量值，则用该常量替换该表列的出现次数。这增加了部分索引成为覆盖索引的可能性。
    2. 禁用视图扫描优化（在 3.40.0 版本 —— 项目 3d 中添加）——因为它导致多个性能回退。取而代之的是，将 DISTINCT 子查询的估计行数减小 8 倍。
  13. SQLite 现在在运行时检测底层硬件是否支持精度高于 double 的 “long double”，并根据检测结果使用相应的浮点例程。
  14. 在支持的 Windows 平台上，命令行界面 现在默认对输入和输出都使用 UTF-8。可以使用 --no-utf8 选项来禁用 UTF8 支持。

来自动作 3.44.1 版本（2023-11-22）之前的更改：

  1. 更改 命令行界面，使其在 Windows 上的控制台 I/O 使用 UTF-16。这样可以在旧的 Windows7 机器上正确显示 Unicode 文本。
  2. 其他一些晦涩的错误修复。

本次补丁发布（3.44.2 版本，2023-11-24）中的更改：

  1. 修复了在 3.44.1 中（前述第 15 项）引入的命令行界面中的一个错误。
  2. 修复了在内部模糊测试中发现的仅在 3.44.1 发布后几分钟才发现的 FTS5 问题。
  3. 修复了模糊器在前一个版本发布后的次日发现的不完整 assert() 语句。
  4. 修复了在使用 GCC 16 的调试构建中出现的几个无害的编译器警告。
