SQLite 发行版本 3.8.4.3 于 2014-04-03

来自版本 3.8.4（2014-03-10）的先前更改：

  1. 进行代码优化和重构以提升性能。
  2. 向命令行工具添加“.clone”和“.save”命令。
  3. 更新命令行工具上的横幅，以提醒新手用户在使用临时内存数据库。
  4. 修复命令行中对 editline 的支持。
  5. 添加对使用 SQLITE_TESTCTRL_VDBE_COVERAGE 术语和 sqlite3_test_control() 进行 VDBE 程序覆盖测试的支持。
  6. 更新 _FILE_OFFSET_BITS 宏以便在 QNX 上重新工作。
  7. 将 SrcList.nSrc 的数据类型从 u8 更改为 int，以规避 AIX C 编译器的问题。
  8. 修复 Cygwin 上扩展加载。
  9. 错误修复：修复 char() SQL 函数，使其在零参数调用时返回空字符串，而不是“内存不足”错误。
  10. 错误修复：DISTINCT 现在识别 zeroblob 和全 0x00 字节的 blob 是相同的。Ticket [fccbde530a]
  11. 错误修复：在 WHERE 子句包含 IS NOT NULL 术语且也包含 OR 术语并使用 SQLITE_ENABLE_STAT4 编译时，计算查询的正确结果。Ticket [4c86b126f2]
  12. 错误修复：确保在普通表和 WITHOUT ROWID 表的连接中，“rowid” 列被正确解析。Ticket [c34d0557f7]
  13. 错误修复：确保在用于实现包含 ORDER BY 子句的复合 SELECT 语句的并发协程中不重复使用相同的临时寄存器，因为那样会导致错误结果。Ticket [8c63ff0eca]
  14. 错误修复：确保“ORDER BY random()” 子句不会被优化掉。Ticket [65bdeb9739]
  15. 错误修复：修复可能在触发器内的子查询中发生的名称解析错误。Ticket [4ef7e3cfca]
  16. 错误修复：修复形式为 “DEFAULT(-(-9223372036854775808))” 的列默认值表达式，使其正确工作，将列初始化为约等于 +9223372036854775808.0 的浮点值。

来自版本 3.8.4.1（2014-03-11）的先前更改：

  1. 规避 C 预处理宏冲突，该冲突会破坏某些使用 Microsoft Visual Studio 的构建。
  2. 在计算 跳过扫描优化 的代价时，考虑到需要多次查找。

来自版本 3.8.4.2（2014-03-26）的先前更改：

  1. 修复在搜索损坏的数据库文件时可能产生的缓冲区越界读取。

此补丁发布版本 3.8.4.3（2014-04-03）的更改：

  1. 增加了一个一字符修复，解决在查询中混合 DISTINCT、包含子查询中的 GROUP BY 以及 ORDER BY 时可能导致错误结果的问题。Ticket 98825a79ce14。

  3. sqlite3.c 的 SHA1: 310a1faeb9332a3cd8d1f53b4a2e055abf537bdc
