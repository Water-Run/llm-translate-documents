SQLite 3.38.2 发行于 2022-03-26

来自 3.38.0 版本（2022-02-22）的先前更改：

  1. 添加了 -> 与 ->> 运算符，以更方便处理 JSON。 这些新运算符与 MySQL 和 PostgreSQL 兼容。
  2. JSON 函数现在是内建的。无需再使用 -DSQLITE_ENABLE_JSON1 编译选项来启用 JSON 支持。 JSON 默认开启。使用新的 -DSQLITE_OMIT_JSON 编译选项可以禁用 JSON 接口。
  3. 对 日期与时间函数的增强：
    1. 添加了 unixepoch() 函数。
    2. 添加了 auto 修饰符 与 julianday 修饰符。
  4. 将 printf() SQL 函数 重命名为 format() 以提高兼容性。为了向后兼容，仍保留 printf() 名称作为别名。
  5. 添加了 sqlite3_error_offset() 接口，必要时可将 SQL 错误定位到输入 SQL 文本的特定字符， 以便应用程序提供更准确的错误提示。
  6. 对 虚拟表 的接口进行了增强，具体如下：
    1. 添加了 sqlite3_vtab_distinct() 接口。
    2. 添加了 sqlite3_vtab_rhs_value() 接口。
    3. 添加了新的运算符类型 SQLITE_INDEX_CONSTRAINT_LIMIT 与 SQLITE_INDEX_CONSTRAINT_OFFSET。
    4. 添加了 sqlite3_vtab_in() 接口（及相关接口），以便虚拟表能够一次性处理 IN 运算符的约束，而不是对 IN 运算符右侧的每个值分别处理。
  7. 命令行工具（CLI） 的增强：
    1. 列式输出模式增强后可正确处理文本中的制表符与换行符。
    2. 为 列式输出模式 添加了例如 "--wrap N"、"--wordwrap on" 与 "--quote" 等选项。
    3. 添加了 .mode qbox 别名。
    4. .import 命令能够自动区分列名。
    5. 使用新的 sqlite3_error_offset() 接口提供更友好的错误提示。
  8. 查询规划器增强：
    1. 使用布隆过滤器提升大规模分析查询的速度。
    2. 使用平衡归并树来执行带有 ORDER BY 子句的 UNION 或 UNION ALL 复合 SELECT 语句。
  9. ALTER TABLE 语句在 PRAGMA writable_schema=ON 时， 会静默忽略 sqlite_schema 表 中无法解析的条目。

来自 3.38.1 版本（2022-03-12）的先前更改：

  1. 修复了新的布隆过滤器优化在某些罕见查询中可能导致错误结果的问题。
  2. 修复了 localtime 修饰符，使 日期与时间函数可以保留秒的小数部分。
  3. 修复了 sqlite_offset SQL 函数，使其在参数是虚拟列或视图列等极端情况时仍可正常工作。
  4. 修复了 行值 IN 运算符对 虚拟表的约束，确保即便虚拟表实现依赖字节码来过滤不满足约束的行时也能正确运行。
  5. 对 assert() 语句、测试用例与文档的其他小修复。 详见 源码时间线。

此补丁版本 3.38.2（2022-03-26）的变更：

  1. 修复了用户发现的布隆过滤器新优化问题，该问题在执行带有 WHERE 子句约束的 LEFT JOIN 时可能导致错误结果， 约束要求 LEFT JOIN 右表的某一列为 NULL。详见 论坛贴 031e262a89b6a9d2。
  2. 其他小修补。详见 时间线。
