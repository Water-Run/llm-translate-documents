SQLite 3.19.2 版本于 2017-05-25 发布

（2017-05-22）3.19.0 版本之前的更改：

  1. 对于查询中未提取任何列的每个引用表，SQLITE_READ 授权回调 将使用空字符串列名调用一次。
  2. 当对表达式使用索引时，尽量重用索引中已有的表达式值，而不是加载原始列并重新计算表达式。
  3. 增强展开优化，使其能够展开 LEFT JOIN 右侧的视图。
  4. 在命令行 shell 的 .dump 输出中，使用 replace() 替代 char() 来转义嵌入的换行符和回车符。
  5. 在不涉及外键约束列的 UPDATE 语句中避免不必要的外键处理。
  6. 在使用索引的 DISTINCT 查询中，当有合适的索引时，尝试借助索引跳到下一个不同条目，而不是逐行遍历。
  7. 在修改无关表时避免不必要地失效 sqlite3_blob 句柄。
  8. 将 HAVING 子句中仅使用 GROUP BY 所列列的条件移到 WHERE 子句，以加速处理。
  9. 如果同一个查询中某个 VIEW 出现多次，则重用相同的视图物化。
  10. 增强PRAGMA integrity_check，使其能够识别具有两个或更多相同行rowid的表。
  11. 增强FTS5 查询语法，允许对任意表达式应用列过滤器。
  12. 增强json_extract() 函数，使其缓存并重用 JSON 输入文本的解析结果。
  13. 添加了 anycollseq.c 可加载扩展，使通用 SQLite 数据库连接可以读取包含未知和/或特定应用的排序序列的模式。

    错误修复：

  14. 修复 REPLACE 中的问题，避免导致数据库损坏并包含两个或多个相同行rowid的行。此修复对应工单 f68dc596c4e6018d。
  15. 修复 PRAGMA integrity_check 中的问题，该问题导致后续 VACUUM 表现不佳。
  16. 修复 PRAGMA foreign_key_check 命令，使其可以正确处理 WITHOUT ROWID 表上的外键。
  17. 修复 b 树逻辑中的错误，该错误可能导致 IN 运算符查询出现错误的重复答案。工单 61fe9745
  18. 禁止 JSON 数值常量有前导零。对应工单 b93be8729a895a528e2。
  19. 禁止 JSON 字符串中出现控制字符。对应工单 6c9b5514077fed34551。
  20. 限制 JSON 对象和数组的递归深度，以避免递归下降解析器的堆栈使用过多。对应工单 981329adeef51011052。

  2. 删除了导致旧版 MSVC 出现问题的多余分号。
