SQLite 3.34.1 版 发布于 2021-01-20

在 3.34.0 版本（2020-12-01）之前的更改：

  1. 添加了 sqlite3_txn_state() 接口，用于报告数据库连接的当前事务状态。
  2. 增强 递归公共表表达式 ，支持两个或更多递归项（如 SQL Server 所做），这有助于使 针对图的查询 更易编写且执行更快。
  3. 改进了 CHECK 约束 失败时的错误消息。
  4. CLI 增强：
    1. .read 点命令现在除了接受文件名外还支持管道。
    2. 在 .dump 点命令中添加了 --data-only 和 --nosys 选项。
    3. 在 .schema 点命令中添加了 --nosys 选项。
    4. .import 点命令现在可以正确引用表名。
    5. generate_series(START,END,STEP) 表值函数扩展现在已内置在 CLI 中。
    6. .databases 点命令现在根据 sqlite3_db_readonly() 和 sqlite3_txn_state() 确定的状态显示每个数据库文件的状态。
    7. 添加了 --tabs 命令行选项，该选项设置 .mode tabs。
    8. 如果 --init 选项的参数所指的文件无法打开，则会报告错误。--init 选项现在也会遵守 --bail 选项。
  5. 查询规划器改进：
    1. 改进了 DISTINCT 运算符运行成本的估算。
    2. 在使用多列索引执行 UPDATE 或 DELETE 且索引的前几列只有少数对索引查找有用时，先评估完所有 WHERE 子句约束再执行主表查找，以便将这些约束与索引中未使用的后续列匹配，从而避免不必要的主表查找。
    3. 当后续列受到 IN 操作符限制时，新的 OP_SeekScan 操作码用于改进多列索引查找的性能。
  6. 即使一个或多个附加的数据库文件为只读，BEGIN IMMEDIATE 和 BEGIN EXCLUSIVE 命令现在仍然有效。
  7. 增强了 FTS5 以支持 三元组索引。
  8. 在同时有数百个连接并发访问相同数据库文件的情况下，WAL 模式 的锁定原语性能得到提升。
  9. 增强了 carray() 表值函数，使其包含一个通过辅助的 sqlite3_carray_bind() 接口绑定的单参数形式。
  10. substr() SQL 函数现在也可以称为 “substring()”，以兼容 SQL Server。
  11. 语法图 现已实现为 Pikchr 脚本，并渲染为 SVG，以提高可读性和维护便利性。

本次补丁发布，版本 3.34.1（2021-01-20）的更改：

  1. 修复处理同时具有相关 WHERE 子句、\"HAVING 0\" 子句且父查询为聚合查询的子查询时可能出现的使用后释放漏洞。
  2. 修复文档错别字。
  3. 修复扩展中的细微问题。
