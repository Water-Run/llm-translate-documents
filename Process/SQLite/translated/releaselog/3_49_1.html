SQLite 3.49.1 发布于 2025-02-18

来自 3.49.0 版（2025-02-06）的先前更改：

  1. 对查询规划器的增强：
    1. 改进查询时索引优化，使其可在WITHOUT ROWID表上生效。
    2. 为大型星型查询连接生成更优的查询计划。这修复了在SQLite论坛报告的三种不同性能回归。
    3. 当两个或多个查询估计成本相同时，选择每行字节更少的那个。
  2. 增强iif() SQL 函数，使其可接受任意大于或等于两个的参数数量。
  3. 增强会话扩展，使其可在使用生成列的数据库上工作。
  4. 删除未正确实现且从未正常工作的 SQLITE_USE_STDIO_FOR_CONSOLE 编译时选项，改为添加 SQLITE_USE_W32_FOR_CONSOLE_IO 编译时选项。此选项仅适用于类似CLI的命令行工具，而非 SQLite 核心。它使控制台I/O使用 Win32 API 而非 stdio。该选项仅影响 Windows 构建。
  5. 为sqlite3_db_config() 添加三个新选项，默认全开。
    1. SQLITE_DBCONFIG_ENABLE_ATTACH_CREATE
    2. SQLITE_DBCONFIG_ENABLE_ATTACH_WRITE
    3. SQLITE_DBCONFIG_ENABLE_COMMENTS
  6. 将Autotools替换为Autosetup，用于预编译汇编包中使用的 configure 脚本。规范源代码的 configure 脚本在前一个（3.48.0）版本中已经改为 Autosetup。仅汇编包中的主 SQLite configure 脚本更改。汇编包中TEA 子目录所使用（已弃用）的配置脚本仍旧依赖 Autotools。
  7. 解决 3.48.0 发行版中发现的各种小型补丁与修复。

该补丁版本 3.49.1（2025-02-18）中的更改：

  1. 提高 Makefile 和 configure 脚本的可移植性。
  2. 修复concat_ws() 函数中的一个bug（在3.44.0 版中引入），如果分隔符字符串大于两兆字节，可能导致内存错误，具体表现为写出分配空间之外。
  3. 增强了SQLITE_DBCONFIG_LOOKASIDE接口，使其更能抵御误用。
