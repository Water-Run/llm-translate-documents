SQLite 3.7.15 版本发布 于 2012-12-12

  1. 添加了 sqlite3_errstr() 接口。
  2. 在由于 SQLITE_SCHEMA 错误自动重新编译语句时，避免多次调用 sqlite3_trace() 回调。
  3. 添加了对 Windows Phone 8 平台的支持。
  4. 增强了 IN 运算符处理，以便利用具有数值亲和力的索引。
  5. 在可能的情况下使用覆盖索引对整个表进行扫描，基于索引体积更小、I/O 更少的理论。
  6. 改进查询优化器，使 ORDER BY 子句更加积极地得到优化，尤其是在连接中 ORDER BY 子句的各项来自连接的不同表时。
  7. 添加了将 FROM 子句子查询实现为协程而不是将子查询体现为临时表的能力。
  8. 增强了命令行外壳：
    1. 添加了 ".print" 命令
    2. ".width" 命令中使用负数时会右对齐
    3. 在以 SQLITE_DEBUG 编译时添加了 ".wheretrace" 命令
  9. 添加了 busy_timeout pragma。
  10. 添加了 instr() SQL 函数。
  11. 添加了 SQLITE_FCNTL_BUSYHANDLER 文件控制，用于允许 VFS 实现访问繁忙处理程序回调。
  12. 内置 VFS 的 xDelete 方法在被删除的文件不存在时现在返回 SQLITE_IOERR_DELETE_NOENT。
  13. 增强了对 QNX 的支持。
  14. 在针对 ARM 的 MSVC 编译器中绕过优化器错误。
  15. 错误修复：避免在 共享缓存模式 中的各种并发问题。
  16. 错误修复：在同时使用 备份 API、共享缓存 和 SQLite 加密扩展时避免死锁或崩溃。
  17. 错误修复：使用 TCL 接口创建的 SQL 函数尊重 "nullvalue" 设置。
  18. 错误修复：修复在数据库超过 16GB 时创建索引发生的 32 位溢出问题。
  19. 错误修复：在 共享缓存模式 中，在 CHECK 约束 或 视图 内使用 COLLATE 运算符 时避免出现段错误。
  20. SQLITE_SOURCE_ID： "2012-12-12 13:36:53 cd0b37c52658bfdf992b1e3dc467bae1835a94ae"
  21. sqlite3.c 的 SHA1：2b413611f5e3e3b6ef5f618f2a9209cdf25cbcff"
