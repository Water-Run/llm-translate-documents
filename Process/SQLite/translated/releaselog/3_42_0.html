SQLite 3.42.0 版本 发布于 2023-05-16

  1. 添加 FTS5 安全删除命令。此选项在内容被删除时会从 FTS5 倒排索引中移除所有取证痕迹。
  2. 增强 JSON SQL 函数 以支持 JSON5 扩展。
  3. 现在允许 在 sqlite3_initialize() 之后 调用 SQLITE_CONFIG_LOG 和 SQLITE_CONFIG_PCACHE_HDRSZ 这两个 sqlite3_config() 的调用。
  4. 新增 sqlite3_db_config() 选项：SQLITE_DBCONFIG_STMT_SCANSTATUS 与 SQLITE_DBCONFIG_REVERSE_SCANORDER。
  5. 查询优化器改进：
    1. 默认启用“视图计数”优化。
    2. 避免在子查询中计算未使用的列。
    3. 改进 WHERE 子句下推优化。
  6. 对 CLI 的增强：
    1. 添加 --unsafe-testing 命令行选项。没有此选项，某些点命令（例如 ".testctrl"）现已被禁用，因为这些命令仅用于测试，误用可能导致故障。
    2. 即使在 --safe 模式下，也允许使用 ".log on" 和 ".log off" 命令。
    3. 将 "--" 作为命令行参数意味着所有后续以 "-" 开头的参数都被解释为普通非选项参数。
    4. 魔术参数 ":inf" 和 ":nan" 分别绑定到浮点字面量 Infinity 和 NaN。
    5. --utf8 命令行选项在交互式会话期间省略 Windows 控制台与 MBCS 之间的所有转换，并在此类会话期间为 UTF-8 I/O 设置控制台代码页。--utf8 选项在其他平台上无效。
  7. 允许 应用程序定义的 SQL 函数 使用与连接关键字相同的名称：CROSS、FULL、INNER、LEFT、NATURAL、OUTER 或 RIGHT。
  8. 对 PRAGMA integrity_check 的增强：
    1. 当 NaN 值存储在 NOT NULL 列中时，检测并抛出错误。
    2. 改进的错误信息输出可以在 b 树内部发现错误时识别其根页面。
  9. 允许 session 扩展 被配置为捕获缺少显式 ROWID 的表的更改。
  10. 为 日期和时间函数 添加了 subsecond 修饰符。
  11. 传递给 sqlite3_sleep() 的负值从此被解释为 0。
  12. JSON 数组和对象的最大递归深度已从 2000 降至 1000。
  13. 扩展了内置 printf() 函数，使 comma 选项 现在在浮点转换中也有效，除了整数转换之外。
  14. 零星错误修复和性能优化
