SQLite 发行版 3.41.0 — 2023-02-21

  1. 查询规划器改进：
    1. 在包含 GROUP BY 子句的聚合查询中利用索引表达式。
    2. 查询规划器更好地意识到某个索引是否为覆盖索引，并相应地调整预测执行时间。
    3. 查询规划器更加积极地使用协程，而不是物化子查询和视图。
    4. 针对内置表值函数 json_tree() 与 json_each() 的查询现在通常会将“ORDER BY rowid”视为无操作。
    5. 即使表达式已被常量传播优化修改，也增强了查询规划器使用索引表达式的能力。（参见论坛线程 0a539c7。）
  2. 添加内置unhex() SQL 函数。
  3. 将 base64 与 base85 应用定义函数作为扩展，并在CLI中包含该扩展。
  4. 添加sqlite3_stmt_scanstatus_v2() 接口。（仅在使用SQLITE_ENABLE_STMT_SCANSTATUS编译 SQLite 时可用。）
  5. 使用sqlite3_deserialize() 创建的内存数据库现在将其文件名报告为空字符串，而不是“x”。
  6. 对CLI的更改：
    1. 添加新的 base64() 与 base85() SQL 函数。
    2. 使用新的sqlite3_stmt_scanstatus_v2()接口增强EXPLAIN QUERY PLAN输出（前提是使用SQLITE_ENABLE_STMT_SCANSTATUS编译）。
    3. “.scanstats est”命令在分析资料中提供查询规划器估算。
    4. 延续提示会指示当前输入是否处于字符串文字、标识符文字、注释、触发器定义等内部。
    5. 加强了 --safe 命令行选项以禁止危险的 SQL 函数。
    6. CLI 构建默认禁用双引号字符串异常特性。旧的用例可以在运行时通过“.dbconfig dqs_dml on”与“.dbconfig dqs_ddl on”命令重新启用该特性。
  7. 增强PRAGMA integrity_check 命令，以检测表中的文本字符串与索引中相同字符串在字节级别不完全相同时的情况。
  8. 增强carray 表值函数，使其能够绑定 BLOB 对象数组。
  9. 添加了sqlite3_is_interrupted() 接口。
  10. 耗时较长的sqlite3_prepare() 等调用现在会调用进度处理回调，并响应sqlite3_interrupt()。
  11. sqlite3_vtab_in_first() 与 sqlite3_vtab_in_next() 函数得到增强，能够可靠地检测其是否在未通过sqlite3_vtab_in()进行多值 IN 处理的参数上调用。此情况下它们将返回 SQLITE_ERROR 而不是 SQLITE_MISUSE。
  12. 解析器现在忽略 IN 运算符右侧子查询外层的多余括号，因此在这方面 SQLite 的行为与 PostgreSQL 保持一致。此前，SQLite 将该子查询视为带隐含“LIMIT 1”的表达式。
  13. 为sqlite3_file_control() API 添加SQLITE_FCNTL_RESET_CACHE 选项。
  14. Makefile 改进：
    1. 新的 makefile 目标“sqlite3r.c”构建包含恢复扩展的融合体。
    2. 新增 makefile 目标“devtest”与“releasetest”，分别用于提交前运行快速开发测试与进行完整发布测试。
  15. 其他性能增强。
