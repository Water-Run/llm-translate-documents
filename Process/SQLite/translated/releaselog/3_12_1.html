SQLite 3.12.1 版于 2016-04-08 发布

从 3.12.0 版（2016-03-29）以来的先前更改：

    潜在破坏性更改：

  1. SQLITE_DEFAULT_PAGE_SIZE 从 1024 提高到 4096。 SQLITE_DEFAULT_CACHE_SIZE 从 2000 改为 -2000， 因此默认使用相同数量的缓存内存。 有关更多信息，请参阅 3.12.0 版页面大小更改 的应用说明。

    性能增强：

  2. 增强了 Lemon 解析器生成器， 以便生成更小更快的 SQL 解析器。
  3. 仅当两个或更多附加数据库全部被修改、且未将 PRAGMA synchronous 设置为 OFF， 并且未将 journal_mode 设置为 OFF、MEMORY 或 WAL 时， 才创建 主日志 文件。
  4. 仅在线程日志大小超过阈值时才创建 语句日志 文件。 否则日志将保存在内存中，不会发生 I/O。该阈值可以在编译时通过 SQLITE_STMTJRNL_SPILL 或在启动时通过 sqlite3_config(SQLITE_CONFIG_STMTJRNL_SPILL) 配置。
  5. 查询规划器能够优化 虚拟表 上的 IN 操作， 即使 xBestIndex 方法并未将 虚拟表在 IN 操作左侧列的 sqlite3_index_constraint_usage.omit 标志置位。
  6. 查询规划器现在能更好地优化在三表或更多联结中涉及 虚拟表 的访问， 即当对虚拟表的约束跨多个联结表时也能高效处理。
  7. 对 应用自定义 SQL 函数 的处理更高效，尤其是在应用定义数百或数千个自定义函数的情况下。
  8. 查询规划器在估算 ORDER BY 的代价时会考虑 LIMIT 子句。
  9. 配置脚本（在 unix 上）会自动检测 pread() 和 pwrite()，如果可用则设置编译时选项以使用这些操作系统接口。
  10. 减少保存模式所需的内存量。
  11. 其他杂项微优化以改善性能并减少内存使用。

    新功能：

  12. 新增 SQLITE_DBCONFIG_ENABLE_FTS3_TOKENIZER 选项到 sqlite3_db_config()， 允许在运行时启用或禁用带有两个参数的 fts3_tokenizer() SQL 函数。
  13. 为 RBU 扩展添加了 sqlite3rbu_bp_progress() 接口。
  14. 现在 PRAGMA defer_foreign_keys=ON 语句也会禁用外键上的 RESTRICT 操作。
  15. 新增 sqlite3_system_errno() 接口。
  16. 新增 SQLITE_DEFAULT_SYNCHRONOUS 和 SQLITE_DEFAULT_WAL_SYNCHRONOUS 编译时选项。 SQLITE_DEFAULT_SYNCHRONOUS 编译时选项替代了已不再支持的 SQLITE_EXTRA_DURABLE 选项。
  17. 增强了 命令行 shell 中的 “.stats” 命令，在可用时显示来自 /proc 的更多 I/O 性能信息。

    错误修复：

  18. 确保在单个语句内来自多个触发器的 sqlite3_set_auxdata() 值不会相互干扰。 问题追踪 dc9b1c91。
  19. 修复类似 “x IN (SELECT...)” 的表达式的代码生成器， 当右侧的 SELECT 语句是相关子查询时。 问题追踪 5e3c886796e5512e。
  20. 修复与 sqlite3_db_readonly() 接口相关的无害 TSAN 警告。

此特定补丁版本 3.12.1（2016-04-08）中的更改：

  1. 修复 3.12.0 版引入的边界条件错误， 该错误在大量使用 SAVEPOINT 时可能导致崩溃。 问题追踪 7f7f8026eda38。
  2. 修复 视图，使其在可能的情况下继承其定义所依据表的列数据类型。
  3. 修复查询规划器，使 IS 与 IS NULL 操作符能够驱动左外联结上的索引。
