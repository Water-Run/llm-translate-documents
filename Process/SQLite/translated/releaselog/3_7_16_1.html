SQLite 3.7.16.1 版本发布于 2013-03-29

  1. 新增了 PRAGMA foreign_key_check 命令。
  2. 为所有 SQLITE_CONSTRAINT 错误添加了新的扩展错误代码
  3. 添加了 SQLITE_READONLY_ROLLBACK 扩展错误代码，用于数据库因只读且需要回滚恢复而无法打开时。
  4. 新增 SQL 函数 unicode(A) 和 char(X1,...,XN)。
  5. 在 PRAGMA incremental_vacuum 的性能上进行了改进，尤其是在空闲页数超过 freelist 单个主干页可容纳数量的情况下。
  6. 改进了包含聚合 min() 或 max() 的查询的优化。
  7. 增强了虚拟表，使其在 WHERE 子句包含 IN 运算符时也可能使用索引。
  8. 允许即便索引的先前字段在 WHERE 子句中被 IN 运算符约束，也能使用该索引进行排序。
  9. 增强 PRAGMA table_info 命令，使得 “pk” 列为递增整数，以显示主键中列的顺序。
  10. 增强查询优化器以利用传递性连接约束。
  11. 查询优化器的性能改进。
  12. 允许 PRAGMA integrity_check 返回的错误信息超过 20000 字节。
  13. 改进了深度嵌套查询的名称解析。
  14. 添加了 test_regexp.c 模块作为实现 REGEXP 运算符的示例。
  15. 改进了 RTREE 扩展中的错误信息。
  16. 增强 命令行 shell，使得对 ".exit" 命令传入非零参数时，shell 会立即退出而不需正常关闭数据库连接。
  17. 改进了 命令行 shell 中对 dot 命令无效布尔参数的错误信息。
  18. 改进了 “foreign key mismatch” 的错误信息，显示涉及的两个表的名称。
  19. 移除了 unix VFS 中所有对 umask() 的使用。
  20. 添加了 PRAGMA vdbe_addoptrace 和 PRAGMA vdbe_debug 命令。
  21. 在比较非零结尾字符串时改为使用 strncmp() 或等价函数代替 memcmp()。
  22. 更新了 cygwin 接口以省略不推荐使用的 API 调用。
  23. 增强 spellfix1 扩展，使编辑距离成本表可以通过在 “command” 字段中插入类似 'edit_cost_table=TABLE' 的字符串在运行时更改。
  24. 错误修复：修复了一个长期存在的问题，该问题可能导致在三表或更多表的连接中，在多个位置将 INTEGER 字段与 TEXT 字段比较时产生错误的查询结果。 工单 fc7bd6358f
  25. 错误修复：如果因过于复杂的查询而导致视图上的 16 位引用计数器溢出，则会发出错误信息。
  26. 错误修复：在深度嵌套的 UNION ALL 查询中，避免对 LIMIT 和 OFFSET 子句造成内存泄漏。
  27. 错误修复：确保在运行 PRAGMA table_info、index_list、index_info 和 foreign_key_list 之前架构是最新的。

  1. 修复了在 3.7.15 版本 中引入的 ORDER BY 优化器的一个 bug，该 bug 有时会在实际上需要排序时优化掉排序步骤。 工单 a179fe7465
  2. 修复了 CAST 表达式 中的一个长期 bug，该 bug 会将 UTF16 字符识别为数字，即便其最高有效字节不为零。 工单 689137afb6da41。
  3. 修复了在 FTS3 中应用于子字段的 NEAR 运算符的一个 bug。 工单 38b1ae018f。
  4. 修复了存储引擎中的一个长期 bug，该 bug（极少情况下）会错误地报告 SQLITE_CORRUPT 错误，但除此之外无害。 工单 6bfb98dfc0c。
  5. 已移除 SQLITE_OMIT_MERGE_SORT 选项。现在合并排序器是 SQLite 的必需组件。
  6. 修复了源代码注释中的大量拼写错误
