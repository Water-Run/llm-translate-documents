SQLite Release 3.7.16 On 2013-03-18

  1. 新增 PRAGMA foreign_key_check 命令。
  2. 为所有 SQLITE_CONSTRAINT 错误添加新的扩展错误代码。
  3. 因数据库需要回滚恢复但处于只读状态无法打开时，新增 SQLITE_READONLY_ROLLBACK 扩展错误代码。
  4. 新增 SQL 函数 unicode(A) 和 char(X1,...,XN)。
  5. 对 PRAGMA incremental_vacuum 进行了性能优化，尤其在空闲页数量大于 freelist 单个主干页可容纳的情况下。
  6. 优化包含聚合 min() 或 max() 的查询。
  7. 增强虚拟表，使其在 WHERE 子句包含 IN 操作符时也可能使用索引。
  8. 即使索引的前置项在 WHERE 子句中被 IN 操作符限制，也允许使用该索引进行排序。
  9. 增强 PRAGMA table_info 命令，使 “pk” 列为递增整数以显示主键中列的顺序。
  10. 增强查询优化器以利用可传递的连接约束。
  11. 查询优化器的性能提升。
  12. 允许 PRAGMA integrity_check 返回长度超过 20000 字节的错误信息。
  13. 改进对深度嵌套查询的名称解析。
  14. 新增 test_regexp.c 模块，展示如何实现 REGEXP 运算符。
  15. 改进 RTREE 扩展中的错误信息。
  16. 增强命令行 shell，使对“.exit”命令传入非零参数时立即退出而不进行数据库连接的正常关闭。
  17. 改进命令行 shell中对点命令无效布尔参数的错误信息。
  18. 改进“外键不匹配”错误信息，显示所涉及的两个表的名称。
  19. 在 unix VFS 中删除所有对 umask() 的使用。
  20. 新增 PRAGMA vdbe_addoptrace 和 PRAGMA vdbe_debug 命令。
  21. 在比较非零终止字符串时改用 strncmp() 或等价函数替代 memcmp()。
  22. 更新 cygwin 接口以省略已弃用的 API 调用。
  23. 增强 spellfix1 扩展，使通过在 “command” 字段插入类似 “edit_cost_table=TABLE” 的字符串可以在运行时更改编辑距离成本表。
  24. 错误修复：修复长期存在的问题，在进行 3 表或更多表连接、并在多个位置将 INTEGER 字段与 TEXT 字段比较时可能导致查询结果不正确。 工单 fc7bd6358f
  25. 错误修复：当视图引用计数器因过于复杂的查询而溢出到 16 位时，会发出错误信息。
  26. 错误修复：避免在深度嵌套的 UNION ALL 查询中的 LIMIT 和 OFFSET 子句泄露内存。
  27. 错误修复：确保在运行 table_info、index_list、index_info 和 foreign_key_list pragma 之前模式是最新的。
