SQLite 3.5.4 版本发布于 2007-12-14

  1. 修复 UPDATE 或 DELETE 中的一个严重错误，当 OR REPLACE 子句或触发器导致同一表中的行作为副作用被删除时会出现该错误。（见工单 #2832。）该错误最可能的结果是段错误，尽管数据库损坏也是可能的。
  2. 使 ORDER BY 的处理在结果别名与表列名冲突的情况下符合 SQL 标准。正确的行为是优先使用结果别名。旧版本的 SQLite 错误地选择了表列。（见工单 #2822。）
  3. VACUUM 命令保留了 legacy_file_format pragma 的设置。 （工单 #2804。）
  4. 将 group_concat() SQL 函数产品化并正式支持。
  5. 更好地优化了一些 IN 运算符表达式。
  6. 增加了通过设置 auto_vaccum pragma 并对数据库执行 VACUUM 来更改数据库的 auto_vacuum 状态的能力。
  7. FTS3 中的前缀搜索效率大大提高。
  8. 放宽命令行界面中 SQL 语句长度的限制，以便包含非常大的 BLOB 和字符串的数据库的“.dump”输出能够重新播放以重建数据库。
  9. 其它小型错误修复和优化。

一个包含 SQLite 发布版本完整列表 的单页和 年表 也可用。 每次提交的详细历史记录可在 SQLite 版本控制站点 获取。
