1. 概述

百分位扩展提供四个聚合函数，用于计算某个分布的百分位得分和/或中位数值。

1.1. 可用性

百分位扩展自 SQLite 3.51.0（2025-11-04）开始就包含在合并源中。然而，该扩展默认被禁用，必须在编译时使用-DSQLITE_ENABLE_PERCENTILE 选项启用。

在 SQLite 3.51.0 之前，百分位扩展位于一个单独的源代码文件中，需要独立编译，并作为可加载扩展处理。

2. 百分位扩展实现的聚合函数

百分位扩展实现了下文所述的聚合 SQL 函数。所有这些函数所使用的算法在空间上为 O(N)，在时间上为 O(NlogN)，其中 N 是非 NULL 输入的数量。

2.1. median(Y) 聚合函数

median(Y) 函数是一个聚合函数，用于计算所有非 NULL 输入 Y 的中位数。如果传入 median() 的任何 Y 输入既非 NULL 又非数值，则会发生错误。如果没有非 NULL 的数值输入，则 median() 的结果为 NULL。

当所有输入排序且输入数量为奇数时，中位数就是中间元素的值。如果输入数量为偶数，则中位数为两个中间输入的平均值。

median(Y) 函数等价于 percentile(Y,50)。

2.2. percentile(Y,P) 聚合函数

percentile(Y,P) 聚合函数计算一个结果 X，该值大于或等于 P 百分比的非 NULL 输入，并且小于或等于 100-P 百分比的输入。参数 P 必须是介于 0.0 到 100.0 之间的数字。P 的值在聚合中的所有项必须相同，且不能为 NULL。Y 输入必须为 NULL 或数值。Y 的 NULL 值会被忽略。任何非 NULL 的 Y 输入如果不是数值，就会引发错误。

percentile() 函数的工作方式是先对非 NULL 输入排序，然后计算从首项到末项最接近 P 百分比的输入。返回值是两个最接近输入的加权平均。

2.3. percentile_cont(Y,P) 聚合函数

percentile_cont(Y,P) 的工作方式与 percentile(Y,P) 类似，只是 P 的取值范围为 0.0 到 1.0 而不是 0.0 到 100.0。因此 percentile_cont(Y,P) 的结果与 percentile(Y,P*100) 相同。

percentile_cont() 函数是 SQL 标准定义的。然而，它并不是简单地写成 "percentile_cont(Y,P)"，而是符合 SQL 标准的语法如下：

SELECT percentile_cont(P) WITHIN GROUP (ORDER BY Y) FROM tab;

这段语法相当冗长，其实表达的意思与下列语法完全相同：

SELECT percentile_cont(Y,P) FROM tab;

SQLite 会支持 SQL 标准语法，但前提是从标准源代码（而非合并源）编译，并使用 -DSQLITE_ENABLE_ORDERED_SET_AGGREGATES=1 编译选项。若不启用该编译选项，则只支持更简单的 "percentile_cont(Y,P)" 形式。鉴于该冗长的 SQL 标准格式并无优势、可读性又极差，且 SQLITE_ENABLE_ORDERED_SET_AGGREGATES 编译选项会使 SQLite 库变大，因此大多数构建都省略了该选项。

本文作者认为该函数名中的 "_cont" 后缀是 "continuous"（连续）的缩写，表明返回值是与实际百分位数最接近的两个输入值的加权平均。这一名称是 SQL 标准，而非 SQLite 开发者自行命名。

2.4. percentile_disc(Y,P) 聚合函数

percentile_disc(Y,P) 的工作方式类似于 percentile_cont(Y,P)，只不过它不是对最近的输入做加权平均，而是始终返回输入值中的某一个——两个候选值中的较小者。percentile_disc(Y,P) 函数同样由 SQL 标准定义。和 percentile_cont() 一样，该冗长的有序集合聚合语法是必须的，但 SQLite 仅在使用 SQLITE_ENABLE_ORDERED_SET_AGGREGATES 编译选项编译时才支持该语法。

本文作者认为该函数名中的 "_disc" 后缀是 "discrete"（离散）的缩写。这个名称是 SQL 标准，而非 SQLite 开发者自行命名。

3. 设计要求

以下要求定义了百分位扩展。

  1. percentile(Y,P) 函数是一个恰好接受两个参数的聚合函数。

  2. 如果 percentile(Y,P) 中的 P 参数在聚合的每一行中不一致，则会抛出错误。前一句中的“相同”意味着值的差异小于 0.001。

  3. 如果 percentile(Y,P) 的 P 参数计算结果不是 0.0 到 100.0（含）范围内的数字，则会抛出错误。

  4. 如果任何传入 percentile(Y,P) 的 Y 参数计算结果非 NULL 且非数值，则会抛出错误。

  5. 如果任何传入 percentile(Y,P) 的 Y 参数计算结果为正无穷或负无穷，则会抛出错误。（SQLite 始终将 NaN 值解释为 NULL。）

  6. percentile(Y,P) 中的 Y 与 P 均可以是任意表达式，包括 CASE WHEN 表达式。

  7. percentile(Y,P) 聚合最多可处理至少一百万（1,000,000）行输入。

  8. 如果 Y 没有非 NULL 值，则 percentile(Y,P) 返回 NULL。

  9. 如果 Y 只有一个非 NULL 值，则 percentile(Y,P) 返回该唯一的 Y 值。

  10. 如果 Y 有 N 个非 NULL 值（N≥2），并且将 Y 值按从小到大排序，然后从 0 到 N-1 绘制一张图表，使得图表在位置 J 的高度为第 J 个 Y 值，并在相邻的 Y 值之间画直线，则 percentile(Y,P) 函数返回图表在 P*(N-1)/100 处的高度。

  11. percentile(Y,P) 函数始终返回浮点数或 NULL。

  12. percentile(Y,P) 的实现是一个单个的 C99 源文件，可编译为共享库或 DLL，并通过 sqlite3_load_extension() 接口加载到 SQLite 中。

  13. 单独的 median(Y) 函数等价于 percentile(Y,50)。

  14. 单独的 percentile_cont(Y,P) 函数返回的结果等同于 percentile(Y,P*100.0)。换言之，percentile_cont() 的第二个参数的分数值应在 0 到 1 之间，而 percentile() 的参数在 0 到 100 之间。

  15. 单独的 percentile_disc(Y,P) 函数类似于 percentile_cont(Y,P)，但它不是返回最近两个输入值的加权平均，而是返回下一个较低的值。因此 percentile_disc(Y,P) 始终返回某个输入值。

  16. median()、percentile(Y,P)、percentile_cont(Y,P) 与 percentile_disc(Y,P) 均可用作窗口函数。
