1.0 SQLite 要求简介

  • 要求由文档中的摘录组成。要求通常是一个句子的摘录，但也可能是一个句子片段、多个句子、一个表格，或一个气泡语法图的 GIF 图像。

  • 要求使用口语化英语编写，并且不使用情态助动词“shall”。这源于要求取自文档。文档的目标读者是应用程序开发人员。“shall”语言适合合同专家、质量审计员和律师，但在面对应用程序开发人员时会妨碍理解。因此，为了最好地服务于目标读者，省略了“shall”语言。

  • 要求足够详细和精确，以便能够进行与 SQLite 完全兼容的洁净室重新实现。

  • 普通英语中“requirement”一词暗示一种顺序：要求先于实现。但 SQLite 要求中并不存在这样的顺序。SQLite 所谓的“要求”更准确地描述为“关于系统行为的可测试事实陈述”。

  • 文档中关于 SQLite 的每一个可测试事实陈述都成为一个要求。

  • 要求编号是该要求本身的 MD5 哈希值。

    1. 要求本质上是不可变的，因为对要求的任何更改都会导致完全不同的要求编号。

    2. 对于文本要求，在计算 MD5 哈希之前会对文本进行规范化：

      • 删除所有前导和尾部空白。
      • 将所有内部空白序列转换为一个空格字符。
      • 将“&lt;”转换为“<”、“&gt;”转换为“>”、“&#91;”转换为“[”、“&#93;”转换为“]”，以及“&amp;”转换为“&”。

    3. 对于 GIF 语法图要求，MD5 哈希是对整个 GIF 图像文件的内容计算的。

    4. MD5 哈希以人类可读的形式表示如下：

      R-N-N-N-N-N-N-N-N
      其中每个 N 是介于 00000 和 65536 之间的 5 位数字，表示 128 位 MD5 哈希的 16 位。

    5. 可以通过其完整要求编号中的任何唯一前缀引用要求。

  • 文档中的单个文本要求标识为位于“^”与第一个句号或“.”之间的文本，或位于“^(”与“)^”之间的文本。

    1. 文本要求由作为文档构建过程一部分运行的脚本自动从文档中提取。

    2. 从文档提取要求后，要求标记“^”、“^(”和“)^”会从文档文本中移除。这由文档构建脚本自动完成。

    3. 为避免与这些要求分隔符冲突，属于要求文本或其他出现在文档中的“^”字符应编码为“&#94;”。

  • 文档中的单个 GIF 语法图要求标识为 HTML 图像标记，格式为
    <img alt="syntax diagram NAME" src="FILE">
    其中 NAME 是语法图的名称，FILE 是包含语法图的 GIF 文件名。

    1. 语法图要求由作为文档构建过程一部分运行的脚本自动从文档中提取。

    2. GIF 文件才是要求，而不是引用 GIF 文件的 HTML 标记或图的名称。

  • 包含要求文本的文档由脚本生成，这些脚本的输入来自文档 fossil 仓库中的文件以及源代码中的注释。
