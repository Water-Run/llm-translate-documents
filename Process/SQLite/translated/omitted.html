SQLite 尚未实现的 SQL 功能

SQLite 实现了 SQL 大多数常见功能。与其尝试列出 SQLite 支持的所有 SQL 功能，不如更容易列出它未实现的功能。下面列出了不受支持的 SQL 功能。

另请参见 SQLite 的怪癖、注意事项和陷阱。

完整的 ALTER TABLE 支持  ALTER TABLE 命令只支持 RENAME TABLE、ADD COLUMN、RENAME COLUMN 和 DROP COLUMN 变体。其他类型的 ALTER TABLE 操作如 ALTER COLUMN、ADD CONSTRAINT 等则未实现。
完整的触发器支持  支持 FOR EACH ROW 触发器，但不支持 FOR EACH STATEMENT 触发器。
向视图写入  SQLite 中的 VIEW 是只读的。不能在视图上执行 DELETE、INSERT 或 UPDATE 语句。但可以创建在尝试 DELETE、INSERT 或 UPDATE 视图时触发的触发器，并在触发器体内完成所需的操作。
GRANT 与 REVOKE  由于 SQLite 读取和写入普通磁盘文件，因此只能应用底层操作系统的正常文件访问权限。客户端/服务器 RDBMS 中常见的 GRANT 与 REVOKE 命令未实现，因为对于嵌入式数据库引擎而言毫无意义。
