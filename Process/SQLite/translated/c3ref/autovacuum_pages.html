SQLite C 接口

自动清理压缩量回调

int sqlite3_autovacuum_pages(
  sqlite3 *db,
  unsigned int(*)(void*,const char*,unsigned int,unsigned int,unsigned int),
  void*,
  void(*)(void*)
);

sqlite3_autovacuum_pages(D,C,P,X) 接口注册一个回调函数 C，该回调在每次数据库文件自动清理之前被调用。回调接收通用数据指针 (P) 的副本、正在自动清理的附加数据库的模式名、数据库文件的页数、空闲页数以及每页字节数。回调应该返回应该由自动清理移除的空闲页数。如果回调返回零，则不会发生自动清理；如果返回值大于或等于空闲页数，则会进行完全自动清理。

如果有多个作为事务提交一部分而被修改的附加数据库文件，那么自动清理页回调会为每一个文件分别调用。

回调不可重入。回调函数不应尝试调用任何其他 SQLite 接口。如果强行调用，可能会发生严重问题，包括段错误和数据库文件损坏。回调函数应当是一个简单的函数，对其输入参数进行一些算术运算并返回结果。

X 参数是 sqlite3_autovacuum_pages(D,C,P,X) 的一个可选析构函数，用于 P 参数。如果 X 不为 NULL，那么在数据库连接关闭或回调被其他 sqlite3_autovacuum_pages() 调用覆盖时，会调用 X(P)。

每个数据库连接只有一个自动清理页回调。每次调用 sqlite3_autovacuum_pages() 接口都会覆盖之前该数据库连接的所有调用。如果 sqlite3_autovacuum_pages(D,C,P,X) 的回调参数 (C) 为 NULL 指针，则自动清理步骤回调被取消。sqlite3_autovacuum_pages() 的返回值通常为 SQLITE_OK，但在出错时可能返回其他错误码。当前实现只会返回 SQLITE_OK 或 SQLITE_MISUSE，但未来版本可能会添加其他返回码。

如果未指定自动清理页回调（常见情况）或为回调提供了 NULL 指针，则默认行为为清理所有空闲页。换句话说，默认行为与以下回调函数相同：

    unsigned int demonstration_autovac_pages_callback(
      void *pClientData,
      const char *zSchema,
      unsigned int nDbPage,
      unsigned int nFreePage,
      unsigned int nBytePerPage
    ){
      return nFreePage;
    }
