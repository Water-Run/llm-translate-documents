SQLite C 接口

从 wal 文件恢复快照

int sqlite3_snapshot_recover(sqlite3 *db, const char *zDb);

如果在所有数据库连接关闭后（无论是通过使用 SQLITE_FCNTL_PERSIST_WAL 文件控制，还是因为最后一个打开数据库的进程在未调用 sqlite3_close() 的情况下退出）WAL 文件仍然留在磁盘上，并且随即对该数据库和 WAL 文件 打开了一个新连接，那么 sqlite3_snapshot_open() 接口只能打开 WAL 文件中最后添加的事务，尽管 WAL 文件中还包含其他有效事务。

此函数尝试扫描与数据库句柄 db 的数据库 zDb 关联的 WAL 文件，并使所有有效快照可由 sqlite3_snapshot_open() 获取。如果数据库上已经打开了读取事务，或者该数据库不是 WAL 模式数据库，则会报错。

成功时返回 SQLITE_OK，否则返回 SQLite 错误代码。

仅当 SQLite 使用 SQLITE_ENABLE_SNAPSHOT 选项编译时才提供此接口。
