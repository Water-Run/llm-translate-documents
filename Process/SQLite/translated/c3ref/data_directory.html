SQLite C 接口

保存数据库文件的文件夹名称

SQLITE_EXTERN char *sqlite3_data_directory;

如果这个全局变量指向一个字符串，该字符串是一个文件夹（也就是目录）的名称，那么所有使用相对路径名指定并且在启用内置 Windows VFS 时由 SQLite 创建或访问的数据库文件都将被假定为相对于该目录。如果该变量是 NULL 指针，那么 SQLite 会假定所有使用相对路径名指定的数据库文件都相对于该进程的当前目录。只有 Windows VFS 使用这个全局变量；Unix VFS 会忽略它。

在数据库连接打开时更改此变量的值可能会导致数据库损坏。

在多个线程同时读取或修改此变量是不安全的。如果在另一个线程中同时使用数据库连接，则也不安全。这个变量的设定意在作为进程初始化的一部分，在调用任何 SQLite 接口例程之前设置一次，并在随后保持不变。

data_store_directory pragma 可能修改这个变量，并令它指向来自 sqlite3_malloc 获取的内存。此外，data_store_directory pragma 总是假设该变量所指向的任何字符串都保存在来自 sqlite3_malloc 获取的内存中，因而该 pragma 可能尝试使用 sqlite3_free 释放这段内存。因此，如果直接修改这个变量，要么将其置为 NULL，要么令其指向来自 sqlite3_malloc 获取的内存，否则应避免使用 data_store_directory pragma。

另请参阅 对象、 常量 和 函数 的列表。
