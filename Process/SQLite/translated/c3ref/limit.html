SQLite C 接口

运行时限制

int sqlite3_limit(sqlite3*, int id, int newVal);

这个接口允许在每个数据库连接上限制各种结构的大小。第一个参数是要设置或查询其限制的 数据库连接。第二个参数是定义要限制大小的一类结构的 限制类别之一。第三个参数是该结构的新限制值。

如果新限制是负数，则限制保持不变。对于每个限制类别 SQLITE_LIMIT_NAME， 都有一个在编译时通过名为SQLITE_MAX_NAME的 C 预处理宏 设定的硬上限。（名称中的 "_LIMIT_" 被改为 "_MAX_"。）试图将限制 提高到硬上限之上时，会被静默截断为硬上限。

无论限制是否被修改， sqlite3_limit() 接口都会返回限制的先前值。因此，如果只想在不更改限制 的情况下查看当前值，只需将第三个参数设置为 -1 调用此接口即可。

运行时限制适用于同时管理自身内部数据库和受不受信任的外部来源控制的数据库的应用程序。例如， 一个浏览器可能有自己的用于存储历史记录的数据库和多个由从互联网上下载的 JavaScript 应用控制的独立数据库。 内部数据库可以使用较大、默认的限制。由外部来源管理的数据库可以使用更小的限制，以防止拒绝服务攻击。 开发者也可能希望使用sqlite3_set_authorizer()接口进一步控制不受信任的 SQL。 可通过max_page_count PRAGMA限制不受信任脚本创建的数据库大小。

未来版本可能会添加新的运行时限制类别。
