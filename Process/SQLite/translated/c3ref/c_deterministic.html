SQLite C 接口

函数标志

#define SQLITE_DETERMINISTIC    0x000000800
#define SQLITE_DIRECTONLY       0x000080000
#define SQLITE_SUBTYPE          0x000100000
#define SQLITE_INNOCUOUS        0x000200000
#define SQLITE_RESULT_SUBTYPE   0x001000000
#define SQLITE_SELFORDER1       0x002000000

这些常量可与首选文本编码一起按位或，作为sqlite3_create_function()、 sqlite3_create_function16()或sqlite3_create_function_v2()的第四个参数。

SQLITE_DETERMINISTIC
SQLITE_DETERMINISTIC 标志表示只要输入参数相同，该函数总是返回相同的输出。 例如，abs() 函数是确定性的，但randomblob()则不是。 函数必须是确定性的，才能在某些上下文中使用，例如在部分索引的 WHERE 子句中，或在生成列中。 SQLite 也可能通过将确定性函数从内部循环中提取出来来优化它们。

SQLITE_DIRECTONLY
SQLITE_DIRECTONLY 标志表示该函数只能从顶层 SQL 调用，不能在视图或触发器中使用，也不能用于诸如CHECK 约束、DEFAULT 子句、 表达式索引、部分索引或生成列等模式结构中。

对于任何具有副作用或可能泄露敏感信息的应用定义 SQL 函数，建议设置 SQLITE_DIRECTONLY 标志。 这可以防止攻击者引导应用程序使用一个其模式被偷偷修改过的数据库文件，从而以有害的方式调用该应用定义函数。

有些人认为在所有应用定义 SQL 函数上都设置 SQLITE_DIRECTONLY 是一种良好实践，无论它们是否涉及安全，因为这样可以阻止将这些函数用于数据库模式中，从而确保数据库可以使用不具备访问应用定义函数能力的通用工具（例如CLI）进行检查和修改。

SQLITE_INNOCUOUS
SQLITE_INNOCUOUS 标志表示即使被误用，该函数也不太可能引发问题。 一个无害函数不应有副作用，也不应依赖除输入参数之外的任何值。 abs() 函数是无害函数的一个例子。 而由于具有副作用，load_extension() SQL 函数则不是无害函数。

SQLITE_INNOCUOUS 与 SQLITE_DETERMINISTIC 相似，但并不完全相同。 random() 函数就是一个无害但不确定性的例子。

一些加强的安全设置（SQLITE_DBCONFIG_TRUSTED_SCHEMA 和 PRAGMA trusted_schema=OFF）会禁用在视图和触发器中以及在模式结构中的 SQL 函数使用，例如在CHECK 约束、DEFAULT 子句、 表达式索引、部分索引和生成列中，除非该函数被标记为 SQLITE_INNOCUOUS。 大多数内置函数都是无害的。建议开发人员不要为应用定义函数设置 SQLITE_INNOCUOUS，除非该函数经过仔细审计并确认没有潜在的安全副作用和信息泄露。

SQLITE_SUBTYPE
SQLITE_SUBTYPE 标志向 SQLite 表示该函数可能会调用sqlite3_value_subtype()来检查其参数的子类型。 此标志指示 SQLite 省略某些角落优化，这些优化可能会破坏sqlite3_value_subtype()函数的操作，导致它返回零而不是正确的子类型。 所有调用sqlite3_value_subtype()的 SQL 函数都应具有此属性。 如果省略 SQLITE_SUBTYPE 属性，则来自sqlite3_value_subtype()的返回值有时可能为零，即使函数参数表达式指定了非零子类型。

SQLITE_RESULT_SUBTYPE
SQLITE_RESULT_SUBTYPE 标志向 SQLite 表示该函数可能调用sqlite3_result_subtype()来为其结果关联一个子类型。 所有调用sqlite3_result_subtype()的函数都应具有此属性。 如果不具备此属性，当函数在表达式索引中的术语使用时，对sqlite3_result_subtype()的调用可能变为无效。 另一方面，从不调用sqlite3_result_subtype()的 SQL 函数应避免设置此属性，因为此属性的目的是禁用与子类型不兼容的某些优化。

SQLITE_SELFORDER1
SQLITE_SELFORDER1 标志表示该函数是一个聚合函数，它会在内部对提供给第一个参数的值进行排序。 带有单个 ORDER BY 项的有序集合聚合 SQL 记法可以用于调用该函数。 如果在缺少该标志的函数上使用有序集合聚合记法，将会引发错误。 请注意，只有在使用编译时选项 -DSQLITE_ENABLE_ORDERED_SET_AGGREGATES 构建 SQLite 时，才可使用有序集合聚合语法。
