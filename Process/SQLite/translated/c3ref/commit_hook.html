SQLite C 接口

提交与回滚通知回调

void *sqlite3_commit_hook(sqlite3*, int(*)(void*), void*);
void *sqlite3_rollback_hook(sqlite3*, void(*)(void *), void*);

sqlite3_commit_hook() 接口注册回调函数，每当事务 提交 时调用。此前对同一数据库连接调用 sqlite3_commit_hook() 设置的任何回调都会被覆盖。sqlite3_rollback_hook() 接口注册回调函数，每当事务 回滚 时调用。此前对同一数据库连接调用 sqlite3_rollback_hook() 设置的任何回调都会被覆盖。pArg 参数会传递给回调函数。如果提交钩子的回调函数返回非零值，则提交会转换为回滚。

sqlite3_commit_hook(D,C,P) 与 sqlite3_rollback_hook(D,C,P) 函数 返回同一 数据库连接 D 上上一次调用相同函数时的 P 参数，首次调用时返回 NULL。

提交与回滚钩子回调不是可重入的。回调实现不得修改 调用回调的数据库连接。任何修改数据库连接的操作 都必须推迟到触发提交或回滚钩子的 sqlite3_step() 调用完成之后。 请注意，执行任何其他 SQL 语句（包括 SELECT 语句），或仅调用 sqlite3_prepare_v2() 与 sqlite3_step() 都会在本段所述的“修改”意义上修改数据库连接。

注册 NULL 函数会禁用该回调。

当提交钩子回调例程返回零时，COMMIT 操作可正常继续。如果提交钩子 返回非零，则 COMMIT 会被转换为 ROLLBACK。 若提交钩子返回非零导致回滚发生，则会像其他回滚一样调用回滚钩子。

就该 API 而言，当明确执行 “ROLLBACK” 语句，或因错误或约束而导致隐式回滚时，事务即被视为已回滚。 如果数据库连接被关闭而自动回滚，则不会调用回滚回调。

另请参阅 sqlite3_update_hook() 接口。
