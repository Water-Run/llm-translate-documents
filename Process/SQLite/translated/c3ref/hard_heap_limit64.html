SQLite C 接口

对堆大小施加限制

sqlite3_int64 sqlite3_soft_heap_limit64(sqlite3_int64 N);
sqlite3_int64 sqlite3_hard_heap_limit64(sqlite3_int64 N);

这些接口会对单个进程中所有数据库连接所使用的堆内存总量施加限制。

sqlite3_soft_heap_limit64() 接口会设置和/或查询 SQLite 可分配的堆内存软限制。 SQLite 会努力在软堆限制下保持堆内存使用，通过在堆内存使用接近限制时减少页缓存中保留的页数。 软堆限制是“软性”的，因为尽管 SQLite 会尽力保持在限制之下，但它宁可超出限制也不会返回 SQLITE_NOMEM 错误。换句话说，软堆限制只是建议。

sqlite3_hard_heap_limit64(N) 接口为分配的内存总量设置 N 字节的硬上限。 sqlite3_hard_heap_limit64(N) 接口与 sqlite3_soft_heap_limit64(N) 类似， 只是当达到硬堆限制时，内存分配会失败。

sqlite3_soft_heap_limit64() 和 sqlite3_hard_heap_limit64() 的返回值 是在调用前堆限制的大小，如果发生错误则为负值。如果参数 N 为负数， 则堆限制不会改变。因此，可以通过调用 sqlite3_soft_heap_limit64(-1) 或 sqlite3_hard_heap_limit(-1) 来确定当前堆限制的大小。

将堆限制设置为零会禁用堆限制机制。

软堆限制不能大于硬堆限制。 如果启用了硬堆限制，并且调用 sqlite3_soft_heap_limit(N) 所指定的 N 大于硬堆限制， 那么软堆限制将被设置为硬堆限制的值。 只要启用硬堆限制，软堆限制就会自动启用。当调用 sqlite3_hard_heap_limit64(N) 并且软堆限制超出 1..N 的范围时，软堆限制将被设置为 N。 在启用硬堆限制时调用 sqlite3_soft_heap_limit64(0) 会使软堆限制等于硬堆限制。

也可以通过 PRAGMA soft_heap_limit 和 PRAGMA hard_heap_limit 来调整内存分配限制。

如果满足以下一项或多项条件，目前的实现不会强制执行堆限制：

  • 限制值设为零。
  • 使用 sqlite3_config(SQLITE_CONFIG_MEMSTATUS,...) 的启动时选项 和 SQLITE_DEFAULT_MEMSTATUS 编译时选项组合禁用内存计数。
  • 使用 sqlite3_config(SQLITE_CONFIG_PCACHE2,...) 指定了替代的页缓存实现。
  • 页缓存从 sqlite3_config(SQLITE_CONFIG_PAGECACHE,...) 提供的自己的内存池中分配，而不是从堆中分配。

在未来版本中，SQLite 强制执行堆限制的情况可能会发生变化。
