SQLite C 接口

定义特殊析构行为的常量

typedef void (*sqlite3_destructor_type)(void*);
#define SQLITE_STATIC      ((sqlite3_destructor_type)0)
#define SQLITE_TRANSIENT   ((sqlite3_destructor_type)-1)

这些是作为像 sqlite3_result_blob() 这样的例程所接收的最后一个参数传入的析构函数的特殊值。如果析构函数参数是 SQLITE_STATIC，表示内容指针是常量，并且永远不会改变。它不需要被销毁。SQLITE_TRANSIENT 值意味着内容在不久的将来很可能会改变，SQLite 应该在返回之前为该内容创建自己的私有副本。

该 typedef 是为了解决某些 C++ 编译器中的问题而必须存在的。
