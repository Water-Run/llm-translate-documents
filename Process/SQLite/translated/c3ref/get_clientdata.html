SQLite C 接口

数据库连接客户端数据

void *sqlite3_get_clientdata(sqlite3*,const char*);
int sqlite3_set_clientdata(sqlite3*, const char*, void*, void(*)(void*));

这些函数用于将一个或多个具名指针与数据库连接关联。 调用 sqlite3_set_clientdata(D,N,P,X) 会将指针 P 附加到使用名称 N 的数据库连接 D 上。随后的 sqlite3_get_clientdata(D,N) 调用将返回指针 P 的副本， 如果之前没有使用相同的 D 和 N 调用过 sqlite3_set_clientdata()，则返回 NULL 指针。 名称通过 strcmp() 进行比较，因此区分大小写。 成功时返回 0，分配失败时返回 SQLITE_NOMEM。

如果 P 和 X 都非 NULL，则析构函数 X 会在下面事件中首次发生时使用 参数 P 被调用：

  • 在尝试注册指针 P 的 sqlite3_set_clientdata() 调用期间发生内存不足错误。
  • 随后再次使用相同的 D 和 N 参数调用 sqlite3_set_clientdata(D,N,P,X)。
  • 数据库连接关闭。SQLite 不保证析构函数调用的顺序，只保证 在数据库连接关闭过程中每个析构函数都会恰好被调用一次。

SQLite 除了在适当时机调用析构函数外，不会对客户端数据做其它处理。 客户端数据的预期用途是为封装库提供一个机制，用于在 SQLite 数据库连接上 存储额外的信息。

单个数据库连接可附加的不同客户端数据指针数量没有限制（仅受可用内存限制）。 然而，当前实现针对只有一到两个不同客户端数据名称的情况进行了优化。 建议应用程序和封装库各自不要使用多个客户端数据名称。

无法枚举与数据库连接关联的客户端数据指针。 可以将 N 参数视为一个秘密密钥，只有知道该密钥的代码才能访问关联的数据。

安全警告：这些接口不应在脚本语言或攻击者可能调用它们的其他场景中公开。 任何能调用这些接口的代理很可能也能控制该进程。

数据库连接客户端数据仅在 SQLite 3.44.0（2023-11-01）及更高版本中可用。

另请参阅：sqlite3_set_auxdata() 和 sqlite3_get_auxdata()。
