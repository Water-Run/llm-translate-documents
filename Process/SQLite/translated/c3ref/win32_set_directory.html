SQLite C 接口

Win32 特定接口

int sqlite3_win32_set_directory(
  unsigned long type, /* Identifier for directory being set or reset */
  void *zValue        /* New value for directory being set or reset */
);
int sqlite3_win32_set_directory8(unsigned long type, const char *zValue);
int sqlite3_win32_set_directory16(unsigned long type, const void *zValue);

这些接口仅在 Windows 上可用。 sqlite3_win32_set_directory 接口用于根据 type 参数的值将 zValue 设置为与 sqlite3_temp_directory 或 sqlite3_data_directory 变量相关联的值。 当 zValue 参数为 NULL 时会通过 sqlite3_free 释放之前的值；当 zValue 非 NULL 时，会先复制到 sqlite3_malloc 获取的内存中再使用。 sqlite3_win32_set_directory 接口在成功时返回 SQLITE_OK，如果不支持 type 则返回 SQLITE_ERROR，如果无法分配内存则返回 SQLITE_NOMEM。 sqlite3_data_directory 变量的值旨在作为 Win32 子平台（如 WinRT 和 UWP）中 当前目录概念缺失时的替代方案。 sqlite3_win32_set_directory8 和 sqlite3_win32_set_directory16 接口的行为与 sqlite3_win32_set_directory 接口完全相同，但字符串参数必须分别为 UTF-8 或 UTF-16。
