SQLite C 接口

编译时库版本号

#define SQLITE_VERSION        "3.51.2"
#define SQLITE_VERSION_NUMBER 3051002
#define SQLITE_SOURCE_ID      "2026-01-09 17:27:48 b270f8339eb13b504d0b2ba154ebca966b7dde08e40c3ed7d559749818cb2075"
#define SQLITE_SCM_BRANCH     "branch-3.51"
#define SQLITE_SCM_TAGS       "release version-3.51.2"
#define SQLITE_SCM_DATETIME   "2026-01-09T17:27:48.405Z"

在 sqlite3.h 头文件中，SQLITE_VERSION C 预处理宏计算为一个字符串字面量，其格式为 “X.Y.Z”，其中 X 是主版本号（SQLite3 始终为 3），Y 是次版本号，Z 是发布号。SQLITE_VERSION_NUMBER C 预处理宏解析为一个整数，值为 \(X*1000000 + Y*1000 + Z\)，其中 X、Y 和 Z 与 SQLITE_VERSION 中使用的数字相同。任意给定发行版的 SQLITE_VERSION_NUMBER 也会大于它所派生的发行版。要么 Y 保持不变且 Z 增加，要么 Y 增加且 Z 重置为零。

自 版本 3.6.18（2009-09-11）以来，SQLite 源代码一直存储在 Fossil 配置管理系统 中。SQLITE_SOURCE_ID 宏解析为一个字符串，用于标识配置管理系统中特定的 SQLite 提交。SQLITE_SOURCE_ID 字符串包含提交的日期和时间（UTC）以及整个源代码树的 SHA1 或 SHA3-256 哈希。如果源代码自最近一次提交以来经过任何修改，则哈希的最后四位十六进制数字可能会被更改。

另请参见：sqlite3_libversion()、sqlite3_libversion_number()、sqlite3_sourceid()、sqlite_version() 和 sqlite_source_id()。
