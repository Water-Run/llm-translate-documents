SQLite C 接口

预写式日志提交钩子

void *sqlite3_wal_hook(
  sqlite3*,
  int(*)(void *,sqlite3*,const char*,int),
  void*
);

sqlite3_wal_hook() 函数用于注册一个回调，该回调在以 wal 模式向数据库提交数据时每次都会被调用。

提交完成并释放数据库的写锁之后，SQLite 会调用该回调，因此实现可以根据需要读取、写入或 检查点 数据库。

回调被调用时传入的第一个参数是注册该回调时传递给 sqlite3_wal_hook() 第三个参数的副本。第二个参数是数据库句柄的副本。第三个参数是写入的数据库名称——要么是 “main”，要么是某个 ATTACH 过的数据库的名称。第四个参数是写前日志文件当前包含的页数，包括刚刚提交的页。

回调函数通常应该返回 SQLITE_OK。如果返回错误代码，该错误会向上传播至触发回调的语句并导致其报告错误，尽管提交已经发生。如果回调返回 SQLITE_ROW 或 SQLITE_DONE，或者返回一个不对应于任何有效 SQLite 错误码的值，则其结果未定义。

单个数据库句柄最多只能同时注册一个写前日志回调。再次调用 sqlite3_wal_hook() 会替换默认行为或之前注册的写前日志回调。

返回值是前一次调用传入的第三个参数的副本（如果有），否则为 0。

sqlite3_wal_autocheckpoint() 接口和 wal_autocheckpoint 指令 都会调用 sqlite3_wal_hook()，并会覆盖任何先前的 sqlite3_wal_hook() 设置。

如果使用此函数设置了写前日志回调，则应定期调用 sqlite3_wal_checkpoint_v2() 或 PRAGMA wal_checkpoint，以防写前日志文件无限增长。

传入 NULL 指针作为回调将完全禁用自动检查点。要重新启用默认行为，请调用 sqlite3_wal_autocheckpoint(db,1000) 或使用 PRAGMA wal_checkpoint 。
