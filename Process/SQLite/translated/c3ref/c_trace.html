SQLite C 接口

SQL 跟踪事件代码

#define SQLITE_TRACE_STMT       0x01
#define SQLITE_TRACE_PROFILE    0x02
#define SQLITE_TRACE_ROW        0x04
#define SQLITE_TRACE_CLOSE      0x08

这些常量标识可以使用 sqlite3_trace_v2() 跟踪逻辑监控的事件类别。sqlite3_trace_v2(D,M,X,P) 的 M 参数是以下一个或多个常量经按位或组合的结果。跟踪回调的第一个参数是以下常量之一。

将来的版本可能会添加新的跟踪常量。

跟踪回调有四个参数：xCallback(T,C,P,X)。T 参数是上面的整数类型代码之一。C 参数是作为 sqlite3_trace_v2() 第四个参数传入的上下文指针的副本。P 和 X 参数是指针，其含义取决于 T。

SQLITE_TRACE_STMT
当准备好的语句第一次开始运行以及在其执行过程中的其他时间（例如每个触发器子程序开始时）会调用 SQLITE_TRACE_STMT 回调。P 参数指向该准备语句。X 参数指向一个字符串，其内容是准备语句的未展开 SQL 文本或表示触发器调用的 SQL 注释。当 X 以“--”开头时，回调可以使用 X 参数来获得与旧的 sqlite3_trace() 接口相同的文本，否则可以调用 sqlite3_expanded_sql(P)。

SQLITE_TRACE_PROFILE
SQLITE_TRACE_PROFILE 回调提供的内容与 sqlite3_profile() 回调相近。P 参数指向准备语句，X 参数指向一个 64 位整数，约表示该准备语句执行所花费的纳秒数。SQLITE_TRACE_PROFILE 回调在语句执行完成时被调用。

SQLITE_TRACE_ROW
每当准备语句产生一行结果时，都会调用 SQLITE_TRACE_ROW 回调。P 参数指向准备语句，X 参数未使用。

SQLITE_TRACE_CLOSE
当数据库连接关闭时，会调用 SQLITE_TRACE_CLOSE 回调。P 参数指向 数据库连接 对象，X 参数未使用。
