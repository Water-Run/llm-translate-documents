SQLite C 接口

数据更改通知回调

void *sqlite3_update_hook(
  sqlite3*,
  void(*)(void *,int ,char const *,char const *,sqlite3_int64),
  void*
);

sqlite3_update_hook() 接口在第一个参数指定的 数据库连接 中注册一个回调函数， 该回调在 rowid 表 中的任何一行被更新、插入或删除时调用。 对同一数据库连接之前的调用设置的任何回调都会被覆盖。

第二个参数是一个指向函数的指针，当 rowid 表 中的某一行被更新、插入或删除时调用该函数。 通过将第二个参数设置为 NULL 来调用 sqlite3_update_hook() 可以禁用更新钩子。 回调的第一个参数是传递给 sqlite3_update_hook() 的第三个参数的副本。 第二个回调参数是 SQLITE_INSERT、SQLITE_DELETE 或 SQLITE_UPDATE 中之一，取决于导致回调被调用的操作。 回调的第三个和第四个参数包含指向包含受影响行的数据库和表名的指针。 最后一个回调参数是该行的 rowid。 在更新的情况下，这个 rowid 是更新完成之后的值。

当内部系统表（即 sqlite_sequence）被修改时，不会调用更新钩子。 当 WITHOUT ROWID 表 被修改时，也不会调用更新钩子。

在当前实现中，如果因 ON CONFLICT REPLACE 子句 而删除发生冲突的行，则不会调用更新钩子。 使用 截断优化 删除行时，也不会调用更新钩子。 本段中定义的例外情况可能会在未来的 SQLite 版本中更改。

更新钩子是在对应更改之前还是之后调用目前尚未规定，并且可能取决于更改的类型。 不要依赖钩子调用的顺序与触发钩子的操作的最终结果之间的关系。

更新钩子实现不得执行任何会修改调用更新钩子的数据库连接的操作。 任何修改数据库连接的操作必须推迟到触发更新钩子的 sqlite3_step() 调用完成之后。 请注意，sqlite3_prepare_v2() 和 sqlite3_step() 都会修改它们的数据库连接，就本段中的“修改”而言。

sqlite3_update_hook(D,C,P) 函数返回同一 数据库连接 D 上前一次调用的 P 参数，对 D 的第一次调用则返回 NULL。

另请参阅 sqlite3_commit_hook()、sqlite3_rollback_hook() 和 sqlite3_preupdate_hook() 接口。
