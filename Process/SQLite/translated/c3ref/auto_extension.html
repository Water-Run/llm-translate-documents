SQLite C 接口

自动加载静态链接扩展

int sqlite3_auto_extension(void(*xEntryPoint)(void));

这个接口会在为每一个新创建的数据库连接调用 xEntryPoint() 函数。 这里的思想是 xEntryPoint() 是要自动加载到所有新数据库连接中的一个 SQLite 扩展的入口点。

尽管函数原型显示 xEntryPoint() 不接受参数且返回 void， SQLite 实际上会用三个参数调用 xEntryPoint() 并期待整数结果，就像入口点的签名如下：

   int xEntryPoint(
     sqlite3 *db,
     const char **pzErrMsg,
     const struct sqlite3_api_routines *pThunk
   );

如果 xEntryPoint 例程遇到错误，它应该让 *pzErrMsg 指向一个恰当的错误消息（来自 sqlite3_mprintf()）并返回恰当的错误码。 SQLite 确保在调用 xEntryPoint() 之前 *pzErrMsg 为 NULL。 当 xEntryPoint() 返回后，SQLite 将对 *pzErrMsg 调用 sqlite3_free()。如果任意 xEntryPoint() 返回错误，则导致该 xEntryPoint() 的 sqlite3_open()、sqlite3_open16() 或 sqlite3_open_v2() 调用都会失败。

使用已经在自动扩展列表中的入口点 X 调用 sqlite3_auto_extension(X) 是无害的无操作。 对于打开的每个数据库连接，没有入口点会被调用超过一次。

另见：sqlite3_reset_auto_extension() 和 sqlite3_cancel_auto_extension()
