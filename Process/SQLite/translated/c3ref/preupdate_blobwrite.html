SQLite C 接口

预更新钩子。

#if defined(SQLITE_ENABLE_PREUPDATE_HOOK)
void *sqlite3_preupdate_hook(
  sqlite3 *db,
  void(*xPreUpdate)(
    void *pCtx,                   /* Copy of third arg to preupdate_hook() */
    sqlite3 *db,                  /* Database handle */
    int op,                       /* SQLITE_UPDATE, DELETE or INSERT */
    char const *zDb,              /* Database name */
    char const *zName,            /* Table name */
    sqlite3_int64 iKey1,          /* Rowid of row about to be deleted/updated */
    sqlite3_int64 iKey2           /* New rowid value (for a rowid UPDATE) */
  ),
  void*
);
int sqlite3_preupdate_old(sqlite3 *, int, sqlite3_value **);
int sqlite3_preupdate_count(sqlite3 *);
int sqlite3_preupdate_depth(sqlite3 *);
int sqlite3_preupdate_new(sqlite3 *, int, sqlite3_value **);
int sqlite3_preupdate_blobwrite(sqlite3 *);
#endif

只有在使用 SQLITE_ENABLE_PREUPDATE_HOOK 编译选项编译 SQLite 时，这些接口才可用。

sqlite3_preupdate_hook() 接口注册一个回调函数， 该函数会在对数据库表执行每次 INSERT、UPDATE 和 DELETE 操作之前被调用。 在单个 数据库连接 上最多只能注册一个预更新钩子；每次调用 sqlite3_preupdate_hook() 都会覆盖之前的设置。 通过将第二个参数设为 NULL 调用 sqlite3_preupdate_hook() 可禁用预更新钩子。 第三个参数会被传递给回调函数作为第一个参数。

预更新钩子仅在对真实数据库表进行更改时激活；对于 虚拟表 或 sqlite_sequence、sqlite_stat1 等系统表的更改不会触发预更新钩子。

预更新回调的第二个参数是注册该钩子的 数据库连接 指针。 第三个参数是常量 SQLITE_INSERT、SQLITE_DELETE 或 SQLITE_UPDATE 之一，用于标识即将执行的更新操作类型。 第四个参数是正在被修改的数据库连接中数据库的名称。主数据库是 “main”，TEMP 表是 “temp”，或者是 ATTACH 语句中 AS 关键字后指定的附加数据库别名。 第五个参数是正在被修改的表的名称。

对于 rowid 表 的 UPDATE 或 DELETE 操作，传递给预更新回调的第六个参数是正在被修改或删除的行的初始 rowid。 对于 rowid 表上的 INSERT 操作或任何 WITHOUT ROWID 表上的操作，第六个参数的值未定义。 对于 rowid 表上的 INSERT 或 UPDATE 操作，第七个参数是正在插入或更新的行的最终 rowid 值。 对于 WITHOUT ROWID 表或 rowid 表上的 DELETE 操作，第七个参数的值未定义。

函数 sqlite3_preupdate_hook(D,C,P) 返回在同一 数据库连接 D 上的上一次调用中传入的 P 参数；对 D 的第一次调用返回 NULL。

sqlite3_preupdate_old()、sqlite3_preupdate_new()、 sqlite3_preupdate_count() 和 sqlite3_preupdate_depth() 接口提供有关预更新事件的附加信息。 这些例程只能在预更新回调中调用。 在预更新回调之外或使用与回调提供的不同的 数据库连接 指针调用这些例程会导致未定义且可能不可取的行为。

sqlite3_preupdate_count(D) 接口返回正在插入、更新或删除的行的列数。

sqlite3_preupdate_old(D,N,P) 接口在 P 中写入一个指针，指向包含被更新前指定行第 N 列值的 受保护 sqlite3_value。参数 N 必须介于 0 到列数减一之间，否则行为未定义。 该接口只能在 SQLITE_UPDATE 和 SQLITE_DELETE 预更新回调中使用；若在 SQLITE_INSERT 回调中使用，则行为未定义。 P 指向的 sqlite3_value 会在预更新回调返回时被销毁。

sqlite3_preupdate_new(D,N,P) 接口在 P 中写入一个指针，指向包含被更新后指定行第 N 列值的 受保护 sqlite3_value。参数 N 必须介于 0 到列数减一之间，否则行为未定义。 该接口只能在 SQLITE_INSERT 和 SQLITE_UPDATE 预更新回调中使用；若在 SQLITE_DELETE 回调中使用，则行为未定义。 P 指向的 sqlite3_value 会在预更新回调返回时被销毁。

sqlite3_preupdate_depth(D) 接口在预更新回调是由直接的插入、更新或删除操作触发时返回 0； 由顶层触发器调用的插入、更新或删除返回 1； 由顶层触发器调用的触发器所造成的更改返回 2；依此类推。

当使用 sqlite3_blob_write() API 更新 blob 列时，预更新钩子会以 SQLITE_DELETE 调用，因为新值尚不可用。 在 op==SQLITE_DELETE 的回调实际通过 sqlite3_blob_write() API 进行写入时， sqlite3_preupdate_blobwrite() 返回正在写入的列的索引。 在其他情况（包括普通的 DELETE）中，预更新钩子由于其他原因被调用时，sqlite3_preupdate_blobwrite() 返回 -1。
