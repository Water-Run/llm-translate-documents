SQLite C 接口

获取 URI 参数的值

const char *sqlite3_uri_parameter(sqlite3_filename z, const char *zParam);
int sqlite3_uri_boolean(sqlite3_filename z, const char *zParam, int bDefault);
sqlite3_int64 sqlite3_uri_int64(sqlite3_filename, const char*, sqlite3_int64);
const char *sqlite3_uri_key(sqlite3_filename z, int N);

这些实用例程适用于自定义 VFS 实现，用于检测数据库文件是否为包含特定查询参数的 URI，如果是，则获取该查询参数的值。

这些接口的第一个参数（以下称为 F）必须为以下之一：

  • 由 SQLite 内核创建并传递给 VFS 实现的 xOpen() 方法的数据库文件名指针，或
  • 从 sqlite3_db_filename() 获取的文件名，或
  • 使用 sqlite3_create_filename() 构建的新文件名。
如果 F 参数不是上述情况之一，则行为未定义并且可能不理想。旧版本的 SQLite 对无效 F 参数比新版本更宽容。

如果 F 是一个合适的文件名（如上一段所述），并且 P 是查询参数的名称，那么 sqlite3_uri_parameter(F,P) 返回 P 参数的值（如果存在），如果 P 未出现在 F 的查询参数中，则返回 NULL 指针。 如果 P 是 F 的查询参数但没有显式值，那么 sqlite3_uri_parameter(F,P) 会返回指向空字符串的指针。

sqlite3_uri_boolean(F,P,B) 假定 P 是布尔参数，并根据 P 的值返回 true (1) 或 false (0)。当查询参数 P 的值任意大小写为 “yes”、“true” 或 “on”，或者值以非零数字开头时，sqlite3_uri_boolean(F,P,B) 返回 true (1)。当查询参数 P 的值任意大小写为 “no”、“false” 或 “off”，或者值以数字零开头时，sqlite3_uri_boolean(F,P,B) 返回 false (0)。如果 P 不是 F 的查询参数或者 P 的值不匹配上述任何情况，则 sqlite3_uri_boolean(F,P,B) 返回 (B!=0)。

sqlite3_uri_int64(F,P,D) 将 P 的值转换为 64 位有符号整数并返回该整数，如果 P 不存在则返回 D。如果 P 的值不是整数，则返回零。

sqlite3_uri_key(F,N) 返回一个指向 F 的第 N 个查询参数的名称（不是值）的指针，如果 N 小于零或大于查询参数数量减一，则返回 NULL 指针。N 的值从零开始，因此 N 为 0 可获取第一个查询参数的名称，N 为 1 可获取第二个参数，依此类推。

如果 F 是 NULL 指针，则 sqlite3_uri_parameter(F,P) 返回 NULL，而 sqlite3_uri_boolean(F,P,B) 返回 B。如果 F 不是 NULL 指针且不是 SQLite 内核传入 xOpen VFS 方法的数据库文件路径指针，则该例程的行为未定义并且可能不理想。

从 SQLite 3.31.0 版（2020-01-22）开始，输入的 F 参数除了主数据库文件之外，还可以是回滚日志文件或 WAL 文件的名称。在 3.31.0 之前，这些例程仅在 F 为主数据库文件的名称时才能工作。当 F 参数是回滚日志或 WAL 文件的名称时，它可以访问与主数据库文件相同的所有查询参数。
