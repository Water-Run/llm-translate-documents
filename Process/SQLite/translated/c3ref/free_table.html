SQLite C 接口

执行查询的便捷例程

int sqlite3_get_table(
  sqlite3 *db,          /* An open database */
  const char *zSql,     /* SQL to be evaluated */
  char ***pazResult,    /* Results of the query */
  int *pnRow,           /* Number of result rows written here */
  int *pnColumn,        /* Number of result columns written here */
  char **pzErrmsg       /* Error msg written here */
);
void sqlite3_free_table(char **result);

这是为了向后兼容而保留的传统接口。不建议使用此接口。

定义：一个 结果表 是由 sqlite3_get_table() 接口创建的内存数据结构。结果表记录了一个或多个查询的完整结果。

该表在概念上有若干行和列。但这些数量并不是结果表本身的一部分。 这些数量是单独获取的。令 N 为行数，M 为列数。

结果表是一个指向以零终止的 UTF-8 字符串的指针数组。 数组中共有 (N+1)*M 个元素。前 M 个指针指向包含列名的零终止字符串。 其余条目全部指向查询结果。NULL 值对应 NULL 指针。其他所有值都以 sqlite3_column_text() 返回的 UTF-8 零终止字符串形式存在。

结果表可能由一个或多个内存分配组成。不能直接将结果表传递给 sqlite3_free()。结果表应使用 sqlite3_free_table() 释放。

作为结果表格式的示例，假设查询结果如下所示：

Name        | Age
-----------------------
Alice       | 43
Bob         | 28
Cindy       | 21

有两列（M==2）和三行（N==3）。因此结果表有 8 个条目。假设结果表存储在名为 azResult 的数组中。 那么 azResult 的内容为：

azResult[0] = "Name";
azResult[1] = "Age";
azResult[2] = "Alice";
azResult[3] = "43";
azResult[4] = "Bob";
azResult[5] = "28";
azResult[6] = "Cindy";
azResult[7] = "21";

sqlite3_get_table() 函数在其第二个参数提供的以零终止的 UTF-8 字符串中执行一个或多个以分号分隔的 SQL 语句，并将结果表返回给第三个参数提供的指针。

在应用程序使用完 sqlite3_get_table() 的结果之后，必须将结果表指针传递给 sqlite3_free_table() 以释放 malloc 分配的内存。 由于 sqlite3_malloc() 在 sqlite3_get_table() 内部的实现方式，调用方不得直接调用 sqlite3_free()。只有 sqlite3_free_table() 能够正确且安全地释放内存。

sqlite3_get_table() 接口是对 sqlite3_exec() 的包装。sqlite3_get_table() 例程无法访问 SQLite 的任何内部数据结构。它只使用此处定义的公共接口。 因此，包装层在内部 sqlite3_exec() 调用之外发生的错误不会反映在后续对 sqlite3_errcode() 或 sqlite3_errmsg() 的调用中。
