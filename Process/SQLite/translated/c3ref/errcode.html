SQLite C 接口

错误代码和消息

int sqlite3_errcode(sqlite3 *db);
int sqlite3_extended_errcode(sqlite3 *db);
const char *sqlite3_errmsg(sqlite3*);
const void *sqlite3_errmsg16(sqlite3*);
const char *sqlite3_errstr(int);
int sqlite3_error_offset(sqlite3 *db);

如果与数据库连接 D 关联的最近一次 sqlite3_* API 调用失败，那么 sqlite3_errcode(D) 接口会返回该 API 调用的数字结果代码或扩展结果代码。sqlite3_extended_errcode() 接口工作方式相同，只不过它总是返回扩展结果代码，即便扩展结果代码被禁用。

sqlite3_errcode() 和/或 sqlite3_extended_errcode() 返回的值可能会随着每次 API 调用而变化。不过，有一些接口保证不会改变错误代码的值。保持错误代码不变的接口包括下列项：

  • sqlite3_errcode()
  • sqlite3_extended_errcode()
  • sqlite3_errmsg()
  • sqlite3_errmsg16()
  • sqlite3_error_offset()

sqlite3_errmsg() 和 sqlite3_errmsg16() 返回描述该错误的英文文本，分别以 UTF-8 或 UTF-16 的形式提供，如果没有可用的错误消息则返回 NULL。（有关此规则的例外，请参阅 SQLite 如何处理无效 UTF。）错误消息字符串所需的内存由内部管理，应用程序无需担心释放该结果。然而，错误字符串可能会被后续对其他 SQLite 接口函数的调用覆盖或释放。

sqlite3_errstr(E) 接口返回描述结果代码 E 的英文文本，格式为 UTF-8，如果 E 不是可提供文本错误消息的结果代码，则返回 NULL。错误消息字符串所需的内存由内部管理，应用程序不得释放。

如果最近的错误引用了 SQL 输入中的特定标记，sqlite3_error_offset() 接口会返回该标记起始位置的字节偏移量。sqlite3_error_offset() 返回的字节偏移量假定输入 SQL 为 UTF-8。如果最近的错误未引用 SQL 输入中的特定标记，那么 sqlite3_error_offset() 函数返回 -1。

当启用序列化线程安全模式时，可能会发生在首次错误和调用这些接口之间的另一个线程上出现第二个错误的情况。若发生这种情况，则会报告第二个错误，因为这些接口总是返回最近的结果。为避免此类情况，每个线程在开始使用数据库连接 D 之前，可以通过调用 sqlite3_mutex_enter(sqlite3_db_mutex(D)) 来独占该连接，并在完成对本文列出接口的所有调用后调用 sqlite3_mutex_leave(sqlite3_db_mutex(D))。

如果某个接口返回 SQLITE_MISUSE，说明应用程序错误地调用了该接口。在这种情况下，错误代码和错误消息可能会被设置，也可能不会被设置。
