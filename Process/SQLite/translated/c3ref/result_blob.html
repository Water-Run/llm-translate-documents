SQLite C 接口

设置 SQL 函数的结果

void sqlite3_result_blob(sqlite3_context*, const void*, int, void(*)(void*));
void sqlite3_result_blob64(sqlite3_context*,const void*,
                           sqlite3_uint64,void(*)(void*));
void sqlite3_result_double(sqlite3_context*, double);
void sqlite3_result_error(sqlite3_context*, const char*, int);
void sqlite3_result_error16(sqlite3_context*, const void*, int);
void sqlite3_result_error_toobig(sqlite3_context*);
void sqlite3_result_error_nomem(sqlite3_context*);
void sqlite3_result_error_code(sqlite3_context*, int);
void sqlite3_result_int(sqlite3_context*, int);
void sqlite3_result_int64(sqlite3_context*, sqlite3_int64);
void sqlite3_result_null(sqlite3_context*);
void sqlite3_result_text(sqlite3_context*, const char*, int, void(*)(void*));
void sqlite3_result_text64(sqlite3_context*, const char*,sqlite3_uint64,
                           void(*)(void*), unsigned char encoding);
void sqlite3_result_text16(sqlite3_context*, const void*, int, void(*)(void*));
void sqlite3_result_text16le(sqlite3_context*, const void*, int,void(*)(void*));
void sqlite3_result_text16be(sqlite3_context*, const void*, int,void(*)(void*));
void sqlite3_result_value(sqlite3_context*, sqlite3_value*);
void sqlite3_result_pointer(sqlite3_context*, void*,const char*,void(*)(void*));
void sqlite3_result_zeroblob(sqlite3_context*, int n);
int sqlite3_result_zeroblob64(sqlite3_context*, sqlite3_uint64 n);

这些例程由实现 SQL 函数和聚合的 xFunc 或 xFinal 回调使用。有关更多信息，请参阅 sqlite3_create_function() 与 sqlite3_create_function16()。

这些函数的行为与用于将值绑定到准备语句中主机参数的参数绑定函数族非常相似。有关更多信息，请参阅SQL 参数文档。

sqlite3_result_blob() 接口将应用定义函数的结果设置为由第二个参数指向、长度为第三个参数 N 字节的 BLOB。

sqlite3_result_zeroblob(C,N) 与 sqlite3_result_zeroblob64(C,N) 接口将应用定义函数的结果设置为一个大小为 N 字节、内容全为零的 BLOB。

sqlite3_result_double() 接口将应用定义函数的结果设置为第二个参数指定的浮点值。

sqlite3_result_error() 与 sqlite3_result_error16() 函数会使得所实现的 SQL 函数抛出异常。SQLite 使用 sqlite3_result_error() 或 sqlite3_result_error16() 的第二个参数指向的字符串作为错误消息文本。SQLite 将 sqlite3_result_error() 的字符串解释为 UTF-8，并将 sqlite3_result_error16() 的字符串按照与sqlite3_bind_text16()相同的字节顺序确定规则解释为 UTF-16。如果 sqlite3_result_error() 或 sqlite3_result_error16() 的第三个参数为负，则 SQLite 会将第二个参数中的所有文本直到第一个零字符作为错误消息。如果第三个参数为非负，则 SQLite 会从第二个参数中取该数量的字节（不是字符）作为错误消息。sqlite3_result_error() 与 sqlite3_result_error16() 在返回前会私下复制错误消息文本，因此调用函数可以在它们返回后安全地释放或修改该文本。sqlite3_result_error_code() 函数在函数出错时更改 SQLite 返回的错误代码，默认错误代码为 SQLITE_ERROR。随后的 sqlite3_result_error() 或 sqlite3_result_error16() 调用会将错误代码重置为 SQLITE_ERROR。

sqlite3_result_error_toobig() 接口会使 SQLite 抛出一个指示字符串或 BLOB 太长无法表示的错误。

sqlite3_result_error_nomem() 接口会使 SQLite 抛出一个指示内存分配失败的错误。

sqlite3_result_int() 接口将应用定义函数的返回值设置为第二个参数给定的 32 位有符号整数。sqlite3_result_int64() 接口将返回值设置为第二个参数给定的 64 位有符号整数。

sqlite3_result_null() 接口将应用定义函数的返回值设置为 NULL。

sqlite3_result_text()、sqlite3_result_text16()、sqlite3_result_text16le() 与 sqlite3_result_text16be() 接口分别将应用定义函数的返回值设置为以 UTF-8、UTF-16 本地字节序、UTF-16 小端或 UTF-16 大端表示的文本字符串。sqlite3_result_text64() 接口将应用定义函数的返回值设置为由第五个（也是最后一个）参数指定编码的文本字符串，该参数必须是 SQLITE_UTF8、SQLITE_UTF16、SQLITE_UTF16BE 或 SQLITE_UTF16LE 之一。SQLite 会使用 sqlite3_result_text* 接口的第二个参数作为来自应用的文本结果。如果任何 sqlite3_result_text* 接口（除 sqlite3_result_text64() 外）的第三个参数为负，则 SQLite 会通过寻找第二个参数中的第一个零字符来自行计算字符串长度。如果 sqlite3_result_text* 接口的第三个参数为非负，则 SQLite 会从第二个参数指向的文本中取该数量的字节（不是字符）作为应用定义函数的结果。如果第三个参数为非负，则它必须是字符串在以 NUL 结尾时 NUL 终止符所在的字节偏移量。如果字符串中在小于第三个参数值的字节偏移处出现任何 NUL 字符，则结果字符串将包含嵌入式 NUL，而且在包含嵌入式 NUL 的字符串上进行的表达式的结果是未定义的。

如果 sqlite3_result_text* 接口或 sqlite3_result_blob 的第四个参数是非空指针，则 SQLite 在使用该文本或 BLOB 结果完成后会调用该函数作为析构函数。如果 sqlite3_result_text* 接口或 sqlite3_result_blob 的第四个参数是特殊常量 SQLITE_STATIC，则 SQLite 会假设文本或 BLOB 结果位于常量空间内，在使用完成后既不复制参数内容也不调用析构函数。如果 sqlite3_result_text* 接口或 sqlite3_result_blob 的第四个参数是特殊常量 SQLITE_TRANSIENT，则 SQLite 会在返回之前将结果复制到由 sqlite3_malloc() 获取的空间中。

对于 sqlite3_result_text16()、sqlite3_result_text16le() 与 sqlite3_result_text16be() 例程，以及当编码不是 UTF8 时的 sqlite3_result_text64()，如果输入的 UTF16 以字节顺序标记（BOM，U+FEFF）开头，则会从字符串中删除 BOM，并根据 BOM 指定的字节顺序解释其余字符串。文本开头的 BOM 所指定的字节顺序会覆盖接口过程所指定的字节顺序。例如，如果在调用 sqlite3_result_text16le() 时输入文本以字节 0xfe、0xff（大端字节顺序标记）开头，则会跳过输入的前两个字节，剩余输入会被解释为 UTF16BE 文本。

对于传递给 sqlite3_result_text16()、sqlite3_result_text16be()、sqlite3_result_text16le() 与 sqlite3_result_text64() 例程的 UTF16 输入文本，如果文本包含无效的 UTF16 字符，则这些无效字符可能会被转换为 Unicode 替代字符 U+FFFD。

sqlite3_result_value() 接口将应用定义函数的结果设置为第二个参数指定的非保护 sqlite3_value对象的副本。sqlite3_result_value() 接口会复制该sqlite3_value，因此在 sqlite3_result_value() 返回后，参数中指定的 sqlite3_value 可以安全地更改或释放而不会造成影响。受保护 sqlite3_value对象始终可以用于需要非保护 sqlite3_value对象的地方，因此这两种 sqlite3_value 对象都可以与此接口一起使用。

sqlite3_result_pointer(C,P,T,D) 接口将结果设置为 SQL NULL 值，就像 sqlite3_result_null(C) 一样，只不过它还将主机语言指针 P 或类型 T 与该 NULL 值关联，以便可以在应用自定义 SQL 函数中使用 sqlite3_value_pointer() 检索该指针。如果 D 参数非空，则它是 P 参数的析构函数指针。SQLite 在完成对 P 的使用后，会以 P 为唯一参数调用 D。T 参数应该是一个静态字符串，最好是字符串字面量。sqlite3_result_pointer() 例程是为 SQLite 3.20.0 添加的指针传递接口的一部分。

如果这些例程在与接收 sqlite3_context 指针的应用定义函数所在线程不同的线程中被调用，则其结果未定义。
