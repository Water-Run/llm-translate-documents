SQLite C 接口

序列化数据库

unsigned char *sqlite3_serialize(
  sqlite3 *db,           /* 数据库连接 */
  const char *zSchema,   /* 要序列化的数据库。比如："main"、"temp" 等 */
  sqlite3_int64 *piSize, /* 如果不为 NULL，则在此写入数据库的大小 */
  unsigned int mFlags    /* 零个或多个 SQLITE_SERIALIZE_* 标志 */
);

sqlite3_serialize(D,S,P,F) 接口返回一个指向内存的指针，该内存是数据库连接 D 上 S 数据库的序列化。如果 S 是 NULL 指针，则使用主数据库。如果 P 不是 NULL 指针，则数据库的字节大小会写入 *P。

对于普通的磁盘数据库文件，序列化只是磁盘文件的副本。对于内存数据库或 “TEMP” 数据库，序列化是该数据库备份到磁盘时将被写入磁盘的相同字节序列。

通常情况下，sqlite3_serialize() 会将数据库的序列化复制到从 sqlite3_malloc64() 获取的内存中，并返回该内存的指针。调用者负责释放返回值以避免内存泄漏。然而，如果 F 参数包含 SQLITE_SERIALIZE_NOCOPY 位，则不会进行任何内存分配，sqlite3_serialize() 函数将返回指向 SQLite 当前用于该数据库的连续内存表示的指针；如果不存在这样的连续内存表示，则返回 NULL。只有在先前曾使用相同的 D 和 S 值调用 sqlite3_deserialize(D,S,...) 的情况下，数据库的连续内存表示才通常存在。即使设置了 SQLITE_SERIALIZE_NOCOPY 位但没有数据库的连续副本，数据库的大小仍会写入 *P。

调用结束后，如果设置了 SQLITE_SERIALIZE_NOCOPY 位，则返回的缓冲区内容在下一次对连接的写操作或关闭连接之前都保持可访问且不变，应用程序不得修改该缓冲区。如果该位未设置，调用后 SQLite 不会访问返回的缓冲区。

即使在参数 F 中省略了 SQLITE_SERIALIZE_NOCOPY 位，如果发生内存分配错误，调用 sqlite3_serialize(D,S,P,F) 也可能返回 NULL。

如果 SQLite 使用 SQLITE_OMIT_DESERIALIZE 选项进行编译，则不会包含此接口。
