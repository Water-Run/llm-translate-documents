SQLite C 接口

查询结果中的数据来源

const char *sqlite3_column_database_name(sqlite3_stmt*,int);
const void *sqlite3_column_database_name16(sqlite3_stmt*,int);
const char *sqlite3_column_table_name(sqlite3_stmt*,int);
const void *sqlite3_column_table_name16(sqlite3_stmt*,int);
const char *sqlite3_column_origin_name(sqlite3_stmt*,int);
const void *sqlite3_column_origin_name16(sqlite3_stmt*,int);

这些例程提供一种方法来确定在一个SELECT语句中某个结果列所属的数据库、表以及表列。数据库、表或列的名称可以作为 UTF-8 或 UTF-16 字符串返回。_database_ 例程返回数据库名称，_table_ 例程返回表名称，而 origin_ 例程返回列名称。返回的字符串在通过sqlite3_finalize()销毁预处理语句之前、在某次运行中首次调用sqlite3_step()导致语句自动重新准备之前，或直到以不同编码再次请求相同信息之前，始终有效。

返回的名称是数据库、表与列最初未加别名的名称。

这些接口的第一个参数是预处理语句。这些函数返回语句所产生的第 N 个结果列的信息，其中 N 是第二个函数参数。对于这些例程最左边的列是列 0。

如果语句返回的第 N 列是表达式或子查询而不是列值，那么所有这些函数都返回 NULL。如果发生内存分配错误，这些例程也可能返回 NULL。否则，它们会返回查询结果列所提取数据的附加数据库、表或列的名称。

与所有其他 SQLite API 一样，名称以 “16” 结尾的函数返回 UTF-16 编码字符串，其它函数返回 UTF-8。

这些 API 只有在库是使用SQLITE_ENABLE_COLUMN_METADATA C 预处理常量编译时才可用。

如果两个或更多线程同时针对相同的预处理语句和结果列调用一个或多个列元数据接口，则其结果未定义。

另请参见 对象、 常量 以及 函数 的列表。
