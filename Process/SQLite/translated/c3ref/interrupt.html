SQLite C 接口

中断长时间运行的查询

void sqlite3_interrupt(sqlite3*);
int sqlite3_is_interrupted(sqlite3*);

此函数会使任何待处理的数据库操作尽早中止并返回。通常在用户执行例如按“取消”或 Ctrl-C 这样的操作，希望长时间运行的查询立即停止时调用此例程。

在与当前执行数据库操作的线程不同的线程中调用此例程是安全的。但若使用已关闭或可能在 sqlite3_interrupt() 返回之前关闭的数据库连接调用此例程则不安全。

如果在调用 sqlite3_interrupt() 时某个 SQL 操作已接近完成，那么可能无法中断并会继续执行直至完成。

被中断的 SQL 操作将返回 SQLITE_INTERRUPT。若被中断的 SQL 操作是显式事务中的 INSERT、UPDATE 或 DELETE，则整个事务将自动回滚。

sqlite3_interrupt(D) 调用将持续生效，直到所有当前在数据库连接 D 上运行的 SQL 语句完成。sqlite3_interrupt() 调用之后且当前运行语句计数归零之前启动的任何新 SQL 语句都将被中断，就像它们在 sqlite3_interrupt() 调用之前已经运行一样。当前运行语句计数归零之后启动的新 SQL 语句不会受到 sqlite3_interrupt() 的影响。当没有正在运行的 SQL 语句时调用 sqlite3_interrupt(D) 是空操作，对该调用返回之后开始的 SQL 语句没有影响。

可以使用 sqlite3_is_interrupted(D) 接口判断数据库连接 D 上当前是否有中断生效。若中断生效，返回 1，否则返回 0。
