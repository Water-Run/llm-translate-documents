SQLite C 接口

修改的总行数

int sqlite3_total_changes(sqlite3*);
sqlite3_int64 sqlite3_total_changes64(sqlite3*);

这些函数返回自数据库连接打开以来所有完成的 INSERT、UPDATE 或 DELETE 语句所插入、修改或删除的行总数，包括作为触发器程序一部分执行的那些。两者的区别仅在于返回值的类型，以及当连接修改的行数超过类型 “int” 支持的最大值时，sqlite3_total_changes() 的返回值将变为未定义。执行其他类型的 SQL 语句不会影响 sqlite3_total_changes() 返回的值。

作为 外键操作 进行的更改计入总数，但在 REPLACE 约束解决过程中进行的更改不计入。被 INSTEAD OF 触发器拦截的视图更改也不计入。

sqlite3_total_changes(D) 接口仅报告针对数据库连接 D 执行的 SQL 语句所导致的行数更改。其他数据库连接的更改将被忽略。若要检测其他数据库连接对同一数据库文件所做的更改，请使用 PRAGMA data_version 命令或 SQLITE_FCNTL_DATA_VERSION 文件控制。

如果另一个线程在 sqlite3_total_changes() 运行期间对同一个数据库连接进行了更改，则返回值不可预测且无意义。

  • changes() SQL 函数
  • data_version pragma
