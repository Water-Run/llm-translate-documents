SQLite C 接口

提取有关表列的元数据

int sqlite3_table_column_metadata(
  sqlite3 *db,                /* Connection handle */
  const char *zDbName,        /* Database name or NULL */
  const char *zTableName,     /* Table name */
  const char *zColumnName,    /* Column name */
  char const **pzDataType,    /* OUTPUT: Declared data type */
  char const **pzCollSeq,     /* OUTPUT: Collation sequence name */
  int *pNotNull,              /* OUTPUT: True if NOT NULL constraint exists */
  int *pPrimaryKey,           /* OUTPUT: True if column part of PK */
  int *pAutoinc               /* OUTPUT: True if column is auto-increment */
);

sqlite3_table_column_metadata(X,D,T,C,....) 例程返回数据库连接 X 上数据库 D 中表 T 的列 C 的信息。sqlite3_table_column_metadata() 接口在指定的列存在的情况下返回 SQLITE_OK，并用适当的值填充最后五个参数中非 NULL 的指针。如果指定的列不存在，sqlite3_table_column_metadata() 接口返回 SQLITE_ERROR。如果 sqlite3_table_column_metadata() 的列名参数是 NULL 指针，则该例程仅检查表是否存在，表存在时返回 SQLITE_OK，不存在时返回 SQLITE_ERROR。如果在 sqlite3_table_column_metadata(X,D,T,C,...) 调用中表名参数 T 为 NULL，则结果为未定义行为。

此函数通过第二、第三和第四个参数确定列。第二个参数是包含指定表的数据库名称（即 “main”、“temp” 或附加数据库）或 NULL。如果为 NULL，则会使用数据库引擎解析未限定表引用时相同的算法，在所有附加数据库中搜索该表。

此函数的第三和第四个参数分别是所需列的表名和列名。

元数据通过写入传递给本函数的第 5 个及后续参数的内存位置返回。这些参数中的任意一个都可以为 NULL，此时对应的元数据项将被省略。

参数输出
类型		说明

第 5 项const char*数据类型
第 6 项const char*默认排序规则名称
第 7 项int如果列有 NOT NULL 约束则为真
第 8 项int如果列是主键的一部分则为真
第 9 项int如果列是 AUTOINCREMENT 则为真

用于声明类型和排序规则名称的字符指针所指向的内存在下一次调用任何 SQLite API 函数之前都是有效的。

如果指定的表实际上是视图，则返回一个 错误代码。

如果指定的列是 “rowid”、“oid” 或 “_rowid_”，并且该表不是 WITHOUT ROWID 表，并且一个 INTEGER PRIMARY KEY 列已被显式声明，则输出参数将针对显式声明的列进行设置。如果没有 INTEGER PRIMARY KEY 列，则 rowid 的输出如下所示：

data type: "INTEGER"
collation sequence: "BINARY"
not null: 0
primary key: 1
auto increment: 0

此函数会读取并解析磁盘上的所有数据库模式（如果尚未完成），并在加载模式时遇到任何错误时返回错误。
