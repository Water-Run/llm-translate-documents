SQLite C 接口

预处理语句扫描状态操作码

#define SQLITE_SCANSTAT_NLOOP    0
#define SQLITE_SCANSTAT_NVISIT   1
#define SQLITE_SCANSTAT_EST      2
#define SQLITE_SCANSTAT_NAME     3
#define SQLITE_SCANSTAT_EXPLAIN  4
#define SQLITE_SCANSTAT_SELECTID 5
#define SQLITE_SCANSTAT_PARENTID 6
#define SQLITE_SCANSTAT_NCYCLE   7

以下常量可用于 sqlite3_stmt_scanstatus(S,X,T,V) 接口的 T 参数。每个常量指定 sqlite3_stmt_scanstatus() 返回的不同指标。

当返回给 V 的值是字符串时，用于存储该字符串的空间由预处理语句 S 管理，并在 S 被析构时自动释放。

并非所有值对所有查询元素都可用。如果某个值不可用，当输出变量为数字时会设置为 -1；如果是字符串（SQLITE_SCANSTAT_NAME），则设置为 NULL。

SQLITE_SCANSTAT_NLOOP
V 参数指向的 sqlite3_int64 变量将被设置为第 X 个循环运行的总次数。

SQLITE_SCANSTAT_NVISIT
V 参数指向的 sqlite3_int64 变量将被设置为第 X 个循环所有迭代检查的总行数。

SQLITE_SCANSTAT_EST
V 参数指向的 "double" 变量将被设置为查询规划器对第 X 个循环每次迭代输出平均行数的估算。如果查询规划器的估算准确，那么该值将近似于 NVISIT/NLOOP 的商，且与具有相同 SELECTID 的所有前置循环的该值的乘积将是当前循环的 NLOOP 值。

SQLITE_SCANSTAT_NAME
V 参数指向的 "const char *" 变量会被设置为一个以 UTF-8 编码的以零结尾的字符串，包含用于第 X 个循环的索引或表的名称。

SQLITE_SCANSTAT_EXPLAIN
V 参数指向的 "const char *" 变量会被设置为一个以 UTF-8 编码的以零结尾的字符串，包含第 X 个循环的 EXPLAIN QUERY PLAN 描述。

SQLITE_SCANSTAT_SELECTID
V 参数指向的 "int" 变量会被设置为第 X 个查询计划元素的 id。该 id 在语句中唯一。select-id 与 EXPLAIN QUERY PLAN 查询第一列输出的值相同。

SQLITE_SCANSTAT_PARENTID
V 参数指向的 "int" 变量会被设置为当前查询元素的父元素 id（如果适用），如果查询元素没有父元素则设置为 0。该值与 EXPLAIN QUERY PLAN 查询第二列返回的值相同。

SQLITE_SCANSTAT_NCYCLE
sqlite3_int64 输出值设为在处理该查询元素期间（根据处理器时间戳计数器）所经过的周期数。该值并非对所有查询元素都可用——如果不可用则输出变量设置为 -1。
