SQLite C 接口

确定虚拟表约束的排序序列

const char *sqlite3_vtab_collation(sqlite3_index_info*,int);

此函数只能在调用xBestIndex方法的虚拟表中被调用。此函数返回一个指向字符串的指针，该字符串表示应在其参数所标识的约束上进行文本比较时使用的适当排序序列名称。

第一个参数必须是传给sqlite3_index_info对象的指针，该对象是 xBestIndex() 方法的第一个参数。第二个参数必须是传入 xBestIndex 的 sqlite3_index_info 结构中 aConstraint[] 数组的索引。

重要提示： 第一个参数必须与传入xBestMethod()方法的指针完全相同。第一个参数不得是指向不同sqlite3_index_info对象的指针，即便其为完全相同的副本。

返回值按如下方式计算：

  1. 如果约束来自包含COLLATE 运算符的 WHERE 子句表达式，则返回该 COLLATE 运算符所指定的排序名称。

  2. 如果不存在 COLLATE 运算符，但约束所针对的列在通过COLLATE 子句定义的列排序序列中指定了替代排序序列，且该列定义来自传入sqlite3_declare_vtab()的 CREATE TABLE 语句，则返回该替代排序序列的名称。

  3. 否则，返回“BINARY”。

另请参阅 对象列表、 常量列表以及 函数列表。
