在默认配置下，SQLite API 例程会返回 30 个整数结果代码之一。然而，实践表明其中许多结果代码粒度过粗，并不能像程序员希望的那样提供足够的问题信息。为了解决这个问题，更新版本的 SQLite（自 2006-10-09 的 3.3.8 及以后版本）加入了对额外结果代码的支持，以便提供更详细的错误信息。这些扩展结果代码可以通过每个数据库连接级别的sqlite3_extended_result_codes() API 启用或禁用。或者，也可以使用sqlite3_extended_errcode()来获取最近一次错误对应的扩展代码。

另请参阅 对象、 常量以及 函数的列表。
