SQLite C 接口

设置 Setlk 超时

int sqlite3_setlk_timeout(sqlite3*, int ms, int flags);

此例程仅在 SQLITE_ENABLE_SETLK_TIMEOUT 构建中有用。如果 VFS 支持阻塞锁，它会为指定数据库句柄在 wal 模式数据库上获取的符合条件的锁设置以毫秒为单位的超时。在非 SQLITE_ENABLE_SETLK_TIMEOUT 构建中，或 VFS 不支持阻塞锁时，此函数不执行任何操作。

向此函数传入 0 会完全禁用阻塞锁。传入 -1 请求 VFS 尽可能长时间阻塞——如果可能则无限期阻塞。传入其他任何负值的结果未定义。

在内部，每个 SQLite 数据库句柄存储两个超时值——忙等待超时（用于回滚模式数据库，或当 VFS 不支持阻塞锁时）和 setlk 超时（用于 wal 模式数据库上的阻塞锁）。sqlite3_busy_timeout() 方法设置两个值，而本函数仅设置 setlk 超时值。因此，要为单个数据库句柄配置不同的忙等待超时和 setlk 超时值，应先调用 sqlite3_busy_timeout()，然后再调用此函数。

每当连接到 wal 模式数据库的连接数从 1 降到 0 时，最后一个连接会对数据库获取独占锁，然后进行检查点并删除 wal 文件。在执行这些操作期间，任何尝试从数据库读取的新连接都会以 SQLITE_BUSY 错误失败。或者，如果向此 API 传入 SQLITE_SETLK_BLOCK_ON_CONNECT 标志，则新连接会阻塞，直到独占锁被释放。
