SQLite C 接口

反序列化数据库

int sqlite3_deserialize(
  sqlite3 *db,            /* The database connection */
  const char *zSchema,    /* Which DB to reopen with the deserialization */
  unsigned char *pData,   /* The serialized database content */
  sqlite3_int64 szDb,     /* Number of bytes in the deserialization */
  sqlite3_int64 szBuf,    /* Total size of buffer pData[] */
  unsigned mFlags         /* Zero or more SQLITE_DESERIALIZE_* flags */
);

sqlite3_deserialize(D,S,P,N,M,F) 接口使数据库连接 D 与数据库 S 断开，然后根据 P 中的序列化内容将 S 重新以内存数据库的方式打开。如果 S 是 NULL 指针，则使用主数据库。序列化的数据库 P 大小为 N 字节。M 是缓冲区 P 的大小，可能大于 N。如果 M 大于 N 且 F 中未设置 SQLITE_DESERIALIZE_READONLY 位，则只要总大小不超过 M 字节，SQLite 就可以向内存数据库添加内容。

如果 F 中设置了 SQLITE_DESERIALIZE_FREEONCLOSE 位，则当数据库连接关闭时，SQLite 会在序列化缓冲区上调用 sqlite3_free()。如果设置了 SQLITE_DESERIALIZE_RESIZEABLE 位，则当数据库写入操作导致缓冲区大小超过 M 字节时，SQLite 会尝试使用 sqlite3_realloc64() 来增加缓冲区大小。

应用程序在数据库连接 D 关闭之前不得修改缓冲区 P 或使其失效。

如果数据库当前处于读取事务中或正在参与备份操作，则 sqlite3_deserialize() 接口会返回 SQLITE_BUSY 错误。

无法将数据反序列化到 TEMP 数据库。如果 sqlite3_deserialize(D,S,P,N,M,F) 的 S 参数为 "temp"，则该函数返回 SQLITE_ERROR。

反序列化后的数据库不应处于 WAL 模式。如果数据库处于 WAL 模式，则任何尝试使用数据库文件的操作都会导致 SQLITE_CANTOPEN 错误。应用程序可以在调用 sqlite3_deserialize(D,S,P,N,M,F) 之前，将输入数据库 P 的文件格式版本号（第 18 和 19 字节）设置为 0x01，以将数据库文件强制置于回滚模式，从而规避此限制。

如果 sqlite3_deserialize(D,S,P,N,M,F) 因任何原因失败，并且参数 F 中设置了 SQLITE_DESERIALIZE_FREEONCLOSE 位，则在返回之前会对参数 P 调用 sqlite3_free()。

如果 SQLite 在编译时使用了 SQLITE_OMIT_DESERIALIZE 选项，则省略此接口。
