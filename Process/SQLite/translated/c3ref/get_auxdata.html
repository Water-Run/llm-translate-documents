SQLite C 接口

函数辅助数据

void *sqlite3_get_auxdata(sqlite3_context*, int N);
void sqlite3_set_auxdata(sqlite3_context*, int N, void*, void (*)(void*));

这些函数可被（非聚合）SQL函数用来将辅助数据与参数值相关联。如果在查询执行过程中相同的参数值被传递给同一个SQL函数的多个调用，在某些情况下关联的辅助数据可能会被保留。一个可能用到此功能的示例是正则表达式匹配函数。正则表达式的编译版本可以作为与模式字符串关联的辅助数据存储。只要模式字符串保持不变，编译后的正则表达式就可以在同一个函数的多次调用中重复使用。

sqlite3_get_auxdata(C,N) 接口返回一个指针，指向由 sqlite3_set_auxdata(C,N,P,X) 函数与应用定义函数的第 N 个参数值关联的辅助数据。N 为最左边的函数参数时为零。如果与函数参数没有关联任何辅助数据，sqlite3_get_auxdata(C,N) 接口返回 NULL 指针。

sqlite3_set_auxdata(C,N,P,X) 接口将 P 保存为应用定义函数第 N 个参数的辅助数据。随后的 sqlite3_get_auxdata(C,N) 调用会返回最近一次 sqlite3_set_auxdata(C,N,P,X) 调用中的 P（如果辅助数据仍然有效）或者在辅助数据已被丢弃时返回 NULL。在每次 sqlite3_set_auxdata(C,N,P,X) 调用且 X 不为 NULL 时，当辅助数据被丢弃时，SQLite 会准确调用一次析构函数 X，并将参数 P 传入。SQLite 可以随时丢弃辅助数据，包括：

  • 当相应的函数参数发生变化时，或者
  • 当 sqlite3_reset() 或 sqlite3_finalize() 针对该 SQL 语句被调用时，或者
  • 当 sqlite3_set_auxdata() 在同一参数上再次被调用时，或者
  • 当原始 sqlite3_set_auxdata() 调用期间发生内存分配错误时，或者
  • 当原始 sqlite3_set_auxdata() 调用期间如果该函数在查询规划阶段而不是查询执行阶段被求值时，就像开启 SQLITE_ENABLE_STAT4 时有时发生的。

特别注意最后两个要点。sqlite3_set_auxdata(C,N,P,X) 中的析构函数 X 可能会被立即调用，甚至在 sqlite3_set_auxdata() 接口返回之前。因此 sqlite3_set_auxdata() 应该在函数实现的末尾附近调用，并且在调用 sqlite3_set_auxdata() 之后函数实现不应再使用 P。此外，紧接着对应的 sqlite3_set_auxdata() 调用之后调用 sqlite3_get_auxdata() 在 sqlite3_set_auxdata() 调用期间发生内存不足或函数在查询规划阶段而不是查询执行阶段被求值时，仍可能返回 NULL。

在实践中，对于编译时常量（包括字面值、参数以及由它们组成的表达式）所对应的函数参数，其辅助数据在函数调用之间会被保存。

这些接口中 N 参数的值应为非负数。未来的扩展可能会利用负的 N 值来定义新的函数缓存行为。

这些例程必须在运行该 SQL 函数的同一线程中被调用。

另请参见：sqlite3_get_clientdata() 和 sqlite3_set_clientdata()。
