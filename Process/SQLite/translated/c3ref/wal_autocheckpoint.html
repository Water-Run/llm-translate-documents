SQLite C 接口

配置自动检查点

int sqlite3_wal_autocheckpoint(sqlite3 *db, int N);

sqlite3_wal_autocheckpoint(D,N) 是围绕 sqlite3_wal_hook() 的一个包装器，它会使数据库连接D上的任何数据库在提交事务后， 如果在预写日志文件中有N个或更多帧，就自动执行检查点。 将N参数设为零或负值会完全禁用自动检查点。

该函数注册的回调会替换任何使用 sqlite3_wal_hook() 注册的现有回调。同样地，通过 sqlite3_wal_hook() 注册回调会禁用由此函数配置的自动检查点机制。

wal_autocheckpoint pragma 可用于从 SQL 中调用此接口。

该机制启动的检查点是 PASSIVE 的。

每个新的数据库连接默认启用自动检查点，阈值为1000页或 SQLITE_DEFAULT_WAL_AUTOCHECKPOINT。

只有当默认设置对于特定应用程序被认为不理想时，才有必要使用此接口。
