SQLite C 接口

确定 SQL 语句是否完整

int sqlite3_complete(const char *sql);
int sqlite3_complete16(const void *sql);

这些例程在命令行输入过程中非常有用，用来判断当前输入的文本是否构成完整的 SQL 语句，或者在将文本提交给 SQLite 解析之前是否还需要额外的输入。 这些例程在输入字符串看起来是完整 SQL 语句时返回 1。 如果语句以分号标记结束并且不是格式正确的 CREATE TRIGGER 语句的前缀，则认为该语句是完整的。 嵌入在字符串字面量、带引号的标识符名称或注释中的分号并不是独立的标记（它们属于所包含的标记），因此不算作语句终止符。 最后一个分号之后的空白和注释会被忽略。

这些例程在语句不完整时返回 0。 如果内存分配失败，则返回 SQLITE_NOMEM。

这些例程不会解析 SQL 语句，因此无法检测语法错误的 SQL。

如果在调用 sqlite3_initialize() 之前 SQLite 尚未初始化，则 sqlite3_complete16() 会自动调用 sqlite3_initialize()。 如果该初始化失败，则 sqlite3_complete16() 的返回值无论输入 SQL 是否完整都将为非零。

sqlite3_complete() 的输入必须是以零结尾的 UTF-8 字符串。

sqlite3_complete16() 的输入必须是以零结尾、符合本机字节序的 UTF-16 字符串。
