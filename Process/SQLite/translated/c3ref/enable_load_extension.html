SQLite C 接口

启用或禁用扩展加载

int sqlite3_enable_load_extension(sqlite3 *db, int onoff);

为了避免在尚未准备好处理扩展加载的旧应用中造成安全漏洞，同时也为了在评估用户输入的 SQL 时禁用扩展加载，提供下面的 API 来打开和关闭 sqlite3_load_extension() 机制。

扩展加载默认是关闭的。将 sqlite3_enable_load_extension() 调用的 onoff 设置为 1 即可开启扩展加载，将 onoff 设置为 0 即可再次关闭扩展加载。

此接口可启用或禁用 C API sqlite3_load_extension() 以及 SQL 函数 load_extension()。使用 sqlite3_db_config(db,SQLITE_DBCONFIG_ENABLE_LOAD_EXTENSION,..) 则只启用或禁用 C API。

安全警告：建议使用 SQLITE_DBCONFIG_ENABLE_LOAD_EXTENSION 方法开启扩展加载，而不是使用此接口，这样 load_extension() SQL 函数保持禁用。这可以防止 SQL 注入让攻击者获得扩展加载的能力。
