SQLite C 接口

动态字符串的状态

int sqlite3_str_errcode(sqlite3_str*);
int sqlite3_str_length(sqlite3_str*);
char *sqlite3_str_value(sqlite3_str*);

这些接口返回 sqlite3_str 对象的当前状态。

如果在 sqlite3_str X 中构造动态字符串的过程中之前发生过任何错误，那么 sqlite3_str_errcode(X) 方法将返回适当的错误代码。sqlite3_str_errcode(X) 方法在发生任何内存不足错误后返回 SQLITE_NOMEM，如果动态字符串的大小超过 SQLITE_MAX_LENGTH 则返回 SQLITE_TOOBIG，如果没有发生任何错误则返回 SQLITE_OK。

sqlite3_str_length(X) 方法返回 sqlite3_str 对象 X 正在构造的动态字符串的当前长度（以字节为单位）。sqlite3_str_length(X) 返回的长度不包括零终止字节。

sqlite3_str_value(X) 方法返回一个指向 X 当前正在构造的动态字符串内容的指针。sqlite3_str_value(X) 返回的值由 sqlite3_str 对象 X 管理，后续对相同 sqlite3_str 对象的任何方法都可能释放或修改它。应用程序在对同一对象调用任何后续方法之后不得再使用 sqlite3_str_value(X) 返回的指针。只要不写入超出 0 到 sqlite3_str_length(X) 范围之外的字节，并且在任何后续 sqlite3_str 方法调用之后不再读写任何字节，应用程序可以修改 sqlite3_str_value(X) 返回的字符串内容。
