SQLite C 接口

一步查询执行接口

int sqlite3_exec(
  sqlite3*,                                  /* An open database */
  const char *sql,                           /* SQL to be evaluated */
  int (*callback)(void*,int,char**,char**),  /* Callback function */
  void *,                                    /* 1st argument to callback */
  char **errmsg                              /* Error msg written here */
);

sqlite3_exec() 接口是围绕 sqlite3_prepare_v2()、sqlite3_step() 和 sqlite3_finalize() 的便利包装，允许应用程序运行多个 SQL 语句而无需编写大量 C 代码。

sqlite3_exec() 接口在作为第 1 个参数传入的数据库连接的上下文中， 执行传入其第 2 个参数的零个或多个 UTF-8 编码、以分号分隔的 SQL 语句。 如果传递给 sqlite3_exec() 的第 3 个参数的回调函数不为 NULL，则对于从评估过的 SQL 语句中输出的每个结果行都会调用该函数。 sqlite3_exec() 的第 4 个参数在每次回调调用时会传递给回调函数的第 1 个参数。 如果传递给 sqlite3_exec() 的回调指针为 NULL，那么将不会调用任何回调函数，结果行也会被忽略。

如果在评估传入 sqlite3_exec() 的 SQL 语句时发生错误，则当前语句的执行停止，后续语句会被跳过。 如果传给 sqlite3_exec() 的第 5 个参数非 NULL，则任何错误信息都写入通过 sqlite3_malloc() 获取的内存并通过第 5 个参数返回。 为避免内存泄漏，应用程序在不再需要通过 sqlite3_exec() 的第 5 个参数返回的错误信息字符串之后，应调用 sqlite3_free()。 如果 sqlite3_exec() 的第 5 个参数非 NULL 且未发生错误，则 sqlite3_exec() 在返回之前会将第 5 个参数中的指针设置为 NULL。

如果 sqlite3_exec() 的回调返回非零值，则 sqlite3_exec() 例程返回 SQLITE_ABORT，不再调用回调函数，也不再执行任何后续的 SQL 语句。

sqlite3_exec() 回调函数的第 2 个参数是结果中的列数。sqlite3_exec() 回调的第 3 个参数是指向字符串的指针数组，就像来自 sqlite3_column_text() 一样，每个列对应一个字符串。 如果结果行的某个元素为 NULL，则 sqlite3_exec() 回调对应的字符串指针为 NULL 指针。sqlite3_exec() 回调的第 4 个参数是指向字符串的指针数组，每个项代表通过 sqlite3_column_name() 获取的相应结果列的名称。

如果传给 sqlite3_exec() 的第 2 个参数是 NULL 指针、指向空字符串的指针，或仅包含空白和/或 SQL 注释的指针，则不会评估任何 SQL 语句，数据库也不会发生更改。

限制：

  • 应用程序必须确保传给 sqlite3_exec() 的第 1 个参数是有效且打开的数据库连接。
  • 应用程序在 sqlite3_exec() 运行期间不得关闭传入 sqlite3_exec() 的第 1 个参数指定的数据库连接。
  • 应用程序在 sqlite3_exec() 运行期间不得修改传入第 2 个参数的 SQL 语句文本。
  • 应用程序在返回后不得解引用传递给回调的第 3 和第 4 个参数所指向的数组或字符串指针。
