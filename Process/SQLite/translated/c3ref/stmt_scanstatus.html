SQLite C 接口

Prepared Statement Scan Status

int sqlite3_stmt_scanstatus(
  sqlite3_stmt *pStmt,      /* Prepared statement for which info desired */
  int idx,                  /* Index of loop to report on */
  int iScanStatusOp,        /* Information desired.  SQLITE_SCANSTAT_* */
  void *pOut                /* Result written here */
);
int sqlite3_stmt_scanstatus_v2(
  sqlite3_stmt *pStmt,      /* Prepared statement for which info desired */
  int idx,                  /* Index of loop to report on */
  int iScanStatusOp,        /* Information desired.  SQLITE_SCANSTAT_* */
  int flags,                /* Mask of flags defined below */
  void *pOut                /* Result written here */
);

这些接口返回有关 pStmt 的预测和实际性能的信息。高级应用可以利用此接口比较预测性能与实际性能，在发现差异时发出警告和/或重新运行ANALYZE。

由于预计很少使用此接口，仅在 SQLite 使用 SQLITE_ENABLE_STMT_SCANSTATUS 编译时选项编译时可用。

“iScanStatusOp” 参数决定返回哪种状态信息。“iScanStatusOp” 必须是scanstatus 选项之一，否则此接口的行为未定义。所请求的测量结果写入 “pOut” 参数指向的变量。

“flags” 参数必须传入标志掩码。目前只定义了一个标志 - SQLITE_SCANSTAT_COMPLEX。如果指定 SQLITE_SCANSTAT_COMPLEX，则所有由 “EXPLAIN QUERY PLAN” 输出报告的查询计划元素都可获得状态信息。如果未指定 SQLITE_SCANSTAT_COMPLEX，则只有对应查询循环的查询计划元素（“EXPLAIN QUERY PLAN” 输出中的 “SCAN...” 和 “SEARCH...” 元素）可用。调用 sqlite3_stmt_scanstatus() 等价于使用 flags 参数为零调用 sqlite3_stmt_scanstatus_v2()。

“idx” 参数标识要检索统计信息的特定查询元素。查询元素从零开始编号。值为 -1 时可能检索整个查询的统计信息。如果 idx 超出范围——小于 -1 或大于或等于用于实现语句的查询元素总数——将返回非零值，pOut 指向的变量保持不变。
