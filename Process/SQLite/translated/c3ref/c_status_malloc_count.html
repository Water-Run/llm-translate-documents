SQLite C Interface

Status Parameters

#define SQLITE_STATUS_MEMORY_USED          0
#define SQLITE_STATUS_PAGECACHE_USED       1
#define SQLITE_STATUS_PAGECACHE_OVERFLOW   2
#define SQLITE_STATUS_SCRATCH_USED         3  /* NOT USED */
#define SQLITE_STATUS_SCRATCH_OVERFLOW     4  /* NOT USED */
#define SQLITE_STATUS_MALLOC_SIZE          5
#define SQLITE_STATUS_PARSER_STACK         6
#define SQLITE_STATUS_PAGECACHE_SIZE       7
#define SQLITE_STATUS_SCRATCH_SIZE         8  /* NOT USED */
#define SQLITE_STATUS_MALLOC_COUNT         9

这些整数常量指定了可以通过 sqlite3_status() 返回的各种运行时状态参数。

SQLITE_STATUS_MEMORY_USED
此参数是通过 sqlite3_malloc() 直接或间接分配的当前内存总量。此数值包括应用对 sqlite3_malloc() 的调用以及 SQLite 库的内部内存使用。由 SQLITE_CONFIG_PAGECACHE 控制的辅助页缓存内存不包含在此参数内。返回值为 sqlite3_mem_methods 中 xSize 方法报告的分配大小之和。

SQLITE_STATUS_MALLOC_SIZE
此参数记录交给 sqlite3_malloc() 或 sqlite3_realloc()（或其内部等价方法）的最大内存分配请求。只有 sqlite3_status() 的 *pHighwater 参数返回的值才有意义。写入 *pCurrent 参数的值未定义。

SQLITE_STATUS_MALLOC_COUNT
此参数记录当前已分配的独立内存块数量。

SQLITE_STATUS_PAGECACHE_USED
此参数返回通过 SQLITE_CONFIG_PAGECACHE 配置的 pagecache 内存分配器 中已使用的页面数。返回值以页面为单位，而非字节。

SQLITE_STATUS_PAGECACHE_OVERFLOW
此参数返回由于 SQLITE_CONFIG_PAGECACHE 缓冲区无法满足而被迫溢出到 sqlite3_malloc() 的页缓存分配字节数。返回值包括因分配大小超过 SQLITE_CONFIG_PAGECACHE 的 "sz" 参数而溢出的分配，以及因页缓存空间耗尽而溢出的分配。

SQLITE_STATUS_PAGECACHE_SIZE
此参数记录交给 pagecache 内存分配器 的最大内存分配请求。只有 sqlite3_status() 的 *pHighwater 参数返回的值才有意义。写入 *pCurrent 参数的值未定义。

SQLITE_STATUS_SCRATCH_USED
不再使用。

SQLITE_STATUS_SCRATCH_OVERFLOW
不再使用。

SQLITE_STATUS_SCRATCH_SIZE
不再使用。

SQLITE_STATUS_PARSER_STACK
*pHighwater 参数记录解析器栈的最大深度。*pCurrent 的值未定义。只有在 SQLite 使用 YYTRACKMAXSTACKDEPTH 编译时，*pHighwater 的值才有意义。

可能会不时添加新的状态参数。
