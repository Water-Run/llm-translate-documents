SQLite C 接口

记录数据库快照

int sqlite3_snapshot_get(
  sqlite3 *db,
  const char *zSchema,
  sqlite3_snapshot **ppSnapshot
);

sqlite3_snapshot_get(D,S,P) 接口尝试创建一个新的 sqlite3_snapshot 对象，用于记录数据库连接 D 中模式 S 的当前状态。 成功时， sqlite3_snapshot_get(D,S,P) 接口会将新创建的 sqlite3_snapshot 对象的指针写入 *P，并返回 SQLITE_OK。 如果在调用本函数时模式 S 上尚未打开读事务，则会自动打开一个。

如果本函数打开了读事务，则可确保返回的快照对象在读事务关闭之前不会被数据库写入者或检查点操作使无效。 如果在调用本函数时读事务已经打开，则不能保证这一点。此时，数据库上的任何后续写入或检查点操作可能使返回的快照句柄失效，即使读事务仍然保持打开状态。

以下条件必须为真才能使本函数成功。如果在调用 sqlite3_snapshot_get() 时以下任何一条为假，则返回 SQLITE_ERROR。 此时 *P 的最终值未定义。

  • 数据库句柄不得处于自动提交模式。

  • 数据库连接 D 的模式 S 必须是WAL 模式数据库。

  • 数据库连接 D 的模式 S 上不得存在打开的写事务。

  • 自当前 wal 文件在磁盘上创建以来必须至少写入过一个或多个事务（由任意连接）。这意味着在首次打开的 wal 模式数据库上立即不能获取快照；必须先写入至少一个事务。

本函数也可能返回 SQLITE_NOMEM。如果以自动提交模式调用数据库句柄但因其他原因失败， 是否在模式 S 上打开了读事务则未定义。

成功调用 sqlite3_snapshot_get() 后返回的 sqlite3_snapshot 对象必须使用 sqlite3_snapshot_free() 释放，以避免内存泄漏。

sqlite3_snapshot_get() 接口仅在使用 SQLITE_ENABLE_SNAPSHOT 编译时选项时可用。
