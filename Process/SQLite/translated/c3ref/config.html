SQLite C 接口

配置 SQLite 库

int sqlite3_config(int, ...);

sqlite3_config() 接口用于对 SQLite 进行全局配置更改，以便将 SQLite 调整为应用程序的特定需求。默认配置对大多数应用程序是推荐的，因此通常不需要此例程。它是为具有不寻常需求的少数应用程序提供的。

sqlite3_config() 接口不是线程安全的。应用程序必须确保在 sqlite3_config() 运行时，其他线程不会调用其他 SQLite 接口。

sqlite3_config() 的第一个参数是一个整数配置选项，用于确定要配置 SQLite 的哪个属性。后续的参数则取决于第一个参数指定的配置选项。

对于大多数配置选项，sqlite3_config() 接口只能在使用sqlite3_initialize()进行库初始化之前或在通过sqlite3_shutdown()关闭之后调用。可以在任何时间调用的特殊配置选项称为“随时配置选项”。如果 sqlite3_config() 在sqlite3_initialize()之后且在sqlite3_shutdown()之前被调用，并且第一个参数不是随时配置选项，那么 sqlite3_config() 调用将返回 SQLITE_MISUSE。需要注意的是，sqlite3_config() 可以作为应用程序定义的sqlite3_os_init() 实现的一部分被调用。

当配置选项设置成功时，sqlite3_config() 返回 SQLITE_OK。如果该选项未知或 SQLite 无法设置该选项，则此例程返回非零的错误代码。
