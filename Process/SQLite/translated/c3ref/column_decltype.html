SQLite C 接口

查询结果的声明数据类型

const char *sqlite3_column_decltype(sqlite3_stmt*,int);
const void *sqlite3_column_decltype16(sqlite3_stmt*,int);

第一个参数是一个预处理语句。 如果该语句是一个SELECT语句，并且该SELECT返回结果集的第 N 列是一个表列（不是表达式或子查询），则返回该表列的声明类型。 如果结果集的第 N 列是表达式或子查询，则返回 NULL 指针。 返回的字符串始终使用 UTF-8 编码。

例如，给定如下数据库模式：

CREATE TABLE t1(c1 VARIANT);

以及要编译的以下语句：

SELECT c1 + 1, c1 FROM t1;

此例中，该例程会针对第二个结果列 (i==1) 返回字符串 “VARIANT”，针对第一个结果列 (i==0) 返回 NULL 指针。

SQLite 使用动态运行时类型。因此，仅仅因为某列声明包含特定类型，并不意味着存储在该列中的数据就一定是该声明类型。SQLite 是强类型的，但类型是动态的，而非静态的。类型与单个值关联，而不是与用来保存这些值的容器关联。
