SQLite C 接口

创建和销毁 VFS 文件名

sqlite3_filename sqlite3_create_filename(
  const char *zDatabase,
  const char *zJournal,
  const char *zWal,
  int nParam,
  const char **azParam
);
void sqlite3_free_filename(sqlite3_filename);

这些接口供 VFS shim 实现使用，在该上下文之外没有实际用途。

sqlite3_create_filename(D,J,W,N,P) 会分配内存，用于保存数据库文件名 D 的一个版本，以及对应的日志文件 J、WAL 文件 W 和包含 N 个 URI 键/值对的数组 P。sqlite3_create_filename(D,J,W,N,P) 的返回值是一个数据库文件名的指针，可安全传递给如下例程：

  • sqlite3_uri_parameter(),
  • sqlite3_uri_boolean(),
  • sqlite3_uri_int64(),
  • sqlite3_uri_key(),
  • sqlite3_filename_database(),
  • sqlite3_filename_journal(), or
  • sqlite3_filename_wal().
如果发生内存分配错误，sqlite3_create_filename() 可能会返回 NULL 指针。从 sqlite3_create_filename(X) 获得的内存在之后必须通过对 sqlite3_free_filename(Y) 的对应调用释放。

sqlite3_create_filename(D,J,W,N,P) 中的参数 P 应该是一个包含 2*N 个指向字符串的指针数组。该数组中的每对指针对应一个查询参数的键与值。如果 N 为零，则 P 参数可以为 NULL 指针。P 数组中的 2*N 个指针都不得为 NULL，键指针也不应为空字符串。sqlite3_create_filename(D,J,W,N,P) 中的 D、J 或 W 参数也都不得为 NULL 指针，但可以是空字符串。

sqlite3_free_filename(Y) 例程释放先前由 sqlite3_create_filename() 获得的内存分配。若 Y 为 NULL 指针，调用 sqlite3_free_filename(Y) 是一个无害的空操作。

如果传给 sqlite3_free_filename(Y) 的 Y 参数既不是 NULL 指针，也不是之前通过 sqlite3_create_filename() 获得的指针，那么可能会发生诸如堆损坏或段错误等严重问题。在调用 sqlite3_free_filename(Y) 后，不应再次使用 Y 的值。这意味着，如果某个 VFS 的 sqlite3_vfs.xOpen() 方法使用 Y 被调用，那么在调用 sqlite3_free_filename(Y) 之前，也应当调用相应的 sqlite3_module.xClose() 方法。
