SQLite C 接口

打开 BLOB 以进行增量 I/O

int sqlite3_blob_open(
  sqlite3*,
  const char *zDb,
  const char *zTable,
  const char *zColumn,
  sqlite3_int64 iRow,
  int flags,
  sqlite3_blob **ppBlob
);

此接口打开位于数据库 zDb 中表 zTable 的第 iRow 行、zColumn 列的BLOB 句柄；换句话说，就是下面 SQL 所选择的同一个 BLOB：

SELECT zColumn FROM zDb.zTable WHERE rowid = iRow;

参数 zDb 不是包含数据库的文件名，而是数据库的符号名称。对于附加数据库，此名称是 ATTACH 语句中 AS 关键字之后的名称。主数据库文件的名称为 "main"。对于 TEMP 表，数据库名称为 "temp"。

如果 flags 参数非零，则以可读写方式打开 BLOB。若 flags 参数为零，则以只读方式打开 BLOB。

成功时返回 SQLITE_OK，并在 *ppBlob 中存储新的BLOB 句柄。否则返回错误代码，除非错误代码为 SQLITE_MISUSE，此时 *ppBlob 被设置为 NULL。这意味着，只要不滥用 API，在此函数返回后调用 sqlite3_blob_close() 释放 *ppBlob 总是安全的。

如果下列任意一项为真，则该函数会以 SQLITE_ERROR 失败：

  • 数据库 zDb 不存在，
  • 表 zTable 在数据库 zDb 中不存在，
  • 表 zTable 是 WITHOUT ROWID 表，
  • 列 zColumn 不存在，
  • 表中不存在第 iRow 行，
  • 指定行 iRow 的列 zColumn 包含的值不是 TEXT 或 BLOB，
  • 列 zColumn 属于索引、PRIMARY KEY 或 UNIQUE 约束，且以读/写方式打开该 BLOB，
  • 启用了 外键约束，列 zColumn 属于子键定义，且以读/写方式打开该 BLOB。

除非返回 SQLITE_MISUSE，否则此函数会设置数据库连接的错误代码和消息，可通过sqlite3_errcode()、sqlite3_errmsg()及相关函数访问。

通过 sqlite3_blob_open() 引用的 BLOB 可以使用sqlite3_blob_read() 接口读取，也可以通过sqlite3_blob_write() 修改。可以使用sqlite3_blob_reopen() 接口将BLOB 句柄移到同一张表的不同一行。然而，在BLOB 句柄打开之后，其列、表或数据库均不能改变。

如果 BLOB 句柄指向的行被UPDATE、DELETE，或由于ON CONFLICT 的副作用而修改，则该 BLOB 句柄会被标记为“已过期”。只要该行的任一列被更改，即便不是 BLOB 句柄所在的列，也会发生这种情况。对已过期 BLOB 句柄调用sqlite3_blob_read()或sqlite3_blob_write()会返回SQLITE_ABORT。在 BLOB 过期前写入的更改不会因过期而回滚。只要事务继续完成，这些更改最终会提交。

使用sqlite3_blob_bytes() 接口确定已打开 BLOB 的大小。此接口无法更改 BLOB 的大小。使用UPDATE SQL 命令更改 BLOB 的大小。

sqlite3_bind_zeroblob() 与sqlite3_result_zeroblob() 接口以及内建的zeroblob SQL 函数可用于创建一个全零 BLOB，以便通过增量 BLOB 接口读取或写入。

为避免资源泄漏，每个打开的BLOB 句柄最终都应通过调用sqlite3_blob_close()释放。

另见：sqlite3_blob_close()、 sqlite3_blob_reopen()、sqlite3_blob_read()、 sqlite3_blob_bytes()、sqlite3_blob_write()。
