SQLite C 接口

同步类型标志

#define SQLITE_SYNC_NORMAL        0x00002
#define SQLITE_SYNC_FULL          0x00003
#define SQLITE_SYNC_DATAONLY      0x00010

当 SQLite 调用一个 sqlite3_io_methods 对象的 xSync() 方法时，会将这些整型值的组合作为第二个参数。

使用 SQLITE_SYNC_DATAONLY 标志表示同步操作只需将数据刷新到大容量存储。目录项信息不必刷新。如果该标志的低四位等于 SQLITE_SYNC_NORMAL，则意味着使用普通的 fsync() 语义。如果低四位等于 SQLITE_SYNC_FULL，则意味着使用 Mac OS X 风格的 fullsync 而不是 fsync()。

不要混淆 SQLITE_SYNC_NORMAL 和 SQLITE_SYNC_FULL 标志与 PRAGMA synchronous=NORMAL 和 PRAGMA synchronous=FULL 设置。synchronous pragma 决定何时调用 xSync VFS 方法，并在所有平台上统一应用。SQLITE_SYNC_NORMAL 和 SQLITE_SYNC_FULL 标志决定同步操作的力度和强度，并且对于 SQLite 默认代码来说，仅在 Mac OSX 上存在区别。（第三方 VFS 实现也可能区分 SQLITE_SYNC_NORMAL 和 SQLITE_SYNC_FULL，但在 SQLite 原生支持的操作系统中，只有 Mac OSX 在意这一差异。）
