SQLite C 接口

设置错误码与消息

int sqlite3_set_errmsg(sqlite3 *db, int errcode, const char *zErrMsg);

将作为第一个参数传入的数据库句柄的错误码设置为 errcode，并将错误消息设置为 zErrMsg 所指向的以零结尾字符串的拷贝。如果 zErrMsg 为 NULL，则错误消息将被设置为与所提供错误码关联的默认消息。随后对 sqlite3_errcode() 和 sqlite3_errmsg() 及类似函数的调用将返回此例程设置的值，取代 SQLite 之前设置的值。

如果成功设置错误码和错误消息，则此函数返回 SQLITE_OK；如果发生内存不足，则返回 SQLITE_NOMEM；如果数据库句柄为 NULL 或无效，则返回 SQLITE_MISUSE。

此例程设置的错误码和错误消息将一直生效，直到通过再次调用本例程或 SQLite 自身为反映后续某个 API 调用结果而更改它们。

此函数旨在供 SQLite 扩展或包装器使用。其意图是让扩展或包装器可以调用本例程来设置错误消息和错误码，从而在应用程序看来更像是 SQLite 核心功能。
