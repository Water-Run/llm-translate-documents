SQLite C 接口

初始化 SQLite 库

int sqlite3_initialize(void);
int sqlite3_shutdown(void);
int sqlite3_os_init(void);
int sqlite3_os_end(void);

sqlite3_initialize() 例程负责初始化 SQLite 库。sqlite3_shutdown() 例程会释放 sqlite3_initialize() 分配的所有资源。这些例程旨在帮助嵌入式系统完成进程初始化和关闭。使用 SQLite 的工作站应用通常不需要调用这些例程。

如果某次 sqlite3_initialize() 调用发生在进程生命周期内第一次调用 sqlite3_initialize() 之后，或者发生在 sqlite3_shutdown() 调用之后首次再调用 sqlite3_initialize()，则该次调用被视为“有效”调用。只有有效的 sqlite3_initialize() 调用才会进行初始化，其余调用都无害且不会产生任何操作。

如果某次 sqlite3_shutdown() 调用发生在最后一次 sqlite3_initialize() 之后首次调用 sqlite3_shutdown()，则该次调用被视为“有效”调用。只有有效的 sqlite3_shutdown() 调用才会执行任何反初始化操作，其余合法调用都是无害的空操作。

sqlite3_initialize() 接口是线程安全的，但 sqlite3_shutdown() 不是。sqlite3_shutdown() 接口只能由单个线程调用。在调用 sqlite3_shutdown() 之前，必须关闭所有打开的数据库连接，并释放所有其他 SQLite 资源。

sqlite3_initialize() 会在其他方面调用 sqlite3_os_init()。同样地，sqlite3_shutdown() 会调用 sqlite3_os_end()

sqlite3_initialize() 例程在成功时返回 SQLITE_OK。如果某种原因导致 sqlite3_initialize() 无法初始化库（例如无法分配所需资源如互斥锁），则会返回除 SQLITE_OK 之外的其他错误代码。

sqlite3_initialize() 例程由许多其他 SQLite 接口内部调用，因此应用程序通常不需要直接调用 sqlite3_initialize()。例如，sqlite3_open() 调用 sqlite3_initialize()，因此在调用 sqlite3_open() 时，如果 SQLite 尚未初始化，会自动进行初始化。但是，如果在编译时启用了 SQLITE_OMIT_AUTOINIT 配置选项，那么对 sqlite3_initialize() 的自动调用将被省略，应用程序必须在使用任何其他 SQLite 接口之前直接调用 sqlite3_initialize()。为了最大限度地提高可移植性，建议应用程序总是在使用任何其他 SQLite 接口之前直接调用 sqlite3_initialize()。将来版本的 SQLite 可能会要求如此。换句话说，在编译时启用了 SQLITE_OMIT_AUTOINIT 时所表现出的行为可能会在未来的某个版本中成为默认行为。

sqlite3_os_init() 例程执行 SQLite 库的操作系统相关初始化。sqlite3_os_end() 例程撤销 sqlite3_os_init() 的效果。这些例程通常完成的任务包括分配或释放静态资源、初始化全局变量、设置默认的 sqlite3_vfs 模块，或使用 sqlite3_config() 设置默认配置。

应用程序决不应直接调用 sqlite3_os_init() 或 sqlite3_os_end()。应用程序只应调用 sqlite3_initialize() 和 sqlite3_shutdown()。sqlite3_os_init() 接口会在 sqlite3_initialize() 中自动调用，sqlite3_os_end() 会在 sqlite3_shutdown() 中调用。在为 Unix、Windows 或 OS/2 编译 SQLite 时，sqlite3_os_init() 和 sqlite3_os_end() 的适当实现会内置于 SQLite 中。当为其他平台构建（使用 SQLITE_OS_OTHER=1 编译时选项）时，应用程序必须为 sqlite3_os_init() 和 sqlite3_os_end() 提供合适的实现。应用程序提供的 sqlite3_os_init() 或 sqlite3_os_end() 实现必须在成功时返回 SQLITE_OK，在失败时返回其他错误代码。
