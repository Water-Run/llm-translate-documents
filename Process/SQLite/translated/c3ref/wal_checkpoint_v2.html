SQLite C 接口

对数据库进行检查点

int sqlite3_wal_checkpoint_v2(
  sqlite3 *db,                    /* Database handle */
  const char *zDb,                /* Name of attached database (or NULL) */
  int eMode,                      /* SQLITE_CHECKPOINT_* value */
  int *pnLog,                     /* OUT: Size of WAL log in frames */
  int *pnCkpt                     /* OUT: Total number of frames checkpointed */
);

sqlite3_wal_checkpoint_v2(D,X,M,L,C) 接口在模式 M 下对数据库连接 D 的数据库 X 运行检查点操作。状态信息写入 L 和 C 指向的整数。M 参数必须是有效的 检查点模式：

SQLITE_CHECKPOINT_PASSIVE
尽可能多地检查点帧而无需等待任何数据库读取者或写入者完成，然后在日志中的所有帧都已完成检查点后同步数据库文件。忙等待处理程序回调在 SQLITE_CHECKPOINT_PASSIVE 模式下从不调用。另一方面，如果存在并发的读取者或写入者，消极模式可能会使检查点未完成。

SQLITE_CHECKPOINT_FULL
此模式会阻塞（调用 忙等待处理程序回调），直到没有数据库写入者并且所有读取者都从最新的数据库快照读取。然后它会检查点日志文件中的所有帧并同步数据库文件。该模式在等待期间阻止新的数据库写入者，但允许新的数据库读取者继续运行而不受影响。

SQLITE_CHECKPOINT_RESTART
此模式的工作方式与 SQLITE_CHECKPOINT_FULL 相同，另外在检查点日志文件之后，它会阻塞（调用 忙等待处理程序回调）直到所有读取者仅从数据库文件读取。这样可以确保下一个写入者将从日志文件开头重新启动。与 SQLITE_CHECKPOINT_FULL 一样，该模式在等待期间阻止新的数据库写入尝试，但不会阻碍读取者。

SQLITE_CHECKPOINT_TRUNCATE
此模式的工作方式与 SQLITE_CHECKPOINT_RESTART 相同，另外它还在成功返回之前将日志文件截断为零字节。

SQLITE_CHECKPOINT_NOOP
此模式始终检查点零帧。调用 NOOP 检查点的唯一原因是通过输出参数 *pnLog 和 *pnCkpt 访问 sqlite3_wal_checkpoint_v2() 返回的值。

如果 pnLog 不为 NULL，则 *pnLog 设置为日志文件中的帧总数，或者在因错误或数据库未处于 WAL 模式 而无法运行检查点时设置为 -1。如果 pnCkpt 不为 NULL，则 *pnCkpt 设置为日志文件中已检查点的帧总数（包括在调用该函数之前已检查点的帧），或者在因错误或数据库未处于 WAL 模式而无法运行检查点时设置为 -1。请注意，在成功完成 SQLITE_CHECKPOINT_TRUNCATE 之后，日志文件将被截断为零字节，因此 *pnLog 和 *pnCkpt 都将被设置为零。

所有调用都会在数据库文件上获取独占的“检查点”锁。如果任何其他进程同时正在运行检查点操作，则无法获取锁并返回 SQLITE_BUSY。即使配置了忙等待处理程序，在这种情况下也不会调用它。

SQLITE_CHECKPOINT_FULL、RESTART 和 TRUNCATE 模式也会在数据库文件上获取独占的“写入者”锁。如果写入者锁无法立即获取，并且配置了忙等待处理程序，则在调用并在忙等待处理程序返回 0 之前会重试写入者锁，直到锁被成功获取。在等待数据库读取者时也会调用忙等待处理程序，如上所述。如果在获取写入者锁之前或在等待数据库读取者时忙等待处理程序返回 0，检查点操作将从该点继续，方式与 SQLITE_CHECKPOINT_PASSIVE 相同——尽可能在不再阻塞的情况下检查点帧。在这种情况下返回 SQLITE_BUSY。

如果参数 zDb 为 NULL 或指向零长度字符串，则尝试对附加到 数据库连接 db 的所有 WAL 数据库执行指定操作。在这种情况下写入到输出参数 *pnLog 和 *pnCkpt 的值是不确定的。如果在处理一个或多个附加的 WAL 数据库时遇到 SQLITE_BUSY 错误，则仍尝试处理剩余附加数据库并在结束时返回 SQLITE_BUSY。如果在处理附加数据库时发生任何其他错误，则立即放弃处理并将错误代码返回给调用方。如果在处理附加数据库时没有遇到任何错误（无论是否为 SQLITE_BUSY），则返回 SQLITE_OK。

如果数据库 zDb 是附加数据库的名称但不处于 WAL 模式，则返回 SQLITE_OK，并将 *pnLog 和 *pnCkpt 都设置为 -1。如果 zDb 不为 NULL（或零长度字符串）且不是任何附加数据库的名称，则向调用方返回 SQLITE_ERROR。

除非它返回 SQLITE_MISUSE，否则 sqlite3_wal_checkpoint_v2() 接口会设置可由 sqlite3_errcode() 和 sqlite3_errmsg() 查询的错误信息。

PRAGMA wal_checkpoint 命令可用于从 SQL 调用此接口。
