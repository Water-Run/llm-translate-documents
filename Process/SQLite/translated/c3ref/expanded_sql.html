SQLite C 接口

检索语句 SQL

const char *sqlite3_sql(sqlite3_stmt *pStmt);
char *sqlite3_expanded_sql(sqlite3_stmt *pStmt);
#ifdef SQLITE_ENABLE_NORMALIZE
const char *sqlite3_normalized_sql(sqlite3_stmt *pStmt);
#endif

sqlite3_sql(P) 接口返回一个指针，指向用于创建准备语句 P 的 UTF-8 SQL 文本副本前提是 P 由 sqlite3_prepare_v2()、sqlite3_prepare_v3()、sqlite3_prepare16_v2() 或 sqlite3_prepare16_v3() 创建。sqlite3_expanded_sql(P) 接口返回一个指针，指向包含准备语句 P 的 SQL 文本、并展开了绑定参数后的 UTF-8 字符串。sqlite3_normalized_sql(P) 接口返回一个指针，指向包含准备语句 P 的标准化 SQL 文本的 UTF-8 字符串。用于标准化 SQL 语句的语义未明确规定，且可能会改变。至少文字值会被替换成合适的占位符。

例如，如果使用 SQL 文本 "SELECT $abc,:xyz" 创建准备语句，且参数 $abc 绑定为整数 2345，而参数 :xyz 未绑定，则 sqlite3_sql() 会返回原始字符串 "SELECT $abc,:xyz"，但 sqlite3_expanded_sql() 会返回 "SELECT 2345,NULL"。

如果可用内存不足以容纳结果，或者结果会超过由SQLITE_LIMIT_LENGTH确定的最大字符串长度，那么 sqlite3_expanded_sql() 接口会返回 NULL。

SQLITE_TRACE_SIZE_LIMIT 编译时选项限制了绑定参数展开后的大小。SQLITE_OMIT_TRACE 编译时选项会导致 sqlite3_expanded_sql() 始终返回 NULL。

sqlite3_sql(P) 和 sqlite3_normalized_sql(P) 返回的字符串由 SQLite 管理，并在准备语句完成时自动释放。另一方面，sqlite3_expanded_sql(P) 返回的字符串来自 sqlite3_malloc()，必须由应用程序通过传递给 sqlite3_free() 来释放。

仅当定义了SQLITE_ENABLE_NORMALIZE 编译时选项时，sqlite3_normalized_sql() 接口才可用。
