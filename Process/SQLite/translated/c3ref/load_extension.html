SQLite C 接口

加载扩展

int sqlite3_load_extension(
  sqlite3 *db,          /* 将扩展加载到这个数据库连接 */
  const char *zFile,    /* 包含扩展的共享库名称 */
  const char *zProc,    /* 入口点。如果为 0，则从 zFile 推导 */
  char **pzErrMsg       /* 如果不为 0，将错误信息放在这里 */
);

该接口从指定文件加载 SQLite 扩展库。

sqlite3_load_extension() 接口尝试加载位于文件 zFile 中的 SQLite 扩展 库。如果无法直接加载该文件，则尝试添加各种操作系统特定的扩展名进行加载。 因此，例如，如果 “samplelib” 无法加载，则可能会尝试 “samplelib.so”、 “samplelib.dylib” 或 “samplelib.dll” 等名称。

入口点是 zProc。 zProc 可能为 0，此时 SQLite 会尝试自行构造一个入口点名称。它首先尝试 “sqlite3_extension_init”。 如果仍然失败，则构造名称 “sqlite3_X_init”，其中 X 由文件名中从最后一个 “/” 到紧随其后的第一个 “.” 之间的所有 ASCII 字母字符的小写等价组成，并忽略任何开头的 “lib”。 sqlite3_load_extension() 接口在成功时返回 SQLITE_OK，如果出现问题则返回 SQLITE_ERROR。 如果发生错误且 pzErrMsg 不为 0，则 sqlite3_load_extension() 接口应尝试填充 *pzErrMsg， 该错误信息文本存储在通过 sqlite3_malloc() 获取的内存中。调用函数应通过调用 sqlite3_free() 来释放这段内存。

调用此 API 之前必须使用 sqlite3_enable_load_extension() 或 sqlite3_db_config(db,SQLITE_DBCONFIG_ENABLE_LOAD_EXTENSION,1,NULL) 启用扩展加载，否则将返回错误。

安全警告：建议仅使用 SQLITE_DBCONFIG_ENABLE_LOAD_EXTENSION 方法启用此接口。 应避免使用 sqlite3_enable_load_extension() 接口。 这样可以保持 SQL 函数 load_extension() 处于禁用状态，并防止 SQL 注入使攻击者获得扩展加载权限。
