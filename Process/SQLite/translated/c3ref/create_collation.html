SQLite C 接口

定义新的排序序列

int sqlite3_create_collation(
  sqlite3*,
  const char *zName,
  int eTextRep,
  void *pArg,
  int(*xCompare)(void*,int,const void*,int,const void*)
);
int sqlite3_create_collation_v2(
  sqlite3*,
  const char *zName,
  int eTextRep,
  void *pArg,
  int(*xCompare)(void*,int,const void*,int,const void*),
  void(*xDestroy)(void*)
);
int sqlite3_create_collation16(
  sqlite3*,
  const void *zName,
  int eTextRep,
  void *pArg,
  int(*xCompare)(void*,int,const void*,int,const void*)
);

这些函数添加、删除或修改与第一个参数指定的排序规则相关联的数据库连接。

排序规则的名称对于 sqlite3_create_collation() 和 sqlite3_create_collation_v2() 是 UTF-8 字符串，对于 sqlite3_create_collation16() 是以本机字节顺序的 UTF-16 字符串。根据 sqlite3_strnicmp() 比较相等的排序规则名称视为相同名称。

第三个参数 (eTextRep) 必须是以下常量之一：

  • SQLITE_UTF8,
  • SQLITE_UTF16LE,
  • SQLITE_UTF16BE,
  • SQLITE_UTF16, 或
  • SQLITE_UTF16_ALIGNED.
eTextRep 参数决定传递给排序函数回调 xCompare 的字符串的编码。 SQLITE_UTF16 和 SQLITE_UTF16_ALIGNED 常量会强制字符串采用本机字节顺序的 UTF16 编码。 SQLITE_UTF16_ALIGNED 的值还会强制字符串以偶数字节地址开始。

第四个参数 pArg 是一个应用程序数据指针，它作为第一个参数传递给排序函数回调。

第五个参数 xCompare 是一个指向排序函数的指针。 可以使用相同名称但不同 eTextRep 参数注册多个排序函数，SQLite 会使用需要最少数据转换的函数。 如果 xCompare 参数为 NULL，则删除该排序函数。当所有具有相同名称的排序函数都被删除时，该排序规则便无法再使用。

排序函数回调会在传入一份 pArg 应用程序数据指针副本以及两个使用 eTextRep 参数指定编码的字符串时被调用。传递给排序函数回调的两个整数参数分别表示两个字符串的字节长度。排序函数必须返回一个整数，该整数在第一个字符串小于、等于或大于第二个字符串时分别为负值、零或正值。对于相同的输入，排序函数必须始终返回相同的结果。如果使用不同 eTextRep 值注册了两个或更多排序函数到同一个排序规则名称，那么在使用等效字符串调用时，所有函数的返回结果必须等价。排序函数对于所有字符串 A、B、C 必须满足以下性质：

  1. 如果 A==B，则 B==A。
  2. 如果 A==B 且 B==C，则 A==C。
  3. 如果 A<B，则 B>A。
  4. 如果 A<B 且 B<C，则 A<C。

如果某个排序函数未能满足上述任何一项约束，且该排序函数被注册并使用，则 SQLite 的行为是未定义的。

sqlite3_create_collation_v2() 的行为与 sqlite3_create_collation() 相同，唯一的区别是当排序函数被删除时会在 pArg 上调用 xDestroy 回调。 当排序函数被后续的排序规则创建函数覆盖或当使用 sqlite3_close() 关闭 数据库连接 时，这些排序函数会被删除。

如果 sqlite3_create_collation_v2() 函数失败，则不会调用 xDestroy 回调。调用 sqlite3_create_collation_v2() 并且传入非 NULL 的 xDestroy 参数的应用程序应检查返回码，并自行释放应用程序数据指针，而不是期望 SQLite 代劳。这与其他所有 SQLite 接口不同。这种不一致虽令人遗憾，但若改变将破坏向后兼容性。

另请参阅：sqlite3_collation_needed() 与 sqlite3_collation_needed16()。
