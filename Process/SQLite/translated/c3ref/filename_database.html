SQLite C 接口

翻译文件名

const char *sqlite3_filename_database(sqlite3_filename);
const char *sqlite3_filename_journal(sqlite3_filename);
const char *sqlite3_filename_wal(sqlite3_filename);

这些例程可供自定义 VFS 实现使用，以在主数据库文件、日志文件和 WAL 文件之间翻译文件名。

如果 F 是 SQLite 内核传递给 VFS 的 sqlite 数据库文件、日志文件或 WAL 文件的名称，则 sqlite3_filename_database(F) 返回对应数据库文件的名称。

如果 F 是 SQLite 内核传递给 VFS 的 sqlite 数据库文件、日志文件或 WAL 文件的名称，或者 F 是从 sqlite3_db_filename() 获取的数据库文件名，则 sqlite3_filename_journal(F) 返回对应回滚日志文件的名称。

如果 F 是 SQLite 内核传递给 VFS 的 sqlite 数据库文件、日志文件或 WAL 文件的名称，或者 F 是从 sqlite3_db_filename() 获取的数据库文件名，则 sqlite3_filename_wal(F) 返回对应 WAL 文件的名称。

在上述所有情况下，如果 F 既不是从 SQLite 内核传递给 VFS 的数据库、日志或 WAL 文件名，也不是 sqlite3_db_filename() 的返回值，则结果未定义，并可能导致内存访问违规。
