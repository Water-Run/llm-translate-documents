SQLite C 接口

启用或禁用共享分页缓存

int sqlite3_enable_shared_cache(int);

此例程启用或禁用数据库缓存与架构数据结构在访问同一个数据库的连接之间共享。 当参数为 true 时分享被启用，为 false 时分享被禁用。

如果 SQLite 使用 -DSQLITE_OMIT_SHARED_CACHE 编译，则该接口会被省略。推荐使用-DSQLITE_OMIT_SHARED_CACHE 编译选项，因为不建议使用共享缓存模式。

缓存共享的启用与禁用对整个进程有效。 这是自 SQLite 3.5.0 版本（2007-09-04）以来的改变。 在更早的 SQLite 版本中，共享是针对每个线程单独启用或禁用的。

该接口设置的缓存共享模式影响之后所有对sqlite3_open()、sqlite3_open_v2()和sqlite3_open16()的调用。 已有的数据库连接继续使用它们打开时所生效的共享模式。

如果共享缓存成功启用或禁用，则此例程返回SQLITE_OK。否则会返回一个错误码。

默认情况下共享缓存是禁用的，建议保持此状态。换句话说，请不要使用此例程。该接口仅为历史兼容性而保留，但不建议使用。任何共享缓存的使用都不被鼓励。如果必须使用共享缓存，推荐仅通过sqlite3_open_v2() 接口并使用SQLITE_OPEN_SHAREDCACHE标志，为单个数据库连接启用共享缓存。

注意：此方法在 MacOS X 10.7 与 iOS 5.0 上被禁用，将始终返回 SQLITE_MISUSE。在这些系统上，应通过sqlite3_open_v2()并与SQLITE_OPEN_SHAREDCACHE一起， 为每个数据库连接启用共享缓存模式。

在写入 32 位整数为原子的处理器上，此接口是线程安全的。
