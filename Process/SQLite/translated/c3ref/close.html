SQLite C 接口

关闭数据库连接

int sqlite3_close(sqlite3*);
int sqlite3_close_v2(sqlite3*);

sqlite3_close() 与 sqlite3_close_v2() 例程是 sqlite3 对象的析构函数。 如果 sqlite3 对象被成功销毁并释放所有关联资源，调用 sqlite3_close() 与 sqlite3_close_v2() 会返回 SQLITE_OK。

理想情况下，应用程序在尝试关闭该 sqlite3 对象之前，应先 finalize 所有 prepared statements、关闭 所有 BLOB 句柄，并完成所有与该对象关联的 sqlite3_backup 对象（通过 finish）。如果数据库连接仍关联着未 finalize 的 prepared statements、未关闭的 BLOB 句柄或未完成的 sqlite3_backup 对象，则调用 sqlite3_close() 会使数据库连接保持打开并返回 SQLITE_BUSY。如果在存在未 finalize 的 prepared statements、未关闭的 BLOB 句柄或未完成的 sqlite3_backups 时调用 sqlite3_close_v2()，它仍会返回 SQLITE_OK，但不会立即释放数据库连接，而是将其标记为不可用的“僵尸”，并在所有 prepared statements 被 finalize、所有 BLOB 句柄关闭以及所有备份完成后自动安排释放数据库连接。sqlite3_close_v2() 接口旨在用于垃圾回收式的宿主语言，这类语言中析构函数的调用顺序是任意的。

如果在事务仍然打开时销毁了 sqlite3 对象，该事务会自动回滚。

传递给 sqlite3_close(C) 与 sqlite3_close_v2(C) 的 C 参数必须是 NULL 指针或从 sqlite3_open()、sqlite3_open16() 或 sqlite3_open_v2() 获得的、尚未关闭的 sqlite3 对象指针。以 NULL 指针调用 sqlite3_close() 或 sqlite3_close_v2() 是一个无害的空操作。
