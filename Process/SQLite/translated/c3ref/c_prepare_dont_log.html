SQLite C 接口

准备标志

#define SQLITE_PREPARE_PERSISTENT              0x01
#define SQLITE_PREPARE_NORMALIZE               0x02
#define SQLITE_PREPARE_NO_VTAB                 0x04
#define SQLITE_PREPARE_DONT_LOG                0x10

这些常量定义了可以传递给 "prepFlags" 参数的各种标志，该参数属于 sqlite3_prepare_v3() 和 sqlite3_prepare16_v3() 接口。

未来的 SQLite 版本可能会增加新的标志。

SQLITE_PREPARE_PERSISTENT
SQLITE_PREPARE_PERSISTENT 标志向查询规划器提示该预处理语句将长期保留并可能重复使用多次。没有这个标志时，sqlite3_prepare_v3() 和 sqlite3_prepare16_v3() 假定预处理语句最多只会使用一次或几次，然后很快通过 sqlite3_finalize() 销毁。当前实现根据此提示避免使用 lookaside 内存，以免耗尽有限的 lookaside 内存。SQLite 的未来版本可能会以不同方式响应该提示。

SQLITE_PREPARE_NORMALIZE
SQLITE_PREPARE_NORMALIZE 标志是一个无操作。这个标志过去是任何希望使用 sqlite3_normalized_sql() 接口的预处理语句所必须的。 然而，现在所有预处理语句无论是否使用该标志都可以访问 sqlite3_normalized_sql() 接口。

SQLITE_PREPARE_NO_VTAB
SQLITE_PREPARE_NO_VTAB 标志使 SQL 编译器在语句使用任何虚拟表时返回错误（错误代码 SQLITE_ERROR）。

SQLITE_PREPARE_DONT_LOG
SQLITE_PREPARE_DONT_LOG 标志防止 SQL 编译器错误被发送到由 SQLITE_CONFIG_LOG 定义的错误日志。 例如，这可用于进行测试编译以确认某个 SQL 语法是否格式正确，当语法不正确时不会在全局错误日志中生成信息。 如果测试编译失败，sqlite3_prepare_v3() 调用在有无该标志的情况下返回相同的错误指示； 只是省略了记录错误的 sqlite3_log() 调用。
