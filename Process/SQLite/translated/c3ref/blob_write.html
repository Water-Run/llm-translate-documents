SQLite C 接口

逐步向BLOB写入数据

int sqlite3_blob_write(sqlite3_blob *, const void *z, int n, int iOffset);

此函数用于将调用者提供的缓冲区中的数据写入一个打开的 BLOB 句柄。从缓冲区 Z 开始，在偏移量 iOffset 处向打开的 BLOB 复制 N 字节的数据。

成功时，sqlite3_blob_write() 返回 SQLITE_OK。否则，返回一个 错误代码 或一个 扩展错误代码。除非返回 SQLITE_MISUSE，该函数会设置通过 sqlite3_errcode() 和 sqlite3_errmsg() 及相关函数可访问的 数据库连接 错误代码和消息。

如果传入的 BLOB 句柄 未以写入方式打开（传给 sqlite3_blob_open() 的标志为零），则此函数返回 SQLITE_READONLY。

此函数只能修改 BLOB 的内容；无法使用此 API 增加 BLOB 的大小。如果偏移量 iOffset 距离 BLOB 末端小于 N 字节，则返回 SQLITE_ERROR，并且不会写入任何数据。可以使用 sqlite3_blob_bytes() 接口确定 BLOB 的大小（从而确定 N+iOffset 的最大值）。如果 N 或 iOffset 小于零，则返回 SQLITE_ERROR，并且不会写入任何数据。

尝试向已过期的 BLOB 句柄 写入会失败，并返回 SQLITE_ABORT 错误代码。BLOB 句柄过期前对 BLOB 所做的写入不会因句柄的过期而回滚，尽管这些更改可能已被导致 BLOB 句柄过期的语句或其他独立语句覆盖。

此例程仅适用于由先前成功调用 sqlite3_blob_open() 创建的且尚未由 sqlite3_blob_close() 关闭的 BLOB 句柄。向此例程传入任何其他指针将导致未定义且可能是不希望的行为。

另请参见：sqlite3_blob_read()。
