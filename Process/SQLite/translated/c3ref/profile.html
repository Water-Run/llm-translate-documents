SQLite C 接口

已弃用的跟踪和分析函数

void *sqlite3_trace(sqlite3*,
   void(*xTrace)(void*,const char*), void*);
void *sqlite3_profile(sqlite3*,
   void(*xProfile)(void*,const char*,sqlite3_uint64), void*);

这些例程已被弃用。请使用 sqlite3_trace_v2() 接口 来代替这里所述的例程。

这些例程注册了可用于跟踪和分析 SQL 语句执行过程的回调函数。

sqlite3_trace() 注册的回调函数会在 sqlite3_step() 运行 SQL 语句的过程中在多个时刻被调用。 当语句第一次开始执行时，会以 UTF-8 形式传入 SQL 语句文本。 在进入每个触发子程序时，sqlite3_trace() 可能会被额外调用。触发器的回调包含标识该触发器的 UTF-8 SQL 注释。

SQLITE_TRACE_SIZE_LIMIT 编译时选项可用于限制 sqlite3_trace() 输出中绑定参数扩展的长度。

sqlite3_profile() 注册的回调函数会在每个 SQL 语句执行完成时被调用。分析回调包含原始语句文本以及该语句运行所花费的墙钟时间估计。分析回调的时间单位是纳秒，然而当前实现只能达到毫秒级分辨率，因此时间的最低六位数字没有实际意义。SQLite 的未来版本可能在分析回调中提供更高的分辨率。调用 sqlite3_trace() 或 sqlite3_trace_v2() 任何一个都会取消分析回调。
