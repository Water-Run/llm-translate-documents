SQLite C 接口

需要排序规则回调

int sqlite3_collation_needed(
  sqlite3*,
  void*,
  void(*)(void*,sqlite3*,int eTextRep,const char*)
);
int sqlite3_collation_needed16(
  sqlite3*,
  void*,
  void(*)(void*,sqlite3*,int eTextRep,const void*)
);

为了避免在使用数据库之前必须注册所有排序规则序列，可以针对数据库连接注册一个回调函数，每当需要未定义的排序规则序列时就会调用此函数。

如果使用 sqlite3_collation_needed() API 注册此函数，则会以 UTF-8 编码的字符串形式传入未定义排序规则序列的名称。如果使用 sqlite3_collation_needed16()，则会以本机字节序的 UTF-16 形式传入名称。调用任一函数都会替换现有的排序规则需求回调。

当回调被调用时，第一个参数是传给 sqlite3_collation_needed() 或 sqlite3_collation_needed16() 的第二个参数的副本。第二个参数是数据库连接。第三个参数是 SQLITE_UTF8、SQLITE_UTF16BE 或 SQLITE_UTF16LE 之一，表示所需排序规则函数最希望的形式。第四个参数是所需排序规则序列的名称。

回调函数应使用 sqlite3_create_collation()、sqlite3_create_collation16() 或 sqlite3_create_collation_v2() 注册所需的排序规则。
