SQLite C 接口

查找 IN 约束右侧的所有元素。

int sqlite3_vtab_in_first(sqlite3_value *pVal, sqlite3_value **ppOut);
int sqlite3_vtab_in_next(sqlite3_value *pVal, sqlite3_value **ppOut);

这些接口仅在 xFilter() 方法 的虚拟表 实现内部有用。 从任何其他上下文调用这些接口的结果 都是未定义的，并且可能有害。

调用 sqlite3_vtab_in_first(X,P) 或 sqlite3_vtab_in_next(X,P) 时的 X 参数应该是调用这些例程的 xFilter 方法的参数之一，特别是先前在 xBestIndex 方法中 使用 sqlite3_vtab_in() 接口为 一次性 IN 约束处理所选择的参数。 如果 X 参数不是为一次性 IN 约束处理而选择的 xFilter 参数， 则这些例程返回 SQLITE_ERROR。

使用这些例程通过如下代码访问 IN 约束右侧的所有值：

   for(rc=sqlite3_vtab_in_first(pList, &pVal);
       rc==SQLITE_OK && pVal;
       rc=sqlite3_vtab_in_next(pList, &pVal)
   ){
     // do something with pVal
   }
   if( rc!=SQLITE_DONE ){
     // an error has occurred
   }

调用 sqlite3_vtab_in_first(X,P) 和 sqlite3_vtab_in_next(X,P) 成功时返回 SQLITE_OK，并设置 *P 指向 IN 约束右侧的第一个或下一个值。 如果 IN 约束右侧没有更多值，则将 *P 设为 NULL，并且这些例程返回 SQLITE_DONE。 在发生故障时，返回值可能是其他值，例如 SQLITE_NOMEM。

这些例程返回的 *ppOut 值仅在下一次调用这些例程之前或从中调用这些例程的 xFilter 方法结束之前有效。如果虚拟表实现需要更久地保留 *ppOut 值，则必须 自行复制。*ppOut 值是受保护的。
