SQLite C 接口

xBestIndex() 中的约束值

int sqlite3_vtab_rhs_value(sqlite3_index_info*, int, sqlite3_value **ppVal);

此 API 只能在xBestIndex 方法的 虚表实现中使用。从 xBestIndex 方法外部调用此接口的结果是未定义的， 并且可能有害。

当在xBestIndex 方法的 虚表实现中调用 sqlite3_vtab_rhs_value(P,J,V) 接口时，其中 P 是传入 xBestIndex 的 sqlite3_index_info 对象指针的副本， J 是 P->aConstraint[] 的零基索引，则此例程尝试将 *V 设置为该约束右侧操作数的值（如果右侧操作数已知）。 如果右侧操作数未知，则 *V 设置为 NULL 指针。只有当 *V 被设置为某个值时，sqlite3_vtab_rhs_value(P,J,V) 接口才返回 SQLITE_OK。如果第 J 个约束的右侧不可用，则接口返回 SQLITE_NOTFOUND。 如果发生错误，sqlite3_vtab_rhs_value(P,J,V) 接口可以返回除 SQLITE_OK 或 SQLITE_NOTFOUND 之外的结果码。

只有在约束的右侧操作数是原始 SQL 语句中的字面值时，sqlite3_vtab_rhs_value() 接口才通常会成功。 如果右侧操作数是表达式、对其他列的引用或 主机参数，那么 sqlite3_vtab_rhs_value() 很可能返回 SQLITE_NOTFOUND。

某些约束，例如 SQLITE_INDEX_CONSTRAINT_ISNULL 和 SQLITE_INDEX_CONSTRAINT_ISNOTNULL，没有右侧操作数。 对于此类约束，sqlite3_vtab_rhs_value() 总是返回 SQLITE_NOTFOUND。

在 *V 中返回的 sqlite3_value 对象是受保护的 sqlite3_value，并在 xBestIndex 方法调用期间保持有效。当 xBestIndex 返回后，由 sqlite3_vtab_rhs_value() 返回的 sqlite3_value 对象会自动释放。

此例程名称中的“_rhs_”是“Right-Hand Side”（右侧）的缩写。
