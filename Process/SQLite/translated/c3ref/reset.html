SQLite C 接口

重置预处理语句对象

int sqlite3_reset(sqlite3_stmt *pStmt);

sqlite3_reset() 函数用于将 预处理语句 对象重置回初始状态，以便重新执行。 任何通过 sqlite3_bind_*() API 绑定值的 SQL 语句变量仍然保留其值。 使用 sqlite3_clear_bindings() 可清除绑定。

sqlite3_reset(S) 接口会将 预处理语句 S 重新置于其程序的起始位置。

sqlite3_reset(S) 的返回码表明先前对预处理语句 S 的评估是否成功完成。 如果 sqlite3_step(S) 从未在 S 上被调用，或者自上一次调用 sqlite3_reset(S) 以来 尚未再次调用 sqlite3_step(S)，则 sqlite3_reset(S) 将返回 SQLITE_OK。

如果最近一次对 预处理语句 S 调用 sqlite3_step(S) 时返回错误， 那么 sqlite3_reset(S) 将返回相应的 错误代码。 即使之前没有错误，但重置预处理语句的过程中出现新错误，sqlite3_reset(S) 接口也可能返回 错误代码。 例如，如果带有 RETURNING 子句的 INSERT 语句仅执行了一次 sqlite3_step(S)，该次调用可能返回 SQLITE_ROW，但整体语句仍可能失败，且 sqlite3_reset(S) 可能会因为锁定限制阻止数据库更改提交而返回 SQLITE_BUSY。 因此，应用在调用 sqlite3_reset(S) 时，即使此前的 sqlite3_step(S) 未表明问题，也应检查其返回码。

sqlite3_reset(S) 接口不会更改 预处理语句 S 上任何 绑定 的值。
