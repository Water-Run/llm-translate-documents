SQLite C 接口

获取 SQL 值

const void *sqlite3_value_blob(sqlite3_value*);
double sqlite3_value_double(sqlite3_value*);
int sqlite3_value_int(sqlite3_value*);
sqlite3_int64 sqlite3_value_int64(sqlite3_value*);
void *sqlite3_value_pointer(sqlite3_value*, const char*);
const unsigned char *sqlite3_value_text(sqlite3_value*);
const void *sqlite3_value_text16(sqlite3_value*);
const void *sqlite3_value_text16le(sqlite3_value*);
const void *sqlite3_value_text16be(sqlite3_value*);
int sqlite3_value_bytes(sqlite3_value*);
int sqlite3_value_bytes16(sqlite3_value*);
int sqlite3_value_type(sqlite3_value*);
int sqlite3_value_numeric_type(sqlite3_value*);
int sqlite3_value_nochange(sqlite3_value*);
int sqlite3_value_frombind(sqlite3_value*);

摘要：

sqlite3_value_blob→BLOB 值
sqlite3_value_double→REAL 值
sqlite3_value_int→32 位 INTEGER 值
sqlite3_value_int64→64 位 INTEGER 值
sqlite3_value_pointer→指针值
sqlite3_value_text→UTF-8 文本值
sqlite3_value_text16→以本机字节顺序存储的 UTF-16 文本值
sqlite3_value_text16be→UTF-16be 文本值
sqlite3_value_text16le→UTF-16le 文本值
   
sqlite3_value_bytes→以字节为单位的 BLOB 或 UTF-8 文本大小
sqlite3_value_bytes16   →  以字节为单位的 UTF-16 文本大小
sqlite3_value_type→值的默认数据类型
sqlite3_value_numeric_type   →  值的最佳数值数据类型
sqlite3_value_nochange   →  如果在针对虚拟表的 UPDATE 操作中该列未改变则返回 true。
sqlite3_value_frombind   →  如果值来自绑定参数则返回 true。

详细信息：

这些例程从受保护的 sqlite3_value 对象中提取类型、大小和内容信息。受保护的 sqlite3_value 对象用于将参数信息传递给实现应用定义的 SQL 函数和虚拟表的函数。

这些例程仅适用于受保护的 sqlite3_value 对象。任何尝试在未受保护的 sqlite3_value对象上使用这些例程的行为都不是线程安全的。

这些例程与相应的列访问函数的工作方式相同，只不过这些例程接受一个受保护的 sqlite3_value对象指针，而不是sqlite3_stmt*指针和整数列号。

sqlite3_value_text16() 接口以主机机器的本机字节序提取 UTF-16 字符串。sqlite3_value_text16be() 和 sqlite3_value_text16le() 接口分别以大端和小端提取 UTF-16 字符串。

如果sqlite3_value对象 V 是通过sqlite3_bind_pointer(S,I,P,X,D)或sqlite3_result_pointer(C,P,X,D)初始化的，并且 X 与 Y 是根据 strcmp(X,Y) 相等的字符串，那么 sqlite3_value_pointer(V,Y) 将返回指针 P，否则 sqlite3_value_pointer(V,Y) 返回 NULL。sqlite3_bind_pointer() 例程是为 SQLite 3.20.0 新增的指针传递接口的一部分。

sqlite3_value_type(V) 接口返回数据类型代码，表示sqlite3_value对象 V 的初始数据类型。返回值为SQLITE_INTEGER、SQLITE_FLOAT、SQLITE_TEXT、SQLITE_BLOB或SQLITE_NULL之一。其他接口可能会更改 sqlite3_value 对象的数据类型。例如，如果初始数据类型是 SQLITE_INTEGER，并且调用 sqlite3_value_text(V) 提取该整数的文本值，那么随后调用 sqlite3_value_type(V) 可能返回 SQLITE_TEXT。是否发生持久的内部数据类型转换是未定义的，并且可能在 SQLite 的不同版本之间发生变化。

sqlite3_value_numeric_type() 接口尝试对该值应用数值倾向。这意味着会尝试将该值转换为整数或浮点数。如果可以在不丢失信息的情况下完成此转换（换句话说，如果该值是看起来像数字的字符串），则执行转换。否则不会发生转换。转换后的数据类型会被返回。

在虚拟表的xUpdate方法中，当且仅当对应于 X 的列在调用 xUpdate 方法所实现的 UPDATE 操作中未发生改变，并且在之前为该列提取值的xColumn方法调用在未设置结果（很可能是因为查询了sqlite3_vtab_nochange()并发现该列未改变）时返回时，sqlite3_value_nochange(X) 接口才返回 true。在xUpdate方法中，对于 sqlite3_value_nochange(X) 返回 true 的任何值，在其他方面都将表现为 NULL。如果 sqlite3_value_nochange(X) 在非 UPDATE 语句的xUpdate 方法调用之外的任何地方被调用，则其返回值是任意且无意义的。

sqlite3_value_frombind(X) 接口在值 X 源自某个sqlite3_bind()接口时返回非零值。如果 X 源自 SQL 字面量值、表列或表达式，则 sqlite3_value_frombind(X) 返回零。

请特别注意，从sqlite3_value_blob()、sqlite3_value_text()或sqlite3_value_text16()返回的指针可能会在随后调用sqlite3_value_bytes()、sqlite3_value_bytes16()、sqlite3_value_text()或sqlite3_value_text16()时失效。

这些例程必须在提供sqlite3_value*参数的 SQL 函数所在的相同线程中调用。

只要输入参数正确，这些例程仅在格式转换期间发生内存不足错误时才会失败。仅以下接口子集可能遭遇内存不足错误：

  • sqlite3_value_blob()
  • sqlite3_value_text()
  • sqlite3_value_text16()
  • sqlite3_value_text16le()
  • sqlite3_value_text16be()
  • sqlite3_value_bytes()
  • sqlite3_value_bytes16()

如果发生内存不足错误，那么这些例程的返回值与该列包含 SQL NULL 值时的返回值相同。可以通过在获取可疑返回值之后并在对同一个数据库连接调用任何其他 SQLite 接口之前立即调用sqlite3_errcode()来区分有效的 SQL NULL 返回值和内存不足错误。
