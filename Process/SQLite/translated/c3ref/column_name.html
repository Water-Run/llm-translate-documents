SQLite C 接口

结果集中的列名

const char *sqlite3_column_name(sqlite3_stmt*, int N);
const void *sqlite3_column_name16(sqlite3_stmt*, int N);

这些例程返回在SELECT 语句结果集中某个特定列所分配的名称。sqlite3_column_name() 接口返回一个指向以空字符终止的 UTF-8 字符串的指针，而 sqlite3_column_name16() 返回一个指向以空字符终止的 UTF-16 字符串的指针。第一个参数是实现该SELECT 语句的预处理语句。第二个参数是列编号。最左边的列编号为 0。

返回的字符串指针在以下任意情况发生前都保持有效：预处理语句被sqlite3_finalize()销毁，或者针对某次执行第一调用sqlite3_step()自动重新准备语句，或者对同一列再次调用 sqlite3_column_name() 或 sqlite3_column_name16()。

如果在任一例程处理过程中 sqlite3_malloc() 失败（例如在 UTF-8 转换为 UTF-16 时），则返回 NULL 指针。

如果该列使用了 AS 子句，则结果列的名称为 AS 子句的值。如果没有 AS 子句，则列名称未指定，并可能随每个 SQLite 版本变化。
