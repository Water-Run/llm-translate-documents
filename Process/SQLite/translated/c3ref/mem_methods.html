SQLite C 接口

内存分配例程

typedef struct sqlite3_mem_methods sqlite3_mem_methods;
struct sqlite3_mem_methods {
  void *(*xMalloc)(int);         /* 内存分配函数 */
  void (*xFree)(void*);          /* 释放之前的分配 */
  void *(*xRealloc)(void*,int);  /* 调整分配大小 */
  int (*xSize)(void*);           /* 返回分配大小 */
  int (*xRoundup)(int);          /* 将请求大小向上舍入到分配大小 */
  int (*xInit)(void*);           /* 初始化内存分配器 */
  void (*xShutdown)(void*);      /* 反初始化内存分配器 */
  void *pAppData;                /* 作为 xInit() 和 xShutdown() 的参数 */
};

这个对象的一个实例定义了SQLite与底层内存分配例程之间的接口。

这个对象在SQLite接口中只在一个地方使用。当配置选项是 SQLITE_CONFIG_MALLOC或 SQLITE_CONFIG_GETMALLOC时， 指向该对象实例的指针作为 sqlite3_config()的参数。 通过创建该对象的实例并在配置期间将其传递给 sqlite3_config( SQLITE_CONFIG_MALLOC) ，一个般程序可以为SQLite所有动态内存需求指定一个替代的内存分配子系统。

请注意，SQLite 带有几个 内置内存分配器，它们对于绝大多数应用程序已经完全足够， 因此只有极少数具有特殊内存分配要求的应用程序才需要这个对象。 这个对象也在SQLite的测试过程中使用，以指定一个模拟内存耗尽情况的替代内存分配器， 从而验证SQLite能否从此类情况中优雅地恢复。

xMalloc、xRealloc 和 xFree 方法必须像标准C库中的 malloc()、realloc() 和 free() 函数一样工作。 SQLite 保证传递给 xRealloc 的第二个参数始终是之前调用 xRoundup 返回的值。

xSize 应该返回先前通过 xMalloc 或 xRealloc 获取的内存分配的大小。已分配的大小总是至少与请求的大小一样大，但可能更大。

xRoundup 方法返回在给定请求大小的条件下内存分配最终会被分配的大小。大多数内存分配器至少将内存分配向上调整到下一个8的倍数。有些分配器会向上调整到更大的倍数或2的幂。每个通过 sqlite3_malloc()或 sqlite3_realloc()发起的内存分配请求都会先调用 xRoundup。 如果 xRoundup 返回0，则对应的内存分配会失败。

xInit 方法负责初始化内存分配器。例如，它可能会分配所需的互斥体或初始化内部数据结构。 xShutdown 方法由 sqlite3_shutdown()（间接）调用，应释放 xInit 获得的资源。 pAppData 指针作为 xInit 和 xShutdown 的唯一参数。

SQLite 在调用 xInit 方法时会持有 SQLITE_MUTEX_STATIC_MAIN 互斥锁，因此 xInit 方法不必是线程安全的。 xShutdown 方法仅由 sqlite3_shutdown()调用，因此也无需线程安全。 对于其他方法，只要 SQLITE_CONFIG_MEMSTATUS 配置选项开启（默认开启），SQLite 就会持有 SQLITE_MUTEX_STATIC_MEM 互斥锁，因此这些方法会被自动串行化。 但如果禁用了 SQLITE_CONFIG_MEMSTATUS， 那么其他方法就必须线程安全，或自行安排串行化。

SQLite 不会在未调用 xShutdown() 的情况下多次调用 xInit()。
