SQLite C 接口

设置 SQL 函数的子类型

void sqlite3_result_subtype(sqlite3_context*,unsigned int);

sqlite3_result_subtype(C,T) 函数使得带有 应用定义 SQL 函数 的结果在 sqlite3_context C 中的子类型为值 T。在当前版本的 SQLite 中， 仅保留子类型 T 的低 8 位；高位会被丢弃。 SQLite 所保留的子类型字节数在未来版本中可能会增加。

每个调用此接口的 应用定义 SQL 函数 在其 SQLITE_RESULT_SUBTYPE 属性 用于注册该 SQL 函数时，应将该属性包含在其文本编码参数中。 如果调用 sqlite3_result_subtype() 的函数省略了 SQLITE_RESULT_SUBTYPE 属性， 则在某些情况下 sqlite3_result_subtype() 可能无法设置结果子类型。

如果 SQLite 是使用 -DSQLITE_STRICT_SUBTYPE=1 编译的，那么任何调用 sqlite3_result_subtype() 接口且未设置 SQLITE_RESULT_SUBTYPE 属性的 SQL 函数都会引发错误。 未来版本的 SQLite 可能会默认启用 -DSQLITE_STRICT_SUBTYPE=1。

另请参见 对象、 常量 和 函数 列表。
