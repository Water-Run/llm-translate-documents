SQLite C 接口

更改预编译语句的 EXPLAIN 设置

int sqlite3_stmt_explain(sqlite3_stmt *pStmt, int eMode);

sqlite3_stmt_explain(S,E) 接口更改预编译语句 S 的 EXPLAIN 设置。如果 E 为 0，则 S 成为普通预编译语句。如果 E 为 1，则 S 的行为就像其 SQL 文本以 "EXPLAIN" 开头。如果 E 为 2，则 S 的行为就像其 SQL 文本以 "EXPLAIN QUERY PLAN" 开头。

调用 sqlite3_stmt_explain(S,E) 可能会导致对 S 重新准备。SQLite 会尽量避免重新准备，但在首次切换到 EXPLAIN 或 EXPLAIN QUERY PLAN 模式时可能需要重新准备。

由于可能需要重新准备，如果 S 是使用 sqlite3_prepare() 而不是较新的 sqlite3_prepare_v2() 或 sqlite3_prepare_v3() 接口创建的，因此没有保存的 SQL 文本可用于重新准备，那么调用 sqlite3_stmt_explain(S,E) 将因为无法重新准备而返回 SQLITE_ERROR。

更改预编译语句的 explain 设置不会更改该语句的原始 SQL 文本。因此，如果原始 SQL 文本以 EXPLAIN 或 EXPLAIN QUERY PLAN 开头，但调用 sqlite3_stmt_explain(S,0) 将该语句转换为普通语句，那么即使该语句现在行为像普通 SQL 语句，sqlite3_sql(S) 输出中仍会包含 EXPLAIN 或 EXPLAIN QUERY PLAN 关键字。

如果 explain 模式成功更改，该例程返回 SQLITE_OK，否则返回错误代码。 explain 模式在语句处于激活状态时无法更改。因此，最好在调用 sqlite3_stmt_explain(S,E) 之前立即调用 sqlite3_reset(S)。
