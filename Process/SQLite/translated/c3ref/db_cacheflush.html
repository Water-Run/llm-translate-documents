SQLite C 接口

在事务中途将缓存写入磁盘

int sqlite3_db_cacheflush(sqlite3*);

如果在调用 sqlite3_db_cacheflush(D) 接口时，数据库连接 D 上正在打开写事务，那么分页缓存中所有当前未被使用的脏页都会被写入磁盘。脏页在被活跃的 SQL 语句创建的数据库游标读取时可能仍在使用，或者它是数据库文件的第一页（第一页始终为“正在使用”）。sqlite3_db_cacheflush(D) 接口会刷新所有模式的缓存——“main”、“temp”以及任何附加的数据库。

如果此函数在将脏页刷新到磁盘之前需要获取额外的数据库锁，它会这样做。如果这些锁无法立即获得且已配置忙等待处理程序回调，就会以通常方式调用该回调。如果仍然无法获得所需的锁，则跳过该数据库并尝试刷新下一个（如果存在）数据库的脏页。如果由于无法获得锁而跳过某些数据库，但没有发生其他错误，则此函数返回 SQLITE_BUSY。

如果在将脏页刷新到磁盘的过程中发生其他错误（例如 IO 错误或内存不足），则会放弃处理并立即向调用者返回 SQLite 错误代码。

否则，如果没有发生错误，sqlite3_db_cacheflush() 返回 SQLITE_OK。

此函数不会设置由 sqlite3_errcode() 和 sqlite3_errmsg() 函数返回的数据库句柄错误代码或消息。
