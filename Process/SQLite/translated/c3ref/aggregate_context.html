SQLite C 接口

获取聚合函数上下文

void *sqlite3_aggregate_context(sqlite3_context*, int nBytes);

聚合 SQL 函数的实现使用此例程来分配内存以存储它们的状态。

第一次针对特定聚合函数调用 sqlite3_aggregate_context(C,N) 例程时，SQLite 会分配 N 字节的内存，将其清零，并返回新内存的指针。在对同一聚合函数实例进行第二次及之后调用 sqlite3_aggregate_context() 时，会返回相同的缓冲区。通常在每次调用 xStep 回调时调用 sqlite3_aggregate_context()，在调用 xFinal 回调时再调用一次。在没有行匹配聚合查询的情况下，聚合函数实现的 xStep() 回调永远不会被调用，而 xFinal() 恰好调用一次。在这些情况下，sqlite3_aggregate_context() 可能首次在 xFinal() 中被调用。

sqlite3_aggregate_context(C,N) 例程在首次调用时会在 N 小于或等于零或者发生内存分配错误时返回 NULL 指针。

sqlite3_aggregate_context(C,N) 分配的空间大小由首次成功调用时的 N 参数决定。在同一聚合函数实例中之后的调用中修改 N 的值不会改变内存分配的大小。在 xFinal 回调中，通常在调用 sqlite3_aggregate_context(C,N) 时将 N 设置为 0，以避免不必要的内存分配。

当聚合查询结束时，SQLite 会自动释放 sqlite3_aggregate_context() 分配的内存。

第一个参数必须是SQL 函数上下文的副本，该上下文是实现聚合函数的 xStep 或 xFinal 回调例程的第一个参数。

此例程必须在聚合 SQL 函数运行的同一线程中调用。
