1. 概述

generate_series(START,STOP,STEP) 表值函数 是包含在 SQLite 源代码树中的 可加载扩展，并编译到 命令行 shell 中。generate_series() 表有一个名为“value”的可见结果列用于保存整数值， 行数由参数 START、STOP 和 STEP 决定。该表的第一行的值为 START，之后的行按 STEP 增量，直到不超过 STOP。

generate_series() 表还有额外的隐藏列，分别名为“start”、“stop”和“step”，它们的值对应于提供的或默认的 START、STOP 和 STEP 的有效值。 它还具有一个 rowid，可通过常规名称访问。

省略的参数会采用默认值。STEP 默认为 1。STOP 默认为 4294967295。从版本 3.37.0（2021-11-27）及以后开始，START 参数是必需的，如果 START 被省略或具有自引用或其他无法计算的值，将引发错误。旧版本对 START 的默认值为 0。可以通过在近期代码中使用 -DZERO_ARGUMENT_GENERATE_SERIES 编译来获得旧的行为。

1.1. 等价的递归公用表表达式

对于正的步长值，可以使用递归公用表表达式模拟 generate_series 表。如果三个参数分别为 \($start\)、\($end\) 和 \($step\)，那么等价的公用表表达式为： 

WITH RECURSIVE generate_series(value) AS (
  SELECT $start
  UNION ALL
  SELECT value+$step FROM generate_series
   WHERE value+$step<=$end
) ...

该公用表表达式无需加载扩展即可工作。另一方面，扩展更易编程且更快。

2. 使用示例

生成所有小于或等于 100 的 5 的倍数： 

SELECT value FROM generate_series(5,100,5);

生成 20 个随机整数值： 

SELECT random() FROM generate_series(1,20);

查找账户编号在 10000 到 20000 之间且是 100 的偶数倍的每位客户的姓名。 

SELECT customer.name
  FROM customer, generate_series(10000,20000,100)
 WHERE customer.id=value;
/* or */
SELECT name FROM customer
 WHERE id IN (SELECT value
                FROM generate_series(10000,20000,200));
