SQLite 中的文件格式变化

SQLite 数据库的底层文件格式不会以不兼容的方式变更。 如今市面上流通着字面上数万亿个 SQLite 数据库文件，SQLite 的开发者 承诺在未来数十年内持续支持这些文件。

在 SQLite 3.0.0（2004-06-18）之前，文件格式确实会在不同版本间发生变化。 但从那时起，文件格式就完全保持向后兼容。

我们说“向后兼容”，是指 新版 SQLite 总能读取并写入由旧版 SQLite 创建的数据库文件。 而通常情况下，SQLite 也具备“向前兼容性”，旧版 SQLite 可以读取并写入 新版 SQLite 创建的数据库文件。但有时会出现向前兼容性的中断。 有时会向文件格式中添加新特性。例如，WAL 模式在 3.7.0 版本（2010-07-21）中被添加。 SQLite 3.7.0 及更高版本能够读取并写入由较早版本的 SQLite 创建的所有数据库文件。 而较早版本的 SQLite 也可以读取并写入由 SQLite 3.7.0 及之后版本创建的数据库文件， 只要数据库未使用 WAL 模式。但在 3.7.0 之前的 SQLite 版本无法读取或写入使用 WAL 模式的数据库文件。

摘要

  • 新版 SQLite 总是可以读取和/或写入由旧版 SQLite 创建的数据库文件，最早可追溯到 3.0.0（2004-06-18）。

  • 只要数据库不使用该旧版本所不支持的新特性，到 3.0.0 为止的旧版 SQLite 也可以读取并写入由新版 SQLite 创建的数据库文件。
