COMPLETION() 表值函数

1. 概述

COMPLETION 扩展实现了一个名为“completion”的表值函数，可在交互式 SQL 输入过程中建议部分输入词的补全。completion 表例如可用于帮助实现制表符补全。

2. 详情

设计的查询接口是：

SELECT DISTINCT candidate COLLATE nocase
  FROM completion($prefix, $wholeline)
 ORDER BY 1;

上述查询将返回以 $prefix 开头的整个输入单词的建议。$wholeline 参数为从行首到插入点的所有文本，$wholeline 参数用于提供上下文。

$prefix 参数可能为 NULL，此时前缀由 $wholeline 推断。或者，如果没有可用的上下文信息或不需要上下文感知补全，$wholeline 参数可以为 NULL 或省略。

completion 表可能会返回重复候选项，而且返回顺序任意。示例查询中添加的 DISTINCT 关键字和 ORDER BY 是为了使结果唯一，并按字典序排序。

2.1. 示例用法

completion 表与 readline 或 linenoise 输入行编辑包一起，在命令行 shell中用于实现制表符补全。参见源码文件https://sqlite.org/src/file/src/shell.c.in了解示例代码。搜索“FROM completion”即可找到相关代码片段。

由于 completion 表内置于命令行 shell 以提供制表符补全，因此可直接在命令行 shell 中对 completion 表运行测试查询。只需输入如上示例所示的查询，填入适当的 $prefix 和 $wholeline 值，观察输出即可。

3. 限制

completion 表专为交互式使用设计，返回速度适合人工输入。在用户界面中只要响应几乎是瞬时的，就不会刻意追求非常高的效率。

截至撰写本文（2017-07-13），completion 虚拟表只查找 SQL 关键字以及模式、表和列名。$wholeline 中包含的上下文完全被忽略。未来的增强计划会尝试返回来自函数名、pragma 名称等其他来源的新补全，并考虑更多上下文。completion 表应被视为仍在开发中的功能。
