3. 内置标量 SQL 函数说明

abs(X)

abs(X) 函数返回数字参数 X 的绝对值。如果 X 为 NULL，则 abs(X) 返回 NULL。 如果 X 是无法转换为数字的字符串或 BLOB，则 abs(X) 返回 0.0。 如果 X 是整数 -9223372036854775808，则 abs(X) 会抛出整数溢出错误，因为 没有与之对应的 64 位二进制补码正值。

changes()

changes() 函数返回最近执行完成的 INSERT、DELETE 或 UPDATE 语句（不含较低级别触发器中的语句） 所影响的数据库行数。changes() SQL 函数是对 sqlite3_changes64() C/C++ 函数的包装， 因此遵循相同的变更计数规则。

char(X1,X2,...,XN)

char(X1,X2,...,XN) 函数返回一个字符串，其字符的 Unicode 代码点值分别由整数 X1 到 XN 提供。

coalesce(X,Y,...)

coalesce() 函数返回第一个非 NULL 参数的副本；若所有参数均为 NULL，则返回 NULL。coalesce() 至少 需要两个参数。

concat(X,...)

concat(...) 函数返回一个字符串，该字符串是其所有非 NULL 参数的字符串表示的连接。如果所有参数 均为 NULL，则 concat() 返回空字符串。

concat_ws(SEP,X,...)

concat_ws(SEP,...) 函数返回一个字符串，它是使用第一个参数的文本值作为分隔符， 将其后所有非 NULL 参数连接而成。如果第一个参数为 NULL，则 concat_ws() 返回 NULL。 如果第一个参数之后的所有参数都是 NULL，则 concat_ws() 返回空字符串。

format(FORMAT,...)

format(FORMAT,...) SQL 函数的行为类似于 C 语言的 sqlite3_mprintf() 函数 和标准 C 库中的 printf() 函数。第一个参数是格式字符串，用后续参数中的值构造输出字符串。 如果 FORMAT 参数缺失或为 NULL，则结果为 NULL。%n 格式会静默忽略且不占用参数。%p 是 %X 的别名。 %z 可以与 %s 互换。如果参数列表太少，则缺失参数视为 NULL，数值格式会转换为 0 或 0.0，%s 则转换为空字符串。 详情参见 内置 printf() 文档。

glob(X,Y)

glob(X,Y) 函数等价于表达式 "Y GLOB X"。注意 glob() 函数中的 X 和 Y 参数 相对于中缀 GLOB 运算符是颠倒的。Y 是字符串，X 是模式。 所以，例如以下两个表达式是等价的： 

     name GLOB '*helium*'
     glob('*helium*',name)

如果使用 sqlite3_create_function() 接口用其它实现重写 glob(X,Y) 函数， 那么 GLOB 运算符将调用该替代实现。

hex(X)

hex() 函数将其参数解释为 BLOB，并返回该 BLOB 内容的全大写十六进制表示字符串。

如果 hex(X) 中的参数 X 是整数或浮点数，则“将其解释为 BLOB”意味着先将数值转换为 UTF8 文本表示， 再将该文本解释为 BLOB。因此 "hex(12345678)" 输出 "3132333435363738"，而不是整数值的二进制表示 "0000000000BC614E"。

另请参见：unhex()

ifnull(X,Y)

ifnull() 函数返回其第一个非 NULL 参数的副本；若两个参数都为 NULL，则返回 NULL。ifnull() 必须 具有且仅具有两个参数。ifnull() 等价于具有两个参数的 coalesce()。

iif(B1,V1,...)
if(B1,V1,...)

iif(B1,V1,...,BN,VN) 函数以成对参数的形式传参。每对的第一个参数是布尔值，第二个参数是在该布尔值为真时返回的值。 iif() 会返回第一个为真的布尔值所对应的值。如果 iif() 的参数数量为奇数，则最后一个参数是在前面所有布尔值为假时返回的值。 如果参数数量为偶数且所有布尔值为假，则返回 NULL。iif() 函数至少需要两个参数。 iif() 实际上是 CASE 表达式的简写。例如 iif(X,Y,Z) 与 CASE WHEN X THEN Y ELSE Z END 在逻辑上等价并生成相同的 字节码。if() 函数只是 iif() 的另一种写法。

iif() 函数使用短路求值。只有在需要时才会计算参数。因此，如果某个值参数涉及昂贵计算（如复杂子查询）， 但对应的布尔值为假，则该昂贵计算永不执行。同样，在第一个布尔值为真之后的布尔参数也不会被评估。

iif() 函数最初要求正好三个参数。支持两个参数的版本以及将函数拼写为 "if()" 是在 SQLite 3.48.0（2025-01-14）中引入的。 允许使用超过 3 个参数是在 SQLite 3.49.0（2025-02-06）中添加的功能。

instr(X,Y)

instr(X,Y) 函数查找字符串 Y 在字符串 X 中的首次出现位置，并返回之前的字符数加 1；如果未在 X 中找到 Y，则返回 0。 如果 X 和 Y 都是 BLOB，则 instr(X,Y) 返回在第一次出现 Y 之前的字节数加 1；如果 X 中未出现 Y，则返回 0。 如果 instr(X,Y) 的两个参数都非 NULL 且不是 BLOB，则都解释为字符串。 如果 X 或 Y 为 NULL，则结果为 NULL。

last_insert_rowid()

last_insert_rowid() 函数返回调用该函数的数据库连接最近一次插入行的 ROWID。 last_insert_rowid() SQL 函数是对 sqlite3_last_insert_rowid() C/C++ 接口函数的包装。

length(X)

对于字符串值 X，length(X) 返回输入字符串 X 中第一个 U+0000 字符之前的 Unicode 代码点数（不是字节数）。 由于 SQLite 字符串通常不包含 NUL 字符，length(X) 通常返回字符串 X 的字符总数。 对于 BLOB 值 X，length(X) 返回 BLOB 中的字节数。如果 X 为 NULL，则 length(X) 为 NULL。 如果 X 是数值，则 length(X) 返回其字符串表示的长度。

注意，对于字符串，length(X) 返回字符串的字符或代码点长度，而非字节长度。 字符长度等于字符串中的字符数。对于 UTF-16 字符串，字符长度总与字节长度不同； 对于含多字节字符的 UTF-8 字符串，字符长度也可能与字节长度不同。若需查找字符串的字节长度，请使用 octet_length()。

对于 BLOB 值，length(X) 始终返回 BLOB 的字节长度。

对于字符串值，length(X) 必须将整个字符串加载到内存中以计算字符长度。但对于 BLOB 值， SQLite 已经知道 BLOB 中的字节数，因此不需要将整个值读入内存。因此，对于多兆字节值， 由于无需加载内容，length(X) 在处理 BLOB 时通常比处理字符串快得多。

like(X,Y)
like(X,Y,Z)

like() 函数用于实现 "Y LIKE X [ESCAPE Z]" 表达式。 如果存在可选的 ESCAPE 子句，则 like() 用三个参数调用；否则只用两个参数调用。 注意 like() 函数中的 X 和 Y 参数相对于中缀 LIKE 运算符是颠倒的。 X 是模式，Y 是用于匹配该模式的字符串。因此以下表达式等价： 

     name LIKE '%neon%'
     like('%neon%',name)

可以使用 sqlite3_create_function() 接口重写 like() 函数，从而更改 LIKE 运算符的行为。在重写 like() 函数时，可能需要同时重写两个参数和三个参数版本， 否则根据是否指定 ESCAPE 子句，可能调用不同的代码来实现 LIKE 运算符。

likelihood(X,Y)

likelihood(X,Y) 函数返回参数 X 本身。likelihood(X,Y) 中的 Y 必须是介于 0.0 与 1.0（含）之间的浮点常量。 likelihood(X) 函数是一个空操作，代码生成器会优化掉，于是在运行时（即调用 sqlite3_step() 时）不消耗 CPU 周期。 likelihood(X,Y) 的目的是为查询优化器提供提示，说明参数 X 是一个以约 Y 的概率为真的布尔值。 unlikely(X) 是 likelihood(X,0.0625) 的简写。 likely(X) 是 likelihood(X,0.9375) 的简写。

likely(X)

likely(X) 函数返回参数 X 本身。likely(X) 是一个空操作，代码生成器会优化掉，因此在运行时（即调用 sqlite3_step() 时）不消耗 CPU 周期。likely(X) 的作用是向查询优化器提示参数 X 是通常为真的布尔值。likely(X) 等价于 likelihood(X,0.9375)。 另见：unlikely(X)。

load_extension(X)
load_extension(X,Y)

load_extension(X,Y) 函数从名为 X 的共享库文件加载 SQLite 扩展，入口点为 Y。 load_extension() 的结果始终为 NULL。如果省略参数 Y，则使用默认入口点名。 如果扩展加载或初始化失败，load_extension() 会引发异常。

如果扩展尝试修改或删除 SQL 函数或排序序列，则 load_extension() 会失败。 扩展可以添加新函数或排序序列，但不能修改或删除已有函数或排序序列，因为这些对象可能在当前运行的 SQL 语句中被其它部分调用。 若要加载会更改或删除函数/排序序列的扩展，请使用 sqlite3_load_extension() C 接口。

出于安全考虑，扩展加载默认被禁用，必须通过提前调用 sqlite3_enable_load_extension() 才能启用。

lower(X)

lower(X) 函数返回字符串 X 的一个副本，其中所有 ASCII 字符都被转换为小写。 默认内置的 lower() 仅对 ASCII 字符有效。若需处理非 ASCII 字符，请加载 ICU 扩展。

ltrim(X)
ltrim(X,Y)

ltrim(X,Y) 函数返回一个字符串，通过从 X 的左侧删除出现在 Y 中的所有字符得到。 如果省略 Y 参数，ltrim(X) 会删除 X 左侧的空格。

max(X,Y,...)

多参数 max() 函数返回最大值参数；如果任一参数为 NULL，则返回 NULL。 多参数 max() 会按从左到右的顺序查找定义排序函数的参数，并在所有字符串比较中使用该排序函数。 如果没有参数定义排序函数，则使用 BINARY 排序函数。注意，拥有两个或多个参数时 max() 是简单函数， 若只给出单个参数，则作为 聚合函数 运算。

min(X,Y,...)

多参数 min() 函数返回最小值参数。多参数 min() 会按从左到右搜索定义排序函数的参数，并在所有字符串比较中使用该排序函数。 若没有参数定义排序函数，则使用 BINARY 排序函数。注意，拥有两个或更多参数时 min() 是简单函数， 仅给出单个参数时则作为 聚合函数 运算。

nullif(X,Y)

nullif(X,Y) 函数如果两个参数不同则返回第一个参数；如果相同则返回 NULL。 nullif(X,Y) 会按从左到右的顺序搜索定义排序函数的参数，并在所有字符串比较中使用该排序函数。 如果两个参数都未定义排序函数，则使用 BINARY 排序函数。

octet_length(X)

octet_length(X) 函数返回文本字符串 X 的编码字节数。如果 X 为 NULL，则 octet_length(X) 返回 NULL。 如果 X 是 BLOB 值，则 octet_length(X) 与 length(X) 相同。 如果 X 是数值，则 octet_length(X) 返回该数值的文本表示所占字节数。

由于 octet_length(X) 返回的是 X 的字节数而非字符或代码点数，因此返回值取决于数据库编码。 如果数据库编码为 UTF16 而非 UTF8，相同输入字符串可能获得不同结果。

如果参数 X 是表列且值类型为文本或 BLOB，则 octet_length(X) 无需从磁盘读取 X 的内容即可计算字节长度， 因为长度可由元数据得知。因此，即便 X 是包含多兆字节文本或 BLOB 的列，octet_length(X) 仍然高效。

printf(FORMAT,...)

printf() SQL 函数是 format() SQL 函数 的别名。format() 函数最初命名为 printf()， 但为了兼容其他数据库产品后来更名为 format()。保留 printf() 名称作为别名是为了不破坏遗留代码。

quote(X)

quote(X) 函数返回一个 SQL 字面量文本，即其参数值的表示，可直接插入 SQL 语句。 字符串会被单引号包围，并在内部引号处加以转义。BLOB 会被编码为十六进制字面量。 含嵌入 NUL 字符的字符串无法表示为 SQL 字面量，因此返回的字面量会在第一个 NUL 之前截断。

unistr_quote(X) 扩展了 quote(X) 的功能，还会对控制字符进行转义。

random()

random() 函数返回一个在 -9223372036854775807 与 +9223372036854775807 之间的伪随机整数。

random() 特意避免生成 -9223372036854775808，以便其值始终可传递给 abs()。 自版本 3.3.5（约 2006 年）起 random() 就这样实现，但此文档直到 2025-07-25 之前错误地指出 -9223372036854775808 可能是返回值。

randomblob(N)

randomblob(N) 函数返回一个包含伪随机字节的 N 字节 BLOB。如果 N 小于 1，则返回一个 1 字节的随机 BLOB。

提示：应用程序可以结合 hex() 和/或 lower() 使用此函数生成全局唯一标识符，例如：

hex(randomblob(16))

lower(hex(randomblob(16)))

replace(X,Y,Z)

replace(X,Y,Z) 函数返回一个字符串，通过将字符串 X 中的每个 Y 替换为 Z 构成。 比较使用 BINARY 排序序列。如果 Y 是空字符串，则原样返回 X。 如果 Z 起初不是字符串，则在处理前先转换为 UTF-8 字符串。

round(X)
round(X,Y)

round(X,Y) 函数返回将浮点值 X 舍入到小数点右侧 Y 位的结果。若省略或将 Y 设为负数，则视为 0。

rtrim(X)
rtrim(X,Y)

rtrim(X,Y) 返回一个字符串，通过从 X 的右侧删除出现在 Y 中的所有字符构成。 如果省略 Y 参数，则 rtrim(X) 会删除 X 右侧的空格。

sign(X)

sign(X) 函数在 X 为数值且为负、零、正时分别返回 -1、0、+1。如果 X 为 NULL 或无法无损转换为数字的字符串/ BLOB，则返回 NULL。

soundex(X)

soundex(X) 函数返回字符串 X 的 soundex 编码。如果参数为 NULL 或不包含 ASCII 字母字符，则返回 "?000"。 该函数默认未包含在 SQLite 中。只有在构建时使用了 SQLITE_SOUNDEX 编译选项时才可用。

sqlite_compileoption_get(N)

sqlite_compileoption_get() SQL 函数是对 sqlite3_compileoption_get() C/C++ 函数的包装。 该例程返回构建 SQLite 时使用的第 N 个编译时选项；若 N 超出范围则返回 NULL。另见 compile_options PRAGMA。

sqlite_compileoption_used(X)

sqlite_compileoption_used() SQL 函数是对 sqlite3_compileoption_used() C/C++ 函数的包装。 当参数 X 是编译时选项名的字符串时，若该选项在构建时使用则返回真（1），否则返回假（0）。

sqlite_offset(X)

sqlite_offset(X) 函数返回用于读取某个值所在记录在数据库文件中的字节偏移量。 如果 X 不是普通表中的列，则 sqlite_offset(X) 返回 NULL。sqlite_offset(X) 返回的值可能引用原始表或索引，具体取决于查询。 如果 X 的值通常从索引中提取，则 sqlite_offset(X) 返回对应索引记录的偏移； 若从原始表中提取，则返回表记录的偏移。

sqlite_offset(X) SQL 函数仅在 SQLite 使用 -DSQLITE_ENABLE_OFFSET_SQL_FUNC 编译选项构建时可用。

sqlite_source_id()

sqlite_source_id() 函数返回标识用于构建 SQLite 库的源代码特定版本的字符串。返回的字符串 由源代码检入的日期与时间以及该检入的 SHA3-256 哈希组成。该函数是对 sqlite3_sourceid() C 接口的 SQL 包装。

sqlite_version()

sqlite_version() 函数返回当前运行的 SQLite 库的版本字符串。该函数是对 sqlite3_libversion() C 接口的 SQL 包装。

substr(X,Y,Z)
substr(X,Y)
substring(X,Y,Z)
substring(X,Y)

substr(X,Y,Z) 函数返回字符串 X 的子串，从第 Y 个字符开始，总计 Z 个字符。 如果省略 Z，则 substr(X,Y) 会返回从第 Y 个字符起至字符串末尾的所有字符。 X 的最左字符编号为 1。若 Y 为负数，则从字符串右侧开始计数找到子串首字符。 若 Z 为负数，则返回第 Y 个字符前的 abs(Z) 个字符。 如果 X 是字符串，则字符索引基于实际 UTF-8 字符；如果 X 是 BLOB，则索引基于字节。

自 SQLite 3.34 起，“substring()”是“substr()”的别名。

total_changes()

total_changes() 函数返回自当前数据库连接打开以来所有 INSERT、UPDATE 或 DELETE 语句导致的行变更总数。 该函数是对 sqlite3_total_changes64() C/C++ 接口的包装。

trim(X)
trim(X,Y)

trim(X,Y) 函数返回一个字符串，通过同时从左右两端删除出现在 Y 中的所有字符构成。 如果省略 Y 参数，则 trim(X) 会删除 X 两端的空格。

typeof(X)

typeof(X) 函数返回一个字符串，表示表达式 X 的 数据类型： “null”、“integer”、“real”、“text” 或 “blob”。

unhex(X)
unhex(X,Y)

unhex(X,Y) 函数返回一个 BLOB，其内容是十六进制字符串 X 的解码结果。如果 X 包含非十六进制字符且不在 Y 中，则 unhex(X,Y) 返回 NULL。 若省略 Y，则视为空字符串，因此 X 必须是纯十六进制字符串。X 中所有的十六进制数字必须成对出现且彼此相邻，否则 unhex(X,Y) 返回 NULL。 如果 X 或 Y 为 NULL，则 unhex(X,Y) 返回 NULL。X 可以混合大小写十六进制数字。Y 中的十六进制数字不会影响 X 的翻译； 只有 Y 中不是十六进制数字的字符才会在 X 中被忽略。

另请参见：hex()

unicode(X)

unicode(X) 函数返回字符串 X 第一个字符对应的 Unicode 代码点数。如果参数不是字符串，则结果未定义。

unistr(X)

unistr(X) 函数解析输入字符串 X 中的反斜杠转义，并返回一个将这些转义转换为实际 Unicode 代码点的新字符串。 该函数在 SQLite 3.50.0（2025-05-29）中引入，旨在与 PostgreSQL、SQL Server 和 Oracle 保持一致。 PostgreSQL 文档指出： “Unicode 字符可以用 \XXXX（4 个十六进制数字）、\+XXXXXX（6 个十六进制数字）、\uXXXX（4 个十六进制数字）或 \UXXXXXXXX（8 个十六进制数字）表示。 要表示反斜杠，请写两个反斜杠。其它字符按字面处理。”

unistr_quote(X)

unistr_quote(X) 函数返回一个 SQL 字面量或常量表达式的文本，其编码了参数 X 的值并可嵌入 SQL 语句。 在多数情况下，unistr_quote(X) 的输出与 quote(X) 相同。 但是，当 X 是包含 U+0001 至 U+001f 范围内的控制字符的文本时，会对这些字符以及 X 中的 "\" 进行 JSON 风格的反斜杠转义， 并将整个输出包裹在 "unistr(..)" 之中。 这样可以让结果文本在解释 ANSI 转义码 的设备上安全显示。

unlikely(X)

unlikely(X) 函数返回参数 X 本身。unlikely(X) 是一个空操作，代码生成器会优化掉，因此在运行时（即调用 sqlite3_step() 时）不消耗 CPU 周期。unlikely(X) 的作用是向查询优化器提示参数 X 是通常为假的布尔值。 unlikely(X) 等价于 likelihood(X, 0.0625)。

upper(X)

upper(X) 函数返回输入字符串 X 的副本，其中所有小写 ASCII 字符都被转换为大写形式。

zeroblob(N)

zeroblob(N) 函数返回一个由 N 个 0x00 字节组成的 BLOB。SQLite 对这些 zeroblob 具有高效的管理能力。 Zeroblob 可用于为一个稍后通过 增量 BLOB I/O 写入的 BLOB 预留空间。 该 SQL 函数通过 C/C++ 接口中的 sqlite3_result_zeroblob() 例程实现。
