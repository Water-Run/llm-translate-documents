在 SQLite 中，数据库对象（表、索引、触发器或视图）由对象名称以及其所处数据库的名称来标识。 数据库对象可以位于主数据库、临时数据库或 附加数据库 中。

DROP TABLE、DROP INDEX、DROP VIEW、DROP TRIGGER、 REINDEX、ALTER TABLE 以及许多其他命令的语法都允许用户仅通过对象名称，或通过对象名称和数据库名称的组合来指定数据库对象。 如果对象引用中未指定数据库，那么 SQLite 将首先在 temp 数据库中搜索，其次在 main 数据库中搜索，再按照附加的顺序在所有附加数据库中搜索具有匹配名称的对象。 引用将解析为找到的第一个匹配项。例如： 

      /* 向临时、主数据库及附加数据库添加名为 't1' 的表 */
      ATTACH 'file.db' AS aux;
      CREATE TABLE t1(x, y);
      CREATE TEMP TABLE t1(x, y);
      CREATE TABLE aux.t1(x, y);

      DROP TABLE t1;         /* 删除 temp 数据库中的表 */
      DROP TABLE t1;         /* 删除 main 数据库中的表 */
      DROP TABLE t1;         /* 删除 aux 数据库中的表 */

如果对象引用中指定了 schema 名称，则该名称必须是 “main”、“temp”，或某个附加数据库的 schema 名称。 与其他 SQL 标识符一样，schema 名称不区分大小写。 如果指定了 schema 名称，则仅在该 schema 中搜索命名对象。

大多数对象引用仅能解析为特定类型的对象（例如 DROP TABLE 语句中的引用只能解析为表对象，而不能是索引、触发器或视图）。 但是在某些上下文（例如 REINDEX）中，对象引用可能会解析为多种类型的对象。 在搜索数据库 schema 以查找命名对象时，引用上下文中无法使用的类型对象将始终被忽略。
