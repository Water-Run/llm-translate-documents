如何下载标准SQLite源代码

1. 引言

大多数程序员使用amalgamation将 SQLite 编译进他们的应用程序。amalgamation是 C 代码，但它不是“源代码”。amalgamation由脚本从源代码生成。

本文档介绍如何获取 SQLite 的标准源代码——构建 amalgamation 的原始源文件。有关获取标准源代码后应如何处理的更多信息，请参阅如何编译 SQLite页面。

2. 直接下载

SQLite 源代码官方发布的快照通常可以直接从 SQLite 网站的下载页面获取。即使下载页面上未列出所需的特定版本，命名规则也相当清晰，因此程序员通常可以猜出某个历史版本的名称并以此方式下载。

3. 直接从版本控制系统获取代码

对于 SQLite 的任何历史版本，都可以从Fossil版本控制系统获取源代码树，既可以下载特定版本的 tarball 或 ZIP 归档，也可以克隆整个项目历史。

SQLite 源代码托管在三个地理位置不同的服务器上：

https://sqlite.org/src (Dallas)
https://www2.sqlite.org/src (Newark)
https://www3.sqlite.org/src (San Francisco)

文档则在相同服务器上的独立源代码仓库中维护：

https://sqlite.org/docsrc (Dallas)
https://www2.sqlite.org/docsrc (Newark)
https://www3.sqlite.org/docsrc (San Francisco)

要下载特定的历史版本，首先通过访问这些服务器之一的时间线页面（例如：https://sqlite.org/src/timeline）找到所需的具体版本。如果您知道希望下载版本的大致日期，可以在“timeline”URL 后添加类似“c=YYYY-MM-DD”的查询参数，以查看以该日期为中心的时间线。例如，要查看 2013 年 8 月 26 日左右的所有提交，请访问https://sqlite.org/src/timeline?c=2013-08-26。如果您寻找的是官方发布版本，请访问年表页面，点击所需发布版本左侧的日期，就会立即跳转到与该发布相对应的提交。

找到具体版本后，点击该版本的超链接以查看“提交信息页面”。然后点击“Tarball”链接或“ZIP archive”链接，以下载完整的源代码树。

4. 验证代码未被修改

源代码树根目录下的“manifest”文件为仓库中每个源文件都包含一个 SHA3-256 或 SHA1 哈希。整棵源代码树的版本名称就是“manifest”文件本身的 SHA3-256 哈希，有时如果该文件的最后一行以 “# Remove this line” 开头，则最后一行会被省略。 “manifest.uuid” 文件应包含 “manifest” 文件的 SHA3-256 哈希。如果上述所有哈希比较都正确，那么您就可以确信自己的源代码树是真实且未被篡改的。有关 manifest 文件格式的详细信息，请参阅Fossil 网站。

检查源代码真实性的过程已由 makefile 自动化： 

make verify-source

在 Windows 上： 

nmake /f Makefile.msc verify-source

使用 makefile 验证源代码完整性可以检测源代码树的意外变更，但恶意变更可能通过同时修改 makefile 而被隐藏。

5. 克隆完整开发历史

要克隆 SQLite 的完整历史，首先访问https://www.fossil-scm.org/download.html页面，下载 Fossil 版本控制程序的预编译二进制文件。也可以在同一页面获取其源代码并自行编译。

从 2017-03-12 起，必须使用 Fossil 2.0 或更高版本才能使以下说明有效。SQLite 仓库自该日期起开始使用 SHA3 哈希命名工件，而不是 SHA1 哈希，因此需要 Fossil 2.0 或更高版本才能识别新的 SHA3 哈希。要查看您正在使用的 Fossil 版本，请输入“fossil -v”。

Fossil 是一个完全独立的程序，因此只需将“fossil”或“fossil.exe”可执行文件放在您的 $PATH 或 %PATH% 中即可安装。安装 Fossil 后，请执行： 

fossil clone https://sqlite.org/src sqlite.fossil

上述命令会将 SQLite 完整开发历史的副本保存到您计算机上的“sqlite.fossil”文件。创建此副本大约需要一分钟并会传输约 32 兆字节的数据。创建副本后，通过输入以下命令“打开”仓库： 

fossil open sqlite.fossil

第二个命令会将 SQLite 源代码树中最新的提交“检出”到当前目录。之后，您可以轻松地通过输入以下命令切换到不同的版本： 

fossil update VERSION

其中 VERSION 可以是分支名（如“trunk”或“session”）以获取特定分支上的最新提交，也可以是某个提交的 SHA1 哈希或 SHA1 哈希前缀，还可以是诸如“version-3.8.8”之类的标签。每当您运行“fossil update”时，它会自动连接到原始仓库 https://sqlite.org/src ，以获取自上次更新以来其他人可能提交的新内容。
