在多线程应用程序中使用 SQLite

1. 概述

SQLite 支持三种不同的线程模式：

  1. 单线程。
    在此模式下，所有互斥锁都被禁用，SQLite 不适合在多个线程中同时使用。

  2. 多线程。
    在此模式下，只要没有单个 数据库连接 或从数据库连接派生的任何对象（例如 预处理语句）在两个或更多线程中同时使用，就可以安全地由多个线程使用 SQLite。

  3. 序列化。
    在序列化模式中，对任何 SQLite 数据库连接 或从该数据库连接派生的任何对象所做的 API 调用都可以安全地由多个线程执行。对单个对象的影响与在单个线程中按同样顺序执行 API 调用是一样的。“序列化”这个名称来自于 SQLite 使用互斥锁对每个对象的访问进行串行化处理。

线程模式可以在编译时（即从源代码编译 SQLite 库时）、启动时（即打算使用 SQLite 的应用初始化时）或运行时（即创建新的 SQLite 数据库连接时）进行选择。一般来说，运行时覆盖启动时，启动时覆盖编译时。不过，一旦选择了单线程模式，就无法再覆盖。

默认模式是序列化。

2. 线程模式的编译时选择

使用 SQLITE_THREADSAFE 编译时参数来选择线程模式。如果没有提供 SQLITE_THREADSAFE 编译时参数，则使用序列化模式。这可以通过 -DSQLITE_THREADSAFE=1 明确指定。使用 -DSQLITE_THREADSAFE=0 时，线程模式为单线程。使用 -DSQLITE_THREADSAFE=2 时，线程模式为多线程。

sqlite3_threadsafe() 接口的返回值是编译时设置的 SQLITE_THREADSAFE 的值。它不会反映通过 sqlite3_config() 接口或作为 sqlite3_open_v2() 第三个参数传入的标志在运行时对线程模式所做的更改。

如果在编译时选择了单线程模式，则构建中会省略关键的互斥逻辑，并且在启动时或运行时无法启用多线程或序列化模式。

3. 线程模式的启动时选择

假设编译时选择的线程模式不是单线程，那么可以在初始化期间使用 sqlite3_config() 接口来更改线程模式。SQLITE_CONFIG_SINGLETHREAD 动词会将 SQLite 设置为单线程模式，SQLITE_CONFIG_MULTITHREAD 动词设置为多线程模式，SQLITE_CONFIG_SERIALIZED 动词设置为序列化模式。

4. 线程模式的运行时选择

如果在编译时或启动时没有选择单线程模式，那么可以在运行时为各个数据库连接选择多线程或序列化。无法将单个数据库连接降级为单线程模式。也无法在编译时或启动时模式为单线程时将某个数据库连接提升为其他模式。

单个数据库连接的线程模式由作为 sqlite3_open_v2() 第三个参数传入的标志决定。SQLITE_OPEN_NOMUTEX 标志会使数据库连接处于多线程模式，SQLITE_OPEN_FULLMUTEX 标志会使连接处于序列化模式。如果既未指定这两个标志，或使用 sqlite3_open() 或 sqlite3_open16() 代替 sqlite3_open_v2()，则将使用由编译时和启动时设置决定的默认模式。
