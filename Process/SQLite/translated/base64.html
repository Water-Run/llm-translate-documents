Base64() SQL 函数

1. 概览

base64() 函数是一个作为 SQLite 可加载扩展实现的 SQL 函数。该函数将二进制 BLOB 转换为等效的 RFC 4648 文本，或将 RFC 4648 文本转换为等效的 BLOB。

base64() 函数并非 SQLite 的标准组成部分。它必须作为单独的扩展加载。base64() 的源代码位于 SQLite 源代码树的 base64.c 源文件 中的 ext/misc/ 文件夹。

base64() 函数不包含在 SQLite 库的标准构建中，但在 CLI 中默认加载。这是 CLI 的典型行为，它加载了许多超出标准 SQLite 库所提供的扩展。

2. 特性

  1. base64() 函数始终只接受一个参数。

  2. 如果 base64() 的参数是 BLOB，则返回值为该 BLOB 的 RFC 4648 编码文本。

  3. 如果 base64() 的参数是 base64 文本，则返回值为该 base64 文本对应的二进制 BLOB。

  4. 如果参数是 NULL，则返回 NULL。

  5. 如果 base64() 的参数不是 TEXT、BLOB 或 NULL，将引发错误。

  6. 如果参数是 TEXT，将忽略前后空白。

  7. 如果参数是具有 base64 外观但包含非 base64 字符的 TEXT，则尽可能多地将输入转换为 BLOB 并返回该 BLOB。

  8. base64() 函数使用标准的 RFC 4648 字母表：“ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/”。
