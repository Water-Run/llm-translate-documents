1. 简介

所谓 “SQLite 存档”，是一种类似于 ZIP 存档 或者 Tarball 的文件容器，但其底层是基于 SQLite 数据库。

SQLite 存档就是一个普通的 SQLite 数据库文件，其模式包含下列表： 

CREATE TABLE sqlar(
  name TEXT PRIMARY KEY,  -- name of the file
  mode INT,               -- access permissions
  mtime INT,              -- last modification time
  sz INT,                 -- original file size
  data BLOB               -- compressed content
);

SQLAR 表的每一行保存一个文件的内容。 文件名（相对于存档根目录的完整路径）存储在 “name” 字段中。 “mode” 字段是一个整数，表示该文件的类 Unix 访问权限。 “mtime” 是文件最后修改时间，以自 1970 年起的秒数表示。 “sz” 是文件原始未压缩的大小。 “data” 字段保存文件内容。内容通常使用 Deflate 压缩，但也不一定总是如此。 如果 “sz” 字段等于 “data” 字段的大小，则内容以未压缩形式存储。

1.1. 数据库作为容器对象

SQLite 存档就是一个更一般概念的示例，即 SQLite 数据库可以表现为一个容纳大量较小数据组件的容器对象。

对于像 PostgreSQL 或 Oracle 这样的客户端/服务器数据库，用户和开发者通常把数据库视作一个服务或“节点”，而不是一个对象。这是因为数据库内容散布在服务器上的多个文件中，或可能分布在集群中的多个服务器上。人们无法指向一个单一文件甚至一个目录来说 “这就是数据库”。

相比之下，SQLite 将所有内容存储在磁盘上的单个文件中。 这个单个文件可以直接指向并说“这就是数据库”。它表现得像一个对象。 SQLite 数据库文件可以被复制、重命名、作为电子邮件附件发送、作为 POST HTTP 请求的参数，或者像图片、文档或媒体文件一样作为其他数据对象处理。

研究表明许多应用已经把 SQLite 作为容器对象。例如， Kennedy （与同名的 SQLite 开发者无直接关系）报告称 14% 的 Android 应用从不向其 SQLite 数据库写入内容。相信这些应用是从云端下载整个数据库，然后按需在本地使用数据。 换句话说，这些应用使用 SQLite，不是将其当作传统数据库，而是作为一种可查询的线传输格式。

1.2. 使用 SQLite 存档的应用

Fossil 分布式版本控制 系统允许用户选择将提交下载为 Tarball、ZIP 存档或 SQLite 存档之一。

2. SQLite 存档的优势

  1. SQLite 存档具有很高的灵活性。 ZIP 存档和 Tarball 只能存储文件。SQLite 存档可以存储文件以及应用可能认为有用的其他表格或关联数据。

  2. SQLite 存档具有事务特性。 即使在更新过程中发生崩溃或断电，更新仍然是原子且可持久化的。 读取者即使在另一个进程同时更新存档，也能看到一致且不会改变的内容版本。

  3. SQLite 存档可以增量更新。 可以在不重写整个存档的情况下单独添加、删除或替换文件。

  4. SQLite 存档可以使用高级查询语言（SQL）查询。 例如：

    • 存档中所有以 “.h” 或 “.cpp” 结尾的文件的总体大小是多少？
    • 有多少百分比的文件压缩率低于 25%？
    • 存档中有多少可执行文件？
    此类问题（以及无数其他问题）都能在无需解压或提取任何内容的情况下得到解答。

  5. 已经将 SQLite 用于其他用途的应用可以通过一个小型扩展 (https://sqlite.org/src/file/ext/misc/sqlar.c) 来支持 SQLite 存档，以处理内容的压缩和解压。 如果存档中的文件是未压缩的，则即使省略这个小扩展也没问题。 相比之下，支持 ZIP 存档和/或 Tarball 通常需要独立的库或大量额外的自定义代码，甚至两者兼备。

  6. SQLite 存档可以绕过防火墙强加的审查。 例如，被认为“危险”的某些文件类型（如 DLL）会被 Gmail 以及可能的许多其他电子邮件服务和防火墙阻止，即便这些文件被包裹在 ZIP 存档或 Tarball 中。 但这些防火墙通常尚未识别 SQLite 存档，因此可以把内容放在 SQLite 存档里以规避审查。

3. SQLite 存档的劣势

  1. SQLite 存档是一种相对较新的格式，首次描述于 2014 年。相比之下，ZIP 存档和 Tarball 已存在数十年，并作为标准格式被广泛使用。 大多数程序员都知道 ZIP 存档或 Tarball 是什么，但如果你说 “SQLite 存档”，更可能听到 “什么？”的回应。处理 ZIP 存档和 Tarball 的工具更有可能预装在常见计算机上。

  2. 由于 SQLite 数据库是一个更通用的格式（它设计之初就比仅仅存储一堆文件复杂得多），因此没有 ZIP 存档或 Tarball 格式那么紧凑。 一个 SQLite 存档通常比等价的 ZIP 存档大约 1%。 Tarball 是作为单一单位压缩，而 SQLite 和 ZIP 存档则是分别压缩每个文件。因此 Tarball 往往比 ZIP 或 SQLite 存档更小。

    例如，下面的表格展示了 SQLite 3.22.0 源代码树中 1,743 个文件的 SQLite 存档、ZIP 存档和 Tarball 的相对体积：

    SQLite 存档10,754,048
    ZIP 存档（使用 Info-ZIP 3.0）10,662,365
    ZIP 存档（使用 zipfile）10,390,215
    Tarball 9,781,109

  3. SQLite 存档仅支持 Deflate 压缩算法。 而 Tarball 和 ZIP 存档支持更多样的压缩方法。

4. 从命令行管理 SQLite 存档

推荐的创建、更新、列出和提取 SQLite 存档的方式是使用 sqlite3.exe 命令行 Shell（SQLite 版本 3.23.0（2018-04-02）或更高）。 该 CLI 支持 -A 命令行选项，可简便管理 SQLite 存档。 SQLite 版本 3.22.0（2018-01-22）的 CLI 提供 .archive 命令 来管理 SQLite 存档，但那需要与 Shell 交互。

要列出名为 “example.sqlar” 的 SQLite 存档中的所有文件，可使用下列命令之一： 

sqlite3 example.sqlar -At
sqlite3 example.sqlar -Atv

要从名为 “example.sqlar” 的 SQLite 存档中提取所有文件： 

sqlite3 example.sqlar -Ax

要创建一个名为 “alltxt.sqlar” 的新 SQLite 存档，其中包含当前目录下所有 *.txt 文件： 

sqlite3 alltxt.sqlar -Ac *.txt

要向现有 SQLite 存档中添加或更新文件： 

sqlite3 example.sqlar -Au *.md

如需使用提示和所有选项的摘要，只需在没有额外参数的情况下给 CLI 添加 -A 选项： 

sqlite3 -A

如果文件名参数是 ZIP 存档而非 SQLite 数据库，上述命令仍按相同方式工作。

4.1. 其他命令行工具

就像有管理 ZIP 存档的 “zip” 程序、管理 Tarball 的 “tar” 程序一样， “sqlar” 程序 用于管理 SQL 存档。 “sqlar” 程序能创建新的 SQLite 存档、列出现有存档的内容、向存档中添加或移除文件，以及从存档中提取文件。 另有一个名为 “sqlarfs” 的程序可以将 SQLite 存档挂载为 Fuse 文件系统。

5. 从应用代码管理 SQLite 存档

应用可以通过连接 SQLite 并包含 ext/misc/sqlar.c 扩展 来轻松读取或写入 SQLite 存档，以处理内容的压缩与解压。 sqlar.c 扩展创建了两个新的 SQL 函数。

sqlar_compress(X)

sqlar_compress(X) 函数尝试使用 默认 算法压缩 X 的副本，并以 blob 返回结果。 如果输入 X 不是可压缩的 blob，则返回 X 的副本。当向 SQLite 存档插入内容时使用此例程。

sqlar_uncompress(Y,SZ)

sqlar_uncompress(Y,SZ) 函数会撤销 sqlar_compress(X) 所做的压缩。 Y 参数是已压缩的内容（来自先前对 sqlar_compress() 的调用），SZ 是生成 Y 的输入 X 原始未压缩尺寸。 如果 SZ 小于或等于 Y 的大小，则表示未发生压缩，因此 sqlar_uncompress(Y,SZ) 返回 Y 的副本。 否则，sqlar_uncompress(Y,SZ) 会对 Y 运行 Inflate 算法以解压并恢复其原始形式，并返回解压后的内容。 当从 SQLite 存档提取内容时使用此例程。

使用上述两个例程，应用可以轻松地向 SQLite 存档插入新记录或提取现有记录。 插入新记录时可以使用如下代码： 

INSERT INTO sqlar(name,mode,mtime,sz,data)
 VALUES ($name,$mode,strftime('%s',$mtime),
         length($content),sqlar_compress($content));

从 SQLite 存档中提取条目时可以使用如下代码： 

SELECT name, mode, datetime(mtime,'unixepoch'), sqlar_uncompress(data,sz)
  FROM sqlar
 WHERE ...;

上述代码适用于一般情况。对于仅存储未压缩或不可压缩内容的特殊情况 （例如只存储 JPEG、GIF 和/或 PNG 图像的 SQLite 存档），则可以在不使用 sqlar_compress() 和 sqlar_uncompress() 函数的情况下插入和提取内容，也无需 sqlar.c 扩展。
