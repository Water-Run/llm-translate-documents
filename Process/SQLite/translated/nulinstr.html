字符串中的 NUL 字符

1. 介绍

SQLite 允许在存储于数据库中的字符串值中包含 NUL 字符（ASCII 0x00，Unicode \u0000）。不过，在字符串中使用 NUL 可能会导致令人意外的行为：

  1. length() SQL 函数 仅计算直到第一个 NUL 之前（不含）的字符。

  2. quote() SQL 函数 仅显示直到第一个 NUL 之前（不含）的字符。

  3. .dump 命令在 命令行界面 中的输出会省略第一个 NUL 字符以及它之后的所有文本。事实上，命令行界面 在所有情况下都会省略第一个 NUL 字符之后的所有内容。

不建议在 SQL 文本字符串中使用 NUL 字符。

2. 意外行为

请考虑如下 SQL： 

CREATE TABLE t1(
  a INTEGER PRIMARY KEY,
  b TEXT
);
INSERT INTO t1(a,b) VALUES(1, 'abc'||char(0)||'xyz');

SELECT a, b, length(b) FROM t1;

上述 SELECT 语句的输出为： 

1,'abc',3

（本文假设 命令行界面 设置了 ".mode quote"。）但如果你运行： 

SELECT * FROM t1 WHERE b='abc';

那么不会返回任何行。SQLite 知道 t1.b 列实际上包含一个 7 字符长度的字符串，而 7 字符字符串 'abc'||char(0)||'xyz' 与 3 字符字符串 'abc' 不相等，因此不会返回任何行。但是用户可能会被误导，因为 命令行界面 的输出似乎显示该字符串只有 3 个字符。这看起来像是一个 bug，但这就是 SQLite 的行为方式。

3. 如何判断字符串中是否包含 NUL 字符

如果你将字符串 CAST 为 BLOB，则会显示字符串的完整长度。例如： 

SELECT a, CAST(b AS BLOB) FROM t1;

输出如下： 

1,X'6162630078797a'

在 BLOB 输出中，你可以清楚地看到 NUL 字符作为该 7 字符字符串的第 4 个字符。

另一种更自动化的方式来判断字符串值 X 是否包含嵌入的 NUL 字符是使用如下表达式： 

instr(X,char(0))

如果该表达式返回非零值 N，则表示在第 N 个字符位置存在嵌入的 NUL。因此，要统计包含嵌入 NUL 字符的行数： 

SELECT count(*) FROM t1 WHERE instr(b,char(0))>0;

4. 从文本字段中移除 NUL 字符

以下示例展示如何从表的列中移除 NUL 字符以及其后所有文本。因此，如果你的数据库文件中包含嵌入 NUL 字符并希望将其删除，可以运行类似如下的 UPDATE 语句： 

UPDATE t1 SET b=substr(b,1,instr(b,char(0)))
 WHERE instr(b,char(0));
