1. 介绍

虚拟表是一个显示 SQL 表接口但并不直接存储在数据库文件中的对象。虚拟表机制是 SQLite 的一项功能，允许 SQLite 使用强大的 SQL 查询语言访问和操作除数据库文件中的位之外的其他资源。

下表列出了一些可用于 SQLite 的虚拟表实现。开发者可以在自己的应用程序中部署这些虚拟表，或者将下述实现用作编写自身虚拟表的模板。

下列列表并不详尽。SQLite 源树以及其他地方存在更多虚拟表实现。下面的列表尝试涵盖一些更有趣的虚拟表实现。

2. 虚拟表

名称描述
bytecode 一个表值函数，显示准备语句的字节码。
carray 一个表值函数，允许将 C 语言的整数、双精度、字符串或二进制数组在查询中用作表。
closure 计算集合的传递闭包。
completion 在交互式 SQL 输入期间为部分输入的词提供补全建议。由CLI用于实现制表符补全。
csv 一个虚拟表，将逗号分隔值（CSV）文件（RFC 4180）表示为只读表，以便在更大的查询中使用。
dbstat 提供有关数据库文件中每页目的和使用的信息。在sqlite3_analyzer实用程序的实现中使用。
files_of_checkin 提供关于 Fossil 版本控制系统中某次提交的所有文件的信息。该虚拟表不属于 SQLite 项目，但因其展示了如何使用虚拟表且用于帮助版本控制 SQLite 源码而被列出。
fsdir 一个表值函数，对主机计算机的选定文件层级中的每个文件返回一行。由CLI用于帮助实现.archive 命令。
FTS3 高性能全文检索索引。
FTS5 更高性能的全文检索索引。
generate_series 一个表值函数，返回一系列整数，仿照 PostgreSQL 中同名的表值函数。
json_each 一个表值函数，用于分解 JSON 字符串。
json_tree 一个表值函数，用于分解 JSON 字符串。
OsQuery 数百个虚拟表，发布主机计算机的各个方面，如进程表、用户列表、活动网络连接等。OsQuery 是由 Facebook 发起的独立项目，托管于GitHub，用于安全分析与入侵检测。OsQuery 并非 SQLite 项目的一部分，但因其演示了如何利用 SQL 语言与 SQLite 虚拟表机制，为重要的现实问题提供优雅解决方案而被列出。
pragma 内置的表值函数，返回PRAGMA 语句的结果，可在普通 SQL 查询中使用。
RTree Guttmann R*Tree 空间索引思想的实现。
spellfix1 实现拼写校正引擎的虚拟表。
sqlite_btreeinfo 此实验性的表值函数提供数据库文件中的单个B 树的信息，例如深度、估计页数与条目数等。
sqlite_dbpage 用于原始数据库文件内容的键/值存储。键是页码，值是二进制页面内容。
sqlite_memstat 提供对sqlite3_status64()与sqlite3_db_status() 接口的 SQL 访问。
sqlite_stmt 一个表值函数，每个准备语句对应一行，该语句关联于打开的数据库连接.
swarmvtab 一个实验模块，通过单个虚拟表抽象按需只读地访问分布于多个数据库的多个表。
tables_used 一个表值函数，显示准备语句访问的表与索引。
tclvar 将TCL 解释器的全局变量表示为 SQL 表。作为 SQLite 测试套件的一部分使用。
templatevtab 一个模板虚拟表实现，对想编写自身虚拟表的开发者而言是有用的起点。
unionvtab 一个实验模块，通过单个虚拟表抽象按需只读地访问分布于多个数据库的多个表。
vfsstat 一个表值函数，配合同包的VFS shim 提供 SQLite 执行的系统调用次数信息。
vtablog 一个虚拟表，在其关键方法被调用时在标准输出打印诊断信息。用于虚拟表接口的交互分析与调试。
wholenumber 一个返回 1 到 4294967295 之间所有整数的虚拟表。
zipfile 将ZIP 归档表示为 SQL 表。支持读取与写入。由CLI用于实现读取与写入 ZIP 归档的功能。
